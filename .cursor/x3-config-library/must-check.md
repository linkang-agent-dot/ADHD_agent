# X3 克隆活动换皮自查清单（must-check）

> 来源：2026 世界杯 / 夏日恋语 / 深海节三节日实战踩坑收编（2026-07-09 审计沉淀）。
> 用法：**开工正查**——克隆任何活动时按环节逐项打勾，不要等导表报错/玩家客诉再倒查。
> 每条后括号标注坑源（哪个节日撞的、撞了几次）。字段级细节见同目录 `table-reference.md`。

## 一、配置克隆（ActvOnline / Pack / Reward）

- [ ] **c33/c44 展示道具**：界面顶部展示的「消耗券|兑换币」必须换成本节日道具对。规律=同节日各活动 c33 一致（尼罗 1128|1129 / 夏日 1134|1135 / 世界杯 1146|1147 / 深海转盘族 1200|1201·大富翁族 1057|1202）。拜访/累充/转盘/酒馆/集市/许愿池/BP/拼图都有这两列。（深海撞 **4 次**才全量收口 6 处——本清单头号条目）
- [ ] **累充白名单**：新建的连锁/锚点/付费包必须追加进本节日累充白名单（AO 累充行 col50；grep 列名会命中 TaskType 表头=假阳性，按 AO 行号定位）。（世界杯 ChainPack701 五档漏加→付费不计累充；夏日也漏过）
- [ ] **Pack c35 主数据名（Name 列）**：这是 dim.iap 用的纯中文名，与 i18n TXT_Pack_Name 是两个东西。抽奖券系礼包命名=所发奖券道具名，克隆时极易照抄源节日名/带尾空格。（深海 13021-24 整名是尼罗残留「女王恩典卷」+ 12 处尾空格）
- [ ] **Pack c8 PackPrice 档位**：真实 IAP 价走 PackPrice.tsv 的 Dollar，**id↔金额非顺序**（105=$4.99/106=$7.99/107=$9.99/111=$19.99/109=$14.99）。克隆必核 col8 与 col7 显示价对得上。（世界杯竞猜模板配错真金白银价，克隆继承）
- [ ] **Pack MainBg(c30)/BottomBg 留空**：拜访门头包/装饰子包/连锁子包的弹窗背景由 prefab 写死，图字段一律留空；排查图层污染要一次看完 Pack 表全部 5-7 个图字段并与历史同类对比。（夏日 210921 填了→紫色三武器弹窗，排查 3 次；深海 211020 守住）
- [ ] **Pack c19 永久售卖窗**：克隆包别误挂 6001 等永久窗（活动窗外可买）。历代拜访包全空。（深海拜访包 211020 误挂）
- [ ] **RuleTips / 带文本 ID 禁共用**：克隆活动若沿用源活动 RuleTips ID（AO col13），i18n 是同一套自动 key——必须新建独立 ID 并 repoint；正文**每一条逐条核语义**（获取途径/玩法描述会留源活动逻辑），不能只换道具名。（世界杯开箱 16031 蹭元旦：第 2 条获取途径全错+韩语整段没换；深海累充/转盘也挂过世界杯/尼罗残留）
- [ ] **Reward 三铁律**：①同 RewardID 内行 id（col1 seq）必须连续，append 文件尾会打断→整组迁 fresh 连续块（RewardID 不变引用方零改动） ②DropPara 必填 10000 ③col1=内部行号 / col2=RewardID 外键别搞混。（世界杯/夏日/深海全撞过）
- [ ] **新 Reward seq 起点必须用 Python 全表精确查 max，禁用 awk 数值比较**：awk `$1>m` 在该表上会漏报真 max（2026-07-09 马戏节实证：awk 报 15905098、真 max=15920370，差 1.5 万——直接撞了回归BP块 176 个 seq）。写后必跑全表 col0 唯一性校验。
- [ ] **克隆奖励组先查 RewardID 是否跨节日共享**：共享则必须克隆新块再改（改共享块=连源节日一起改）。（世界杯 BP 组 140 与元旦组 132 共享 4028xxx→克隆 4099xxx；深海 BP 同理）
- [ ] **BP 奖励行 id=组×100+等级 硬契约**：BattlePassScoreReward 行 id 违反公式→点升级无响应但奖励显示正常（极易误判成积分错）。（世界杯 142xx 段、深海 14121 段都撞）
- [ ] **锚点礼包(PackType15) 要挂 ItemObtain**：只 clone Pack 行不注册 ItemObtain + 券道具 ObtainID = 「再获取」入口不显示。（世界杯漏、深海补 1200 时记住了）
- [ ] **锚点礼包内容对齐 A 线**：钻石=券×125、VIP=券×1.25（券 20/80/200/400→钻 2500/1w/2.5w/5w+VIP 25/100/250/500）。历史老节日 B 线纯券是 BUG 已全量补齐，别再克隆出纯券包。
- [ ] **跨节日共享 ScoreID/奖池组要改内容先建独立 ID**：酒馆 ScoreID 717 跨节日共享→深海建 719；许愿池奖池组 105 与春节共用（原位改=接受共改）。改前先查引用方。**且别把"原位改共享组"当唯一解**：新活动壳指向新组=纯新增零污染，可当场做、不用等上届节日下线（马戏节 07-10 实证：许愿池 5014→独立组 106，解除了"等深海下线才能改组 105"的排期依赖）。"上线前才能动"只适用于**改上届在用的行本身**（如周卡复用同一 AO 的奖池组行）。
- [ ] **自选池/奖池道具审计五件套**：存在性 + i18n + 图标 + 全表流通度 + 功能出口。骨架照抄≠内容合理。（深海酒馆照抄夏日骨架带出「红宝石之血」「英雄之证」两个零联动死道具）
- [ ] **进度礼包/FinalReward 类必配 MailID(c17) 兜底**：手动领+活动结束零兜底=终奖无人领事故。（深海 102993 → X3NEW-1829 客诉，83 人补偿）
- [ ] **cfg 行乱序补在表末尾时，回查所有按线性/顺序假设引用它的下游**。（世界杯 4 队框补表尾→线性公式指错框，「解锁错框」隐蔽 bug）

## 二、i18n（换皮独立一步，导表不校验）

- [ ] **克隆新 ID 必按新 ID 建全部 TXT_ 自动 key**：客户端不读配置表单元格中文（那只是备注），走 `TXT_{表}_{字段}_{行ID}` 查 i18n bytes。导表绿≠标题不掉。界面文本空→去 CfgProtoTextEx.cs 查 UI 读哪个 key（如装饰阶梯 HUD 页签读 TXT_ChainPack_Name 不是 ActvName）。
- [ ] **逐语言扫，别只信中英**：①「cn+en 译好其余 14 语照抄英文」=伪完整，能骗过粗查 ②clone swap 只改 cn/en 其余语言残留源节日词（繁中「尼羅之輝」）③en 丢限定词会撞名（世界杯表情丢国家名→ARG/SUI 同角色撞名）。审计工具：`festival_i18n_completeness.py`（母版门控）+ `i18n_leak_audit.py --grep {节日}`（0 leak/0 空/0 撞名才过）。
- [ ] **并发 i18n 防孤儿**：基于旧 worktree 视图重复建规则 key 会造孤儿，动手前先 fetch 看队友是否已建。（深海 15018/15019 孤儿）
- [ ] **同名不同表的 key 族别猜，用真源反查**：岛屿皮肤 Skin__Skin 名 key=`TXT_Skin_Name_{id}`，英雄皮肤 Hero__HeroSkin=`TXT_HeroSkin_Name_{id}`——两表 id 空间重叠（都有 1016/1017），按错误 key 族建会撞已有英雄皮肤 key。（马戏节 07-10 实证）反查法=拿源行中文名 grep Text.tsv 拆复合 key。
- [ ] **带数字的按钮/文案 key 跨语种对数字**：同一 key 部分语种把数字翻错能潜伏多代活动（大富翁单抽 `Text_ActvVoyage_Btn_1` cn=「探索1次」但 en/jp/it/pl/po/tr/th 七语=x10，从首版错到 2026-07-10 玩家才报）。机检法=按 key 逐语种扫阿拉伯数字是否一致（对照组 `Text_ActvVoyage_Btn_2` 全语种 x5 一致）。

## 三、美术 / DK / 客户端资源

- [ ] **Item 道具 DK_Icon(col20)**：克隆道具的图标列带着源节日图（深海券 1200/1201 显示情人节信封/玫瑰→玩家报「道具没出」）。转盘类换皮过 10 项继承坑清单（背景/HUD/col33/消耗道具/奖池/RankCfg/OtherReward/券获取渠道/col16 残留名/DK_Icon）。
- [ ] **活动类 DK 双补**：手编 Path_Activity 后必补 Display_Activity（runtime 真源），否则 Ctrl+T 全量重导会清 Path-only 条目；Ctrl+T Save 前先 git pull。周卡背景多一处 tableResInfo(master)。
- [ ] **HUD 图标 vs 道具图标是两套 namespace**：ActvIcon(col21) 读 Path_Activity、道具 icon(col20) 读 Path_Item，跨 namespace 填了不显示。
- [ ] **DK/png 的 client 提交要跟配置同分支族走**：配置同步 dev 时 client 资源没跟=not found（深海 102993 icon 只在 dev_festival）。查 client DK 真实状态必用 `git grep -c "<DK>" <ref> -- <file>`（git show|grep 不可靠，四格矩阵法见 reference_x3_client_resources）。
- [ ] **新 client 资源「没生效」先想 AB 包**：症状=只新加的不对、老的都正常；数据全对（DK 解析对+bytes grep 到）就是 AB 没重建。修=正式构建出包/本地重打 AB。别再翻配置。
- [ ] **GIF 表情透明铁律**：PIL 存 GIF `optimize=False`，保 transparency==background==255，否则游戏黑底。（世界杯四强表情）
- [ ] **透明图必差分法验真透明**：gpt「transparent」会把棋盘格画进 RGB，须 remove_background 真抠。
- [ ] **克隆玩法验收「按钮显示道具=实扣道具」逐个核**：大富翁(Voyage)单抽按钮图标被 UIActvVoyage.cs 重复赋值覆盖成 LotteryItemID2(1058海神罗盘)、5连按钮图标从未被代码赋值全靠 prefab 烤死默认图——首版(75682e206)即错，2026-07-10 MR!709 修。共用 UI 代码修一次历代 Voyage 全受益（含马戏节大富翁克隆），但换皮验收仍要逐按钮对配置 LotteryItemID1/2；配置顺序本身没错时别去改表。
- [ ] **老图当构图锚生图，逐张检查源节日语义元素残留**：参考图里的节日专属道具/梗会被模型照搬进新图（马戏节实证：深海每日礼包 bg 的藏宝图+罗盘被原样带进马戏版）——c33 展示道具残留坑的美术版。prompt 写"replace ALL <源节日> elements"不够，出图后人工过一遍语义物件清单。
- [ ] **出图顺序=核心道具先行**（2026-07-10 用户纠正）：活动背景的中心物件多半是本节日抽奖道具（箱子/券/转盘），**先出道具定稿图→再拿道具图当参考锚出背景**；跳过这步=背景中心物件全错要整批重做（马戏节实证：10 张背景先出，3-4 张中心是深海寻宝语义全废）。玩法形态也要先锁死（转盘 vs 开箱）再出图。
- [ ] **外显配置行建完必回填美术排产清单（配置↔美术两清单勾稽）**：纪念卡卡面/铭牌图/皮肤头图标这类"配置一行+图一张"的外显资产，最容易掉在配置清单与美术排产的夹缝里——配置建了行≠图进了出图批次（马戏节实证 07-14：纪念卡 81 卡面+铭牌 106 图两张漏排，用户抓漏才补）。收口时按"占位 DK 清单"反查出图批次，一一对上。

## 四、数值

- [ ] **跨游戏换皮 ROI 用目标游戏口径**：X2 成就礼包 ≈2×，X3 福利/大奖档 ~700%/8×，别套源游戏。
- [ ] **券面价锚**：抽奖券 $0.25/张、宝石 $0.002、A 线 ROI≈2.25×（尼罗白皮书，见 numerical-manual.md）。
- [ ] **BP 满级门槛对齐活跃度**：任务无上限+满级太低+双 BP 共任务=上线即满级（深海 16400=世界杯 27%→D1 满级 1.8%）；夏日反例=门槛太高满级率 5.8% depth 没起来。克隆 BP 必重算 Count/满级分。
- [ ] **凡大奖是英雄皮肤，必核目标服玩家能否获得本体**：皮肤无本体=废奖（世界杯足球宝贝 5304001 vs 本体 50040 传奇限定）。延伸：消耗券→消耗活动是否部署；外显→客户端 DK 资源包；视频皮肤→客户端展示程序版本。

## 五、部署 / 上线

- [ ] **服列表别信脚本硬编码**：上 prod 前必查线上现存同类活动真实 servers[] 作 ground truth（iGame activity/list）。（世界杯 QF 漏 28 新服全撤回）
- [ ] **签到(Type14)按 7 天多期部署**，不能配一个长整段（整段=只签 7 天后面空）。
- [ ] **硬时长约束先查**：多阶段酒馆=阶段和须等于活动时长；每日礼包 Type29=固定 7 天；拼图=登录任务 max 天数下限；跨服(Type50 等)必须跨服+绝对时间对齐、最不灵活先排它。
- [ ] **子活动(TC=0) 要不要单独 iGame 部署看 AO TimeCycle 是否为空**：ActvGroupSchedule 只管 HUD 分组不管激活。（深海拼图 101828 差点漏部署）
- [ ] **配置生效 vs 服务器镜像**：部署时间早于配置 build 完成时间=跑旧配置；判据=LastDeployTime vs build 完成时刻。客户端所见非所得的改动（周卡奖池等读本地 Cfg 渲染）两端必须同批，别热更单边。
- [ ] **iGame 活动列表聚合类工具必须翻页**（bp_snowflakes 只读第 1 页漏 60 老服）。GMPrintServerActivityByCfgId 查单服活动不可靠，验证走 Mongo ServerActivity。

## 六、结算 / 运营（竞猜类专用）

- [ ] **礼包 id 跨轮复用必按 settle_from 时间窗隔离**，否则把上一轮下注算进来重复发奖；跨轮间隔近时用 created_at 时间戳非日粒度。
- [ ] **淘汰赛对阵等 ESPN 真实队名，绝不按占位序号推**；开球时间权威源=ESPN（只认 ET 锚+12h 换北京），别信 Wikipedia。

## 七、分支 / 协作（改 gdconfig 前）

- [ ] `git branch --show-current`（并发 agent 会切分支）；配置审计读 `git show origin/{分支}:tsv/...` 远端真源，别读工作树。
