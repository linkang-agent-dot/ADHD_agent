## 2. ROI 价值锚框架（换皮核心尺子）

### 公式
```
售价钻石价值 = 售价(USD) × 500          （$49.99→24995；$99.99→49995）
礼包总钻石价值 = Σ(每个道具钻石价值 × 数量)
ROI = 礼包总钻石价值 ÷ 售价钻石价值
```

### 钻石 ↔ 美刀换算（官方兑换比例 1 USD = 500 钻石）
**确认口径：1 美元 = 500 钻石**（用户确认的官方兑换比例）。`钻石充值` 页签（Pack 1001-1006）佐证：$4.99→2500、$14.99→7500、$29.99→15000、$59.99→30000、$99.99→50000 均为 1:500（$0.99→500 算出 ~505 是小额档取整，首充双倍是额外加成）。
```
美刀价值     = 钻石价值 ÷ 500              （1 钻石 = $0.002）
礼包总美刀价值 = 礼包总钻石价值 ÷ 500
美刀ROI      = 礼包总美刀价值 ÷ 售价(USD)
```
- **美刀ROI ≡ 钻石ROI（同值）**：美刀ROI = (礼包总钻石价值÷500) ÷ 售价 = 礼包总钻石价值 ÷ (售价×500) = 钻石ROI。分子分母同时 ÷500 抵消，比值不受单位影响。两个口径并列只为直观——钻石口径看"装了多少钻"，美刀口径看"装了多少钱"，ROI 数字一致。
- 例：水手招募 $49.99 档 → 礼包总钻石价值 ~451666 → 礼包总美刀价值 ~$903 → 美刀ROI = 903÷49.99 ≈ **18.1x**（与钻石ROI 18.1x 相同）。

### 换皮铁律：ROI 不变
换皮 = 换道具/皮肤/文案，**不改礼包的性价比定位**。替换道具后到 `礼包总览`（或用录入模版）重算 ROI：
- ✅ 落在模板同档区间 → 价值守恒，OK
- ❌ 偏离 → 换错了等值物，回去调数量或换等值道具

> 观测样本：水手招募系列 $49.99 / $99.99 档 ROI 普遍 **17~19x**（同价位同玩法 ROI 高度一致）。各礼包类型 / 价位的标准 ROI 带可从 `礼包总览` 按 `礼包类型`+`售价` 透视得出，换皮时以**被换模板自己的 ROI** 为锚最稳（不必跨类型比）。

## 4. 换皮数值微调规则

承接 `reskin-numerical-framework.md`（P2/X2 框架）的判断标准，X3 适配：

| 维度 | 规则 |
|------|------|
| **变现侧（锁死）** | 礼包 ROI、随机礼包期望值、掉落权重 —— 无数据支撑不动。换皮保持 ROI 同档 |
| **流量侧（可调）** | 投放节奏、累充档位门槛、限购次数、活动时长 —— 按运营节奏可调 |
| **占位行** | Reward 模板常藏 `MinNum=0` 装饰性占位（52003 信物 / 11002 加速等），换皮后**扫全部 MinNum=0 行**清掉，否则面板显 ×0（见 `table-reference.md §4.2`） |
| **等值替换** | 换道具优先选 `道具表` 已估值的等值物；重算 ROI 守在模板区间 |
| **随机礼包** | floor/锚点变动不自动联动期望值；无数据不调 drop 权重 |

## 5. 推币机/大富翁类活动换皮 ROI 方法论

> §1-4 是「单礼包」等值尺子；本节是「整套抽奖活动（推币机 / 大富翁 / 转盘）」的 ROI 尺子——玩家用罗盘/代币循环抽，产出散落在棋盘各格 + 跑圈 + 阶段奖。换皮这类活动不是看单礼包 ROI，而是看**全程产出钻价值 ÷ 全程实付钻**。
> 案例来源：2026 深海节「航海之路大富翁」(ActvType=28) 数值方案，2026-06-02。

### 5.1 全局 ROI 定义（整活动口径）
```
ROI = 全程产出钻价值 ÷ 全程实付钻
全程实付钻 = 抽奖次数 × 单抽消耗道具钻价值
           （例：买 659 罗盘 → 164,750 钻；700 抽 ≈ 实付 $330）
全程产出钻价值 = Σ(各产出来源标称钻价值)
           道具钻价值仍走 §3 道具表；标称值口径 500钻=$1（同 §2）
```
- **评估锚点用一个固定抽数算**（案例用 **700 抽** 做全活动评估锚，约对应一个深 R 玩家的完整投入）。换皮时固定同一抽数，前后对比 ROI。
- **目标 ROI 锚定同类玩法**：转盘/抽奖类锚 **尼罗节 / 春节转盘**。案例原始砍升级后 ROI ≈11.7x，目标压到 **8x**（同价值锚思路：换皮/改版后落在同档区间，深 R 不越抽越赚）。换皮时以**被换模板自己的同类玩法 ROI** 为锚最稳。

### 5.2 命中率均匀模型（怎么把"落格随机产出"折成确定钻价值）
棋盘 N 格（案例 24 格），每抽掷骰均匀落点：
```
每格落点次数 ≈ 总抽数 ÷ 格数         （700抽 ÷ 24 ≈ 每格 29.17 次落点）
某格 700抽产出钻 = 每格落点次数 × 该格单档期望 E（钻/落点）
```
- **E 值（每次落点的期望钻产出）是主旋钮**：改 E → 实时重算该格 700抽产出 → 汇总得全局 ROI。换皮调数量时只调 E，不动玩法。
- **起始岛跨越是隐藏大头，必须单独算**：代码 `landIndex>24` 即触发跨越，`REWARD_COUNT=2`（跨一次给 2 个奖励）。700抽 ≈ 跑 102 圈 → ≈204 次额外抽取，按可升级岛均值结算。漏算跨越奖励会严重低估产出 ROI。

### 5.3 ROI 加速器诊断（推币机/大富翁的通病 + 怎么压平）
**病理：投入线性、产出加速 → ROI 越抽越高**（每抽成本恒定，但产出有放大机制，深 R 越抽越赚，与"边际递减"目标相反）。两大加速器：
| 加速器 | 机制 | 处理 |
|--------|------|------|
| ① 岛屿/格子升级 | 停留累经验升级，lv1→5 产出暴涨 3-6× | 换皮可**整体砍掉**（岛屿不升级、不显等级 UI、不跑升级代码）→ 直接消除加速器。连带：多档产出**塌成单档**(定单档 E)、升级相关事件(经验+2)失效需顺手调、变色等表现优化作废 |
| ② 阶段奖励后置 | 按抽数 8 档(案例阈值 20~700)，后段价值是前段 10× | 砍升级后这是**剩余主加速**；压平靠**价值前移 / 拉平梯度** |
- 能下刀的杠杆（修边际递减）：升级曲线、单档产出 E 值、阶段奖励梯度、各产出池权重·价值、神秘宝藏大池权重。**调权重让期望更平**是不改内容只改概率的最小改动。

### 5.4 珍珠贝累计进度奖设计（线性递增 + 封顶续值）
进度道具（案例"珍珠贝"，机制复用神秘宝箱：刷格→停留领→补刷）累计满阈值领阶段奖：
- **阈值序列递增**：每档所需进度数逐档递增（线性递增梯度），**末值自动续**——到末档后按末档增量持续可领（**封顶不再涨、但可一直领**）。这是抗后期断档、给长线玩家持续正反馈的设计。
- **产出速率折算**：进度产出速率 = 常驻数 × 跑圈量（案例常驻 4 个珍珠贝 × 102 圈 ≈ 拾取 117 个）→ 用速率反推各档阈值卡领取节奏。
- **与抽数阶段奖并存**：珍珠贝进度条（按拾取数）与 §5.3② 抽数阶段奖（按抽数）**两套并行、互不影响**，分别独立结算钻价值进 ROI。

### 5.5 「神秘宝箱 → 道具进度阶段奖」改造思路
原"神秘宝箱"（随机刷格、到达额外开箱）改造为**进度阶段奖**的载体：
- 玩法层：保留宝箱的"刷格→停留拾取→补刷"机制不变，但拾取物从"直接发奖"改为"+1 进度"，**累计满阈值时才开宝箱**（5.4 的阈值序列）。
- 数值层：原本散落在格子里的宝箱即时产出，**整体转入进度阶段奖统一结算**（案例把"信物/金币/加速"从随机踩中转为进度档奖励）→ 产出从"随机不可控"变"按进度可控"，更好压 ROI 曲线。

### 5.6 换皮这类活动的 checklist
1. 固定评估锚抽数（如 700），算全程实付钻（抽数×单抽道具钻价值）。
2. 列全产出来源：各格 E×落点次数 + 起始岛跨越(REWARD_COUNT) + 阶段奖 + 进度奖 + 神秘宝藏，逐项折钻价值。
3. 汇总算全局 ROI，对齐同类玩法锚（转盘锚尼罗/春节）。
4. 偏离则优先**调 E 值 / 调权重**（不改内容），其次砍加速器（升级 / 后置梯度）。
5. 改完重算 ROI 落回目标档区间。
