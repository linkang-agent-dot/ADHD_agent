# P2→X2 数值换皮决策框架

> **本文件**：数值决策层——P2→X2 时数值要不要调、触达率逻辑（WHEN/WHY to adjust）
> **配套文件**：[reskin-rules.md](./reskin-rules.md)——换皮操作层（HOW to do reskin，字段级规则）

## 核心判断标准

**这个改动能不能提升触达率？能 → 改，不能 → 不改。**

P2→X2 换皮的根本问题是用户基数差异（X2 高R约为 P2 的 1/9），导致触达率低。所有改动决策围绕一个目标：提升触达率 → 提升付费人数 → 提升收入。

```
P2/X2 差异（用户基数悬殊）
  → 触达率低
    → 付费人数少
      → 收入低
```

改触达相关参数 → 触达率↑ → 付费人数↑ → 收入↑ → 有效
改礼包内容/ROI → 触达率不变 → 付费人数不变 → 收入不变 → 无效

## 两层决策：道具层 + 数值层

### 第一层：道具 ID 换不换？

按道具绑定的系统层级判断：

| 层级 | 判断标准 | 操作 | 典型道具 |
|------|---------|------|---------|
| 游戏系统道具 | ID 前缀 11117xxx 且名称带"合成挖矿" / 属于基础养成线 / 不带节日名称 | **直接沿用** | 研究药水、时光药水、英雄经验、加速、资源箱 |
| 节日品牌道具 | 道具名含具体节日名 / 类型为 battle_pass_exp / item_card_asset | **换 ID** | 装饰物、涂饰、节日自选箱、BP积分、卡包 |
| 项目特定内容 | 特定英雄碎片 / P2 独有道具 | **换 ID** | 英雄碎片-万斯 → X2 对应英雄 |
| 系统配置 ID | 不在 1111 表，属于 activity/quest/event 配置层 | **映射新 ID** | 矿洞条件 ID、砍价 A_INT_id |

**快速验证法**：拿到 P2 道具列表后，逐个在 X2 1111 表中查 ID。存在且名称不含其他节日名 → 沿用；存在但名称含其他节日 → 需要本节日新版本；不存在 → 需要新建或替代。

> - 修复=按占星映射 `11116110→11111082`（已做）；`11116112` 的节日替代待定。
> - ⚠️ **但别据此断言"X2 没有 11116110/112"**：这俩在 X2 英雄养成礼包(英雄达到N星-罗尼贾/美妮 201310023~037、美妮专属礼包 21351208~221)里**被大量引用**，到底是 X2 真有此料(占星换只是设计取向) 还是那些包也是死配置——**存疑，以客户端实测/用户确认为准**。换节日礼包时统一对齐占星(换 11111082)即可，不必先判定"有没有"。

### 第二层：数值改不改？

按数值影响的环节判断：

| 环节 | 与触达率的关系 | 操作 | 典型参数 |
|------|-------------|------|---------|
| 流量侧（门槛/难度/排名） | 直接影响「有多少人能玩、玩多深」 | **要改** | 开启条件、关卡矿石等级、排名积分门槛、触达率目标 |
| 变现侧（价格/ROI/结构） | 不影响触达率，跟 P2/X2 差异无关 | **不改** | 价格阶梯、礼包ROI、连锁结构、砍价比例、药水单价 |

**判断口诀**：
- 因为有差异 → 所以要提高触达 → 改触达对付费有影响 → 改
- 改礼包内容对触达没影响 → 不改

---

## 流量侧：哪些参数要改、怎么改？

| 参数 | 改动方向 | 改动依据 |
|------|---------|---------|
| 开启条件 | 降低门槛 | X2 同等门槛过滤掉的人比例更大 |
| 关卡难度 | 前期大幅降、后期少动 | 人少→排行稀疏→没追赶动力；降难度让更多人「玩进去」才有消费场景 |
| 排名积分门槛 | 按 X2 人数规模重设 | 进榜线要让 ~60% 参与者够得着，否则排行榜空荡荡 |
| 触达率目标 | 保守一档 | X2 用户对新玩法的习惯尚未建立 |

**关卡难度改法原则**：开局降多（利用复利效应）、后面少动（保持挑战感）。具体矿石等级降幅见各活动案例。

## 变现侧：哪些参数不改、为什么？

| 参数 | 不改理由 |
|------|---------|
| 价格阶梯 $1.99~$99.99 | App Store/Google Play 标准档位，跟项目无关 |
| 礼包 ROI 300%~490% | 跨节日交叉验证过的平衡点，改了不影响触达 |
| 连锁结构（免费→付费交替） | 付费转化的通用设计，跟用户基数无关 |
| 砍价阶梯 $49.99→$1.99 | 「占便宜」促销心理，跟项目无关 |
| 每日限购 1次/天 | 分散付费、制造稀缺感，跟用户基数无关 |
| 药水单价 | 挖矿内循环的锚定价格，改了会破坏产出-消耗平衡 |

## 实操清单（换皮启动时过一遍）

1. **列道具清单** → 在 X2 1111 表逐个核实 → 分"沿用/换ID/新建"三桶
2. **识别节日专属道具** → 装饰物、涂饰、自选箱、BP、卡包 → 标记【待填】
3. **识别系统配置 ID** → 矿洞条件、砍价配置等非 1111 表 ID → 标记【待映射】
4. **评估触达率差异** → 查 P2 vs X2 的高R人数比、当前触达率 → 决定流量侧参数改幅
5. **变现侧参数** → 默认全部沿用 P2，除非有明确数据表明 X2 付费分层与 P2 显著不同
6. **写入配置表** → 沿用的直接复制，待填的标记占位符，改的附上改动依据
