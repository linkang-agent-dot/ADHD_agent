# 卡册配置踩坑汇总

持续追加，每次搓完新节日补充。

---

## PowerShell + gws JSON 引号
Windows PowerShell 无法正确传递嵌套 JSON 给 `gws` CLI。
用 Python `subprocess.run()` 包装，彻底回避。

## 大批量 gws 写入静默失败
单次 `values.update` 超过 ~50 行含 JSON 内容的数据时，可能无报错但返回空。
拆成 10-20 行一批，失败时逐行重试。

## 资源 ID 不一定连续分配
科技节卡片 ID 和主题 ID 连续（81卡 + 9主题 = 90连号），但复活节的主题 ID 跳了一段。
永远从实际数据读取 ID，不要用 `max(卡片ID) + 1` 推算。

## 复活节比科技节多一个空列
复活节页签在 Col E 插了"AI出图"空列，导致后续所有列索引 +1。
每次处理新节日必须先分析实际列结构，不能直接复用上一节的列号。

## 简体 vs 繁体
表头可能写"中文名稱"（繁体风格），但实际内容用简体。
除非策划明确要求繁体，一律用简体中文。如需繁体用 `opencc-python-reimplemented` 的 S2T 转换。

## 行共享区域容易漏改
原始页签的多个逻辑区域共享同一批行号（仅用不同列区分），非常容易漏改某个区域。
建议先做完整的列分析，列出每个区域的确切行列范围，再逐区域操作。

## 已被占用的 ID
`111110008`、`111110104`、`111110105` 是卡册完成奖励道具，分配新道具 ID 时要跳过。
