# P2 配置表模块知识库

> **项目**：P2（别名后缀 `_dev` / `_P2`）
> X2 项目见 [modules-x2.md](./modules-x2.md)
>
> 本文件定义 P2 所有配置表的**大定义**（表是什么、什么时候需要它）和**子定义**（在具体组装中怎么用）。
> P2 和 X2 的大定义相同，但**列号、列名、子定义有差异**，必须分开维护。

---

## 使用方法

1. **确认项目** → 见 SKILL.md S-1（P2 用本文件，X2 用 modules-x2.md）
2. **接到任务** → 理解任务本质（涉及道具？付费？展示？活动？装饰？）
3. **逐表过触发条件** → 每张表的「何时需要」命中就拉进来
4. **读子定义** → 确认这张表在当前上下文中具体改什么字段
5. **如果所有表都不命中** → 可能需要新建配置表或新模块

## 2112 — 活动主体表 (activity_config)

### 大定义

定义活动的顶层入口：时间窗口、banner、入口图标、内容引用（串联任务和礼包）。**所有活动配置的追踪起点。**

**何时需要**：只要有「活动入口」（玩家能在活动界面看到的东西），就需要这张表。

### 重点字段

| col | 字段名 | 角色 | 换皮处理 |
|-----|--------|------|---------|
| [1] | `A_INT_id` | 活动行 ID | **换** — 分配新 ID |
| [3] | `A_STR_constant` | 活动组件类型（如 `event_gird_gacha_daily`） | **不换** — 决定前端用哪个模板 |
| [6] | `A_INT_base_activity_id` | 父活动 ID | **换** — 指向新的父活动 |
| [7] | `A_MAP_filter` | 前置条件（建筑等级等） | **换** — 如有新建筑 ID |
| [8] | `A_MAP_text` | 活动标题/描述 LC key | **换** — 改为新主题 LC key |
| [9] | `A_ARR_activity_components` | 内容引用：task→2115, package→2135 | **换** — 所有 ID 指向新行 |
| [10] | `A_MAP_description` | 规则 LC key | 同类活动**可不换** |
| [11] | `A_INT_ui_template` | 2119 UI 关联 | 同类活动**可不换** |
| [14] | `S_STR_banner_url` | Banner 图片路径 | **换** — 新主题素材 |
| [18] | `A_INT_show_hud` | HUD 入口图标 ID | **换** — 新 ID |

### 子定义

| 组装上下文 | 用法说明 |
|-----------|---------|
| **A 普通活动礼包** | components 含 task→2115 + package→2135，show_hud 换新 ID |
| **B 预购连锁礼包** | 同 A 结构，但 package 指向的 2135 为 chain 模式（带 condition.actvstart） |
| **C 装饰物** | 装饰物不直接挂 2112（通过道具系统触发），但如有装饰相关活动入口则需要 |
| **D 集卡册** | components 引用集卡册专属组件类型 |
| **E 机甲累充** | components 含 `iap_show` + `mecha_skin_select` + `jump_link`→2121 + 11个 task→2115；换皮改 group_label LC + task IDs + jump_link ID + show_hud；banner/mini_banner/ui_template/default_displaykey 可复用 |
| **F 多条件连锁礼包** | **2112 整行复用不改**；仅改 2115 任务行的 reward 道具 ID；通用道具（进度道具/加速）保留，节日绑定道具（BP箱/卡包）替换 |

### 依赖关系

```
2112 → 2115 (通过 components 的 task ID)
2112 → 2135 (通过 components 的 package ID)
2112 → 2121 (通过 components 的 jump_link ID)
2112 ← 2011 (2011 的 actv_id 反向引用 2112)
```

## 2115 — 活动任务表 (activity_task)

### 大定义

定义活动内的任务条件与奖励：达成什么条件、发什么奖励。每行一个任务档位。

**何时需要**：只要活动有「做 XX 得 YY」的任务机制，就需要这张表。

### 重点字段

| col | 字段名 | 角色 | 换皮处理 |
|-----|--------|------|---------|
| [2] | `A_INT_id` | 任务行 ID | **换** — 分配新 ID |
| [4] | `A_MAP_showcond` | 展示条件（建筑等级、活动开始时间等） | **换** — 如有新建筑 ID |
| [5] | `A_MAP_fincond` | 完成条件（如累计购买 N 次） | **换** — 内含 2013 ID 引用，必须同步改 |
| [7] | `A_ARR_reward` | 奖励道具列表 | **换** — 道具 ID 和数量 |
| [8] | `A_STR_task_desc` | 任务描述 LC key | **换** — 新主题 LC key |
| [9] | `A_MAP_task_title` | 任务标题 LC key | **换** |

### 子定义

| 组装上下文 | 用法说明 |
|-----------|---------|
| **A 普通活动礼包** | fincond 引用 2013 ID（如累购礼包），reward 引用道具 ID（注意 9 位 vs 8 位 BUG） |
| **B 预购连锁礼包** | 结构同 A |
| **E 机甲累充** | 11 档任务，fincond 为累充金额阈值（1250~500000 + 每日1500）；reward 主奖是**节日玩法货币**（非机甲/喷漆），每节日需换对应货币道具 ID（春节=`11112392`骰子、情人节=`11112900`探宝券）；固定特殊道具（染色自选箱`11112163`、机甲自选`11112280`）跨节日可复用 |
| **F 多条件连锁礼包** | 94 行级批量操作，**任务 ID 不变，只改 reward 字段**。reward 中区分：节日绑定道具→替换（如 BP 箱 `11119772`→`11112500`、卡包 `111110002`→`111110305`）；通用道具→保留（进度道具 `11119262`、加速 `11111155`）。批量可用 Python str replace |

> ⚠️ 2115 表的 ID 在 **col[1]**（col[0] 是 `A_INT_group`），用 `gsheet_query` 查询时须加 `--id-col 1`。

### 依赖关系

```
2115 ← 2112 (被 2112 的 components task 引用)
2115 → 2013 (fincond 内含 2013 ID)
2115 → 1111 (reward 内含道具 ID)
```

## 2135 — 活动礼包表 (activity_package)

### 大定义

将礼包挂载到活动：head row 定义礼包组，chain row 串联多档位/多阶段。连接「活动」和「礼包规则」的桥梁。

**何时需要**：只要活动中有「礼包购买入口」（不管免费还是付费），就需要这张表。

### 重点字段

| col | 字段名 | 角色 | 换皮处理 |
|-----|--------|------|---------|
| [1] | `A_INT_id` | 礼包行 ID | **换** — 分配新 ID |
| [3] | `A_INT_iap` | 关联 2011 ID | **换** — 指向新的 2011 行 |
| [4] | `A_INT_free_pkg_condition` | 免费礼包条件 | 按需换 |
| [6] | `A_ARR_get_items` | 礼包内含道具（chain 模式用） | **换** — 道具 ID |
| [8] | `A_STR_banner_url` | 礼包 banner 图 | **换** |
| [11] | `S_MAP_filters` | 条件过滤（chain 模式含 condition.actvstart） | **换** — 内含活动 ID |

### 子定义

| 组装上下文 | 用法说明 |
|-----------|---------|
| **A 普通活动礼包** | head row col3 指向 2011 ID，走标准 2013 模板；结构简单，多数列为空/默认值 |
| **B 预购连锁礼包** | head row col3 指向 2011 ID（同 A），但 chain row 的 filters 带 `condition.actvstart.id`（必须同步改为新 2112 ID）；col11 填 display_key（从 1111 查），**跳过 2013** |

### 依赖关系

```
2135 ← 2112 (被 2112 的 components package 引用)
2135 → 2011 (col3 指向 2011 ID)
2135 → 2112 (chain 模式的 condition.actvstart 反向引用 2112 ID)
```

## 2011 — 礼包规则表 (iap_config)

### 大定义

定义礼包的触发条件与购买规则：何时展示、累充关联、限购次数、服务器分发策略。控制「玩家在什么条件下能看到并购买这个礼包」。

**何时需要**：只要涉及 IAP 付费购买（玩家花真钱买的东西），就需要这张表。

### 重点字段

| col | 字段名 | 角色 | 换皮处理 |
|-----|--------|------|---------|
| [1] | `A_INT_id` | 礼包规则 ID（A_STR_constant） | **必须新增** — 不能复用旧 ID |
| [3] | `A_STR_function` | 类型（special / normal） | **不换** |
| [4] | `A_STR_pkg_type` | 子类型（normal / chain 等） | **不换** |
| [7] | `S_MAP_server_info` | 服务器 schema 分发 | **不换**（除非分服策略变化） |
| [8] | `A_INT_priority` | 排序权重 | **换** — 新分配 |
| [9] | `A_MAP_time_info` | 展示时机（含 `actv_id` 指向 2112） | **换** — actv_id 必须指向新 2112 ID |
| [11] | `A_MAP_triggers` | 触发条件 | 按需换 |
| [12] | `A_ARR_iap_status` | 礼包状态（含 `recharge_actv` 累充关联） | **看情况** — 本活动累充 ID 保留，外部累充需确认 |

### 子定义

| 组装上下文 | 用法说明 |
|-----------|---------|
| **A 普通活动礼包** | `time_info.actv_id` 指向 2112 控制展示时机；`iap_status` 含 `recharge_actv` 累充关联 |
| **B 预购连锁礼包** | 同 A 的 `actv_id` 逻辑（换皮必须同步改）；`iap_status` 内含大量本活动累充 ID（保留不改） |

### 依赖关系

```
2011 → 2112 (time_info.actv_id 引用 2112 ID)
2011 ← 2013 (被 2013 的 config_id 引用)
2011 ← 2135 (被 2135 的 col3 引用)
```

## 2013 — 礼包内容模板表 (iap_template)

### 大定义

定义礼包的具体道具组合模板：包含哪些道具、数量、价格、展示标题、支付渠道。是「玩家打开礼包弹窗看到的内容」的配置源。

**何时需要**：只要礼包需要「道具组合模板」（标准礼包走这里；预购连锁不走），就需要这张表。

### 重点字段

| col | 字段名 | 角色 | 换皮处理 |
|-----|--------|------|---------|
| [1] | `A_INT_id` | 模板行 ID | **换** — 分配新 ID |
| [3] | `A_INT_config_id` | 关联 2011 ID | **换** — 必须指向新 2011 ID |
| [5] | `N_STR_temp_desc` | 礼包描述 | **换** |
| [6] | `A_STR_pkg_title` | 礼包标题 LC key | **换** — 新主题 LC key |
| [8] | `A_FLT_price` | 价格 | 按需换（同档不换） |
| [9] | `A_ARR_price_info` | 支付渠道产品 ID 列表 | **不换**（同价格沿用） |
| [10] | `A_MAP_limit` | 购买限制（次数/周期） | 按需换 |
| [14-18] | `A_ARR_*_items` | 各类道具组（CD/加速/资源/PVP/其他） | **换** — 道具 ID 和数量 |

### 子定义

| 组装上下文 | 用法说明 |
|-----------|---------|
| **A 普通活动礼包** | config_id 必须指向新 2011 ID；道具组合按策划需求调整；支付渠道同价格可复用 |
| **B 预购连锁礼包** | **不涉及**（预购连锁不走 2013 模板，直接在 2135 chain row 定义内容） |

### 依赖关系

```
2013 → 2011 (config_id 指向 2011 ID)
2013 ← 2115 (被 2115 fincond 内含的 2013 ID 引用)
2013 → 1111 (道具组内含道具 ID)
```

## 1111 — 道具定义表 (item)

### 大定义

定义游戏中所有道具的基础属性：ID、分类、效果、显示资源引用、使用规则。是所有道具的「身份证」。

**何时需要**：只要涉及「新道具」或需要「查道具属性」（display_key、效果等），就需要这张表。

### 重点字段

| col | 字段名 | 角色 | 换皮处理 |
|-----|--------|------|---------|
| [0] | `A_INT_id` | 道具 ID（注意 9 位 vs 8 位） | **换** — 新道具分配新 ID |
| [2] | `A_STR_constant` | 道具唯一标识 | **必须新增** |
| [3] | `A_STR_class` | 道具分类（statue_decorate / event / decorate_paint 等） | **不换** — 同类型沿用 |
| [6] | `C_INT_display_key` | 关联 1511 的 DK ID | **换** — 指向新 DK |
| [9] | `A_MAP_lc_name` | 道具名 LC key | **换** — 新主题 LC key |
| [10] | `C_MAP_lc_desc` | 道具描述 LC key | **换** |
| [22] | `A_INT_source` | 道具来源标记 | 按需换 |
| [23] | `A_INT_get_access_group` | 关联 1168 获取途径组 ID | **换** — 指向新 1168 行 |

### 子定义

| 组装上下文 | 用法说明 |
|-----------|---------|
| **A 普通活动礼包** | 不改此表，只读 display_key 作为奖励道具图标引用源 |
| **B 预购连锁礼包** | 需查 col6 的 `C_INT_display_key` 来填 2135 chain row col11 |
| **C 装饰物** | 新增 3 类道具行：主道具(`statue_decorate`) + 升级道具(`event`) + 涂饰道具(`decorate_paint`)；涂饰道具 effect 含 `decorate_paint` 触发 + 自回收 |

### 依赖关系

```
1111 → 1511 (display_key 引用 1511 ID)
1111 → 1168 (get_access_group 引用 1168 组 ID)
1111 ← 2013 (被 2013 道具组引用)
1111 ← 2115 (被 2115 reward 引用)
1111 ← 2148 (被 2148 unlock_item / paint_item 引用)
1111 ← 1127 (被 1127 unlock_cost 引用)
```

## 1118 — 建筑升级表 (building)

### 大定义

定义建筑/装饰的星级升级规则：每★一行，含升级消耗、属性增长、等级上限。控制「从 N 星升到 N+1 星需要什么」。

**何时需要**：只要有「星级升级机制」（装饰物升星、建筑升级），就需要这张表。

### 重点字段

| col | 字段名 | 角色 | 换皮处理 |
|-----|--------|------|---------|
| [0] | `A_INT_id` | 行 ID（组×100+星级） | **换** — 新组新 ID |
| [1] | `A_INT_building_id` | 建筑/装饰组 ID | **换** — 新组 ID |
| [6] | `C_INT_display_key` | 每星外观 DK | **换** — 新 DK |
| [7] | `A_MAP_lc_name` | 建筑名 LC key | **换** |
| [11] | `A_INT_lvl` | 当前星级 | 按序填 |
| [12] | `A_INT_max_lvl` | 最大星级 | **扩星时必改** — 如 3→10 |
| [14] | `A_ARR_cost_asset` | 升级消耗（道具 ID + 数量） | **换** — 指向新道具 |
| [15] | `A_ARR_add_asset` | 升级获得 | 按需换 |
| [17] | `A_ARR_status` | 状态 BUFF | 按需换 |

### 子定义

| 组装上下文 | 用法说明 |
|-----------|---------|
| **C 装饰物** | 每颗星一行；ID 规则: 组 ID = `1118xxx`，行 ID = `组×100+星级`（如 111821901）。扩星时旧行改 `max_lvl`，新增★4+ 行。 |

### ID 编排

```
组 ID:  1118xxx（如 1118219）
行 ID:  组ID × 100 + 星级（如 111821901 = 1118219 组的 ★1）
```

### 依赖关系

```
1118 → 1111 (cost_asset 引用道具 ID)
1118 → 1511 (display_key 引用 1511 ID)
1118 ← 1127 (被 1127 building_ids 引用)
1118 ← 2148 (被 2148 building 字段引用)
```

## 2148 — 装饰等级表 (event_decoration_level)

### 大定义

定义装饰物每颗星的详细属性：BUFF 数值、涂饰开关、涂饰道具引用、涂饰技能引用。**装饰物的核心数据表。**

**何时需要**：只要涉及「装饰物的星级属性、BUFF、涂饰」，就需要这张表。

### 重点字段

| col | 字段名 | 角色 | 换皮处理 |
|-----|--------|------|---------|
| [0] | `A_INT_id` | 行 ID（组×100+星级） | **换** — 新组新 ID |
| [2] | `A_INT_group_id` | 装饰组 ID | **换** |
| [3] | `A_INT_building` | 关联 1118 建筑组 ID | **换** |
| [4] | `A_INT_unlock_item` | 解锁道具 ID（1111） | **换** |
| [5] | `A_INT_paint` | 涂饰开关（0/1） | 扩星加涂饰时 0→1 |
| [6] | `A_INT_star` | 当前星级 | 按序填 |
| [7] | `A_INT_star_max` | 最大星级 | **扩星时必改** |
| [8] | `C_INT_display_key` | 每星 DK | **换** |
| [11] | `A_MAP_lc_name` | 装饰名 LC key | **换** |
| [14] | `A_ARR_BUFF` | 属性 BUFF（按星递增） | **换** — BUFF ID + 递增值 |
| [15] | `C_ARR_paint_item` | 涂饰道具 ID | **换** — 指向新 1111 涂饰道具 |
| [16] | `A_ARR_paint_buff` | 涂饰 BUFF（每星+100） | **换** — BUFF ID |
| [17] | `A_ARR_upgrade_cost` | 升级消耗 | **换** — 道具 ID |
| [19] | `A_INT_decroation_paint_skill` | 涂饰技能行 ID（2171） | **换** — 指向新 2171 行 |

### 子定义

| 组装上下文 | 用法说明 |
|-----------|---------|
| **C 装饰物** | 核心表。col[5]=paint 标志 0/1；col[14]=BUFF 按 `每星+100, ★10跳1000` 模板递增；col[16]=paint_buff 统一用 `121140935`，val=`★×100`；扩星时旧行改 star_max + paint 标志 + 追加涂饰字段。 |

### BUFF 递增模板（col[14]）

```
★1:  buff_a=100,                  citybeauty=200
★2:  buff_a=100, buff_b=100,      citybeauty=1000
★3+: 每星 buff_a/b 各+100,        citybeauty 每星+1000
★10: buff_a=1000, buff_b=1000（跳增，非900）, citybeauty=9000
```

### ID 编排

```
组 ID:  2148xx（如 214847）
行 ID:  组ID × 100 + 星级（如 21484701 = 214847 组的 ★1）
```

### 依赖关系

```
2148 → 1118 (building 字段引用 1118 组 ID)
2148 → 1111 (unlock_item / paint_item 引用道具 ID)
2148 → 2171 (decroation_paint_skill 引用 2171 行 ID)
2148 → 1511 (display_key 引用 1511 ID)
2148 ← 1168 (被 1168 关联引用)
```

## 2171 — 装饰涂饰技能表 (装饰技能)

### 大定义

定义涂饰激活后的技能效果：按组分组，每★一行，描述该星级涂饰的具体能力（兵种增益等）。

**何时需要**：只要装饰有「涂饰」功能（激活涂饰后获得额外技能效果），就需要这张表。

### 重点字段

| col | 字段名 | 角色 | 换皮处理 |
|-----|--------|------|---------|
| [0] | `A_INT_id` | 行 ID（组×10+级别） | **换** — 新组新 ID |
| [1] | `A_INT_group` | 技能组 ID | **换** |
| [5] | `C_MAP_lc_name` | 技能名 LC key | **换** |
| [8] | `A_INT_lv` | 技能等级（对应装饰星级） | 按序填 |
| [9] | `A_INT_max_lv` | 最大等级 | 与装饰最大星级一致 |
| [12] | `A_ARR_status` | 技能效果（BUFF 数组） | **换** — BUFF ID + 数值 |

### 子定义

## 1011 — 本地化文本表 (i18n)

### 大定义

存储所有 LC key 的多语言翻译文本。是所有玩家可见文字的「翻译源」。

**何时需要**：只要有「玩家能看到的文字」（道具名、活动标题、任务描述、技能名、解锁提示...），就需要这张表。

### 重点字段

标准四列格式：

| 列 | 含义 |
|---|---|
| LC_KEY | 完整 LC key（如 `LC_ITEM_2026easter_dec2_item_2`） |
| key | 短 key（如 `2026easter_dec2_item_2`） |
| 中文 | 中文文本 |
| English | 英文文本 |

后续列为其他语言翻译。

### 子定义

| 组装上下文 | 用法说明 |
|-----------|---------|
| **A 普通活动礼包** | 活动标题/描述 + 礼包标题 LC key |
| **B 预购连锁礼包** | 活动标题 LC key（无单独礼包标题） |
| **C 装饰物** | 道具名/描述 + 涂饰道具名/描述 + 升级道具描述 + 解锁提示 + 涂饰技能名 LC key |
| **D 集卡册** | 6+1 区域结构：卡片名 / 卡包名 / 卡册主题名（结构最复杂） |

### LC Key 命名模板

```
道具名:     LC_ITEM_{theme}_{item_name}
道具描述:   LC_ITEM_{theme}_{item_name}_desc
涂饰名:     LC_ITEM_{theme}_paint_{dec_name}_name    →  "{装饰名}"粉刷
涂饰描述:   LC_ITEM_{theme}_paint_{dec_name}_desc    →  使用后可激活"..."装饰物带来的增益状态。
升级道具描述: {desc}。首次获得可以解锁{装饰名}，后续获得可用于升级{装饰名}。
活动标题:   LC_EVENT_{theme}_{activity_type}
技能名:     LC_EVENT_{theme}_dec_paint_skill_name
```

### 依赖关系

```
1011 ← 所有含 LC key 的表（2112 text/description, 2115 task_desc/task_title,
       2013 pkg_title, 1111 lc_name/lc_desc, 1118 lc_name, 2148 lc_name/lc_desc,
       2171 lc_name, 1127 unlock_desc）
```
