# P2 拓荒节集卡册 — 流水线总览（与 x2 分离说明）

## 1. 为何与 x2-media 分开

- **x2-media** 面向 **X2 工程**（含 CardGallary 640×900 等图鉴体系）。  
- **P2 拓荒节集卡册** 使用**另一套**尺寸、目录命名、批处理脚本，历史上曾复用 GRFal 能力但**不应**再挂在 x2-media 的「集卡/图鉴」类型下。  
- 本 pipeline 的 Agent 入口为 **`p2-card-album-media`**；生成工具仍走 **grfal-api**。

## 2. 两条交付物

| 类型 | 常见规格 | 说明 |
|------|----------|------|
| 套内卡 | 240×320 | 每「册」9 张，多册主题（工地/冒险/牛仔/掘金/大富翁/弹珠/挖孔/推币等） |
| 卡册主题图（册封面块） | 304×328 透明 | 白底 1:1 出图后 rembg + 描边 + 落入安全区；与单张 240 卡是**不同**产物 |

## 5. 启动 Checklist 标准格式

```
[前置检查]（自动检测，无需用户操作）
- Bearer token 状态
- Roger / Scott / Penguin 参考图路径
- 历史封面参考目录
- assets 目录（棋盘UI / 描边风格参考）

[角色分配 checklist]（自动推荐，用户确认）
规则：
- Roger 主导（最熟悉角色），占比约 50%
- Scott 出现在体力/竞技/对弈场景，占比约 20%
- Penguin 出现在轻松/游戏场景，占比约 8%
- 双主角（Roger+Scott / Roger+Penguin）出现在重要/对话场景，占比约 18%
- 整体 72 张不超过 55% 同一角色

输出格式：
| 组 | 主题 | 角色分配 | 逻辑 |
每组汇总 + 全局汇总（各角色总数）
确认后直接生成，无需逐卡确认
```
