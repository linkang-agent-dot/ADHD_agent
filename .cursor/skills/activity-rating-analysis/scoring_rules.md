# 评分规则详解

## 评分体系总览

基于1041游戏付费效果分析报告设计，核心发现：
- ARPPU是活动效果差异的关键（$32-$45差距40%）
- 付费率各活动差异不大（10-12%）
- 超R贡献60-72%是付费主力

## 维度权重

| 维度 | 权重 | 指标 | 核心逻辑 |
|-----|------|------|---------|
| 付费深度 | 40% | ARPPU | 让付费用户花更多 |
| 转化广度 | 30% | 付费率 | 拉动更多人付费 |
| 综合变现 | 20% | ARPU | 整体变现效率 |
| R级健康度 | 10% | 超R收入占比 | 60-70%最健康 |

## 付费深度评分（ARPPU）

```python
def score_arppu(x):
    if x >= 100: return 100   # 顶级
    elif x >= 70: return 90   # 优秀
    elif x >= 50: return 80   # 良好
    elif x >= 35: return 70   # 中等
    elif x >= 25: return 60   # 及格
    elif x >= 15: return 45   # 偏低
    else: return 30           # 较差
```

**参考值（报告数据换算）：**
- 科技节：$45.4 → 游戏内约100+
- 周年庆：$39.6 → 游戏内约80-90
- 音乐节：$32.5 → 游戏内约60-70

## 转化广度评分（付费率）

```python
def score_pay_rate(x):
    if x >= 15: return 100    # 顶级
    elif x >= 12: return 90   # 优秀
    elif x >= 10: return 80   # 良好
    elif x >= 8: return 70    # 中等
    elif x >= 6: return 55    # 及格
    elif x >= 4: return 40    # 偏低
    else: return 25           # 较差
```

**参考值（报告数据）：**
- 周年庆：12.2%（最高）
- 科技节：11.1%
- 春节：10.3%

## 综合变现评分（ARPU）

```python
def score_arpu(x):
    if x >= 20: return 100
    elif x >= 15: return 90
    elif x >= 10: return 80
    elif x >= 7: return 70
    elif x >= 5: return 60
    elif x >= 3: return 45
    else: return 30
```

**参考值（报告数据换算）：**
- 科技节：$5.06
- 音乐节：$3.75

## R级健康度评分（超R收入占比）

```python
def score_r_health(x):
    if 60 <= x <= 70: return 100   # 最佳区间
    elif 55 <= x < 60: return 85   # 略低
    elif 70 < x <= 75: return 85   # 略高
    elif 50 <= x < 55: return 70   # 偏低
    elif 75 < x <= 80: return 70   # 偏高
    elif 45 <= x < 50: return 55   # 较低
    elif 80 < x <= 85: return 55   # 较高
    elif x > 85: return 40         # 过度依赖
    else: return 40                # 过低
```

**报告数据：**
- 超R贡献60-72%是正常范围
- 科技节72.2%、周年庆71.3%偏高
- 登月节60.2%、音乐节61.0%较健康

## 综合得分计算

```python
综合得分 = (
    付费深度得分 * 0.40 +
    转化广度得分 * 0.30 +
    综合变现得分 * 0.20 +
    R级健康度得分 * 0.10
)
```

## 等级与梯队

### 梯队

| 梯队 | 综合得分 | 含义 |
|-----|---------|-----|
| T1 | ≥80 | 标杆活动 |
| T2 | 65-79 | 表现稳定 |
| T3 | <65 | 需优化 |

## 多期活动处理

同名活动多期合并时：
- **单次平均付费额** = 累计总额 ÷ 上线次数
- ARPPU、付费率、ARPU取**平均值**
- 超R占比用累计值计算（结果相同）
