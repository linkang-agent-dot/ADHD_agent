# 节日活动效果评级分析

## 适用场景

- 评估单个/多个活动的付费表现
- 新活动数据与历史活动对标
- 诊断活动的付费结构问题

## 四维度评估框架

| 维度 | 看什么 | 一句话解读 | 权重 |
|-----|-------|----------|-----|
| **变现力** | 付费ARPU | 每个用户平均贡献多少钱 | 30% |
| **转化力** | 付费率 | 多少人愿意掏钱 | 25% |
| **鲸鱼依赖度** | 超R收入占比 | 收入是不是全靠超R撑 | 25% |
| **付费分层清晰度** | 超R÷小R付费率 | 越有钱的人是不是越愿意买 | 20% |

## 评级标准

### 综合评级

| 等级 | 综合得分 | 含义 |
|-----|---------|-----|
| **S** | ≥85 | 标杆活动 |
| **A** | 70-84 | 优秀 |
| **B** | 55-69 | 合格 |
| **C** | 40-54 | 待优化 |
| **D** | <40 | 需重点改进 |

### 各维度评分标准

详见 [scoring_rules.md](scoring_rules.md)

## 使用流程

### 步骤1：获取数据

用户需提供以下数据（Excel或文本均可）：

```
必需字段：
- 活动名称
- 各R级（小R/中R/大R/超R）的：付费总额、付费人数、付费率

可选字段：
- 投放内容描述
- 整体付费率、整体ARPPU、整体付费ARPU
```

### 步骤2：计算指标

从原始数据计算四维度指标：

```python
# 变现力
付费ARPU = 总付费额 / 总用户数

# 转化力
付费率 = 付费人数 / 总用户数 * 100

# 鲸鱼依赖度
超R收入占比 = 超R付费额 / 总付费额 * 100

# 付费分层清晰度
分层清晰度 = 超R付费率 / 小R付费率
```

### 步骤3：运行分析脚本

```bash
python scripts/rating_analysis.py --input <数据文件路径>
```

### 步骤4：输出结果

脚本将生成：
- `活动评级分析结果.xlsx` - 完整数据和评分明细
- `活动评级分析图表.png` - 4张可视化图表

```
    度   │  高依赖     │  低依赖      │

```

### 常见问题诊断

| 问题 | 表现 | 建议 |
|-----|------|-----|
| 鲸鱼依赖过高 | 超R收入占比>85% | 增加中大R的付费点 |
| 分层失效 | 清晰度<5倍 | 检查超R奖励吸引力 |
| 转化力低 | 付费率<7% | 优化活动入口/定价 |
| 变现力弱 | ARPU<5 | 增加付费深度设计 |

## 新活动对标

提供新活动R级数据后，输出：

1. **四维度得分**：与评分标准对比
2. **综合评级**：S/A/B/C/D
3. **对标活动**：找出最相似的历史活动
4. **诊断建议**：具体优化方向
