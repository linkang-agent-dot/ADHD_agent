# 节日活动付费效果评级系统

## 适用场景

- 评估单个活动的付费表现
- 对比多个活动的效果排名
- 为新活动方案提供效果预估参考
- 诊断活动的付费结构问题

## 评分体系（基于1041报告）

| 维度 | 指标 | 权重 | 核心逻辑 |
|-----|------|------|---------|
| **付费深度** | ARPPU | 40% | 让付费用户花更多（报告发现：活动效果差异主要体现在ARPPU） |
| **转化广度** | 付费率 | 30% | 拉动更多人付费 |
| **综合变现** | ARPU | 20% | 整体变现效率 |
| **R级健康度** | 超R收入占比 | 10% | 60-70%最健康，过高风险大 |

### 报告核心发现

1. **ARPPU是活动效果差异的关键**：科技节$45.4 vs 音乐节$32.5，差距40%
2. **付费率各活动差异不大**：各活动付费率在10-12%，波动仅2pp
3. **超R贡献60-72%是付费主力**：超R占DAU仅3-6%，但贡献P2付费额60-72%
4. **活动效果好坏取决于能不能让超R花更多钱**

## 梯队与等级

### 梯队划分（参考报告T1/T2/T3）

| 梯队 | 综合得分 | 含义 |
|-----|---------|-----|
| **T1** | ≥80分 | 标杆活动，可复制推广 |
| **T2** | 65-79分 | 表现稳定，有优化空间 |
| **T3** | <65分 | 需重点优化 |

### 等级划分

| 等级 | 综合得分 | 含义 |
|-----|---------|-----|
| **S** | ≥85 | 标杆活动 |
| **A** | 75-84 | 优秀 |
| **B** | 65-74 | 合格 |
| **C** | 50-64 | 待优化 |
| **D** | <50 | 需重点改进 |

## 评分标准

### 付费深度（ARPPU）- 权重40%

| ARPPU区间 | 得分 | 评价 |
|----------|------|------|
| ≥100 | 100 | 顶级 |
| 70-100 | 90 | 优秀 |
| 50-70 | 80 | 良好 |
| 35-50 | 70 | 中等 |
| 25-35 | 60 | 及格 |
| 15-25 | 45 | 偏低 |
| <15 | 30 | 较差 |

### 转化广度（付费率）- 权重30%

| 付费率 | 得分 | 评价 |
|--------|------|------|
| ≥15% | 100 | 顶级 |
| 12-15% | 90 | 优秀 |
| 10-12% | 80 | 良好 |
| 8-10% | 70 | 中等 |
| 6-8% | 55 | 及格 |
| 4-6% | 40 | 偏低 |
| <4% | 25 | 较差 |

### 综合变现（ARPU）- 权重20%

| ARPU区间 | 得分 | 评价 |
|----------|------|------|
| ≥20 | 100 | 顶级 |
| 15-20 | 90 | 优秀 |
| 10-15 | 80 | 良好 |
| 7-10 | 70 | 中等 |
| 5-7 | 60 | 及格 |
| 3-5 | 45 | 偏低 |
| <3 | 30 | 较差 |

### R级健康度（超R收入占比）- 权重10%

| 超R收入占比 | 得分 | 评价 |
|------------|------|------|
| 60-70% | 100 | 最佳区间 |
| 55-60%或70-75% | 85 | 略偏离 |
| 50-55%或75-80% | 70 | 偏离 |
| 45-50%或80-85% | 55 | 较偏离 |
| <45%或>85% | 40 | 极端/风险高 |

## 特殊处理

### 同名活动多期合并

系统自动识别同名活动的多期（如"挖孔小游戏-第1期"、"第2期"），合并后：
- **单次平均付费额** = 累计总额 ÷ 上线次数（用于评级）
- **累计总付费额** = 所有期数累加（供参考）
- ARPPU、付费率取平均值

### 模糊匹配规则

自动合并以下模式：
- `XXX-第1期`、`XXX-第2期` → `XXX`
- `XXX-圣诞节`、`XXX-复活节` → `XXX`
- `XXX（BP排行榜）-周年庆` → `XXX`

## 使用流程

### 步骤1：准备数据

用户需提供Excel文件，包含以下字段：

```
必需字段：
- 活动名称
- 付费总额
- 付费人数
- 付费率
- 付费ARPU
- ARPPU
- R级数据（小R/超R的付费率、付费额）

可选字段：
- 投放内容描述
- 节日时间
```

### 步骤2：运行分析

```bash
python scripts/rating_analysis.py --input <数据文件路径>
```

### 步骤3：查看输出

脚本生成：
- `单活动评级_结果.xlsx` - 完整数据和评分明细
- `单活动评级分析图表.png` - 4张可视化图表

## 输出图表说明

| 图表 | 用途 |
|-----|------|
| 综合评分排名 | 单活动得分排名，显示梯队+等级 |
| 四维度雷达图 | Top活动四维度对比，看短板 |
| ARPPU vs 付费率散点图 | 付费深度与转化广度关系，气泡=单次平均付费额 |
| 维度得分对比 | Top活动各维度得分柱状图 |

## 常见问题诊断

| 问题 | 表现 | 建议 |
|-----|------|-----|
| 付费深度低 | ARPPU<35 | 增加高价值礼包、提升付费上限 |
| 转化力低 | 付费率<8% | 优化活动入口、降低付费门槛、增加首充奖励 |
| R级依赖过高 | 超R占比>80% | 增加中大R档位礼包降低风险 |
| 综合变现弱 | ARPU<5 | 同时优化ARPPU和付费率 |
