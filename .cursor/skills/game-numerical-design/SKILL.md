# 游戏数值设计讨论框架

## 触发条件
当用户讨论以下内容时触发：
- 活动数值设计、玩法数值设计
- ROI计算、投放设计、坑深设计
- 玩家体验设计、付费分层设计
- 提到"数值"、"ROI"、"投放"、"坑深"、"玩家体验"等关键词

## 核心框架

### 1. 设计目的层（必须首先明确）

在开始任何数值讨论前，必须先确认：

```
┌─────────────────────────────────────────────────────────┐
│  核心问题：要解决什么问题？                              │
│  ↓                                                      │
│  设计目标：达成什么效果？（可量化）                      │
│  ↓                                                      │
│  设计约束：有哪些不能动的限制？                          │
└─────────────────────────────────────────────────────────┘
```

**提问模板：**
- "这个设计要解决的核心问题是什么？"
- "目标用户是谁？（超R/大R/中R/小R/免费）"
- "有哪些约束条件？（不能改的、必须保持的）"

### 2. 玩家循环层（体验视角）

任何玩法都要拆解玩家循环：

```
进入活动 → 了解玩法 → 核心游玩 → 获得奖励 → 兑换/消耗 → 持续/退出
    ↑                                                        │
    └────────────────── 循环 ──────────────────────────────┘
```

**每个阶段要回答：**
| 阶段 | 设计点 | 核心问题 |
|-----|-------|---------|
| 进入活动 | 玩家动力 | 为什么要玩？吸引力是什么？ |
| 核心游玩 | 体验设计 | 玩起来感觉怎么样？有没有爽感？ |
| 获得奖励 | 价值感知 | 玩家能感知到奖励的价值吗？ |
| 兑换/消耗 | 奖励获取 | 能换到想要的东西吗？成本合理吗？ |
| 持续/退出 | 长线留存 | 有没有阶段性目标？会不会无聊？ |

### 3. 数值验证层（经济视角）

**ROI计算框架：**
```
玩家投入（成本）
    ↓
行为产出（道具/券/积分）
    ↓
兑换价值（能换到什么）
    ↓
ROI = 兑换价值 / 玩家投入
```

**分层验证：**
| 玩家档位 | 投入 | 产出 | 能换到 | ROI | 体验感受 |
|---------|-----|------|-------|-----|---------|
| 小R | $X | ... | ... | ... | ... |
| 中R | $X | ... | ... | ... | ... |
| 大R | $X | ... | ... | ... | ... |
| 超R | $X | ... | ... | ... | ... |

## 讨论沉淀规则

### 每次讨论必须记录：

1. **问题**：讨论的是什么问题？
2. **方案**：最终选择了什么方案？
3. **原因**：为什么这么做？（这是最重要的）
4. **否决项**：讨论过但否决的方案，以及否决原因

### 沉淀格式：

```markdown
### [设计点名称]

**问题**：xxx

**方案**：xxx

**原因**：xxx

**否决项**：
- 方案A：xxx（否决原因：xxx）
- 方案B：xxx（否决原因：xxx）
```

## 讨论流程

### 开始新讨论时：

1. 先确认设计目的（核心问题、目标、约束）
2. 画出玩家循环
3. 按循环阶段逐个讨论设计点
4. 每个设计点讨论完，立即沉淀

### 继续已有讨论时：

1. 先回顾已沉淀的内容
2. 提醒用户"上次我们讨论到xxx，确定了xxx"
3. 继续未完成的设计点

### 讨论结束时：

1. 总结所有设计点和决策
2. 输出完整的设计文档
3. 更新可视化网页（如有）
4. 沉淀到Obsidian KB

## 提醒机制

当用户可能遗忘时，主动提醒：

- "我们之前确定了[设计目的]是xxx，现在讨论的这个点是否符合这个目标？"
- "这个方案和之前确定的[约束条件]有冲突，需要重新考虑"
- "上次我们否决了类似的方案，原因是xxx，这次情况有变化吗？"

## 输出物

1. **设计文档**（Obsidian MD）
   - 设计目的
   - 玩家循环
   - 设计点与决策
   - 核心数值

2. **可视化网页**（HTML）
   - 设计目的
   - 玩家循环图
   - 设计点卡片（问题/方案/原因）
   - 数值表格

3. **数值表格**（Google Sheet）
   - 基础数值
   - ROI计算
   - 分层验证
   - 模拟验证
