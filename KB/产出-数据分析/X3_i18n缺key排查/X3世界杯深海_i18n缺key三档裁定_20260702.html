机制纠正（关键）：缺 key 的表现是「空白」，不是「海外原样显示中文」
配置文本列（Label/Name/Desc/Tag/RankName…）在导表时被剥离，根本不进 client config bytes——proto 生成类里没有这些字段的原始 string 成员（实证 CActvExchangeCfg 等只有 int/bool/数组）。文本的唯一访问器是 CfgProtoTextEx.cs 生成的 key-getter：
getter → CfgHelper.CheckAndConvertCfgTxtToEmpty(key) → LocalizationMgr.Get(key) 为空 ? "" : key
UI: TFWText.text = getter结果 → 空串直接显空白，无全局兜底
∴ key 没建 → getter 返回空串 → 该位置空白，且影响所有语言（含简中）。单元格里的中文从不下发，谈不上"原样显示中文"。
但"空白"还是"无害"取决于 UI 有没有读这个 getter、空了会不会 SetActive(false) 隐藏或 fallback —— 由此分三档。
⚠️「隐藏 ≠ 无害」：② 档字段空值时 UI 会 SetActive(false) 隐藏该元素——意味着策划写在单元格、想显示的角标/描述对全语言（含简中）都没显示出来，是设计意图静默丢失。尤其 营销角标（Pack.Tag「限定/120倍」、ActvExchange.Label「专属」）是礼包价值钩子，缺 key = 角标对谁都不显示，变现上有损失，商业上应修。
