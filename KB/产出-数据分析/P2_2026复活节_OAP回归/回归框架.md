## 活动机制摘要

- 储值期投入猿猴币 → 锁定 → 分2期返还本金+节日自选道具
- 仅活动期间充值获得的猿猴币可参与（历史屯币不可用）
- 6个储值档位：2000/5000/10000/20000/30000/50000 猿猴币
- 设计ROI：110.5% ~ 135.7%（档位越高ROI越高）
- 付费墙新增5档猿猴币礼包
- 设计目的：提高猿族币收入占比，增加猿猴币活动曝光与内部返利
- 打点：ope_coin_store（参与储值）/ ope_coin_store_refund（储值返还）

## 回归点 1：整体收入与付费表现

| 模块 | 内容 |
|------|------|
| **目的** | 验证 OAP 是否达成"提高猿族币收入占比"的设计目标，评估活动对节日整体盘子的贡献量级 |
| **数据** | ① 活动总收入（美元）② 节日付费人数 & 活动付费人数 ③ 活动ARPU（活动收入/节日付费人数）④ 活动付费率（活动付费人数/活动期DAU）⑤ 与上期OAP（2025感恩节二期）对比：收入、付费率、ARPU 变化 |
| **优化** | 判断OAP在节日中的收入占比是否合理，是否有继续提升的空间；对比历史期次看趋势是上升还是衰退 |

## 回归点 2：档位选择分布 × R级分层

| 模块 | 内容 |
|------|------|
| **目的** | 验证6档定价梯度是否有效分层不同付费能力玩家，识别是否存在"设计了但没人买"的空档 |
| **数据** | ① 各档位购买人数、购买次数、收入贡献占比 ② 分R级（小R/中R/大R/超R）的档位选择分布 ③ 各档位人均储值次数（是否有人重复参与同一档） |
| **优化** | 哪些档位是主力档？低档位是否有效触达小R？高档是否承接超R需求？如果某档位渗透极低，判断是定价问题还是ROI吸引力不足 |

## 回归点 3：转化漏斗（曝光→充值猿猴币→参与储值）

| 模块 | 内容 |
|------|------|
| **目的** | 拆解从"看到活动"到"实际完成储值"的全链路转化，找出流失最大的环节 |
| **数据** | ① 活动页面曝光/打开人数 ② 猿猴币购买人数（付费墙新页签"猿猴币补给"） ③ 实际参与储值人数 ④ 各步骤转化率：曝光→购买→储值 ⑤ 分R级看每步转化率差异 |
| **优化** | 如果"购买猿猴币→参与储值"这步流失大，说明储值机制理解成本高或ROI不够有吸引力；如果"曝光→购买"流失大，说明活动入口或猿猴币认知有问题 |

## 回归点 4：ROI兑现与领奖行为

| 模块 | 内容 |
|------|------|
| **目的** | 验证分期返还机制是否有效（玩家是否回来领奖），实际ROI是否与设计一致 |
| **数据** | ① 第1期/第2期领奖率（领奖人数/储值人数） ② 未领奖流失比例及原因（忘了？还是弃号？） ③ 各档位实际ROI vs 设计ROI ④ 补发邮件触发量（活动结束后未领取的补发数据） |
| **优化** | 领奖率低说明分期机制可能增加了操作成本而非粘性；判断2期返还的时间间隔是否合理；补发量大说明机制需要简化 |

## 回归点 5：猿猴币生态影响

| 模块 | 内容 |
|------|------|
| **目的** | 回到设计初衷——OAP是否真的拉动了猿猴币收入占比？是增量还是存量挤压？ |
| **数据** | ① 活动期间猿猴币总充值额 vs 非活动期猿猴币充值额 ② 活动期间美元直充收入是否下降（是否存在渠道替代效应） ③ 猿猴币充值渗透率变化（活动期 vs 活动前后） |
| **优化** | 如果猿猴币充值上涨但美元直充同比例下降，说明OAP只是把同一批钱从美元换成猿猴币，增量有限；反之则证明OAP确实创造了新增付费 |

## 回归点 6：新增猿猴币礼包（付费墙5档）表现

| 模块 | 内容 |
|------|------|
| **目的** | 评估付费墙新增的5档猿猴币礼包定价和转化效果 |
| **数据** | ① 5档礼包各自销量、收入、购买人数 ② 人均购买次数（是否有人反复购买低档凑高档） ③ 与常规猿猴币充值对比：单价是否有优势？ |
| **优化** | 5档中哪些是主力档？是否存在某档定价不合理导致跳档？是否需要调整档位间距 |
