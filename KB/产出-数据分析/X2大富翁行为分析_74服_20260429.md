# X2 大富翁活动效果分析报告

X2玩家行为循环：
* 日常活跃行为
		* shop小游戏（8天63000次）
		* 完成任务并领奖（8天12000次）
			* 加速
			* 资源
			* 战利品
* * PVE


## 一、分析背景

**目标**：探究74服大富翁活动期间（2026.4.29~5.6），参与玩家的行为特征，定位大富翁参与度低的原因，并提出优化方向。

**数据范围**：
- 游戏：X2（game_cd=1089）
- 服务器：1007402（74服）
- 活动期：2026-04-29 ~ 2026-05-06（8天）
- 样本：47个大富翁参与玩家
- 数据源：TRINO_HF，schema `iceberg.v1089`

---

## 二、分析路径

```
第1层：reason_id 粗分类（玩家做了哪些大类行为）
    ↓
第2层：reason_sub_id 细拆（每个大类里具体做了什么子任务）
    ↓
第3层：ods_user_event counter级（客户端真实操作事件，最底层）
    ↓
第4层：按活跃度分群（登录≥7天 vs 全体，看核心玩家的行为差异）
```

## 四、第2层：高频行为细拆

### 4.1 event_task 按 reason_sub_id 拆分

#### 数据来源
表：`ods_user_asset` LEFT JOIN `dim_asset`

#### SQL
```sql
SELECT a.reason_sub_id,
       COUNT(DISTINCT a.user_id) AS participants,
       COUNT(*) AS total_actions,
       SUM(a.change_count) AS total_gain,
       ARRAY_AGG(DISTINCT b.asset_name) AS reward_names
FROM iceberg.v1089.ods_user_asset a
LEFT JOIN iceberg.v1089.dim_asset b ON a.asset_id = b.asset_id
WHERE a.server_id = '1007402'
  AND a.partition_date BETWEEN '2026-04-29' AND '2026-05-06'
  AND a.reason_id = 'event_task'
  AND a.change_type = '1'
  AND a.user_id IN (/* 47个user_id */)
GROUP BY a.reason_sub_id
ORDER BY total_actions DESC
LIMIT 30
```

#### 结果（按奖励类型归类）

| 任务类型        | 参与人数   | 8天总次数   | 奖励内容            |
| ----------- | ------ | ------- | --------------- |
| 加速道具任务      | 37~47人 | ~3,200次 | 5分钟加速、5分钟建筑加速   |
| 资源采集任务      | 38~47人 | ~2,800次 | 粮食、钢铁           |
| 宝箱任务        | 38~47人 | ~635次   | 战利品宝箱-绿/蓝、高级抽奖券 |
| 电池+英雄经验任务   | 21~36人 | ~789次   | 电池、英雄经验-5000    |
| 混合资源任务（阶段性） | 24~36人 | ~876次   | 粮食、钢铁 + 未知道具    |

#### 结论
**加速道具是玩家最渴望的资源**，加速任务频次（~3,200次）远超资源任务（~2,800次）。任务参与呈漏斗结构：47人基础 → 43~46人进阶 → 28~39人深度 → 21~30人高难。


## 五·三、消耗行为：资源流向分析

> 数据来源：`ods_user_asset`，`change_type = '2'`（消耗方向）

### SQL

```sql
WITH monopoly_users AS (
  SELECT DISTINCT user_id FROM iceberg.v1089.ods_user_asset
  WHERE server_id = '1007402'
    AND partition_date BETWEEN '2026-04-29' AND '2026-05-06'
    AND reason_id = 'monopoly_gacha'
)
SELECT asset_id, reason_id,
       SUM(ABS(change_count)) AS total_consume,
       COUNT(*) AS total_ops,
       COUNT(DISTINCT user_id) AS users
FROM iceberg.v1089.ods_user_asset
WHERE server_id = '1007402'
  AND partition_date BETWEEN '2026-04-29' AND '2026-05-06'
  AND change_type = '2'
  AND user_id IN (SELECT user_id FROM monopoly_users)
GROUP BY asset_id, reason_id
ORDER BY total_consume DESC
```

## 五·四、同期个人积分活动：消耗途径得分排名

> 数据来源：`ods_user_activity`，activity_type=`211200075`，attribute4=`21222097`
> 这是活动期间同步运行的**个人积分活动**（非大富翁），记录玩家消耗行为触发的积分变动。

### 分析方法

通过 LAG 窗口函数计算每次 activity_score 变动量，按 reason_id 汇总实际积分贡献，反映各消耗行为在积分活动中的实际权重。

### SQL

```sql
WITH monopoly_users AS (
  SELECT DISTINCT user_id FROM iceberg.v1089.ods_user_asset
  WHERE server_id='1007402'
    AND partition_date BETWEEN '2026-04-29' AND '2026-05-06'
    AND reason_id='monopoly_gacha'
),
scored AS (
  SELECT user_id, reason_id, activity_score,
         LAG(activity_score,1,0) OVER (
           PARTITION BY user_id ORDER BY created_at
         ) AS prev_score
  FROM iceberg.v1089.ods_user_activity
  WHERE activity_type='211200075'
    AND attribute4='21222097'
    AND user_id IN (SELECT user_id FROM monopoly_users)
)
SELECT reason_id,
       COUNT(*) AS events,
       COUNT(DISTINCT user_id) AS users,
       SUM(activity_score - prev_score) AS total_score
FROM scored
WHERE activity_score > prev_score
GROUP BY reason_id
ORDER BY total_score DESC
```

## 八、综合结论

### 8.1 大富翁为什么不行？

| 维度 | 玩家喜欢的行为（PVE/任务/推币机） | 大富翁 |
|------|-------------------------------|--------|
| 进入门槛 | 低（箱子是副产品，英雄闲着就能派，推币机免费投） | 高（骰子稀缺） |
| 确定性 | 高（做了就有东西拿） | 低（随机走格） |
| 反馈速度 | 即时~中等 | 慢（需要骰子，走完才有奖励） |
| 正反馈循环 | 有（产出→提升战力→做更多） | 无（奖励与战力弱关联） |
| 操作频次 | 高（50~200次/天） | 低（~5次/天） |

### 8.2 对标：推币机 vs 大富翁

| 维度 | 推币机 | 大富翁 |
|------|--------|--------|
| 参与率 | 62%（29/47人） | 100%（但被动） |
| 人均频次 | 69次/天 | ~5次/天 |
| 反馈速度 | 即时（投币→看结果） | 慢（掷骰→走格→等奖励） |
| 操作成本 | 低（点一下投币） | 高（需要骰子，骰子有限） |
| 视觉刺激 | 强（物理模拟，硬币掉落） | 弱（走格子） |

### 8.3 优化方向

1. **降低门槛**：骰子免费化，或绑定日常行为自动产出骰子（打野怪50次送1个骰子）
2. **加速反馈**：走一格就有即时奖励弹出，不要等走完一圈
3. **奖励对接核心需求**：格子上放加速道具（玩家最渴望的）、光谱芯片、机甲经验（高级关卡才掉的稀缺资源）
4. **增加"差一点"感**：显示下一格是什么大奖，制造"再来一次就中了"的心理
5. **嵌入主线循环**：完成加速任务N次 → 获得骰子 → 大富翁产出加速道具 → 继续做任务，形成闭环
6. **无限次机制**：参考推币机，让大富翁变成可以反复玩的"填充时间"行为
