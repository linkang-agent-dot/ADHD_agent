模块 1 · gacha（弹珠机）26拓荒节弹珠

原页签：gacha(26拓荒节弹珠)

一句话玩法：付费弹珠机。$30 买 50 颗弹珠打普通关（掉普通券），累计打满 50 颗触发一次福利关（专属弹珠，掉高级券），高级券进高级商店换皮肤 / 养成道具。福利关是这版的新增设计。
普通弹珠×50($30) → 触发福利关卡 → 获得50福利弹珠 → 打出高级兑换券 → 高级商店兑换

每一环为什么这么设计（原案原文）

模块设计目的
普通弹珠 50 个触发门槛，$30 一次让超R有明确目标感，中低频率保持稀缺性
触发福利关卡累计触发而非随机，给玩家确定性预期
获得 50 福利弹珠仪式感设计，专属弹珠+视觉效果，让玩家感知特别关卡
打出高级兑换券价值载体，通过小件兑换建立价值锚点
高级商店兑换核心奖励出口，三条路径：皮肤（情感）、养成（刚需）、普通币（兜底）

模块 2 · BP 集结奖励（基金）Battle Pass（拓荒节）-2026

原页签：Battle Pass（拓荒节）-2026

一句话玩法：三档通行证（免费 / $6.99 / $19.99）+ 40 级奖励轨道，攒「纪念钻头」升级。外挂一个全服集结奖励：全服购买通行证的人数每到一档（100/200/…/1000 人），全服发一波奖——「集结奖励 / 基金」这名字就是指这个全服解锁玩法。
买礼包/做任务得纪念钻头($0.1/个) → BP升级40级·三档领奖 → 满级进循环宝箱 ∥ 全服购买人数达标 → 全服发奖

模块 3 · 掉落转付费通用模板（拓荒用的就是这份）

原页签：掉落转付费-通用（没有拓荒专属页签，逐节日换投放道具复用这份）

一句话玩法：把日常活跃（打野/采集）变成付费入口的一条链。你工作清单里的三件套对应关系：掉落活动 = 打野掉十字镐；主活动 = 捐献攒积分 + 阶段宝箱 + 兑换商店；礼包 = 卖金色十字镐和付费代币。
pve/gve/采集 → 掉探宝十字镐 → 捐献攒积分(+付费金镐加速) → 阶段宝箱 → 双币兑换商店 ← 礼包(卖金镐+付费币)

③ 礼包

卖金色十字镐（$0.30/个）和纯净水晶（付费币）。明细挂在另一张落地表（本次没访问到）；X3 版礼包定价由林康重出，程序只需要知道：礼包卖的是"加速积分的付费道具 + 付费币"这两样。

每月换皮点（原案修改记录）：兑换商店里的「行军表情」和「gacha 券」逐节日更换成当期道具——搬 X3 时同理，这两个坑位换成 X3 拓荒节的当期投放。
给程序的要点
  • 三个独立子系统拼成一条链：分层掉落（按主城等级三档 + 每日上限）、捐献计分+阶段宝箱、双代币商店——搬的时候按子系统逐个对 X3 载体
  • X3 现成载体初判：掉落、兑换商店往期节日都有；「捐献攒分开箱」贴近 X3 积分活动（ActvScore）——三块可能都不用新写，重点是串起来
  • 所有宝箱/商店 JSON 策划案直接给了格式，落配置照抄结构、换 X3 道具 ID
  • 掉落是流量侧（ROI 4~5 倍故意大方）；宝箱概率和商店定价是变现侧——前者可调，后者等林康的数值表
