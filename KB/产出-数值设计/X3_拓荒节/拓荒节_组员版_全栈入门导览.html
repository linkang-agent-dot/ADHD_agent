② 三条规矩 // 记住这三条就够

1. 开工前找我拿「目的卡」，别猜。

一页纸写清：这模块干嘛的、哪些数值能动哪些不能动、搬过来要改什么、上线看什么数据。没拿到就问我要。

2. 我的代码 MR 必须你审，该打回就打回。

我的代码是 AI 写的，能跑但可能不合理——这块你是守门员，别客气。

3. 验到道具真实进背包才算做完。

配置写对了、动画播了、日志有了，都不算。历史上三次线上补发事故全是栽在这：宝箱开了东西没进包、换皮漏了兑换商店、首次登录/长离线没测。

③ 就一个概念要懂 // 其他的边做边学

挣钱的数值不许动，拉人的数值随便调。
高档礼包、gacha 奖池、BP 付费档 = 挣钱的，数值我出你照抄，别顺手优化；免费掉落、小额入门包 = 拉人的，可以改。类比：前者是对外 API 签名，后者是内部实现。
另外「搬 ≠ 照抄」：X2 把付费点拆成前7天+后7天，结果两头都没做起来——每个模块要改什么，看目的卡就行。
