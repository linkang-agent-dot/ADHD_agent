📌 第一步：怎么把活动弄到游戏里（不上出来啥都测不了）
活动不会自己出现，得先「上」出来才能在游戏里看到。两种上法，看你连的是哪个服 —— 非竞猜模块用 A 最快；竞猜只能走 B。
A · 本地测试服，自己用 GM 开（最快）
前提：游戏连的是本地服 + 知道一个本服在线玩家的 uid（不知道就问开发，当前可用 27798，报「LoadPlayerFail」就换一个）。把下面命令逐条发给 GM，发完进游戏，活动就出现在世界杯入口里。
活动活动ID复制这条命令（uid 换成你的）
开箱101516!gm @27798 GMAddServerActivityByCfgId 101516 0
付费 BP102240!gm @27798 GMAddServerActivityByCfgId 102240 0
签到101403!gm @27798 GMAddServerActivityByCfgId 101403 0
累充100597!gm @27798 GMAddServerActivityByCfgId 100597 0
兑换101339!gm @27798 GMAddServerActivityByCfgId 101339 0
⚠️ 竞猜不能这么开 —— GM 开出来「没有对阵两队」（GM 命令带不了对阵参数），竞猜只能走下面 B。
B · 让运营在 iGame 后台部署（竞猜必走这条）
你自己没 iGame 权限的话，把「测哪些活动 + 哪个服 + 起止时间」发给运营，他在 iGame 后台按活动名搜出来填好部署即可。部署成功后你直接进游戏玩，不用再做别的。
竞猜场次要额外告诉运营「这场打哪两队」—— 在活动参数 customParam 里填两个礼包号，格式：
customParam：{"packIdA":894071, "packIdB":894211}
这例是 巴西 vs 德国（$4.99 档）。换对阵 = 改这两个号重新部署，不用动配置表、不用热更。
📋 customParam 参数范围（填错活动建不出来）
• packIdA / packIdB 取值：894010 ～ 894483 之间的礼包号，两个必须是不同的队。
• 礼包号公式：894000 + 队号×10 + 档位。队号=国家英文名字母序 01–48（=助威头像框道具号 −80299，全表见下方外显速查）；档位 0=免费 / 1=$4.99 / 2=$9.99 / 3=$19.99。
  例：德国队号21、$4.99 → 894000+210+1 = 894211。
• 一个活动只出两个包（两队各一、同一个档位）。想让同一场比赛上多个价格档 = 部署该场的多个实例（每个实例填同一对队、不同档位的号），不是在一个实例里填 4 个号。

⚽ 01 每日竞猜 type64
竞猜界面能打开，显示对阵的两支队
进竞猜界面，背景是体育场，左右两支队的队名、队徽、横幅都显示正确、清晰，不空白/问号。
一场比赛=一个活动实例；两队信息靠后台填的两个礼包号决定，队名/队徽/横幅由礼包配置带出来在界面渲染。队徽=足球+国旗盾合体勋章，48 队定稿。
只能押一支队，押了就锁、不能改 重点
买了 A 队的礼包后，B 队那边就置灰锁住，不能再买、不能改押注（防止两边都买稳赢）。
买竞猜礼包必得抽奖券
买任意一档礼包（免费 / $4.99 / $9.99 / $19.99），都立刻发对应数量的世界杯抽奖券；猜对的额外奖励由运营事后补发。
免费场能领一次
每个比赛日有 1 场免费场，花 1 钻购买领取，押注后记录所选结果。
标题、规则、文案各语言都正常 查
活动标题、描述、规则弹窗、加送提示，中文和其它语言都正常，没有空白或乱码 key。
