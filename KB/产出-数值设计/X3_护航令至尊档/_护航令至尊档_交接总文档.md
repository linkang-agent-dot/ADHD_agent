# 护航令·至尊档（简易战令 3 档化）— 接管总文档

> **这是本模块的唯一入口。** 接手人从「修 BUG / 接管模块 / 切图换皮」任一入口进来，先读本文 → 再点对应产物。
> 案子类型：系统改造案（单模块·规则深化）。状态：策划案草稿 v0.1 + 真素材原型已出，**未建正式 GSheet、未开发、未切图**。

---

## 0. 一句话定位

把 X3 **简易战令 `ActvType=11`** 从「免费/付费两档硬编码」改成 **数据驱动可配 N 档**（抄积分战令 `BattlePassScore` 的位标志引擎 `Free=1 / Advance=2 / Super=4`），第 3 档 = 至尊轨。
**配 1 包=2 档（老 BP 行为不变），配 2 包=3 档，未来加列即 N 档。** 一劳永逸，老 BP 零回归。

价值模型最简：至尊档只是「更高价的第 3 条奖励轨」，superset（买终极=进阶+终极两轨都解锁），**不送等级、不双倍免费奖励、不要钻石跳级**。

## 3. 决策记录（为什么这么定 + 被否决的方案）

| 决策 | 拍板结论 | 为什么 / 否决了什么 |
|---|---|---|
| 价值模型 | **仅「更高价的第 3 条奖励轨」** | 最简价值模型，对大盘付费率/留存零冲击。否决了「至尊档送等级/双倍免费奖励」 |
| 改造范围 | 简易战令 `ActvType=11` **改成数据驱动可配 N 档** | 一劳永逸、老 BP 零影响。否决了「为至尊档单独做一套活动类型」 |
| 解锁方式 | 纯靠买至尊礼包解锁（superset） | 否决了 **「钻石升级」**（积分 BP `BattlePassScore.UpgradePrice` 那套花钻跳级机制）—— 不引入花钻跳级，避免与现付费结构冲突 |
| 命名 | 免费 / 进阶 / 终极 | 现界面列头「至尊」改名「进阶」，新增顶档叫「终极」。每个 BP 实例主题名仍按活动自身，不强加统一品牌名 |
| 价格 | 终极 ≈ 2× 现价（$19.99 / $24.99） | 现付费天花板 $14.99 太低，抬 ARPPU 上限 |
| 落点 | 新建独立 GSheet | 按 X3 策划案模板誊入 |
| 兼容铁律 | 老 BP 不配至尊包 + 不填 SuperReward 列 = 维持 2 档 | 零回归硬约束 |

## 5. 关键背景（摸底结论，接手人必知）

X3 有**两套** BattlePass：
- **简易战令 `ActvType=11`** ← 本案改造对象。配置 `ActvBattlePass`+`BattlePassReward`（只有 `FreeReward`/`PackReward` 两列）；客户端 prefab 写死 2 轨；服务端 2 分支硬编码。
- **积分战令 `ActvType=22/33`**（最佳酒馆/集结令/赛季 BP）：配置 `ActvBattlePassScore` 已有 `FreeReward`/`AdvanceReward`/`SuperReward` 三列；服务端已是数据驱动位标志 `Free=1/Advance=2/Super=4`。**已是 3 档，本案不动 —— 它就是改造的现成范式**。

**核心打法 = 把简易战令改成积分战令那套位标志引擎。** 客户端抄世界盃通行證三列形态，服务端抄 `BattlePassScore.cs`。
