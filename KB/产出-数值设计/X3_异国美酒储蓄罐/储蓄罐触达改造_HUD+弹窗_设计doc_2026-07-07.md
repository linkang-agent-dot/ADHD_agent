# 异国美酒储蓄罐 · 触达改造设计（HUD 挂件 + 登录弹窗 + BI 打点）

> 2026-07-07 立项，超哥需求（触达两缺口：没弹窗、没专属按钮）。设计已过用户（方案 B）。
> 分支 `feature/x3-piggybank-hud`（x3-project，dev 基底）。纯客户端，服务端/配置表零改动。

## 需求原文
1. 新增 HUD 挂在异国美酒界面下方：储蓄罐图标 + 时间（储蓄罐均进入 CD 时显示 CD 时间），且增加日志
2. 点击 HUD 进入获取途径界面
3. 弹窗触发：玩家进入该界面后，有可买储蓄罐礼包且无异国美酒道具 → 弹获取途径（每登录一次）

## 需求澄清结论（用户已拍板）
| 问题 | 拍板 |
|---|---|
| HUD 位置 | 限定英雄抽卡界面（UIHeroLottery）内部下方，非主城 |
| 非 CD 态显示 | 红点（每日刷新），不显示文字 |
| 日志 | BI 数据打点（非客户端诊断日志） |
| 实现形态 | 方案 B：独立 UIPart 挂件（可复用），非写死在界面类 |

## 设计

### 1. HUD 挂件 `UIPiggyBankHudEntry`（UIPart）
- **挂载**：`UIHeroLottery.prefab` 下方角落加节点组（图标 + 红点 + 倒计时 TFWText），`WidgetContainer.AddSingle<UIPiggyBankHudEntry>` 挂载（UIBtnPurchase 同款模式）。
- **可见条件**：当前池 `gachaCfg.CostType == GiftConst.DAILY_RESET_PIGGY_RES_ID(7002)` && 储蓄罐礼包 `CheckCanShowGift` 可见（等级≥3）。否则整体隐藏。
- **状态双态**（UTC 自然日口径，同 `UIPiggyBankContent.IsDailyResetPiggy` / 服务端 `isExoticWineDailyPiggy`）：
  - 有档今日未砸（可买）→ 图标 + 红点。红点=当日未点击过 HUD 才亮，点击即熄、次日重亮（按角色隔离本地存）。
  - 两档今日均砸 → 图标 + 倒计时至明日 UTC 0 点（`UIHelper.StartTiming`）；停留跨 0 点走完自动切回红点态。
- **点击** → `UIHelper.ShowItemObtain(7002, 1)`（获取途径界面，内含储蓄罐卡可直接买）。

### 2. 登录弹窗
- 触发点：UIHeroLottery OnShown（挂件刷新时检查）。
- 条件：美酒池界面 && 有档可买 && `StorageMeta.GetItemNum(7002) == 0` && 本次登录未弹过。
- 动作：自动 `ShowItemObtain(7002, 1)`。
- "每登录一次" = 静态会话标记，重登/切角色重置（范式参考 `UIHelper.TryPushRedPackLoginPopup`，但比它简单，无每日次数）。

### 3. BI 打点（`TgsHelper.Instance`，字符串 ID 进 BIConst）
| 事件 | API | 参数 |
|---|---|---|
| HUD 曝光 | `TraceApplicationLog("piggybank_hud_show")` | state=buyable/cd |
| HUD 点击 | `TraceUserClick("UIHeroLottery.PiggyBankHud")` | — |
| 弹窗触发 | `TraceApplicationLog("piggybank_obtain_popup")` | — |

### 4. 改动清单
| 文件 | 改动 |
|---|---|
| `UI/Gacha/UIPiggyBankHudEntry.cs`（新） | UIPart：状态机/红点/倒计时/点击/打点/弹窗检查 |
| `UIHeroLottery.prefab` | 下方加节点组（图标+红点+倒计时） |
| `UIHeroLottery.cs` | OnLoad 挂载挂件 + 切池时刷新（~2 行） |
| `BIConst` | 3 个事件常量 |
| 美术 | 储蓄罐小图标 1 张（124×136 透明底参考活动入口规格）；临时先裁获取途径卡现图占位 |

### 5. 边界
- 非美酒消耗的抽卡池不显示 HUD、不触发弹窗。
- 弹窗与红点互不依赖（弹过窗红点仍按点击规则亮灭）。
- 玩家等级 < 3（礼包不可见）→ HUD 隐藏、弹窗不触发。
