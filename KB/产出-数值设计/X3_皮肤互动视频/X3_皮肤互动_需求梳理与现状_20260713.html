二、产品设计构想 ★继续聊的核心

2.1 商业结构（已聊定的三条）

  • 本体皮肤互动 = 随皮肤解锁：作为皮肤高级卖点，抬皮肤价值，驱动首购
  • 换装本质就是皮肤：不同服装 = 不同皮肤 SKU，各自带互动，驱动复购（不做成皮肤之外的独立商品）
  • 互动分基础 / 高级两档（见 2.2）

2.2 两档皮肤体系（周末想清楚的核心设计）

基础皮肤（节日标配）高级皮肤（溢价款）
互动1 个特殊互动，不换场景多场景，每场景互动形式不同（更衣室 / 球场…）
未拥有时直接可玩（作为标配卖点直接看到）高级互动位只显示静态图预览
解锁后——视频 + 互动全开
解锁方式随皮肤解锁（购买）皮肤
作用让「互动」成为所有节日皮肤标配，抬整体皮肤价值高价溢价款，鲸鱼向复购

2.3 可见性 / 解锁逻辑

基础互动 → 直接看到能玩（节日皮肤标配）
高级互动 → 未解锁只能看到静态图（预览勾引）→ 想要视频&互动 → 解锁皮肤才能看

2.4 为什么这套设计跟我们技术优势完美咬合（省成本关键）

  1. 静态预览图 = 白捡的：高级互动未解锁显示的静态图，就是 AI 出视频时的首帧场景图（更衣室那张我们已经有了）。预览图零额外成本，出视频顺带就有。
  2. 换场景 = 换背景，AI 几乎白送：玩家感知里「更衣室→球场」是实打实高级内容，对我们只是抠人物换背景重生成，同一套热区/对白结构照搬。玩家觉得贵，我们做着便宜——正是溢价款该有的成本结构。
  3. 代码扩展小：现在路径 SM_{皮肤}_{slot} → 扩成 SM_{皮肤}_{场景}_{slot} 就支持多场景，改动很小。

2.5 映射到现有 demo

我们做的足球宝贝·更衣室这一整套= 高级皮肤的「一个场景」样本。
→ 再加个「球场场景」(换背景 + 一套 idle/act) = 完整高级皮肤；
→ 砍成单场景 = 基础皮肤。

2.6 ★简化：一场景 = 一互动 = 一视频（2026-07-13 定）

三部位×低高档是过度设计——玩家点头/手/身感知不到差异，反把"点一下有惊喜"的爽感稀释，还让产能翻 3 倍。改为 一个场景 = 一个互动 = 一段完整撩人演出。
每场景仅 2 支核心作用
idle 待机循环角色"活"着看着你，进界面就播（钩子）
act 互动视频点角色 → 播一段完整撩人演出 → 回 idle

命名规范（清晰）：SM_{皮肤ID}_{场景}_{idle|act|preview}，场景用英文语义词。preview.png=act首帧(未解锁显示的静态图，白捡)。

例 SM_104001_lockerroom_idle.mp4 / _lockerroom_act.mp4 / _lockerroom_preview.png · SM_104001_field_idle.mp4 …
reject 砍掉（静态图预览接管转化）。现有 9 支归置：留 idle + 挑最撩的 touch_body_high 当 act，其余 7 支退役 → 更衣室即成简化版完整场景，不用重做。

2.7 圆形场景选择器（设计中）

互动界面视频下方一排圆形按钮（原英雄详情底部 tab 位思路），圆内 = 场景 preview 裁圆： 
   ( ● )  ( ○ )  ( 🔒 )
   更衣室  球场   ???
默认选中皮肤默认场景(高亮描边)→进界面播它 idle；点其他圆切场景；未解锁场景显 🔒 → 点它弹静态大图 + 解锁引导（不播视频）。静态图预览机制正好挂这里。

四、待决策清单 ★继续聊要定的

#待定项影响我的初步建议
1高级皮肤起步几个场景？产能 vs 高级感2-3 个（太多产能跟不上，太少不够"高级"）
2基础 vs 高级的体验落差多大？基础互动做多深？撑不撑得起差价基础=进场+1~2个触摸；高级=多场景×完整三部位。落差要肉眼可见
3静态图预览要不要加锁标+"解锁皮肤查看完整"引导？转化力要，锁标+价格+跳皮肤礼包
4静态图预览 vs 试看3s，还是都要？钩子强度点静态图→播3s→卡付费（比纯静态更勾）；静态图做常态展示，3s做点击后的进阶钩子
5基础互动要不要回补到已有的节日皮肤？工作量只从新皮肤起，老皮肤择优回补（挑卖得好的）
6接不接好感体系？要不要动服务端首版纯客户端(拥有=全解锁)，二期再接好感做递进解锁
7正式化深度 & 提交方式上线配置表化+多语言必做；单独 feature 分支+X3NEW 单，不混马戏节
