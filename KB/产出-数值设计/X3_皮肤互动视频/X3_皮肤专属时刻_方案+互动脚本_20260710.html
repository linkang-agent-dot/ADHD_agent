X3 皮肤专属时刻（Skin Moment）· 功能方案 + 互动脚本 v0.1

2026-07-10 · 骨架 = 女仆俱乐部(HeroClub) · 首发实例 = 足球宝贝·爱莉希雅(104001) · 状态：方案稿待过审

一、一句话定位

给视频化皮肤加一层「可互动的专属剧情视频」：HUD 入口点进去 = 一段不透明满幅场景视频，玩家点角色不同部位触发对话 + 反应视频 + 音效。
商业钩子：未拥有皮肤 → 只能看「挑逗版拒绝反应」，引导购买；拥有 → 全解锁。皮肤从"立绘换装"升级成"有情感回报的内容"，直接抬皮肤付费价值。

二、玩家体验流程

  1. 主城 HUD 出现皮肤专属时刻入口图标（带红点）
  2. 点入 → 满屏场景视频：greet 进场问候（播一次）→ idle 待机（循环），有环境音/呼吸/角色偶尔看镜头
  3. 点击角色 头 / 手 / 身体 三个热区 → 触发对应反应视频 + 对白气泡 + 音效，播完回 idle 循环
  4. 按「拥有皮肤与否（或好感档位）」分 低档/高档/拒绝 三层反应，拒绝层 = 转化话术

三处改动（delta）

#改动关键点
1不透明视频（vs HeroClub 的透明SBS）文件名不带 _sbs 后缀（UIVideoPlayer.cs:238 按 EndsWith("sbs.mp4") 判透明）；m_IsTransparentVideo=false（HeroClub 的 DoPlayEmbedVideo 里写死 true，新代码这一个参数改掉）。视频 = 满幅 1080×1920 带场景，格式锚 egypt_sphinx.mp4（h264/yuv420p/24fps）。好处：省掉整条透明化流水（rembg+SBS导出），AI 直出即可用，场景叙事感更强。
2HUD 入口（vs HeroClub 藏在 UIRole 二级）推荐挂活动 HUD 走 ActvOnline 配置驱动（可控上下线/分服/day 锁，运营灵活）；界面本体 = 新窗口 UIHeroSkinMoment，标准 4 件套（业务类/Auto绑定/prefab/路由），prefab 结构照抄 UIHeroClub.prefab 的 VideoRoot+Dialogue+Mask 三节点。备选：常驻入口放皮肤预览界面(UIHeroSkinPreview)加"互动"按钮——两个入口不冲突可都做。
3解锁维度 = 皮肤拥有（vs HeroClub 的好感等级）档位判定改成：未拥有皮肤→reject 层；拥有→high 层（或拥有+好感两轴都用，字段结构本来就支持）。reject 层是转化面，必须做得最勾人。
坐标映射注意：视频满幅 + RawImage ScaleToFit，遮罩图必须按视频同分辨率 1080×1920 制作，热区颜色区跟视频里角色实际站位对齐；角色在反应视频里位移不大（锁镜头）才能保证热区不漂——生产时 prompt 锁镜头锁站位（已有成熟措辞，见皮肤视频生产 KB §六）。

五、互动脚本 · 首发实例「足球宝贝·爱莉希雅 — 赛后加时」

场景设定：黄昏的球员通道/更衣室门口，比赛刚赢，欢呼声从远处传来。她抱着球等你——这段时间只属于指挥官。
人设基调：自信带撩、主动权在她（沿用皮肤视频已验证的"高冷带撩"人设，撩点=征服欲不是谄媚）。
尺度标尺：全程双关+留白，无露骨词汇，以已上线的女仆俱乐部为红线基准。足球术语天然适合做双关（加时赛/贴身盯防/犯规/点球），性暗示全部走隐喻。

音效层设计（三层）

  1. 视频内嵌音轨（主）：环境音+动作foley+配音全烧进 mp4，沿用 HeroClub 做法，m_PauseBgm=false，视频音轨自带 BGM ducking 空间。seedance 默认出音频这里反而是优点；配音走 GRFal TTS 多角色配音后 ffmpeg 混轨。
  2. UI 点击反馈音：点中热区瞬间播一个轻 UI 音（心跳"咚"半声），盖住视频切换的 0.2s 加载感。
  3. 高档触摸的沉浸增强（可选P1）：高档反应视频播放时 UI 层加心跳震动（手机 vibrate 短脉冲）+ 屏幕边缘粉色 vignette 渐入渐出。

六、视频生产链（全复用已有 pipeline，还更简单）

  1. 主稿：GRFal 出「更衣室场景+爱莉希雅站位」场景图（不透明，带背景，比透明皮肤视频省掉白底约束）
  2. 生成：seedance 首帧驱动（模型选型结论：2D立绘脸 seedance 稳、kling 嘴必崩），锁镜头锁站位措辞照抄皮肤视频 prompt 库；idle 用 fflf 收循环
  3. 跳过 透明化三步（rembg/SBS导出全免）——不透明直出
  4. 配音：GRFal TTS → ffmpeg 混进音轨（本地 ffmpeg 只做混轨/几何，最终压缩交 GRFal video_compression）
  5. 包体账：8 支视频 ×3-5s 不透明 h264 ≈ 每支 0.5-1.5MB，全皮肤 <10MB，且走 OnDemand 不占首包

七、风险与红线

风险对策
尺度过审（iOS/各地区）①对白/画面以女仆俱乐部已上线尺度为上限 ②复用现成的和谐视频闸门机制（ResolveHarmonyVideo 双闸：版本+创角时间），iOS/新号可整体屏蔽入口或换和谐版视频——机制现成，成本≈0
热区漂移（角色动出遮罩区）反应视频锁镜头+位移≤10%；遮罩色块画大一圈容错
AI 视频脸崩/嘴崩seedance 已验证嘴稳；对白配音不追求口型精确对齐（气泡承担语义，口型只要"在说话"）
范围蔓延P0 只做：HUD入口+1个皮肤×8支视频+点击对话，不做好感/奖励/服务端；好感轴字段预留、以后接 HeroClub 好感体系
