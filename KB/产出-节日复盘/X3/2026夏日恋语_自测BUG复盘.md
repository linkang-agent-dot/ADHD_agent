## 经验教训

### 1. 合并后审计是刚需

dev-summer-love-song 合 dev 时**至少 2 处数值字段被错误覆盖**（1701 / 1802），公式缓存丢失 1 处（ActvScoreMulti R21-R26）。这次靠 user 肉眼看才发现 1802。**应该程序化扫描合并前后差异**。审计模板见 `workflow_x3_merge_conflict_audit.md`。

### 2. 多层渲染链路不能只看一个字段

弹窗背景排查时被 MainBg 误导了一次（其实 MainBg 是个透明 PNG），最终是 BottomBg 紫色框。下次类似 BUG 要**一次性看完 Pack 表所有 5-7 个图字段** + 跟历史同类礼包对比（拜访礼包模板字段都是空）。

### 3. 公式缓存丢失 vs 数据丢失要分清

`xlrd` 读公式列只拿缓存值，缓存丢 → 拿到 None → 看着像数据丢。修法是 Excel COM `CalculateFull` + `Save`，不是改数据。这条误判过 2 次（717 和 read_only 漏读 ActvTask 594）。

### 4. 配置层与代码层职责区分

- `client/.../Path_Activity.asset` = DK 注册（客户端代码层）— 改了要 push x3-project + MR
- `gdconfig/data/*.xlsx` = 配置层 — 改了 push gdconfig + jolt
- 缺一环都不生效。这次 DK_img_Activity_summer_bg_12 走完整链路用了 4 个仓库操作

### 5. AI 出图迭代要带 reference

第一版 GPT 出图没注入 reference，结果太"实"。注入官方柔情海湾岛屿素材作 reference 后，第二版完美贴合现有视觉风格 + 水彩渐隐质感。
