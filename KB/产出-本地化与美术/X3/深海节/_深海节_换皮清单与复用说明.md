## 三、复用资源到底有哪些（消除"复用一脸懵逼"）

**一个投放物的复用资源 ≠ 只有一张 icon**，通常有三类：
1. **icon**（UI 预览图）：`Assets/Res/UI/Spirits/...`（家具在 `Furniture/Actv/icon_*.png`）→ 最常用，当 subject 参考。
2. **贴图**（产品表面，比 icon 更全）：`Assets/Res/Furniture/Model/.../Texter/*.png` 或 `Prefabs/.../Textures/*.png` → 要画准木纹/花色时喂这个。
3. **3D 模型**（fbx/prefab/mat）：`Furniture/Model/...` + `Furniture/Prefabs/...` → **美术做终稿可直接在 Unity 摆 prefab 渲染截图，比 AI 准；⚠️拷出 Unity 会断引用，留原位**。

## 四、换皮三铁律（方法 · 踩过坑才总结的）

1. **双参考**：每张换皮图喂两张——`老图`=夏日/尼罗对应界面截图（只取**构图框架/尺寸**，prompt 明确 ERASE 旧配色与 UI）+ `新图`=**实际投放物**。
2. **投放物必须忠实入画**（卖具体物件的模块）：把真实现成资源当**主体陈列**出来，**禁止只喂一张贴图让模型自由再创作**。
   - ❌ 教训：拜访礼包第一版只喂 Door 贴图 → 模型画成了不存在的海底金宫门；改喂拱门 icon+地板 icon + 强约束「showcase these EXACT items, do NOT redesign」后才对。
3. **礼包展示视频 = 动画化已批准的方向稿，别套别人的构图公式**（教训：我把夏日"拱门框景"硬套到装饰礼包→被指"背公式，装饰又没拱门"）：
   - **构图/取景跟该模块自己批准的 banner/方向稿走**。拱门框景**只适合卖"门头/建筑"的模块(07拜访)**——因为它的投放物本身就是拱门门头；卖家具/外显的模块(06装饰=遮阳椅)**直接 animate 批准的主题 banner 即可**，别强加拱门。
   - 尺寸跟所用方向稿的比例走（装饰用了 3:4 banner 就 3:4；client 尼罗/夏日视频是 9:16 仅供参考节奏/打磨，不是强制比例）。
   - 流程：用**已选定方向稿当首帧** → image-to-video(vidu, 缓推镜+海浪/光斑/气泡/植物微摇, seamless loop, 保持画面 identity 不变)。
   - 参考节奏/质感：client `Video/VideoRes/egypt_sphinx.mp4`尼罗 / `summer_love_song.mp4`夏日（5s/24fps）。

## 四之补 · ⚠️配置换皮必建 i18n key（为什么第一次换皮没做完 → 标题/名称全掉）

**现象**：深海换皮第一批配置搬完、导表也过，但游戏里 **6 个活动标题（深海罗盘/深海宝藏集市/海滨之约/许愿池/远航酒馆/周卡特惠）+ 礼包名 + 道具名 全显示空白**。世界杯 BP 换号 102240→102243 时也同样掉了标题。

**根因（换皮最易漏的一环）**：
- 客户端运行时 **不读配置表里填的中文**！活动标题/礼包名/道具名走 i18n key `TXT_{Table}_{Field}_{新id}`（如 `TXT_ActvOnline_ActvName_101025`），配置表 ActvOnline.ActvName / Pack.Name / Item.Name 里填的中文**只是编辑器母版/备注**，客户端统统不读（见 [[reference_x3_i18n_workflow]]）。
- 换皮 clone 活动/礼包/道具 = 新建了一批**新 id**。配置搬运只复制了配置表的行，**没在 i18n Text 表按新 id 建 TXT_key** → 新 id 的 key 在字典里不存在 → 标题/名称查不到 → **显示空（掉）**。
- ❗**复用活动也要建**：许愿池/酒馆即便复用模板玩法，新 id（105013/10071704）≠ 模板 id（105012/10071702），key 按新 id 建，**不会自动继承模板 id 的翻译**（只能抄模板译文到新 key）。
- 为什么第一次没做完：i18n 是**独立的 Text 表**，跟配置表分两套流程；换皮时注意力全在「配置搬对没/导表过没」，导表又**不校验 i18n key 存在**（#1114 绿但标题照掉）→ 配置层全绿给了"做完了"的错觉，i18n 这步被整个跳过。

**换皮配置铁律（补第 4 条）**：clone 活动/礼包/道具后，**必到 `tsv/i18n/Text__Text.tsv` 按新 id 建全部 TXT_key**——活动 `TXT_ActvOnline_ActvName/ActvDesc_{id}`、礼包 `TXT_Pack_Name/Desc_{id}`、道具 `TXT_Item_Name_{id}`、规则 `TXT_RuleTips_*_{id}`、铭牌/纪念卡/头像框等所有走 i18n 的字段，16 语言（复用同 cn 现成译文优先，专属新译走 [[reference_x3_i18n_workflow]]）。
**收尾验证（脚本扫，别靠肉眼）**：换皮 id 全列出 → 逐个查 `TXT_{Table}_{Field}_{新id}` 是否在 Text 表 → 缺的补。深海这次扫出缺 22 条（已补 commit 12941b0）。**导表绿 ≠ 标题不掉，i18n 审计是换皮独立一步。**
