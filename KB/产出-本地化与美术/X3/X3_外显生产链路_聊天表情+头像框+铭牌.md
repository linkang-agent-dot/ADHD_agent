# X3 外显生产链路：聊天表情 / 头像框 / 铭牌(头衔)

> 用途：准备「做一波聊天表情 + 铭牌 + 头像框」时，照此文档直接落地。
> 三者都是 **纯 2D PNG 外显**，是 x3-media/GRFal **能端到端产成品**的轻量模块（出 PNG → 注册 DK → 填配置表即进游戏，无需美术做 Spine/3D/特效）。
> 链路 = ①AI/美术出 PNG → ②Unity Ctrl+T 注册 DK（写 `Path_*.asset`）→ ③填对应 tsv 配置表 → ④（头像框还要 Item 包一层 / 铭牌挂活动发放）。

## 0. 三模块一览（先看这张表）

| 模块 | 配置表(tsv 真源) | 资源路径 | 实测尺寸 | 发放方式 | 是否带数值Buff |
|------|------------------|----------|----------|----------|----------------|
| **聊天表情** | `ChatEmojyReply__ChatEmojyReply.tsv` | `Res/UI/Spirits/Chat/img_cm_emoji_*.png` | **72×72** | 全员内置(无获取/不售卖) | ❌ |
| **头像框** | `Personalize__PersonalizeAvatarFrameCfg.tsv` + `Item__Item.tsv`(80xxx) | `Res/UI/Spirits/Personalise/AvatarFrame/Img_Player_AvatarFrame_*.png` | **256×256** | Item_80xxx 道具 / 自选礼包(Item 1080) → 活动奖励 | ✅(站位Buff+虚战力) |
| **铭牌/头衔** | `PlayerTitle__PlayerTitle.tsv` | title板 `Res/UI/Spirits/ActvKvk/`(及 Activity/) `*_icon_title.png`；小图标 `Res/UI/Spirits/ItemIcons/icon_*.png` | title板 **752×192**；小图标≈item icon | 活动配置直接挂(ActvKvk.PlayerTitleID 等) | ✅(站位Buff+品质0/1/2/3) |

### 1.2 ⭐ 可售卖聊天表情 Emoticons（节日通行证/礼包卖的就是这套）
聊天里发的**大表情气泡**，按英雄/节日主题出，**可卖可发**——festival 表情（圣诞/元旦/尼罗/情人节/春节…）都走这套。现有 22 个（ID 100-240）。

#### 主配置表 `Emoticons__Emoticons.tsv`
| 字段 | 说明 |
|------|------|
| ID | 表情ID（100-240，新增往后开新段，如世界杯用专属段） |
| Res | `DK_<名>` 资源DK（→ `Res/UI/Spirits/Emoticons/Icon/icon_global_<名>.png`） |
| Pack | 关联礼包ID（直接绑礼包发放，可空） |
| Name | TXT_名字（如 `[欢庆]`，走 i18n） |
| ShowType | `0/空`=一直显示 / `1`=获得后才显示（问答活动/付费投放的配1） |
| 备注 | 获取途径（活动-秘密问答 / 礼包投放 / 尚未投放） |

#### 资源 & 发放
⚠️**双DK结构（现有22个清一色动图GIF、零静态）**：
- **发到聊天的表情 = 动图 GIF**：`Res` 字段 = `DK_<名>` → `Res/UI/Gif/<名>.bytes`（+ `.gif` 源，~100-270KB）。这是会动的。
- **面板缩略图 = 静态 PNG**：`DK_icon_global_<名>` → `Res/UI/Spirits/Emoticons/Icon/icon_global_<名>.png`，**256×256 透明**。
- 气泡底图：`Res/UI/Spirits/Emoticons/ui_chat_memebg_1~9.png`（共用）
- DK 注册：`Path_Emoticons.asset`（两套 DK 都注册）
- → **做新表情要两份**：动图 GIF→.bytes（Res，会动）+ 静态 icon PNG（面板）。只出静态 = 这套里唯一不动的（纯静态表情只在快捷回复 §1.1 那套）。
- **解锁/售卖**：① **Item 154xx「表情包」道具（ItemType 24）** "使用后获得聊天表情"，道具 param=Emoticons.ID，道具icon=DK_icon_global_<名>、bg=DK_Bg_CM_Item4 → 这个道具进**通行证/竞猜礼包/兑换商店**奖励即"卖表情"；② 或 Emoticons.Pack 字段直接绑礼包
- **使用**：`UIChatEmojiPanel` 聊天表情面板发送；`SocialMeta.Emoji.cs` 管解锁；ShowType=1 的买了才在面板出现

#### 做一波可售卖表情的步骤
1. 出 256×256 透明 PNG（主题表情）→ `icon_global_<名>.png` 放 `Res/UI/Spirits/Emoticons/Icon/`
2. Ctrl+T 注册 DK `DK_<名>` → `Path_Emoticons.asset`
3. `Emoticons__Emoticons.tsv` 加行：ID（新段）/ Res / Pack(可空) / Name(TXT) / ShowType(卖的配1)
4. 配 Item 154xx 表情包道具（ItemType 24，param=Emoticons.ID）→ 挂进通行证/礼包奖励
5. Name 走 i18n + 导表

## 1.5 英雄羁绊贴纸 HeroStickers ⚠️（不是聊天表情，是英雄羁绊收集册）
**重要归类**：名字带 Sticker，实为**英雄 CP/羁绊剧情收集册**（"相册-羁绊"红点，关联 DateMemory 约会回忆系统），养成+外显，**不发到聊天、也不直接售卖**。主题立绘式贴纸（双角色+道具，如贝拉&阿米娜痛饮），**27 个**，512×512 独立 PNG。

### 资源
- 贴纸本体 `Res/UI/Spirits/HeroStickers/icon_bond_<n>.png`（512×512）+ `img_bond_bg_*`（6 底）+ `img_bond_icon_copper/silver/gold/diamond`（积分档位等级图）
- DK 注册：`Res/Config/DisplayKey/Path_Stickers.asset`（`DK_icon_bond_<n>`）
- UI：`UIHeroStickers.prefab`（全屏窗口，英雄/相册入口）/ `StickerBoard.prefab`（玩家摆贴纸的展示板）

### 配置（5 张表，真源 tsv，**改 xlsx 会被导表覆盖**）
| 表 | 作用 |
|----|------|
| `HeroStickers__HeroStickers.tsv` | 主表：ID/Name/Desc(剧情)/DK_Sticker/ShowSort/LightCondition→点亮条件组/LightRewardPoints点亮积分/LightRewardBuff→Buff/LightReward点亮道具/RelatedHeros关联英雄组/ShowGroup→显示条件组 |
| `HeroStickers__LightConditionGroup.tsv` | **点亮条件**：ConditionType 1=拥有英雄/2=拥有皮肤/3=英雄达指定星级；参数=英雄id/皮肤id/英雄id\|星级 |
| `HeroStickers__ShowConditionGroup.tsv` | **显示(投放)条件**：ShowType 0无条件/1=TimeCycle/2=玩家等级/3=前置贴纸/4=获得指定英雄/5=英雄星级/6=活动时间内/7=获得任一配置英雄 |
| `HeroStickers__LightPointsLevel.tsv` | **积分档位奖励**：25铜/50银/75金/100金/150钻，每档领道具(15900716) |
| `HeroStickers__StickerBuff.tsv` | **点亮Buff**：PropType+PropNum(万分比)，如猎人攻击+15%/射手防御+30% |

### 机制（服务端 `HeroStickerMeta.cs`）
- **投放**：跟英雄绑定的常驻系统，新英雄/CP 上线配套出贴纸；节日投放用 ShowConditionGroup 的 TimeCycle(1) 或活动时间内(6) 控制贴纸组出现。
- **获取=点亮**（非购买）：满足 LightCondition（拥有英雄/皮肤/英雄星级）自动点亮 `UnlockHeroSticker` → 给积分+永久Buff+道具。本质是**英雄/皮肤拥有度衍生奖励**，间接绑 gacha 付费。
- **积分档位**：累计点亮积分 → LightPointsLevel 档位领奖。
- **使用**：UIHeroStickers 收集册查看；点亮贴纸给**永久数值Buff**（INumericProvider）；玩家可把贴纸摆到 StickerBoard 板上展示。
- **做一波步骤**：①出 512×512 立绘式主题贴纸（多角色+道具+底环，配英雄CP剧情）→ `icon_bond_<新n>.png` ②`Path_Stickers.asset` 注册 DK（两段同序）③ 5 张 tsv 配齐（主表+点亮条件+显示条件+Buff，按英雄绑定）④导表

## 2. 头像框 (Avatar Frame)

### 是什么
玩家头像外圈的装饰框，**带站位 Buff + 虚战力**，按节日出皮（Egypt/Christmas/ValentinesDay/SpringFestival/anniversary 等命名）。当前 **29 个** PNG。

### ⚠️ 两表协同（关键）
头像框是 **「定义表 + 道具表」双层**：

**A. 定义表 `Personalize__PersonalizeAvatarFrameCfg.tsv`** — 定义这个框长啥样、给什么属性
| 字段 | 说明 |
|------|------|
| ID | 框编号（10000=初始默认，10001+ 起） |
| Order | 排序 |
| Unlock | 0=初始解锁 / 1=道具解锁 |
| ImgRes | `DK_Img_Player_AvatarFrame_<名>` 资源 DK |
| SourceDesc | TXT_来源描述 |
| PropType / PropNum | 属性类型 / 万分比数值（如 220000 全水手攻击 +200=2%） |
| Power | 虚战力（普遍 20000） |
| Name | TXT_名字（如"时间旅者"/"暗夜皇冠"） |
| FramePrefab | DK_头像框预设（动态框才填，静态可空） |

**B. 道具表 `Item__Item.tsv`（Item ID 80xxx）** — 把框包成可发放道具
- 例：`80000 时间旅者` Item 参数 `10001|-1`（= 定义表框ID|时长，-1=永久），引用 `DK_Img_Player_AvatarFrame_03`
- `80001 征服者（2天）` 参数 `10004|172800`（172800 秒=2 天限时框）
- `1080 头像框自选礼包` = 打开后多选一（自选礼包形态）

> 发放：活动/排行榜奖励发 **Item_80xxx**（或自选礼包 1080）→ 玩家获得 → 系统读 Item 参数解锁定义表对应框。

### 资源
- 路径：`Res/UI/Spirits/Personalise/AvatarFrame/Img_Player_AvatarFrame_<名>.png`，**256×256**，透明 PNG
- 节日命名示例：`_Egypt` / `_Christmas_1` / `_ValentinesDay` / `_SpringFestival` / `_MF`(人鱼) / `_anniversary` / `_DOTD` / `_AFD`
- 配套：头像本体 `Res/UI/Spirits/Personalise/Avatar/Img_Player_Avatar_*.png`（**176×176**），定义表 `PersonalizeAvatarCfg`

### 🎯 头像框标准尺寸限制（生成强制遵守 · 2026-06-14 定标，实测 30 个现有框）
**这是自产头像框的权威尺寸标准**，AI 生图 / 美术出图 / 后处理 resize 都按此对齐。量法：256×256 画布，沿水平中线左右扫 alpha。

| 参数 | **标准值** | 允许容差 | 占边长比例 | 约束 |
|------|:--:|:--:|:--:|:--:|
| 画布 | **256×256** 透明PNG·居中对称 | 固定 | — | 🔴硬 |
| 头像透空内径 | **Ø165 px** | 150–180 | 0.62–0.68 | 🔴硬（必须真透空） |
| **环带厚度(单边)** | **45 px** | 40–55 | 0.16–0.20 | 🟡软（按风格） |
| 环外径 | **Ø248 px** | 244–252 | 0.95–0.98 | 🟡软 |
| 装饰最大外扩 | **256 px 顶满即止** | 不出血 | 1.0 | 🔴硬 |

**三条硬规则：**
1. **透空圆 Ø150–180 必须真透明**（差分法验真），且**比头像本体(176×176)略小** → 环压住头像边缘 3~10px、遮硬边不留缝；**严禁 >180**（头像漏边）。
2. **环本体止于外径~248（留 2~4px 边）**；只有冠/翅膀/飘带等装饰才突破圆顶外扩，**最多到画布边 256、不得出血**。
3. **左右镜像对称**（顶部冠/角标可非对称，但环带本体对称）。

**环厚怎么定（软约束，40–55 内挑）**：简洁框取下限 ~40（浪涛之巅29/世界之战34）；华丽节日框取上限 50–64（圣诞63/酿酒52/mosheng64）；**拿不准就用标准值 45**。
**换尺寸照搬**：内径 = 0.62–0.68×边长，单边环厚 = 0.16–0.20×边长，环外径留 2~4px 边。
> 全量逐框测量明细 + 可视化画廊见同目录 `外显图库_表情头像框铭牌\`（`index.html`），复测脚本 `_measure_frame_geometry.py`。

### 做一波头像框的步骤
1. 出 256×256 透明 PNG（**只画框、中间透空**给头像漏出）→ `Img_Player_AvatarFrame_<节日名>.png`
2. Ctrl+T 注册 DK：`DK_Img_Player_AvatarFrame_<节日名>`
3. `PersonalizeAvatarFrameCfg`：加框定义行（ID/ImgRes/属性/Power/Name）
4. `Item__Item.tsv`：加 80xxx 道具行（Item 参数=框定义ID|时长，引用同一 DK）；如要自选礼包另配
5. 活动/排行榜奖励里发 Item_80xxx
6. i18n（Name/SourceDesc）+ 导表

## 3. 铭牌 / 头衔 (Player Title)

### 是什么
玩家名字下方挂的 **头衔铭牌**（一条 752×192 的横牌 + 一个小图标），**带站位 Buff + 品质(蓝/紫/橙)**，主要来自 **KvK / 世界征服类活动**奖励（"风暴领主""海皇""魔海之主"）。当前 3 个在用。

> 注意：X3 没有独立的"铭牌"个性化系统目录——它就是 **PlayerTitle（头衔）**。白皮书 8 大外显模块里其实没单列铭牌，铭牌=头衔的视觉呈现（752×192 title 板）。

### 配置表 `PlayerTitle__PlayerTitle.tsv`
| 字段 | 说明 |
|------|------|
| ID | 头衔编号（101/102/103…） |
| Name | TXT_名字 |
| DK_Icon | **铭牌横板** DK（如 `DK_kvk_icon_title` / `DK_nest_icon_title`），**752×192** |
| PositionBuff | 站位 Buff，格式 `属性ID,万分比|…`（如 `220000,1000|11020,5000`） |
| Reimburse | 重复获取钻石补偿 `单日补偿|最高补偿`（如 `100|1000`） |
| ObtainDesc | TXT_获取方式（"风暴逐鹿活动获取"） |
| DK_SmallIcon | 小图标 DK（如 `DK_icon_global_nesticon_1`，在 ItemIcons/） |
| Quality | 0=默认 / 1=蓝 / 2=紫 / 3=橙 |
| Order | 排序 |

## 4. 三者通用：DK 注册（必做，否则配置引用不到图）

所有 `DK_*` 名都要在客户端注册成 GUID 映射，否则配置表写了名也加载不到图。
- 注册表：`C:\x3-project\client\Assets\Res\Config\DisplayKey\Path_*.asset`（`Path_*` 按类分）
- **正路**：Unity 菜单 `Config / Display Key`（**Ctrl+T**）→ 加 DK 名 + 拖 sprite → 自动写 `keys`/`values` 两段（不会错位）
- 应急手改 yaml：`keys` 段与 `values` 段**两处同序追加**，运行期靠 `keys[i]==values[i].key` 平行对应
- ⚠️ **禁止**整文件全量重导出 Path_*.asset 提交（会静默删掉别人后加的 DK，跨节日资源批量空白）；只追加自己那几条，重导前先 `git pull`
- 客户端仓 protected dev → feature branch + MR；详见 memory `reference_x3_client_resources` / `workflow_x3_protected_branch_mr`

## 5. AI 生产可落地度（x3-media / GRFal）

三者都是**纯 2D 透明 PNG**，属于 GRFal **端到端能出成品**的模块（一个 worker 出 PNG + 抠图/超分到目标尺寸即可入库）：
- 聊天表情 72×72 / 头像框 256×256 / 铭牌横板 752×192 / 小图标 → `generate_image` 出图 → `remove_bg` 透明化 → `resize` 对齐尺寸 → 差分法验真透明（见 feedback `transparent_asset_diff_check`）
- 触发 x3-media skill 生产；画风务必跟现有同类资源对齐（拿现有 PNG 做参考锚）
