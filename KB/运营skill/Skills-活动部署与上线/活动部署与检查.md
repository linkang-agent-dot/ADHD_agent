# 活动部署与检查

## 一、活动部署（igame-actv）
**路径**：`~/.claude/skills/igame-actv/`
**触发**：`部署活动` `活动部署` `甘特图` `排期` `节日活动+上线/部署`

### 工作流
1. 从 Google Sheet 甘特图读取节日活动排期
2. 解析活动配置（schema/灰度/跨服分组/多期）
3. 通过 iGame API 批量部署
4. 部署前确认项检查

### 支持功能
- schema 分服部署
- 灰度服配置
- 跨服分组
- 多期活动

## 二、活动 ID 检查（activity-check）
**路径**：`~/.claude/skills/activity-check/`
**触发**：`活动检查` `ID检查` `引用检查` `检查QA表` `验证配置`

### 校验范围
验证活动配置 JSON 中各类 ID 是否在对应 Google Sheet QA 表中存在：
- `task` — 任务 ID
- `drop` — 掉落 ID
- `exchange` — 兑换 ID
- `rank` — 排名 ID
- `retake` — 回收 ID
- `jump_link` — 跳转链接
- `actv_show_rank` — 显示排名
- `drop_gacha` — 抽卡掉落
- `exchange_pkg` — 兑换礼包
- `preview_reward` — 预览奖励

### 工作流
1. 接收活动配置 JSON
2. 提取所有引用 ID
3. 逐类别查 QA 表验证
4. 输出检查报告
