# 换皮规则详解

> 源文件：`.cursor/config-library/reskin-rules.md`

---

## 一、列类型处理规则

| 列类型 | 换皮处理 |
|--------|---------|
| `base-ID` | **不换** |
| `A_STR_constant` | **必须新增**，不能复用 |
| `A_MAP_filter` | 一般不换 |
| 活动行 ID | 换 — 分配新 ID |
| 父活动 / 关联 ID | 换 |
| 前置条件建筑 ID | 换 |
| 本地化 LC key | 换（改为新主题前缀） |
| content 里的 task/package ID | 换 |
| Banner 图片路径 | 换 |
| 规则 LC key（`rule` 字段） | 同类型活动可不换 |
| 2119 关联 ID | 同类型活动可不换 |
| `A_INT_show_hud` | 换 |
| 常量数值 / 空值 / NULL | 不换 |

---

## 二、换皮完整追踪链（以 2112 为起点）

```
2112（主活动配置）
 ├── {"typ":"task","id":xxx}      → 2115（检查 condition 里的 2013 ID）
 ├── {"typ":"package","id":xxx}   → 2135（检查 2011 引用）
 │         └── col3 (2011 ID)    → 2011（检查 actv_id、recharge_actv）
 │                  └── condition (2013 ID) → 2013（确认道具配置）
 └── A_INT_show_hud               → 换新 ID
```

**规则：每层引用都要向下追踪，不能只看当前行。**

## 三、预购连锁礼包特殊规则

| 字段 | 普通礼包 | 预购连锁礼包 |
|------|---------|------------|
| 2135 head row col2 | 2011 ID | 2011 ID（相同） |
| 2013 模板行 | 需要 | **不需要** |
| 2011 `A_MAP_time_info` | `actv_id` 指向 2112 | 同上，换皮必须同步 |
| 2135 chain row col11 | 模板 ID | 1111 道具的 `C_INT_display_key` |

**col11 获取**：gws 读 1111 表 `item` sheet，列 G 为 `C_INT_display_key`，按道具 ID 匹配。

---

## 四、已知注意事项

- `A_STR_constant`（2011 ID）**必须新增**，不能沿用旧节日
- 2115 的 condition 直接引用 2013 ID → 2013 换了 2115 也要同步
- 累充条件 `recharge_actv` 在 2011 表 → 换皮需单独确认
- **复用 2112 ID 时**：2011 的 `actv_id` 和 2135 的 `condition.actvstart.id` 必须一起改
- 预购连锁礼包 2011 行有大量 `recharge_actv` ID → 换皮时保留

---

## 五、配置行 GSheet 输出格式

所有单独成格的 `""` → 替换为 `'""`
- `\t""\t` 或 `\t""` 行尾 → 替换
- JSON 内部引号 → 保持

## 六、奖励道具图标 BUG 排查

### 排查链路
```
2112 activity_config
 └── A_ARR_activity_components → {"typ":"task","id":xxx}
       ↓
 2115 activity_task
       └── A_ARR_reward → [{"asset":{"typ":"item","id":道具ID,"val":数量}}]
             ↓
 1111 x2_item（按道具 ID 查 display_key → 图标是否正确）
```

### 常见 BUG：9 位 vs 8 位 ID
| 写法 | item 表 | 结果 |
|------|---------|------|
| `11110026`（8位） | 不存在 | 错误图标 |
| `111110026`（9位） | 存在（金羽） | 正常 |

## 七、装饰物换皮（7 张表联动）

### 涉及表

| 表号 | 作用 |
|------|------|
| 1111 | 道具定义 |
| 1118 | 建筑升级（每★一行） |
| 1127 | 放置解锁（subtab=4） |
| 1511 | display_key 视觉资源 |
| 2148 | 装饰等级（核心表） |
| 2171 | 装饰涂饰技能 |
| 1168 | 道具获取途径 |

### ID 编排规则
```
1111:  11112xxx   — 装饰道具
1118:  1118xxx    — 建筑组 ID
1127:  1127xxxx   — 放置解锁
1511:  151105xxx  — display_key
2148:  214847     — 装饰组 ID
2171:  2171017    — 技能组 ID
1168:  11684xxx   — 获取途径
```

| 2 | 100 | 100 | 1000 |

### 涂饰 buff（paint_buff）
`[{"typ":"buff","id":121140935,"val":★×100}]`

### 扩星 + 涂饰要点
1. 1118 `max_lvl` 改大、2148 `star_max` 改大、`paint` 0→1
2. 现有 2148 行追加涂饰字段
3. 新增：1118/2148 高星行、2171 技能行、1168 获取途径
4. 涂饰道具 type=`decorate_paint`，含自回收
5. 新装饰加 1127 行，扩星不用
