# 🟠 V2-B · 特殊字符测试 ★

- [ ] **在做什么**：准备一堆脏数据样本挨个试，看特殊字符写进 GSheet 会不会变形
- **为什么这是本周最值钱的活**：GSheet API 会通、连通性不用验证，**字符兼容性才是真坑**。以往踩过的坑都在这里
- **要测哪些样本**：
  - [ ] 活动名带双引号 `"限时"福利` → 会不会被吃掉或变成 `&quot;`
  - [ ] 英文 LC 带撇号 `don't miss it` → 会不会转义错
  - [ ] 多行描述带 `\n` → 会不会变成一行
  - [ ] 字段以 `=` 开头 → 会不会被 GSheet 当公式执行
  - [ ] emoji 🎁 / 日文 / 阿拉伯文 → 会不会乱码
  - [ ] 超长字段（>1000 字符）→ 会不会截断
- **每条样本**：记"输入 → 实际写入结果"对比，出问题的标红
- **估时**：90 分钟
- 📅 2026-04-15 上午 🔺 **#核心**

# 🟠 V2-C · 写失败怎么回滚

- [ ] **在做什么**：写一个"写到一半失败就清回去"的机制
- **为什么要想清楚**：几张表连着写，写到 T2 失败时，前面写进 T1 的几行就是脏数据。要清掉。
- **难点**：**要清的行可能已经被别的表引用了** — 直接删会破坏数据。得先看有没有人引用，没人引用才能删
- **怎么做**：
  1. 写入每张表前先记"这张表最后一行 ID"
  2. 失败时按 ID 倒序删
  3. 删前检查：这些 ID 在别的表里有没有被引用
- **估时**：60 分钟
- 📅 2026-04-15 下午 🔼
