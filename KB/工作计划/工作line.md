# 工作 Line（节日运营 SOP × 项目 timeline）

> 这是我的"活的工作线"单一事实来源。
> 早上 9:00 读它 → 算出今日高优先级；晚上 21:00 日报 → 拿它对完成度。
> **维护方式**：由每晚 21:00 日报自动写回（只改三/五两节）；偶有外部排期变更需手动纠。
> **可视化**：同目录 `工作line_可视化.html`（主工作流→分支→每件事，可折叠甘特）；数据=`工作line_tree.json`，渲染=`python _render_workline_html.py`，日报回写时自动同步。
> 最后更新：2026-07-08（手动刷新——06-24 后自动回写断档，07-07 晚 API 断连事故已修，今晚 21:00 验证恢复）

## 四、每日挂钩规则

- **早 9:00（高优先级推送）**：按 deadline 逼近程度 + 节点卡点，列今天最该推进的 3 件事。
  优先级口径：封包/上线日越近越高；卡在阻塞节点(配表/QA)优先；两期并行时当期 deadline > 下期出方案。
- **晚 21:00（完成度回看）**：日报里除"做了啥"，再对照本文件 → 标注今天推进了哪些节点、哪些 timeline 项目仍 pending。
