## 一、表结构总览

X2 集卡册涉及 4 张表：

| 表号 | 表名 | 说明 | 典型行数 |
|------|------|------|---------|
| **1108** | CardGallaryBook | 集卡册主表（一个节日一行） | 1行 |
| **1107** | CardGallaryGroup | 卡组定义（每册9组） | 9行 |
| **1123** | CardGallary | 单张卡片（每组6张，共54张） | 54行 |
| **1109** | CardGallaryStore | 卡包商店（3种卡包） | 3行 |

> ⚠️ **X2 全部表的 col[0] 是 `p2_title`（元数据列，不导出）**，所有字段列号比 P2 多 1。

## 四、卡包道具配置（1111 item）

每个节日需在 1111 表新建 **3个卡包道具**，格式参照圣诞/占星节。

| 道具 | 占星节ID | 星级范围 | Drop概率 | DisplayQuality |
|------|---------|---------|---------|---------------|
| 低星包 | 11114017 | ★3-4 | ★3×9000 + ★4×1000 | 1511010121 |
| 中星包 | 11114018 | ★3-5 | ★3×8000 + ★4×1300 + ★5×200 | 1511010121 |
| 高星包 | 11114019 | ★4-5 | ★4×9500 + ★5×500 | 1511010122 |

**CategoryParam 关键字段：**
- `drop.random_add` → 指向本节日 **1108 Book ID**（每次必换）
- `drop.drop_items` → 包含本节日所有54张 1123 卡片ID，按星级分组赋权重

## 五、换皮标准流程

### Step 0 — 准备工作

1. 确认本节日设计文档：卡组数量、卡名列表、每组奖励设计
2. 确认美术 DK 已分配（1511表）：Book DK×4 + Group DK×9 + Card DK×54
3. 确认本节日活动ID（OpenReq 和 1109 RequirementPurchase 绑定用）

### Step 1 — 读参照行

从上一期（圣诞庆典 11081002）读取所有表的完整行，作为模板：

```bash
# 读 1108 参照行
gws sheets 读 1108表 圣诞庆典行

# 读 1107 参照行（9行）
gws sheets 读 1107表 11072xxx系列

# 读 1123 参照行（取任意几行，确认列结构）

# 读 1109 参照行（3行：11090011-013）
```

### Step 2 — 分配新 ID

| 表 | 操作 |
|---|------|
| 1108 | Book ID = 上期 +1 |
| 1107 | Group ID = 查最大已用ID +10（或新段） |
| 1123 | Card ID = 按组分配，每组6个，组间留10空位 |
| 1109 | Store ID = 上期 +10（或新段） |
| 1111 | 卡包道具 ID = 查最大已用ID后续分配 |

### Step 3 — 逐表生成配置

**1108 Book（1行）**
- 替换：Id / Comment / GroupID / DisplayKey×4 / LcName / CardPackID / Rewards（装饰道具ID） / OpenReq（活动ID）

**1107 Group（9行）**
- 替换：Id / Comment / CardID / DisplayKey / LcName / DisplayOrder / RewardsFirst（按组类型设计）/ RewardsFollow

**1123 Card（54行）**
- 替换：Id / Comment组名 / Comment2卡名 / Star / DisplayKey（每张唯一）/ LcName / DisplayOrder / SendHonorCoin / Recycle
- 不换：Category(11071001) / DisplayKey1(1511020219) / LcDesc / DropItems

**1109 Store（3行）**
- 替换：Id / Comment / GoodsId（卡包道具ID）/ Price / Limit / DisplayOrder / RequirementPurchase（活动ID）

**1111 卡包道具（3行）**
- 新建3条道具，`CategoryParam.drop.random_add` 改为本节日 1108 ID

### Step 4 — 本地化

新增 LC Key：
- **册名**：`LC_EVENT_item_card_book_title_X`（X = 上期序号+1）
- **组名**：`LC_EVENT_item_card_collection_title_XX`（续接上期序号）
- **卡名**：`LC_EVENT_item_card_name_XXX`（续接上期序号）
- **卡包名**：`lc_item_card_pack_name_XX`（续接上期序号）

写入 X2 i18n 表（SheetID: `1YGVYGjiDxJ2Rx3MEUv4o-xIEzLuG6NL5wW06bisZSyg`）。

> ⚠️ X2 i18n 暂存区 tab 名为 `AI翻译页签`（sheetId=338656897），含中文 tab 名需用 gid 或 Python API 写入。

### Step 5 — 写入 GSheet 并验证

```bash
# 写入顺序（避免外键先于主键写入）
# 1. 1111 卡包道具（先建道具）
# 2. 1123 Card（54行）
# 3. 1107 Group（9行）
# 4. 1108 Book（1行）
# 5. 1109 Store（3行）
```

**写完必做验证：**
- 行数对：1108×1 / 1107×9 / 1123×54 / 1109×3
- 回读 Book 的 GroupID 是否包含全部9个 Group ID
- 回读每个 Group 的 CardID 长度是否为6
- 回读 Store 的 GoodsId 是否指向本节日卡包道具
- 回读 1111 卡包道具的 `CategoryParam.drop.random_add` 是否为本节日 1108 Book ID

## 六、避坑清单

| # | 坑 | 后果 | 规避方法 |
|---|---|------|---------|
| 1 | 1108 Rewards 里的装饰道具ID未换成本节日 | 发了上一期节日装饰给玩家 | 确认 1111 注释含本节日关键词 |
| 2 | 1107 G7-G9 的 Rewards 照搬上期（专属装饰道具） | 同上 | 逐组对照 1111 表验证道具 |
| 3 | OpenReq 的活动ID未换 | 集卡册入口条件绑了旧活动，可能无法开启 | 确认 id 字段值 = 本节日 2112 活动ID |
| 4 | 1109 RequirementPurchase 的活动ID未换 | 卡包商店购买条件异常 | 同上 |
| 5 | 1111 卡包道具的 random_add 未换 | 卡包开出上一期卡片 | 改为本节日 1108 Book ID |
| 6 | 1123 Card ID 间隔不对 | 组间无间隔，下期扩展困难 | 保持每组6张，组间留10空位（如001-006, 011-016） |
| 7 | DisplayKey 列错位 | 美术资源无法加载 | X2 所有列 +1（p2_title占一列），写入前确认列头 |
| 8 | gws 写入含中文tab名失败 | 静默写入空/写到错误位置 | 用 gid 参数或 Python API |
| 9 | 大批量写入（>50行）静默失败 | 数据缺失 | 分批写，每批≤20行，写完回读验证 |
