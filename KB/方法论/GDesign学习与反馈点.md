# GDesign（DesignDeck）学习 & 反馈点

> 暂存我们对 GDesign 的学习/反馈点，讨论完再集中反馈给 GDesign 团队。
> 分两类：【已落地】= 我们已照搬进自己工作流；【待反馈】= 给 GDesign 产品的建议。

## 【已落地】checker（独立验收员）下沉到各流程

GDesign 最值得学的点：把"判定是否合格"从执行者手里拿走，交给一个**无写权限、无对话历史、按客观清单跑 block/warn 的独立验收 agent**（执行/验收物理分权）。

我们已自建一套 `quality-gate`（位置 `C:\ADHD_agent\.claude\quality-gate\`）：
- 通用验收员 `task-checker`（按任务 type 选清单）+ 收工 Stop hook 守门（产物没验收就拦住，fail-closed）。
- 已覆盖：策划案 / 配置·换皮 / i18n（加新类型 = 只写一份 `<type>-checklist.md`）。
- 客观项机器判（页签齐/占位残留/分支正确/引用ID/Reward seq/MailID/key命名…），主观项（颗粒度/数值合理性/文案）只列出逼人看。

## 【已落地】交互模块工作流（学 DesignDeck 的原型/分析层）

学它 L1 的"交互原型 + analysis.md 交互分析"，产品化成更顺的形态：**一份「活原型 + 实时说明」一体 HTML**——左能点、右实时讲，点哪讲哪，不用两边对照。
- 原型先行：可点原型 → 点出问题定待定 → 照原型 1:1 写交互说明 → 合成一体 HTML → 落策划案。
- 固化在 `KB/方法论/X3交互模块工作流.md` + 同目录范本，挂在策划案「界面交互」槽下。

## 【已落地】透明化skill（★双底差分是好东西）

DesignDeck 两套透明化方法很妙，但**整机编排原来埋在它编译进 main.jsc 的主进程(jimp)里**，skill 只暴露了零件——我们把整机补成本地脚本，脱离 DesignDeck 也能一条命令跑：
- **★双底差分**（生成新透明资产，强烈推荐）：同元素白底+黑底各画一次 → 反推 alpha，**避开 removebg 吃毛发/留白边**，思路简单又狠。→ `transparify_asset.py`，已设为 icon/行军表情/DK 类**默认**透明化（写进 x3-media SKILL 硬规则）。
- **精细拆图**（从现成图拆多层、100% 对齐）：mask inpainting 填绿 → 去绿。→ `ui_extract_fine.py` + `make_bbox_mask.py` + `chroma_key.py`（含 --preview 人工确认节点）。

# 给 GDesign 的反馈（用下来的：3 个搬走的好东西 + 2 点小建议）

> 口吻：用户视角分享体验，不评判对错——先夸再提，"不一定对，供参考"。

这工具我们用得挺爽，先说**三个直接搬进自己工作流的好东西**（你们这几处设计太对了 👍）：

1. **独立验收员（checker）** —— 把"判合格"从干活的人手里拿走、交给一个只读、按客观清单挑错的独立角色，这个"执行/验收分权"思路太对了。我们照它给自己搭了一套，现在策划案 / 配置 / i18n 收工前自动过一遍，"写完不检查"这毛病基本根治了。

2. **交互原型 + 交互分析那层** —— 你们"先出可点原型、再配交互分析"的路子启发很大。我们顺着做成了"活原型 + 实时说明一体"（点哪讲哪），跟人对齐交互的效率一下就上来了。

3. **★双底差分抠图** —— 这个最惊艳：不赌 AI 一次出透明，而是白、黑两底各画一次、纯数学反推 alpha，毛发光晕半透明全保住、还可复现。简单又狠，我们直接搬来当 icon / 资产透明化的默认。

下面是用下来攒的策划视角的**小不顺**，不一定对，供你们参考~

**① "先把需求问全 / 定全再往下做"这套，对我们有点不太顺。**
我们写策划案常常是信息很少就先开干、边讨论边补、案子是慢慢聊出来的。阶段一"问全才开干"有时会打断这种边做边想的节奏。
我们这边是把"对齐确认"放到**快出正式案那一刻**再做一次（把讨论里定下来的汇总一遍），前面让它自由探索。也顺带觉得，整条线要是能**按子功能一块块迭代**（交互的先出原型、数值的先填内容），会比"全定完再统一做"更贴我们的实际。供参考~
