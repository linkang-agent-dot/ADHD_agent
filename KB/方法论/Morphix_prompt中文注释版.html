★ 12 个通用 prompt 工程手法

这 8 段 prompt 反复用同一套手法。先吃透这 12 条，你自己写换皮/生图 prompt 就有了套路——下面每个功能页签都是这些手法的具体落地。
手法怎么写实现什么 / 不这么写会怎样
1. 职责切分给每张参考图钉死角色：「图#1 = 布局来源，图#2 = 视觉来源」让模型分清"抄谁的什么"。不切分 → 模型默认把两张图平均混合，出来四不像。
2. ERASE 强否定不只说"用#2的风格"，还显式说「图#1 的视觉必须全部擦除，连进度条/开关的颜色都不留」模型对原图有视觉惯性，会偷偷留底色/边框。必须显式清零才换得干净。
3. 正反双约束既列 STRICT(必须做)，又列 FORBIDDEN(绝不做)，并点名具体偷懒动作堵死具体退化模式。如横竖转换里"禁止只拉宽不翻轴"——光说正面要求，模型仍会走捷径。
4. 量级框定「跟随 reference 的程度，subtle 保持 subtle，prominent 保持 prominent，不夸张不强化」防特征被放大。X16 教训：分"外框/内部"两区 → 模型拉成夸张对比，外框过重内部过扁。
5. 枚举防退化列举多种装饰让模型按 reference 灵活挑，不单点名某一种X3 教训：单点名"L 形角包" → 模型把所有装饰退化成 L 角包，死板。枚举才有多样性。
6. 唯一性强制「挑唯一一个最匹配的角色还原，禁止融合/平均多个角色特征」防多角色糊一起。不强制 → 模型把角色库里几个特征混成一个怪东西。
7. 语义而非像素「轴向翻转是语义重排，不是把像素旋转 90°，每个档位要按新方向重新排版」防几何偷懒。不点破 → 模型直接转 90° 像素，文字躺倒、布局别扭。
8. 思考步骤前置给一段 INTERNAL STEPS：先清点所有元素 → 再逐个定位 → 最后画强制规划。不前置 → 模型边画边漏，常丢掉次要元素(进度条/角标)。
9. 禁脑补默认值「不要基于'游戏 UI 常见默认值'自由发挥；reference 没显示的视觉处理都不引入」防模型用训练里的"通用游戏UI"脑补，偏离参考。
10. 中间产物文本化图标换皮：先用 describe_media 把参考图榨成 ~80 词风格文字，再套用到每张图文字比图参考更稳定、可批量复用。直接丢多图参考 → 每张结果风格漂移。
11. 后处理友好约束拆分要求 #B8B8B8 灰底而非白底为下游抠图服务：灰底让抗锯齿边缘融进灰色，去底后不留白边光晕。
12. 配方最小化「参考图能说明的，就不要在 prompt 里说」——stylePrompt 只写参考图传达不出的(量级/分布约束)避免文字命令和参考图打架。写多了反而把模型从参考带偏。
一句话总纲：换皮类 prompt = 钉角色(手法1) + 清原图(手法2) + 抄参考(整套design system) + 双向约束堵退化(手法3-9) + 为下游服务(手法10-11)，能让图说话就别用文字(手法12)。

A 换皮 · 中文全文

我提供两张参考图，这是一次完整的视觉风格迁移任务。 【参考图 #1 = 布局与内容来源】 只从图#1 提取这些： · 画面结构布局（每个元素在哪、整体构图） · 文字标签及其内容（标题、分区名、按钮文字、数值标签） · 每个图标/头像在语义上代表什么（一个箭头、一枚金币、一张怪物头像…） · 每个控件的位置（滑条、勾选框、开关、按钮） 图#1 的其余一切全部抛弃：背景色、面板色、边框样式、装饰、按钮形状/颜色、滑条颜色、勾选框样式、开关样式、边框装饰、字体、字色、整体配色、图标渲染风格——图#1 的每一处视觉特征都必须被【擦除】。只有"是什么/在哪里"保留，"长什么样"绝不保留。 【参考图 #2 = 完整视觉风格来源】 把图#2 的每一处视觉都搬到输出上： · 背景与面板的颜色/质感 · 边框样式、框饰、转角装饰、分隔线样式 · 分区标题排版（字重、颜色、字号、标题周围的装饰） · 按钮渲染：形状、圆角、渐变/纯色、立体/阴影、描边、内高光——完整照搬图#2 的按钮视觉语言 · 滑条轨道与滑块样式 / 勾选框样式 / 开关样式 · 图标渲染风格（线宽、填充、颜色） · 子面板渲染（内阴影、高光边、分隔样式） · 整体配色（每个颜色都来自图#2 或由其调色板推导） · 字体族/字重的感觉 把图#2 当作设计系统。输出的每个 UI 元素都必须用图#2 的视觉语言，连确认/取消按钮、开关开/关态、进度指示都是——哪怕图#1 里这些是别的颜色，输出也用图#2 的处理。 【目标】输出应当看起来 100% 像图#2 的作者亲手设计的——用他的风格从零重画，但装的是图#1 的布局和内容。 输出：干净的移动端 UI 稿，竖屏，字体和图标渲染锐利，高分辨率。

原句要点实现什么不这么写会怎样
"只提取布局/文字/语义/控件位置"把图#1 锁死成"骨架供应商"模型会连图#1 的颜色质感一起继承，换皮不彻底

B 主题转换 · 中文全文（最贴节日换皮）

这是一次【创意主题再想象】任务。把一个游戏 UI 界面重新设计成一张主题活动海报，可以大胆发挥来把节日氛围拉满。 【参考图 #1 = 原始 UI（内容来源）】 必须从图#1 保留（这些一定要活下来）： · 所有文字标签、按钮文案、数字、等级标识——逐字保留，不翻译、不改措辞 · 界面的总体用途（活动/商店/设置…）和各区域的大致角色（购买按钮还是购买按钮，奖励格还是奖励格） · 货币/资源图标的语义含义（宝石图标、爱心图标——含义保留，视觉可重塑） 【创意自由——这不是严格的 1:1 换皮】输出应当像一张全新的主题活动海报，而不是原图贴几张贴纸： · 构图和布局【可以重排】来更好地表现主题——主角可以换位置/换姿势，面板可以挪，装饰道具可以加在边角 · 大量添加图#1 里原本没有的主题氛围元素：飘落的雪花、花瓣、悬挂的灯笼、飞舞的丝带、闪烁的星光、烟火、植被等——尽情堆主题点缀 · 把吉祥物/主角重新设计成主题角色（探险家变成圣诞老人、或女巫、或赛龙舟的人——彻底重构，不是换套衣服） · 戏剧性地重做背景场景、光照和整体氛围来贴合主题 · 边框装饰可以完全换新（松枝花环、蜘蛛网、梅花、金龙） · 配色彻底改成主题色板 · 周围道具（灯笼、蜡烛、气球、礼物）可以自由添加 · 把整个界面当作主题活动的营销海报来做——视觉冲击力优先 【目标主题】：[用户输入的主题文字 / 若没填则从参考图#2 推断] 【目标】一张精致、有戏剧感、有沉浸感的主题 UI，保留图#1 的文字内容，但感觉是为目标主题全新设计的。氛围和主题表达是第一优先级。 输出：丰富、有氛围、精致的游戏 UI 稿，字体图标渲染锐利，高分辨率。 [若有参考图] 参考图 #2 = 主题视觉锚点。用它取：目标主题的美术风格、配色、装饰母题、角色气质。【不要】用图#2 的布局——只取它的视觉主题线索。

💡 逐点注释 — 它和普通换皮的区别全在这些措辞上

原句要点实现什么对比普通换皮
"创意主题再想象…当海报来做"给模型松绑，开放创意换皮是"继承风格"，主题是"重新创作"——定调完全相反
"NOT a strict 1:1 reskin"明确否定主动告诉模型"不用对齐原图"换皮反复强调严格保持，这里反着来
"布局可重排 / 主角可换姿势位置"解锁构图自由换皮严禁动布局
"大量添加原本没有的氛围元素"+列举主动要求无中生有加节日装饰，并枚举给灵感换皮不加新元素
"角色彻底重构，不是换衣服"允许把吉祥物重塑成节日形象换皮只换皮不换形
"文字逐字保留 + 区域语义角色不变"放飞的同时守住底线：可读性和功能不丢这是两者唯一的共同约束
"参考图#2 只取主题线索，不取布局"手法1 职责切分的变体——参考图只供"风味"换皮的图#2 是要整套抄的
给你做节日换皮的启示：想"原界面轻改"就用换皮 prompt；想"做一张节日海报级界面"就用主题 prompt。后者的"列举氛围元素 + 角色重塑 + 守住文字底线"三段，可直接套你的节日包装需求。

C 横竖转换 · 中文全文（竖→横；横→竖镜像）

游戏 UI 布局换向——为新的屏幕比例重新设计游戏 UI，遵循目标方向下专业游戏 UI 的惯例。 参考图 #1 = 当前为【竖屏】（纵向、高）的游戏界面。 你的任务：把【同一个】界面为【横屏】（横向、宽）画布重新设计。这【不是】拉伸、【不是】旋转、【不是】裁剪，而是一次完整的布局【重设计】，你要扮演一个懂横屏布局惯例的资深游戏 UI 设计师。 【横屏游戏 UI 布局惯例】（当作设计铁律，不可协商）： · 左上角：返回/关闭按钮（←）。横屏绝不放底部。 · 右上角：玩家资源/货币，横向一排。 · 顶部居中：界面标题 + 下方副标题。 · 底部居中：主操作按钮/页签切换，横向等距排列、水平居中。 · 主体中央区：核心内容占最大面积。 · 左侧或右侧：纵向侧边栏页签/导航。 · 横向装饰条 → 变成通栏顶部条，绝不能悬浮在画面中部。 · 竖屏 3 列的卡片网格 → 横屏变 4-6 列（吃掉多出的横向空间）。 · 角色立绘 → 放左侧或右侧，不居中。 【可滚动列表/轮播/页签条——不要把所有项硬塞进可见区】 · 只显示新方向下舒服放得下的数量，其余隐含（屏外、可滚动）。 · 保持每项相同的视觉密度，不要压缩到把 N 项全塞下。 【轴向/线性组件——翻转其轴向以对齐新屏幕的长边】 （进度条、奖励阶梯、赛季通行证轨道、关卡路径、时间轴、分段轨） · 竖→横：纵向的奖励阶梯必须变成【横向】阶梯，横跨大部分宽度。每个"档位"从纵向堆叠重排成横向并列；当前高亮锚点仍沿横轴标记进度。 · 翻转是【语义】的，不是把像素转 90°——每个档位要按新方向重新排版成合适的单元格。 · 12 档纵向阶梯转横屏，不必显示全部 12 档——显示 5-7 档 + 隐含可滚动才是对的。 【内部思考步骤（动笔前先做）】：清点图#1 每个元素（含轴向组件）→ 按目标方向惯例为每个元素定新位置 → 重排卡片网格 → 翻转每个线性组件的轴向 → 重设计装饰条 → 移动返回键 → 居中底部操作按钮 → 重新平衡留白与比例。 【严格要求】保留每个结构元素；文字逐字保留；语义角色不变（按钮还是按钮、面板还是面板）。 【禁止行为】拉伸/挤压/扭曲；旋转 90° 抄近路；裁掉/丢弃任何元素；翻横屏时还把纵向阶梯/进度条保持纵向（这是偷懒拉伸的头号标志，明令禁止）；返回键放非惯例位置；装饰条悬浮中部；改任何文字。 [有风格参考图] 参考图 #2 = 可选视觉风格锚点。把它的美术风格、配色、装饰母题应用到重设计后的布局上。输出 = 图#1 内容 + 图#2 风格 + 新方向。 [无] 沿用图#1 原本的视觉风格——只改布局和位置，不改外观。 输出：一张精致、专业排版的横屏游戏 UI 稿。

D 元素拆分 · 中文全文

这是一次 UI 美术资源提取任务（界面元素拆分），输出是给美术/程序用的一张雪碧图。 输入：一张完成的游戏 UI 稿。 【要提取什么——只要图形美术资源】 按钮（每个按钮单独一件，不含文字标签）/ 图标 / 装饰美术（叶子、花、宝石、丝带、灯笼、齿轮、纹饰）/ 面板·边框·底板（容纳内容的圆角板，但不含里面的内容）/ 角色立绘与头像 / 滑条轨道（只要条，不要滑块）/ 滑块（只要那个圆把手）/ 勾选框（空框与勾选两态若有则分开）/ 开关背景与开关把手 / 进度条填充与轨道 / 标题栏（装饰板，不含文字）/ 小角标（只要那个色块形状，不含里面数字）。 【排除什么——绝不当作元素】 文字串（标题、标签、按钮文案、数字、等级）——绝不把文字当精灵图提取 / 纯数字元素（"x12""900""Lv4"）——这些是运行时数据不是美术资源 / 复合组合控件——按下面规则拆解。 【关键：把复合控件拆成原子部件】 · 带加减按钮的滑条 必须拆成 4 件：减号按钮、加号按钮、空轨道、滑块。不能合成一条。 · 带图标和文字的按钮 拆成：纯按钮底板（无图标无文字）+ 可复用的图标。 · 奖励格 拆成：纯色框/背景 + 图标/插画（不含数量角标或数字）。 · 开关 拆成：胶囊背景 + 圆把手。 【关键：严格去重】 每个唯一元素只出现一次，不论它在原图里出现多少次。判为重复（只留一个）：同形同渲染的元素（即使位置不同）/ 互为旋转或镜像的副本（4 个转角同纹饰转 0/90/180/270° → 只留一个）/ 因多个同类奖励格而重复的同一图标 / 同形同色的空面板板。保留有意义的不同变体：勾选框的勾选态与未勾选态都留 / 同形不同色的按钮各留一个 / 同控件的常态/按下/禁用态各留一个。不要为填满版面而硬凑。 【排版要求】按类别成行（装饰→面板/框→按钮→图标→角色→滑条/滑块/开关）；元素间留 ≥20px 等距间隔；保留每个元素原本的样式、颜色、光影；不要重画、重新上色或改风格。 【输出背景——关键】用统一的浅灰纯色背景 #B8B8B8（RGB 184,184,184）铺满整个画面元素之间和周围。绝不要用：白色或近白（抗锯齿边缘会与白底混合，去底后留白边光晕）/ 偏灰、暖灰、冷灰、深灰、渐变 / 米色、米白、纸纹 / 棋盘格、网格线、点、任何花纹。 输出：一张高分辨率雪碧图，元素原子化、已去重，铺在统一 #B8B8B8 浅灰底上。无文字、无数字、无复合控件、无重复、无花纹。

E 图标换皮 · 中文全文（两步）

第 1 步 · 提风格（describe_media）

详细描述这张图的【视觉美术风格】。只关注风格，不关注画的是什么物体。要覆盖：渲染技法（3D/2D/手绘/厚涂/像素…）、确切的材质和质感（水晶/玻璃/金属/黏土/纸…）、配色与饱和度、光照风格与反射、光泽或半透程度、描边/轮廓处理、细节繁简程度、情绪氛围。输出一段约 80 词的英文，写成一条可复用的美术风格指令。不要提及任何具体物体。

第 2 步 · 逐张换皮（generate_image）

用下面描述的【新视觉风格】重画这个图标。保持图标的主体身份（它的剪影和所画之物必须仍清晰可辨），但【彻底替换】它所有的视觉处理——颜色、材质、表面质感、渲染技法、光照、边缘、细节程度——换成目标风格。 要应用的目标风格： [第1步榨出的风格描述] 关键：彻底抛弃图标原本的美术风格。输出在材质和渲染上必须与输入根本不同，但仍画的是同一个主体。想象目标风格的画师从零重画这个主体。 背景：完全透明（PNG alpha 通道）。图标居中。无文字、无标签、无额外装饰。

F 线框转效果图 · 中文全文

这是一次线框稿转效果图任务，提供两张参考图。 参考图 #1 = 线框稿（带文字标注的低保真草图）。它包含： · 占位框（常带 × 叉号或空矩形），标示成品 UI 元素将放的位置 · 文字标注（多为中文），描述每个占位处该放什么（如"篮子、紫色宝石、900"= 一篮标着 900 的紫色宝石） · 展示所有元素位置和比例的布局结构 你对图#1 的任务：只提取布局结构和每条标注的含义。把【每一个】占位框、每个 × 矩形、每个线框元素，替换成与其标注描述相符的、完整渲染的插画或 UI 组件。彻底丢掉线框线、占位框和粗糙草图质感。把中文标注翻译成实际渲染内容（如"篮子、紫色宝石、900"→ 画一个装满紫宝石的精美篮子，配数量标签 900）。 参考图 #2 = 完整视觉风格目标。把图#2 的每一处视觉应用到输出：整体美术风格、配色、面板/边框设计、按钮渲染、图标风格、排版感觉、背景氛围、任何角色/吉祥物。 目标：一张精致、可直接上线的高保真 UI 稿——(1) 严格匹配线框布局；(2) 把每个占位/×框替换成符合其标注的渲染插画；(3) 用图#2 的完整视觉语言。输出绝不能看起来像线框。 输出：干净的游戏 UI 稿，渲染锐利，高分辨率。

G 自由画布 · 中文全文

我提供 {n} 张参考图，按以下顺序给出： · 参考图 #1 就是用户所说的"图1"。 · 参考图 #2 就是用户所说的"图2"。…（逐张列举） 当用户在下面提到 图1、图2…图{n} 时，分别指我按此顺序提供的参考图 #1、#2…#{n}。 仔细阅读下面用户的指令，并【完全照其字面执行】。用户的措辞是权威——无论他们描述什么操作（保留某张图为底图、替换主体、迁移风格、组合元素、修改某个特定区域等），以及把哪张图指派成什么角色，都精确照做。【不要】默认"把所有参考图融合在一起"；只有用户明确要求才融合或组合。用户说某处保持不变，就保持不变；说只从图N 取 X，就只取 X。尊重用户陈述的每一条约束、图片指派和保留规则。 下面是用户指令（中文逐字）： {用户输入} 输出：一张遵循上述用户指令的高质量图片。不要添加用户没要求的元素，不要漏掉用户要求保留的元素。

H 局部修改 · 中文全文（图文结合）

[在自由画布 prompt 基础上，嵌入选区约束] 图#1 = 要编辑的原图，这是底图。 图#2..#N = 用户上传的参考图。"图1"=图#2；"图2"=图#3…（顺序映射，防止模型混淆） 编辑区域（唯一【可能】改变的小范围）：位于图#1 的 {区域} 部位，约由多边形顶点 {坐标列表}（坐标为图#1 的 0–100%）围成。只有这个多边形【内部】的像素可以与图#1 不同。多边形【外部】的像素必须与图#1 完全一致。 规则：① 底图是图#1，整张基本保持不变；② 只有多边形内部可变；③ 只从参考图借用用户明确指定的元素（如"换成图1角色的帽子"）；④ 禁止把参考图当底图替换图#1；⑤ 示例："把选区角色的帽子换成图1角色的帽子"= 只改帽子，其余保持原样。
