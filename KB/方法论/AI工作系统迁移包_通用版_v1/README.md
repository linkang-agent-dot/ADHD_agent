# AI 工作系统迁移包 · 通用版 v1

这是一套**生产级 AI 工作系统的可迁移骨架**——把一个重度使用 AI（Claude Code / Cursor 等）干真实生产工作的人，两年里踩坑长出来的**方法论、防线、验收机制、沉淀标准**抽出来做成的迁移包。

> 📦 与市面上「AI 记忆/人格类」迁移包（如"影子迁移包"）的分工：那类包解决"AI 记得住你、像个协作者"；**本包解决"AI 干的活能过生产验收、知识能被下一个人接管"**。两者不冲突，可叠加使用。

## 这套系统的一句话

> AI 干活的质量不靠模型自觉，靠**三段防线 + 判定与执行物理分离的验收**；
> 知识的沉淀标准不是"记下来了"，而是**"新 agent 冷启动能直接接管"**；
> 知识的最终形态不是笔记，是**可执行的工具**。

## 阅读顺序

1. **先读 `_换皮清单.md`**——它告诉你这套系统哪些直接能用、哪些必须你自己长、怎么长。这是全包最重要的一份文件。
2. 再按 `00-先读我` 装好内核，开始用。
3. `01-方法论/` 和 `02-模式库/` 不用一次读完——它们是你遇到对应场景时的参考书。

## 三个诚实声明

1. **骨保证起跑线，不保证战斗力**。这套系统的真正威力来自原主人 199 条实证记忆 + 50 多个业务工具——那些是"皮"，带不走，你得用同样的方法自己长（长法在换皮清单里）。
2. **皮不是填表填出来的，是事件逼出来的**。别指望装好当天就有铁律和验收清单——第一次翻车才长第一条铁律，这是正常节奏。包的作用是：让你翻第一次车时，教训有地方放、放的姿势是对的。
3. 本包已**中性化**：原主人的行业（游戏研发运营）具体业务全部抽掉或匿名为"项目A/B/C"，只保留可迁移的结构与方法。案例里的事故是真实的，细节做了脱敏。
