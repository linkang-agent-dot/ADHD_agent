# 模式库 · 使用说明

## 模式是什么
方法论（01-）是"全骨"——换任何行业一字不改。模式是"**骨+皮**"——从原体系的真实事故里抽出的通用结构，**带一个脱敏案例**帮你识别"我这也是这种情况"。

模式的用法不是通读，是**对号入座**：遇到"多真源系统 / 要合并分支 / 要切换数据源 / 要动生产"这类场景时，来这里查有没有现成的坑防。

## 条目结构（新增模式时照这个写）

```
# 模式名
## 通用形          ← 一句话，不含任何业务名词
## 识别信号        ← 什么特征说明"你的场景命中了这个模式"
## 动作            ← 命中后必须/禁止做什么
## 案例（脱敏）     ← 原体系的真实事故：背景→翻车→教训
## 反模式          ← 最常见的错误做法
```

## 入库判据（换公司测试）
一条候选内容问："**换一家公司/换一个行业，它还成立吗？**"
- 结构成立、参数不成立 → 抽成模式进这里（参数留在案例段）。
- 连结构都不成立 → 那是业务事实，进你的 memory/reference，不进模式库。
- 参数都成立 → 那是方法论，升去 01-。

## 有进有出
- 你自己的翻车抽出新模式 → 按上面结构加文件。
- 某模式两年没被命中 → 移到归档段（可能你的业务形态就没有这种坑）。

## 随包 8 模式速览

| # | 模式 | 一句话 |
|---|---|---|
| 01 | 标识符禁硬抄 | 多真源 id 空间重叠时，标识符必须现场解析 |
| 02 | 合并三铁律 | 自定义合并链路：预检驱动 / 禁批量选边 / 完后双向审计 |
| 03 | 真源切换断旧链 | 数据真源迁移后，旧生成链路必须物理断开 |
| 04 | 端到端反查 | 多层数据链改动，每层独立验证，不信传递 |
| 05 | 删改防线 | 删除先清单确认；但自产临时物默认清不问 |
| 06 | 生产操作先报再动 | 影响真实用户的操作，先报告再执行 |
| 07 | 假设必标注 | 推断型结论行内标"未验证"，验证后删标 |
| 08 | 唯一入口 | 一个案子只有一个"从这里进"的文档 |
