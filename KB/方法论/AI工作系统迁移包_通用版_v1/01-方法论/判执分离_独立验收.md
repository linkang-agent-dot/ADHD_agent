# 判执分离：独立验收，不信任执行者自评

## 核心原则
**干活的 AI 和判定合格的 AI 必须物理分离。** 执行者的"我检查过了"是自我声明，不是验收。第二遍检查的价值全在**视角独立/对抗**，不在"看了两遍"——同一个视角查两遍只能抓错别字。

## 架构（骨，别动）

```
执行 agent（干活） → 产物 → 独立 checker agent（沙箱、只读、不写产物）
                              ↓ 按该产物类型的 checklist 跑 block/warn 检查
                              ↓ 输出结构化结论
                    Stop hook 质检门：有未过验收的产物 → 拦住收工
```

- checker **只读**：不修产物、不动状态标记——判定和修复也要分离，否则 checker 会"顺手改到能过"。
- 每类产物一份 checklist（原体系：配置/设计文档/翻译/数值/方案质量五类）；新增产物类型 = 新增一份清单文件，架构不变。

## 清单设计四原则（写/优化任何 checklist 时套用）

> 根因认识：清单「只有作者能用」和「检查了还漏」是**同一个病**——清单写的是结论（"检查引用链"），依赖操作者脑里的上下文。改成「为什么+怎么查+对错长什么样」，就同时变得**可移交**（别人/别的 agent 能跑）+ **可靠**（可证伪）。

1. **工作姿态 preamble（最高杠杆）**：把 checker 从"检查是否通过"扭成**默认有罪**——"假设它已经坏了，找不到证据证明没坏才 pass；别去找它对的证据"。外加"只信真源不信执行者的摘要"、"按失败模式驱动检查"。不加一条新规则，就升级了每一条的跑法。
2. **每条 block 强制留证据**：pass 必须**打印实际值/命中位置原文**（分支名、序列值、三层各自的值）。没贴实际值的 pass 一律记 `cannot_verify`。目标：能"响亮地失败"。
3. **内联"防什么事故"**：每条 check 把它防的坑写进条目本身（"序列跳号 → 服务端静默吞奖励"），不要只写指针。条目自包含 → 任何人/agent 拿来直接能跑。
4. **开放项 + 剪枝（防老化+防肥胖）**：
   - 开放项：每轮必问"有没有清单**没覆盖**的新失败方式"，捞到的**报人确认后才加**（不自动膨胀）；
   - 剪枝：每条记"上次命中日期"，连续多轮没 fire 的非关键项评审降级/删除。清单**有进有出**。

## 什么时候建（别过早）
某类产出**第二次**出严重质量问题时建该类的 checklist。第一次写 feedback 就够——一次性的坑不值得制度化。
