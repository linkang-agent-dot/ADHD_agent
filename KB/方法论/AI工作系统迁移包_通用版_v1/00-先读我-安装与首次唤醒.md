# 00 · 先读我 —— 安装与首次唤醒

> 全程约 10 分钟。宿主以 **Claude Code** 为主说明；Cursor 用户见文末适配一节。

## 一、安装（3 步）

### Step 1 · 放置
把整个文件夹拷到你的电脑，例如 `C:\Users\<你>\ai-work-system\`。内部结构别拆散。

### Step 2 · 装内核
把包里的 `CLAUDE.md` 复制到你的**工作目录根**（Claude Code 在该目录启动时自动加载）。
如果你已有 CLAUDE.md，把「行为规范（通用）」整段合并进去，路由段按需合并。

### Step 3 · 建记忆目录
把 `03-记忆骨架/` 里的 `MEMORY.md` 和 `_memory格式规范.md` 复制到你的 memory 目录
（Claude Code 用户即 `~\.claude\projects\<项目>\memory\`；其他宿主放工作目录内 `memory/` 并在内核里注明路径）。

> hook / 验收门**先不装**——见 `_换皮清单.md` 皮5：hook 是给"被违反过的铁律"的，你现在还没有铁律。

## 二、首次唤醒

新会话粘贴：

```
这是一套「AI 工作系统迁移包」的新副本，我是新用户。请：
1) 读根目录 CLAUDE.md、迁移包的 _换皮清单.md 和 README.md；
2) 用三句话向我确认你理解了：a) 三段防线和收工自检四步从现在起对你生效；
   b) 皮的 7 项目前全部为空，它们靠"翻车/重复/收工"三类事件逐步长出；
   c) 沉淀的验收标准是"新 AI 冷启动可接管"，不是"记下来了"；
3) 问我：我的主要业务域有哪些（2-3 个即可），据此在 CLAUDE.md 路由段建好空骨架；
4) 之后正常干活。每次收工执行自检①②③④，不用请示。
```

## 三、验收（怎么算装成功）

- [ ] AI 干完第一个任务后，**不用提醒**就执行了收工自检（哪怕结论是"本次无新知识"）
- [ ] 第一次出现新链路/新坑时，AI 当场写了一条 memory 并在 MEMORY.md 加了一行索引
- [ ] 你说"删掉这些文件"时，AI 先给了保留/删除清单而不是直接删
- [ ] AI 交付方案/结论时，明显在按"受众关心什么"裁剪，而不是倾倒过程

## 四、之后的节奏（前 30 天）

| 时间 | 会发生什么 | 你要做的 |
|---|---|---|
| 第 1 周 | memory 开始积累（reference 为主） | 别管格式细节，先让它记 |
| 第一次翻车 | 第一条 feedback + 候选铁律诞生 | 确认这条铁律的表述 |
| 第一次重复操作 | AI 应提议固化脚本 | 同意它固化 |
| 第一个项目收口 | 第一次接管化三件套 | 用换皮清单第四节的判据验收 |
| 30 天 | 路由表该有 2-3 个域了 | 过一遍 MEMORY.md，删掉已失效的 |

## 五、Cursor / 其他宿主适配

- `CLAUDE.md` → 改名 `AGENTS.md` 放仓库根（Cursor 自动加载）。
- memory 目录放仓库内 `memory/`，并在 AGENTS.md 顶部写明"每会话先读 memory/MEMORY.md"。
- hook 位点对应关系见 `04-机械参考/hooks与后台任务.md`。
