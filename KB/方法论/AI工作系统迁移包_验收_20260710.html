02核心：换皮清单的骨 vs 皮

皮（必须接手人自己长）为什么带不走触发事件
皮1 · 触发路由表业务域/关键词/真源只有他有某域第三次出现；或 AI 用错知识当场补
皮2 · 铁律铁律=真实事故的疤翻车→feedback；同类第二次→升铁律
皮3 · memory 事实层你的199条全是业务实证每次收工自检①（默认动作）
皮4 · 验收清单每条check防一个具体事故同类产出第二次出质量问题
皮5 · hook 脚本闸门逻辑全是业务判断铁律被违反的那一次（自觉为主hook兜底）
皮6 · 工具层封装的是他的系统调用/鉴权同一操作第二次手工做
皮7 · 归档路径与唯一入口目录跟产出类型走第一个 3 天以上项目收口时
长皮总原则（写进清单第三节）：皮是事件逼出来的，不是填表填出来的——
翻车
→
feedback / 铁律 / hook（皮2·5）
｜
重复
→
路由 / 工具（皮1·6）
｜
收工
→
memory / 清单 / 归档（皮3·4·7）
验收自测用体系自己的接管化判据：冷启动 AI 能否在「修问题/接管模块/复刻项目」任一入口直接开工。
