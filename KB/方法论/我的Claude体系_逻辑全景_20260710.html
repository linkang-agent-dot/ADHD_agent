03机械层：3 个 hook + 后台自动任务

SessionStartcheck_flags.ps1

开session 检查旗标/待办状态，把该带进上下文的提醒注入。

PreToolUsex3_config_isolation_gate.py

拦在 Edit/Write/Bash 前面的物理闸门：X3 配置仓隔离检查，防止多 agent / 错分支误写生产配置——不靠 AI 自觉，靠拦截。

Stoppending_verify_gate.py

收工时的质检门：待验收产物未过 checklist 就拦停。配 5 类清单：config / design-doc / design-merit / i18n / numerical。

后台自动任务（Windows 计划任务 + 无头 claude CLI）：策略游戏雷达（每日）、竞品监控日报、BUG 巡检（工作日每小时）、token 周报+归纳清单（周五）、晨报/日报规划链路。 → 影子包停在"AI 记得住"，这一层解决的是"AI 自己转起来"。清单见 memory reference_background_scheduled_tasks。

05三段防线 + 独立验收（判定与执行物理分离）

⛔ 操作前

  • 写配置前：先追完整引用链≥2层
  • 改 X3 / 提配置仓前：先 git branch --show-current
  • 删文件前：先出「保留/删除」清单等确认
  • 合并开工先验 merge driver 非空（空=静默丢行）
  • 大改动先给方案+影响范围等拍板

🔒 操作中（铁律）

  • X2 SheetID 一律 resolve 现解，禁硬抄
  • X3 直改 tsv 不碰 xlsx；禁对已改表重跑生成
  • 合并禁批量选边（--ours/--theirs）
  • PreToolUse 闸门物理拦截 X3 配置误写
  • 生产操作先报再动；补偿文案先确认

✅ 操作后

  • 端到端反查：GSheet + 本地 TSV + 分支推送三层一致
  • push 后 jolt_verify.py 自动验证构建
  • 合并必跑双向丢行审计 + ExportTable exit0
  • 配置写完反查上下游引用
独立验收器：task-checker（沙箱只读，按 type 读对应清单跑 block/warn 检查，不写产物不动 marker）+ l1-game-ux-checker + Stop hook quality-gate。 设计原则：不信任执行者的自评——干活的 agent 和判定的 agent 物理分离。对比影子包的「L0 承接报告」是自我声明，这里是第三方守门。

06沉淀标准：接管化归纳（体系的灵魂）

收工自检四步（默认动作，不问用户）：
① 反馈循环
新规律/坑/方法论 → 当场进 memory
→
② 归档
产物 → KB 固定路径
→
③ 复用
工具固化进知识库，不留一次性脚本
→
④ 接管化归纳
①②③兜"有没有沉淀"，④兜"能不能被接管"
判据：一个新 agent 冷启动，能否在「修BUG / 接管模块 / 换皮」任一入口立刻拿到全部 keypoint——不能就当场归纳。
案子级三件套：决策记录（按模块·为什么+被否决方案）/ 换皮清单（骨vs皮·copy-and-swap）/ 产物目录 FINAL·废弃标注，挂进该案子唯一入口文档。
禁止只留时间线流水账——"编年史给旁观者，不给接手人；理由埋在时间线里 = 做完就没了"。
与影子包 S3 的本质差：它的永久层标准是"写成带双链的永久笔记"（记忆达标）；这里的标准是"陌生 agent 能凭它直接干活"（交接达标）。它的回归协议移交的是 AI 人格，这里移交的是整条业务。范式实例：KB\产出-数值设计\X3_世界杯\。
