# 节日活动跨服归因分析框架（匹配对照法）

## 一、问题背景：为什么不能直接用"实验组 vs 全量对照组"？

新服按批次上线，各批次服务器的玩家质量存在**系统性差异**：
- 早批（对照组）往往保留了更多高付费玩家，付费率虚高
- 晚批（实验组）整体质量偏低，付费率偏低

X3 尼罗活动实测：
- 对照组（70–77服）W2 付费玩家占比均值 **16.3%**
- 实验组（80–86服）W2 付费玩家占比均值 **12.5%**

直接对比 W2→W3 变化会混入**批次差异**，无法准确归因节日贡献。

---

## 二、解决方案：从对照组中筛"匹配对照服"

**核心思路**：不用全量对照组，而是从对照组中挑 W2 基准相近的服，作为实验服的配对基准。

### 匹配维度（三维，缺一不可）

| 维度 | 参考标准 | 说明 |
|------|----------|------|
| W2 付费玩家占比 | ±2ppt 以内 | 最核心指标，反映服务器整体付费质量 |
| W2 DAU | ±15% 以内 | 控制服务器规模差异 |
| daR + chaoR 人数 | 大体相近 | 高净值玩家分布，防止少量大R拉偏收入均值 |

> **为什么 xiaoR/zhongR 占比也要看？**
> feiR 大量流失后，xiaoR/zhongR 比例会"被动提升"，属于成分漂移而非活动效果。
> 匹配时确认对照服与实验服成分相近，才能做有效比较。

### 配对结果示例（X3 尼罗，2026-05）

| 实验服 | 配对对照服 | W2 付费率差 | 匹配质量 |
|--------|-----------|------------|----------|
| 82服(1810) | 75服(1740) | +0.72ppt | ★★★ 最佳 |
| 84服(1830) | 75服(1740) | +0.62ppt | ★★☆ 良好 |
| 85服(1840) | 77服(1760) | +1.79ppt | ★★☆ 可用 |
| 80/81服 | — | >2.5ppt，无匹配 | ❌ 排除 |

聚合后（82+84+85 vs 75+77）W2 付费率：**7.04% vs 7.02%，基准几乎完美对齐**。

## 三、归因计算公式

```
预期 W3 收入 = W2 收入 × (1 + 对照组自然变化率)
自然变化率   = (对照组W3收入 - 对照组W2收入) / 对照组W2收入

总净增量 = 实验组实际W3收入 - 预期W3收入
```

**X3 尼罗实测数字（82+84+85服 vs 75+77服）：**
- W2 基准（两组各）：~$3,069
- 对照组自然变化率：-29.6%（W2→W3 无活动情况下收入自然衰减）
- 实验组实际变化：+10.1%
- **总净增量：+$4,307**

## 四、节日 vs 非节日收入拆解

```sql
-- 节日收入（iap_id LIKE '2106%' 为例，需按活动调整前缀）
sum(CASE WHEN cast(o.iap_id as varchar) LIKE '2106%'
         THEN coalesce(i.iap_price, 0) ELSE 0 END) AS festival_usd

-- 非节日收入
sum(CASE WHEN cast(o.iap_id as varchar) NOT LIKE '2106%'
         THEN coalesce(i.iap_price, 0) ELSE 0 END) AS non_festival_usd
```

**注意：必须用 `dim_iap.iap_price`（标准 USD 单价 × 订单数），不能用 `ods_user_order.pay_price`（约高7-8倍）。**

**X3 尼罗拆解结果（W3周，82+84+85服）：**
| 类别 | 金额 | 占净增量比例 | 解读 |
|------|------|------------|------|
| 节日礼包直接收入 | $904 | **21%** | 邀请函、家具礼包、主城礼包直接拉动 |
| 非节日净增量 | $3,403 | **79%** | 节日激活整体消费意愿，带动非节日消费提升（光环效应） |

**核心洞察：节日的主要价值不是节日礼包本身，而是通过活动机制激活了玩家整体消费意愿（节假日效应/光环效应）。**

## 五、坑点汇总

| 坑点 | 问题 | 正确做法 |
|------|------|---------|
| `pay_price` 字段 | 不是 USD，约高7-8倍 | 用 `dim_iap.iap_price × 订单数` |
| 对照组质量偏差 | 早批服付费率系统性偏高 | 从对照组中筛匹配服，而非用全量均值 |
| R 级去重 | `dl_user_rlevel_all_info` 一个用户多条记录 | 用 MAX 序号编码 CTE 去重（见下） |
| 单服异常污染 | 某一服有高净值玩家集中付费（如85服异常） | 单独标注，必要时拆分85服单独分析 |
| 活动道具窗口 | W3 = D21-D28（7天），DAU 和收入必须用同一窗口 | CASE WHEN server_day BETWEEN 21 AND 28 THEN 'W3' |

### R 级去重 CTE 模板

```sql
rlevel_dedup AS (
  SELECT user_id,
         CASE MAX(CASE rlevel WHEN 'chaoR' THEN 4 WHEN 'daR' THEN 3
                              WHEN 'zhongR' THEN 2 WHEN 'xiaoR' THEN 1
                              ELSE 0 END)
              WHEN 4 THEN 'chaoR' WHEN 3 THEN 'daR'
              WHEN 2 THEN 'zhongR' WHEN 1 THEN 'xiaoR'
              ELSE 'feiR' END AS rlevel
  FROM v1090.dl_user_rlevel_all_info
  GROUP BY user_id
)
```

## 六、适用场景与局限

**适用：**
- 活动分批次上线（不同服务器、不同时间），有天然对照组
- W2 付费率在 5%–15% 范围（中等质量服），对照服可找到匹配

**局限：**
- 对照服数量少时（如只有1-2个合适匹配），统计置信度有限
- 85服这类异常需要单独对待，不宜直接纳入聚合均值
- "光环效应"的非节日净增量中，仍可能有非活动因素（如某周大R集中充值），需结合服务器层面审查

---

## 七、结论输出模板

> **活动净增量：+$X，其中节日礼包直接贡献 $A（X%），非节日光环效应 $B（Y%）。**
> 对照组自然变化率 -Z%，实验组逆势增长 +W%，节日有效逆转了自然衰减并额外拉升。
> 节日的核心价值在于激活整体消费意愿，而非节日礼包本身的销售量。

## 八、付费率归因（W3 第一单法）

### 算法
对每位 W3 付费玩家，取其 W3 期间时间戳最早的一笔订单，判断是否为节日礼包（iap_id LIKE '2106%'）：
- **节日开单**：W3 首单 = 节日礼包 → 节日是其当周付费的直接触发器
- **非节日开单**：W3 首单 = 非节日礼包 → 存量付费行为

### 指标
- **节日开单率（/付费者）** = 节日开单人数 / W3 总付费人数（反映节日在付费群体中的触发比例）
- **节日开单率（/DAU）** = 节日开单人数 / W3 DAU（付费率贡献口径，可与 pay rate 直接相加）

### SQL 核心逻辑

    WITH w3_orders AS (
      SELECT server_id, user_id, cast(iap_id as varchar) AS iap_id,
             ROW_NUMBER() OVER (PARTITION BY server_id, user_id ORDER BY created_at ASC) AS rn
      FROM v1090.ods_user_order o
      JOIN v1090.dim_open_server so ON o.server_id = so.server_id AND so.server_type = '0'
      -- + 标准过滤（排测试号、排TOKEN）
      -- + server_day BETWEEN 21 AND 28
    )
    SELECT server_id,
           COUNT(*) AS w3_payers,
           SUM(CASE WHEN rn=1 AND iap_id LIKE '2106%' THEN 1 ELSE 0 END) AS festival_open,
           round(100.0 * SUM(CASE WHEN rn=1 AND iap_id LIKE '2106%' THEN 1 ELSE 0 END) / COUNT(*), 1) AS pct_festival_open
    FROM w3_orders
    WHERE rn <= (SELECT MAX(rn) FROM w3_orders WHERE server_id = w3_orders.server_id)
    -- 实际上取 rn=1 的行再聚合
    GROUP BY server_id

### 尼罗活动实测结论（2026-05，X3 80-85服）

| 指标 | 数值 |
|------|------|
| W3 聚合付费率 | 7.38% |
| 节日开单率（/DAU）| 0.96ppt |
| 节日开单率（/付费者）| 13.0% |
| 对照组自然变化 | -0.35ppt |
| 付费率总净贡献 | +1.45ppt |
| 节日对付费率的直接贡献 | 0.96ppt（约66%） |

**结论：节日礼包直接贡献 W3 付费率的约 13%（0.96ppt），其余付费行为来自存量基础逆转自然衰减。节日对付费率的主要作用是"触发当周开单"，而非系统性拉新付费。**

### 分服规律
高质量服（W2 付费率 >6.5%）节日开单率 12-20%，低质量服（W2 <6%）节日开单率仅 4-5%。节日礼包对存量付费意愿强的玩家触发效果更显著。
