## 🔑 重大发现：dev_x3_jhl 分支有现成 X2 pay-gacha 移植脚手架
`origin/dev_x3_jhl` = 同事在移植 X2「惊喜藏品屋 CollectionPayGacha」（ActvType 69，**另一个 X2 活动，不是扭蛋机**）：服务端 3 handler（`ActivityMeta.PayGacha.cs`）+ 独立 `activityPayGacha.proto` + 3 配置表 + 客户端全套 UI + `AIDocs\X2_to_X3_Migration\CollectionPayGacha\` 12 篇迁移文档（含 ActvType 注册草案/屏蔽符号登记）。**借它的模式（pattern），不借它的号**——它的盲盒不放回抽奖核不适用，我们是加权可放回。

## 实现路线（定案）
**PayGacha 脚手架模式 + MechaWheel 加权双池抽奖核 + LuckyWheel 排行接入**：
- 抽奖核照抄 `ActivityMeta.MechaWheel.cs`（已有免/付双池 `*RewardGroupFree` vs `*RewardGroup`、10连 `count`+`ItemNum/ItemNum10`、日免费上限、触发礼包 `DrawCardNum→CreateActivityPack`）——改成单圈双池 + 付费分支**双消耗**（币1211+券1212 两个 ReduceItem）
- 排行照抄 `ActivityMeta.LuckyWheel.cs` 的 `UpdateActivityRankScore(MainRankID/ServerRank)` + `ServerActivityRankMeta.cs` 加 case
- 积分任务发币**不新写**：复用 ActvTask(ActvType=5) 独立子活动发 Item 1211（14 档门槛=数值定稿 v2.5.1，计分=酒馆费率卡）
