# AI 工作进化路径

> 来源：Zoe 内部分享 PPT（2026.04）

## 五阶段进化

```
偷懒 → 变成我 → 变成同事 → 模拟用户 → Skill 闭环
```

### 阶段一：偷懒（重复自动化）
把**烦死了**的重复任务交给 AI。

| 自动化任务 | 输出到 | 频率 |
|-----------|--------|------|
| 竞品日报（8 款游戏） | Obsidian（178 篇笔记） | 每天 |
| 社区舆情（3 个 Discord） | 钉钉群（早 7 点推送） | 每天 |
| 数据日报（DAU/收入/ARPU+环比） | 钉钉群 | 每天 |
| 行业日报（SLG 出海+AI 动态） | Obsidian | 每天 |

**核心理念**：减少前额叶负担——规划、记时间、checklist 交给 AI，脑子留给真正需要判断的地方。

### 阶段二：变成我（分析/追踪/排查）
AI 开始做"我才能做"的事：

| 能力 | 案例 |
|------|------|
| 帮我分析 | X3 付费优化 19→21 SKU、投放数据回归 |
| 帮我追踪 | 海域导量 3 节点自动提醒、P2 六月 28 功能甘特图 |
| 帮我排查 | 新服留存下降定位、付费异常拆到 SKU 级别 |

### 阶段三：变成同事（角色扮演）
AI 扮演不同岗位帮我检查：

| 角色 | 做什么 |
|------|--------|
| 数值策划 | 翻配置表算数值、对比投放节奏、验证 SKU 定价 |
| QA | 版本验收分析录屏、检查配置、排查数据异常 |
| PM | 排版本排期甘特图、追踪里程碑、管理功能列表 |
| 数据分析师 | 写 SQL 拉数据、做留存/付费回归、画趋势图 |
| 发行 | 查提审状态、推灰度、回复商店评论 |

**本质**：让别的部门的人来帮你审——策划配完 → AI 扮数值检查 → 扮 QA 自查 → 出错率大幅下降。

### 阶段四：模拟用户（预测/审视）
- **玩家视角**："如果我是新服第 7 天的中R玩家，这个礼包我买不买？"
- **数据预测**：新服 7 留掉了 25%~31%，14 留会跌多少？ARPPU 能补回来吗？
- **竞品预判**：竞品活动节奏分析 → 预判对手下一步

### 阶段五：Skill 闭环（串联工作流）
```
读活动排期(GSheet) → 读配置表(SVN) → 拉线上数据(Trino) → 三角验证 → 输出优化建议
```
以前运营做一整天的分析，现在一条命令跑完。

## 三种任务模式

| 模式 | 触发方式 | 价值 | 示例 |
|------|---------|------|------|
| 被动：自动跑 | 定时/Cron | 省时间·不遗漏 | 舆情/日报/巡检 |
| 半自动：AI 提醒 | 到时间问你 | 管注意力·不忘事 | 导量追踪节点 |
| 主动：问一句 | 随时对话 | 放大能力·降门槛 | "查 WOS 上周收入" |

## Skill 写作方法论

```
1. 发现一件事反复在做（关键：你得知道这件事该怎么做）
2. 跟 AI 说"帮我把这个流程做成 Skill"
3. AI 写脚本和配置
4. 试一下，告诉它哪里不对
5. 改几轮，Skill 就能用了
```

**例**：竞品日报最初每天手动让 AI 跑 → 一周后流程稳了 → 沉淀成 Skill → 之后全自动。

> 40 个 Skill 不是一天建的，是半年里一个一个攒出来的。
