# 工具选择与 tokenmaxxing

## 工具箱

| 工具 | 定位 | 用来做 |
|------|------|--------|
| **Claude Code** | 主力 | 跑任务、调 API、写脚本、串联一切 |
| **ChatGPT（小龙虾）** | 聊天搭子 | 写东西、Code Interpreter 分析表格 |
| **Cursor** | 改代码帮手 | 需要精细改代码的时候 |
| **Gemini** | 视频+长上下文 | 看视频、NotebookLM |
| **Sider** | 浏览器侧边栏 | 看网页随手问 |
| **悟空** | 中文搜索 | 中文搜索 |
| **妙计** | 快速文案 | 快速生成方案文案 |
| **Obsidian** | 知识库 | 本地知识库，一切归这里 |

> 小龙虾是聊天搭子，Cursor 是改代码帮手，Claude Code 是雇佣兵——脏活累活都丢给它。

## 为什么主力是 Claude Code

| 原因 | 说明 |
|------|------|
| 模型最合我 | 长文本理解、指令遵循、代码质量 |
| CLI 能接任何工具 | Skills/MCP 接一切系统 |
| 记忆系统 | 记得项目、术语、偏好 |
| 迭代快 | 桌面/VSCode/Web 都有 |

**网页版 AI vs CLI 版 AI 的差别**：
- 网页版："帮我查各服 DAU" → "你可以用 SQL 查哦"
- CLI 版："帮我查各服 DAU" → 写 SQL → 连数据库 → 跑完 → 返回结果+解读

## 模型怎么选

| 模型 | 擅长 | 我用来 |
|------|------|--------|
| Claude | 代码、复杂推理、Agent | 主力干活 |
| GPT-4o | 全能、看图、分析表格 | Code Interpreter + 写东西 |
| Gemini | 看视频（原生）、超长上下文 | 竞品视频、NotebookLM |

> 不纠结谁更强，看场景选工具，混着用就对了。

## 省 token 四招

| 策略 | 说明 |
|------|------|
| 简单事用便宜模型 | 翻译用豆包，别拿 Opus 干 |
| 一次说清楚 | 模糊 Prompt = 3 倍浪费 |
| 被动任务用模板 | 日报 Prompt 固定复用 |
| 本地资源替代 | 能搜 Obsidian 就不让 AI 猜 |

## 知识库理念：资源比模型重要

所有人都能用 Claude/GPT/Gemini，差别在你喂给它的东西。

| 只靠模型 | 模型 + 你的资源 |
|---------|----------------|
| "WOS 最近活动怎么样" → 过时的笼统废话 | 104 篇竞品笔记 + SensorTower + Discord 原文 → 精准分析 |

### AI 能读到的资源
- Google Drive（项目/配置文档）
- SVN（游戏配置表）
- Git（代码/脚本）
- 本地文档（Excel/分析报告）
- **Obsidian**（178 篇笔记：竞品/舆情/日报）

### Obsidian 优势
- 本地存储不怕泄露
- AI 每天自动写 5-6 篇
- 需要时直接读做分析
- 导入 NotebookLM 做问答
