🎨 X3 美术出图工作流总览

给所有跟 X3 美术出图打交道的同事的上手地图：我要出某类图 → 走哪条线 → 怎么跑起来 → 出完往哪放。 统一入口是 x3-media skill，后端是 GRFal。冷启动看这一篇就够上手。
统一入口 x3-media 后端 GRFal 异步 media-worker 更新 2026-06-24
4 条贯穿所有线的信条（先记这个，比记流程重要）：
  1. 方向先商量、不急着出图——排版/配色/元素先用文字或 ASCII mockup 跟需求方对齐，再烧 GRFal 时间，省返工。
  2. 每轮只改一处——锁住已认可的部分当 reference，prompt 只动一个变量，否则整体漂移。
  3. 出的都是给美术对齐方向的概念稿（纯 2D 端到端类除外）——最终资产美术会重绘，micro 细节别死磕。
  4. 真源 = 生成脚本——改图改脚本重跑，不手改产物；下次复制最接近的脚本改参数。

1 这套能帮你做什么（先看落地度）

GRFal 只能产 静态 2D 图 / 视频 / 3D mesh 毛坯 / 序列帧 / 音频；产不了 Spine 2D 骨骼动画，也产不了能直接进游戏的特效 prefab。先按落地度判断 AI 能不能帮你省美术：

落地度模块AI 能做到哪
✅ 端到端头像框 / 纪念卡 / 聊天表情 / 活动图标 / banner / 道具·资源·小图标 / 按钮 / UI框一个 worker 出成品 PNG + DK 注册即进游戏
🟡 只能做上游英雄皮肤原画 / 岛屿·主城皮肤 / 家具 / 装饰三件套（3D 模型）出原画锚 / FBX 毛坯当概念参考，带绑点·LOD·规范的 prefab 仍靠美术重做
❌ AI 做不了英雄皮肤 Spine 骨骼动画 / 航迹等特效 prefab搭骨骼+蒙皮+K帧 / 引擎里搭粒子，绕不过美术
⚠️ 换皮场景优先「找现成」不是生成：换皮投放的皮肤/家具/船皮肤绝大多数是游戏里已有的资源，去配置表(HeroSkin/FurnitureSkin/Item)+客户端搜出来；真搜不到才标「🔧 需新出」。

A. 单件 2D 图标/素材 → x3-media 的 type 路由

说人话给 x3-media，它自动匹配 type 走对应配方：

C. UI 切图 / 拆素材 → 先分清「切现成的」还是「切生成的」

「切图」有三条路线，第一反应应该是去游戏里找现成切片，不是 AI 切——能拿现成就别拆：

路线什么时候用怎么做
① 从 prefab 反查真实 sprite
(不切，直接拿)		目标元素游戏里已有（改造现有界面、拼真组件原型）目标界面 prefab 抽 m_Sprite GUID → 扫 .meta 反查 PNG → 拷进 _assets/。详见《X3客户端GUI知识》控件模式库
② ui_extract / ui_extract_fine
(AI 拆一张现成参考图)		有一张参考图(竞品/旧界面)，要从里面抠出 UI 元素x3-media type=ui_extract：去文字 + 分档 resize + pngquant 量化到 1-80KB；毛发/半透明边缘走 ui_extract_fine（双底差分反推 alpha）
③ Morphix E 元素拆分四步
(把自己生成的效果图切成切片)		效果图认可后，要拆成 F 固定层的可用切片交美术/程序见下方四步流水
③ Morphix E 元素拆分四步流水（全 AI+脚本，零手裁；2026-06-11 竞猜界面 F 固定层首跑即成 8 件）：
  1. 雪碧图：generate_image(gpt) + 效果图当 reference + buildSplitPrompt（关键：#B8B8B8 灰底防白边 / 原子化 / 严格去重 / 无文字版 / 行排列 20px 间距 / 不重绘）。
  2. 去底：remove_background（sync 可用，⚠️ result 是嵌套 list [[url],null] 要解开；验真透明 RGBA alpha 0-255）。
  3. 切块：PIL 投影递归分割（行带 gap18 → 列段 → 粘连块二次行分割）；脚本 split_blocks.py。
  4. 归档：按槽位命名（F1_xx_九宫格_无字.png）+ README 映射规格表（目标尺寸）。
★ 切图 vs 拆槽别混：这里的「切图」=把一张图拆成独立素材切片（F 固定层用）；§5.3 的「S6 拆槽」=把效果图按更新频率分四层（F固定/T主题/B背景/TXT文本）的交付策略。先拆槽定层，再对 F 固定层用本节的切图流水。

认证（2026-06-12 起）走 ~/.config/grfal-api/token_store.json 的 Bearer access_token（长效到 2036），什么都不用设，call_grfal 自动 fallback。旧 cookie 注入法已废

认证铁律：脚本开头 os.environ.pop('GRFAL_COOKIE', None) 兜底（残留过期 cookie 优先级更高会顶掉好 token→401）；下载结果也带 Authorization: Bearer。验活 GET /api/tasks?page=1&page_size=1 带 Bearer 应 200。

执行模式：异步 media-worker（强制）

  • 主 agent 不得直调 call_grfal / 生成脚本——一律派 media-worker 子 agent 后台异步跑。
  • 一次原子 GRFal 调用 = 1 个独立 worker（3 张图 = 3 个并发 worker，不在一个 worker 里循环）。
  • 派发时 run_in_background: true + model: "sonnet"（必须显式指定，否则继承不可用模型整批挂掉）。
  • task_id = 单一事实来源，主 agent 只读 state/tasks/<id>.json，不接子 agent 长文本。

GRFal 调用骨架（worker 内）

os.environ.pop('GRFAL_COOKIE', None)
sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8', errors='replace')  # 防 GBK 控制台崩

# generate_image / generate_video 是 long_running:
#   必须 --submit-only 拿 task_id → --check-task 轮询；禁 --sync(300s 硬超时)
#   gpt 带参考图常 4-9min，video 5-12min
#   --file reference_images=path  可重复写 = 多图参考(append)
#   result 可能嵌套 list 要递归 flat；URL 拼 base，下载带 Bearer

模型选型速查

模型强项用在
gpt保角色身份/质量好，但慢(4-9min/2张)角色换装、风格保真要求高的 reskin
gemini快/稳/保 UI 布局好UI 换皮、结构确认稿、改造现有界面 reskin
seedance2D 立绘脸稳 + 运动足皮肤展示视频首选
klingfflf 首尾帧锁得紧、循环无缝视频需严格无缝循环时（但 2D 脸必张嘴/warp）

★ 出图后默认核验（所有换皮出图后必做）

拿对应参考图(复用源)核 4 项，不只看尺寸（脚本 PIL getbbox() 算 bbox+四边 margin+占比逐项对参考）：

① 尺寸
W×H = 参考。gpt 默认出 1024~2048 大图，必须缩到目标
② 格式/透明
图标=RGBA 真透明，背景=RGB。gpt「透明底」常是假透明，验 alpha.min()<255
③ 四边边距
= 参考，决定 sprite 在槽位的对齐
④ 主体占比
= 参考，不能简单居中（岛底座要落格子同位）

4 四条重型工作流详解

A. 角色皮肤换装线（gpt）

  1. 选基础英雄 → 找客户端立绘 Role/FullLength/Role_F_{id}.png。
  2. 本人立绘当 reference，prompt =「保身份(脸/发/身形/画风)，只换造型(服饰/道具/姿势)」→ 出 R1。
  3. 后续每轮用上一版选中图当 ref，逐轮只改一处，版本名带语义后缀。
  4. 用户给的「质量好但带违规元素」的图：以它当质量/姿势锚 ref，prompt 逐条列要改的设计点（红→白金 / 国徽→奖杯 / 去脸颊国旗）——既得质量又合规，FINAL 常这么来。

坑：基底选错会白跑，动手前确认 ref 是哪一版；IP 合规用原创风格化不精确复刻商标。

B. 活动 UI 换皮效果图 + S1–S6 编排（gemini 为主）

S1 结构设计(不出图)→ S2 结构确认稿→ S3 风格化→ S4 细节迭代→ S5 回填闭环→ S6 拆槽
  • S1：收参考图 → 拆「每张取什么基因」→ ASCII mockup + 要素清单对齐 → 拍板布局分区/交互状态机/红线(IP·国家元素)。
  • S2(gemini 快)：多参考合成出 2 候选，确认「结构对」。只验结构不追风格。
  • S3(Morphix buildReskinPrompt 职责切分)：图#1=S2 结构稿(视觉全 ERASE 只留 WHAT/WHERE) + 图#2=本游戏真实 UI 截图(design system 整套照搬)，模型 gpt。图#2 优先用验收截图库里与目标框架最接近的一张。
  • S4：每轮只改一处；★ 变体覆盖检查——规则里写的 UI 元素(角标/锁定态/结算态)必须有图可对，只出默认态 = 漏交付。
  • S5：效果图路径回填策划案「界面交互」段 +「美术需求」页签（双参考交付）。
  • S6：拆槽（见 §5.3）。

C. 改造现有界面效果图 = 真组件拼装 → AI reskin 五步法（首选）

给「已上线界面加/改元素」出落地效果图，比纯 AI 整张生成(布局飘)和纯 CSS 拼装(接缝/光影不统一)都强。

  1. 拆真组件：目标 prefab 抽 m_Sprite GUID → 反查 PNG → 真 sprite 拷进原型 _assets/。
  2. CSS 拼装定精确布局(=图#1 结构稿)：HTML 用真组件按真实坐标拼，新控件也摆到位；chrome headless 截设备区。
  3. (选) 出适配新控件的底图。
  4. ★ AI reskin 合成成品（关键步）：reference_images=[图#1拼装截图, 图#2真实界面]，gemini，9:16；图#1 只取布局/文字/控件位置(视觉全 ERASE)，图#2 = design system 整套抄；新元素显式声明「按图#2 视觉语言原生渲染」。
  5. S6 拆槽交付。
★ 血泪教训：拼装(图#1)看着糙完全没关系（它的视觉在第 4 步全被 ERASE，只取布局）——别在第 2 步死磕拼装精度，赶紧进第 4 步。把结构锚当成品交 = 必烂。
变体：图#1 来源是竞品横屏截图但要竖屏 → 先 buildFlipPrompt 横竖转换再 reskin（两段式）。

D. 皮肤展示视频线（替代 Spine）

  • 目标格式：SBS 透明视频(左半彩色+右半 alpha 烧成白模 on 黑底)，Unity shader 重建 alpha。不透明 showcase 走普通满幅 mp4。
  • 统一 1080×1920(9:16) 直出，不做后期裁切（客户端拍板）。
  • 模型 seedance 首选（脸稳+运动足；kling 给 2D 脸必张嘴/warp 压不动）；kling 仅用于需严格无缝循环。
  • 无缝循环：kling ref_types="首帧图像,尾帧图像" 同图传两次。
  • 透明化链路：generate_video → video_remove_background(.webm) → export_sbs_video（quality=CRF 不是百分比！q12 是甜点，传 95 会糊成马赛克）。
  • 演出心法：撩不撩的根在起点图；眼神给镜头=死活线；写「一条贯穿全身的动作线」不写零件清单(否则糖人)；压制词(minimal/never)会叠加压死动作；prompt 长度=注意力预算。
  • 视频处理一律用项目自带 client/Tools/VideoTools/ffmpeg/ffmpeg.exe（cv2 写不了可控码率，且中文路径静默失败）。

5.1 归档路径

产物根路径

5.2 命名与版本

  • 命名 {活动名}_{模型}_{时间}_v{N}.png，版本带语义后缀(_r5白金加徽章)，定稿标 _FINAL。
  • 真源=生成脚本：改图改脚本重跑，不手改产物；下次复制最接近的脚本改 REF/PROMPT/OUT。

5.3 ★ S6 图槽模板化拆解（效果图认可后、交美术前必做）

效果图别直接交美术——先拆槽，否则文字/渐变烧死在图里 = 美术量 × 场次 × 语言爆炸：

层内容更新成本
F 固定层1 套切图，无文字，永不换一次性
T 主题层按复用单元出（对称设计一图翻转省一半）每主题一次，维表驱动
B 背景层整图按阶段几张（最便宜的仪式感）低
TXT 文本层全 TextKey 叠加，i18n 一次翻，不烧图近零

成本判断：「出图便宜，改程序贵」——渐变/材质烧进图里批量出，不做程序染色。槽位示意图独立画(PIL 线框 / HTML+chrome 截图)，别叠在效果图上；配槽位尺寸规格表(px/比例/数量/基准 1080×1920)。

5.4 落地到 X3 客户端的链路

1. GRFal 生图 → 下载到 KB\产出-本地化与美术\X3\{主题}\
2. 选定 → 复制到 client\Assets\Res\UI\Spirits\{对应目录}\
3. Unity import 生成 .meta(GUID)
4. 编辑 DisplayKey asset 加 DK→path（或 Unity Ctrl+T 面板）
5. git commit + push（feature branch + MR，dev 受保护）
6. gdconfig 配置表字段 = 新 DK
7. push gdconfig + jolt 自动导表

5.5 ★ 交美术的双参考（换皮活动缺一不能出图）

  • 老图参考·格式结构 = 复用源旧界面图（告诉美术版式/尺寸怎么对齐）。
  • 新图参考·投放物元素 = 本期实际投放新东西的资源图（告诉美术新主题该画成啥样）。背景类无独立投放物 → 挂节日主视觉锚。

投放物绝大多数是游戏现成资源，去配置表+客户端搜出来，写完逐路径验存在。

5.6 知识库回写（收工自检）

  • 新规律/踩坑/新链路 → 当场沉淀进知识库/本文档。
  • 大案子产三件套：决策记录(按模块·为什么+被否决方案) / 换皮清单(骨vs皮) / 产物目录 FINAL 标注。
  • 判据 = 新人冷启动能否在「修BUG/接管模块/换皮」任一入口秒懂；禁止只留时间线流水账。
