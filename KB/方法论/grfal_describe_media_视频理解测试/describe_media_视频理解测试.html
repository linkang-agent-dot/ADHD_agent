颗粒度结论

维度表现实测证据
分镜 / 场景切换强用 transitioning to / switching to / returning to 标出 5 个镜头，按时序串成情节
动作序列强逐动作 gerund 流（开炮→落币→吐币→闪光→推币→连击→刷分），先后正确
具体数值 / 文字强OCR 出 “99 silver coins / 1,980 pink gems”、连击数、分数刷新
UI 元素 / 位置细按钮、slider、popup、右下连击、余额刷新都点名
美术 / 镜头风格有“3D animated game style” / “static high-angle camera”
边界（自己搭时要补的）： ① 输出是一整段事件流 caption，没有逐秒时间码（要 [0-2s][2-5s] 得另用 seedance_prompt_reverse 或抽帧烧时间戳）。 ② 无 prompt 通道——只客观描述，不做问答 / 不判断意图（“是不是付费诱导”这种它不答）。 ③ 长视频是否丢镜头未测，建议配 extract_video_frames + 场景检测补帧。
