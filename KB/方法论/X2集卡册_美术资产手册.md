## 二、单卡图（54张）生成规范

### 2.1 尺寸与格式

| 项目 | 规格 |
|------|------|
| 尺寸 | **640 × 900 px** |
| 比例 | 3:4（竖版） |
| 格式 | PNG，无边框 |
| 边框 | **禁止**，游戏客户端自动叠加共用边框（DK: 1511020219） |

### 2.2 AI 生图流程（x2-media skill）

触发 `x2-media` skill，类型选 `card_gallery`。

**默认 prompt（一字不差拼在用户描述前）：**

```
game collectible card, vertical card format, no border no card frame no decorative frame, 3D rendered stylized cartoon, character full body or half-body centered, fantasy game character design, rich costume and prop details, themed background with soft environmental or dramatic lighting, rich color palette optional gold or metallic accents, avoid overwhelming bokeh or confetti particles keep effects subtle, clear silhouette and expressive face, album card illustration, consistent game UI asset style, full body visible in frame adequate margin no severe cropping, character centered with safe zone on all sides do not place subject at frame edge, face and expression consistent with reference character design
```

**生图参数：**

```json
{
  "aspect_ratio": "3:4",
  "reference_images": ["构图参考图（见下方）", "人物立绘/设定图（若有）"]
}
```

**构图参考图（必传）：**

```
C:\ADHD_agent\.cursor\skills\x2-media\scripts\grfal-api\card_gallery_reference\template_640x900.png
```

若不存在，先生成：
```bash
py -3 C:\ADHD_agent\.cursor\skills\x2-media\scripts\grfal-api\card_gallery_reference\gen_template.py
```

### 2.3 生图注意事项

- **系列卡动作要变化**：同组6张卡，每张人物动作/场景不重复（举手/持物/微笑/转身等）
- **人物全身入画**：保留上下边距，避免头顶或脚部被裁切
- **有人物立绘时**：务必传入人物参考图，确保面部一致
- **风格参考目录**：`{X2_CLIENT}\Assets\x2\Res\UI\TextureNew\MiniGame\pic\CardGallary_*.png`
  - 要求：环境光、叙事姿势、多样场景；避免过度粒子/光斑

### 2.4 星级与画面调性参考

| 星级 | 出现组 | 画面调性建议 |
|------|--------|------------|
| ★3 | G1 全部；G2-G7 前几张 | 日常轻松，普通场景 |
| ★4 | G2-G9 中段 | 有叙事张力，动态感 |
| ★5 | G5-G9 末位 | 高光时刻，戏剧性光效，但特效要克制 |

## 三、DK 类型与规格

### 3.1 单卡图 — Portrait 类型

| 项目 | 规格 |
|------|------|
| DK 类型 | **Portrait** |
| 存放路径 | `{X2_CLIENT}\Assets\x2\Res\UI\TextureNew\MiniGame\pic\` |
| 文件命名 | `CardGallary_{节日缩写}_{序号}.png` |
| maxTextureSize | **2048** |
| textureFormat | **51**（ASTC_6x6） |

> ⚠️ Portrait 类型的 meta 规格与 Icon 不同，**不能**用 Icon 的 256/51 参数，会导致高清图被压坏。

### 3.2 卡包图标 — Portrait 类型（同上）

卡包图标（3张）也用 Portrait 类型，但实际是小尺寸图标，注意：

| 项目 | 规格 |
|------|------|
| DK 类型 | **Portrait**（按现有案例） |
| 存放路径 | `{X2_CLIENT}\Assets\x2\Res\UI\DKUpload\Portrait\DK_{key}.png` |
| 原始尺寸 | 256×256（参考圣诞卡包图标） |
| textureFormat | 50（PVRTC_RGBA4）|

> 占星节卡包图标 DK: 1511031091/1511031092/1511031093（交付时为临时圣诞占位图，待美术替换专属图标）

### 3.3 Book / Group 图 — 具体类型待确认

Book 和 Group 的封面/背景图通常是活动专属大图，可能走 `Portrait` 或专用 Atlas 类型。**每次确认美术提供的图片路径，再用 x2-dk-manager 自动判断类型**。

## 五、DK 录入流程（x2-dk-manager skill）

### 5.1 前提

- Unity 编辑器已打开 x2client/client 工程
- 在正确分支上（集卡册专用分支）

### 5.2 单卡图批量录入（54张）

```
触发：录入DK Portrait, {图片路径}
或丢图片：[粘贴图片] Portrait
```

**批量时每次确认 key 是否连续，防止间隔错误。**

典型命令流：
1. `x2-dk-manager` 自动取全局最大 key + 1
2. 复制图片到 `{X2_CLIENT}\Assets\x2\Res\UI\DKUpload\Portrait\DK_{key}.png`
3. 等 Unity 生成 .meta，读取 GUID
4. 修正 .meta → **textureType:8 / 2048 / 51**（Portrait·集卡册卡片规格）
5. 写入 `Display_Portrait.asset`
6. Ctrl+Shift+E 导出
7. 验证 `Path_Portrait.asset` 是否有对应 key

### 5.3 配置表写入时机

**美术先出图 → DK录入完成 → 拿到 DK key → 再填配置表**

不要在美术出图前就填占位 DK，否则配置和实际资源对不上。

## 六、美术资产与配置的对应关系

```
美术提供图片
    ↓
x2-dk-manager 录入 DK
    ↓ 得到 key（如 1511031037）
    ↓
填入 1123 CardGallary 的 DisplayKey 列
    ↓
1123 中所有卡共用边框 DisplayKey1 = 1511020219（不换）
    ↓
1107 Group 的 DisplayKey 填该组封面 DK key
    ↓
1108 Book 的 DisplayKey / DisplayKey1/2/3 填册主图 DK key
```
