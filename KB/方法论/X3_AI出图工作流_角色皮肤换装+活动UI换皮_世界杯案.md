## 1. 整体流程（两条独立线，可并行）

```
角色皮肤换装线:  英雄本人立绘 → 换装概念稿(多轮迭代) → 主稿_FINAL
活动UI换皮线:    原活动UI截图 + 皮肤主稿 + 道具图 → 换皮效果图(多轮迭代) → 效果图_FINAL
```

**核心原则（贯穿两条线）**：
1. **方向先商量、不急着出图**：排版/配色/元素这类方向问题，先用文字(或ASCII mockup)跟用户对齐，再出图。省GRFal时间+少返工。
2. **每轮只改一处**：锁住已认可的部分(reference图)，prompt里只动一个变量。改多了会整体漂移。
3. **都是方向概念稿**：明确告诉用户这是给美术外包对齐方向的稿，最终资产美术会重绘——micro细节(姿态自然度等)不必死磕。

## 4. ★踩坑清单（最值钱）

**GRFal 调用层**：
1. `generate_image` 是 **long_running** → 必须 `--submit-only` 拿 task_id 再 `--check-task` 轮询；加 `--sync` 会300s硬超时。
2. 轮询脚本里 `print` 中文 → Windows GBK 控制台 `UnicodeEncodeError` 崩(任务其实没崩,只是print炸)。必须 `sys.stdout=io.TextIOWrapper(sys.stdout.buffer,encoding='utf-8',errors='replace')` 或 `PYTHONIOENCODING=utf-8`。
3. cookie 要手工注入 `GRFAL_COOKIE` env(call_grfal不自动读config)；下载也要带cookie否则401。
4. **多图参考**：`--file reference_images=` 重复写多次=多图(append生效)，可同时喂 UI布局+角色+道具。
5. 模型选择：**gpt** 保角色身份/质量好但慢(4-9min/2张)；**gemini** 快/稳/保UI布局好。→ 角色换装用gpt，UI换皮用gemini。

**角色换装层**：
6. **基底选错会白跑**：本次R4错用红版当基底→TaskStop掉重来。动手前确认 reference 是哪一版。
7. 「质量好但带违规元素」的图怎么用：用户常会给一张**渲染质量最好但违规**(带国徽/国旗/红队色)的图。解法=**以它当质量/姿势锚 reference**，prompt里**逐条列出要改的设计点**(红→白金/国徽→奖杯/去脸颊国旗/去商标字)——既得质量又合规。本次FINAL就这么来的。

**UI换皮构图层（最容易翻车）**：
8. **角色易被画太大** → 顶到/遮挡顶部UI文字 + 比例不协调。解法=**用原UI当人物尺寸位置锚**+prompt明确「中等场景插画、置于顶栏+标题文字带**下方**、不overlap任何UI文字、不oversize」。
9. **角色与道具(礼盒)都居中会叠住**。解法二选一：①角色偏一侧+道具居中 ②角色保持比例居中、道具**缩小+下移**到脚前(只压脚踝不挡身体)。本次最终用②(因偏侧会把人物缩小)。
10. **中间留白太空** → 加个**小**道具(小奖杯)填背景，但务必小、不抢主焦点。

**IP / 合规层**：
11. 赛事符号(大力神杯)用**原创风格化**(奖杯+月桂)，**不精确复刻** FIFA 商标造型，规避IP。
12. 用户说「去国家元素」= 去**国旗/国徽/国家队配色**；**赛事奖杯印花不算**国家元素，可保留。

**协作节奏层**：
13. 方向问题(排版/配色)**先商量再出图**(用户明确说过"先不出图,商量")。
14. 每轮**只改一处**，版本文件名带**语义后缀**(`_r5白金加徽章` / `_v7_V5比例+礼盒下移+小奖杯`)，主稿标 `_FINAL`，便于回溯。

## 7. Morphix buildReskinPrompt 实战补充（2026-06-10 竞猜界面 X3 风格化）
结构图→目标游戏 UI 风格的换皮，用 Morphix 职责切分法（prompt 全文在 `KB\方法论\Morphix换皮工具逆向_prompt库.html`）：
- **图#1=布局/内容源**：只抽结构排布+文字+元素语义，"every visual aspect must be ERASED. Only the WHAT and WHERE survives"
- **图#2=design system 源**：目标游戏**真实 UI 截图**（如 orig_ui.png），视觉语言整套照搬
- **X3 调优**（templates.js 笔记）：BAN "painterly/笔触"(涂抹感)；金色装饰列举多种别单点名(否则全退化成一种)；**参考图能说明的不写进 prompt**（只写量级/分布约束）
- 流程：先用通用参考出**结构确认稿**（用户点头"结构很对"）→ 再用结构稿当图#1 做风格换皮。结构与风格分两步迭代，比一步到位稳。

## 8. ★策划案内嵌出图完整流程（S1-S5 编排 · 2026-06-10 固化）
> 写策划案到「界面交互」段、需要给新界面出效果图时，按此流程走（Morphix 各功能的挂点也在这）。

**S1 结构设计（对齐,不出图）**
收集参考图（竞品/其他游戏,存 `KB/产出-本地化与美术/X3/{主题}/参考N_*.png`）→ 拆「每张参考各取什么基因/不取什么」→ ASCII mockup + 结构要素清单跟用户对齐 → 拍板结构三件套：布局分区 / 交互状态机 / 红线（IP·国家元素等）。

**S2 结构确认稿（通用风格,快）**

**S3 风格化（Morphix 换皮法）**
buildReskinPrompt 职责切分：图#1=S2结构稿（视觉全 ERASE,只留 WHAT/WHERE）+ 图#2=**本游戏真实 UI 截图**（design system 整套照搬）+ X3 调优（§7）。模型 gpt（风格保真）。

★**改造现有界面(非新界面)时的特例(2026-06-18 英雄养成手册双版本案)**：当目标是「在某个已上线界面上加/改元素」(如养成手册加双页签)，**真实界面截图=唯一结构底，走 img2img 忠实复刻+只插新元素，别整张 AI 重画**。整张重画(哪怕喂真实截图当参考之一)会"风格对但底飘"——重排格阵/换主视觉/丢真实细节(实测 v1 用 gemini 整生即飘)。正解：①拿真实截图当**主参考**(prompt 明确"reproduce FAITHFULLY, the ONLY change is 插入X")；②若 img2img 仍重画，升级到 `image_mask_edit` 真·局部 inpaint(只动那块像素，其余不碰)；③或 Morphix E 元素拆分把真实 UI 拆成原子件→素材化 HTML/PIL 拼装(§11+交互原型素材化工作流)。真实界面截图优先去 `C:\Users\linkang\Pictures\X3验收\` 找(常有该功能的实拍图)。

## 9. ★效果图→落地资产:图槽模板化拆解法(S6 · 2026-06-10 竞猜界面实战)
效果图被认可后别直接交美术——先做**拆槽**,否则文字/渐变烧死在图里=美术量×场次×语言爆炸。
- **四层拆解**: F固定层(1套切图,**无文字**,永不换) / T主题层(按复用单元出,如按队32张,同队全赛程复用;**对称设计一图翻转**省一半) / B背景层(整图按阶段几张,"换背景=换氛围"最便宜的仪式感) / TXT文本层(全TextKey叠加,i18n一次翻完,**不烧图**)
- **成本判断(用户原话级教训)**: "出图便宜,改程序贵"——渐变/材质效果**烧进图里**按复用单元批量出,不做程序染色/gradient(我第一反应推荐程序染色被否,错判成本结构)
- **维表驱动**: 主题层做成维表(如 TeamID|名TextKey|横幅DK|徽DK),运营每场只填ID,视觉/文本自动带出 → "只换主题视觉+本地化文本=完成一场设计"
- **槽位示意图独立画、不叠在效果图上**(draw_slots_v2.py 模板在 KB/产出-本地化与美术/X3/世界杯竞猜界面/): PIL纯线框=左侧四层色块槽位+右侧产出清单表+维表schema+运营三步——叠在效果图上画框会乱(被用户否),独立线框才干净。交付两件套=拆解示意图(结构/产出)+方向定稿FINAL(观感)。
- **槽位规则化+规格表(2026-06-11补)**: 效果图的主题槽位(VS区/主视觉)要用**规则矩形**(轴对齐圆角,无斜切)——方便整图直填槽;交付美术前必须配**槽位尺寸规格表**(尺寸px/比例/九宫格vs整图/数量/出图基准1080×1920+落地以prefab实测回校),否则美术按效果图自由发挥尺寸全乱。
- 接 §8 流程: 此为 **S6**——S5回填后、交美术前的工程化拆解步。

## 11. ★卡片/UI 金框换皮：用 image_layered 抠框，别手工色键(2026-06-17 纪念卡实战)
换皮时要把「劣质/AI直出金框」换成标准金框，**抠框这步交给 GRFal `image_layered` 工具**，别用 PIL：
- **✅ 对的链路**：`image_layered`(任一张干净历史卡)→ 输出层里那张稀疏的 = **干净金框层(双线+四角对称卷草+透明中心，零污染)**，存为可复用模板 → `image_resize` 把新画面调到同尺寸 → 两层 alpha 叠合(金框在上盖住旧框)→ `image_resize` 调回目标尺寸。
- **❌ 别走的三条**：① PIL 跨卡中位数+色键抠框→历史人物身体污染四角、金色糊；② 程序重绘双线框+镜像角→角纹不够精；③ 反向「拿干净历史卡换中心」→ host 卡内底颜色从金框内缘渗出色边。
- **⚠️ `synthesize_card` 填不进内容**：它把输出 alpha 设成 frame 的 alpha(中心透明)，不做「洞里填图」；合成老实用分层抠+alpha叠合，别指望这工具。
- 实战归档：`KB/产出-本地化与美术/X3/世界杯纪念卡/_目录说明.md`(含模板 `_layered/L0.png`)。

## 12. ★改造现有界面效果图：真组件拼装 → AI reskin 五步法（2026-06-18 养成手册双版本案·验证通过·首选）
> **场景**：要给"某个已上线界面加/改元素"(如养成手册加双档切换+封顶格)出落地效果图。**这是该场景的首选链路**，比"纯AI整张生成"(布局飘离真实界面)和"纯CSS/真素材拼装"(接缝/光影不统一、活DOM叠烤死槽对不齐)都强——**各取所长：CSS拼装负责"位置全对"，AI reskin负责"成品观感统一"**。
>
> **核心洞察**：**真素材拼装截图 = 远胜线框的"图#1结构稿"**。线框要AI猜内容/比例，而真组件实摆的拼装=位置/文字/比例全是确定的，AI只需换一层皮。

**五步**：
1. **拆真组件**(prefab库)：目标界面 prefab 抽 `m_Sprite` GUID → 扫 .meta 反查 PNG → 真实 sprite 拷进原型 `_assets/`。方法详见 `活动程序开发\X3客户端GUI知识.md` 控件模式库「从prefab反查真实sprite」。
2. **CSS拼装定精确布局**(=图#1结构稿)：HTML 用真组件按真实坐标拼出布局；新控件(切换栏/封顶宽格)也摆到位。可顺带加交互(购买/二选一互斥/领取)做活原型供"定交互"。chrome headless 截设备区 = 图#1。
3. **(选)出适配新控件的底图**：原界面没有放新控件的位置时，以老底图为模板 img2img 加槽位(如 header 凹槽带)；活DOM叠烤死槽对不齐→PIL竖扫测槽坐标精确贴入+去活元素自身背景框(详见 GUI知识控件模式库)。
4. **★AI reskin 合成成品**(Morphix buildReskinPrompt 职责切分 · 关键步)：
   - `reference_images=[图#1拼装截图, 图#2真实游戏界面]`，模型 **gemini**(保布局；要更强风格保真可加跑 gpt 一版对比)，aspect 9:16。
   - **图#1 只取 布局/文字/控件位置/图标语义，视觉全 ERASE**；**图#2 = design system 整套抄**(背景/面板/边框/按钮/格子/配色/字体)。buildReskinPrompt 全文见 `Morphix换皮工具逆向_prompt库.html`。
   - **新元素显式声明**：图#1 里有而图#2 没有的元素(切换栏/封顶宽格)，prompt 明确"按图#2 视觉语言原生渲染"(如木金分段控件嵌进羊皮纸 header / 封顶=图#2 奖励格的加宽金紫发光版)。
   - **X3 调优**(必加)：禁 painterly/笔触；金饰列举多种让模型按 ref 灵活挑(别单点名 L 角包)；中文文案逐字保留；参考图能说明的别写进 prompt。
   - → AI 把精确布局用真实视觉重绘成**一张统一无缝成品**，自动消除拼装的接缝/对齐/光影不统一。
5. **S6 拆槽交付**：成品认可后→四层拆解(F固定/T主题/B背景/TXT文本)+槽位规格表，给美术/程序(见 §9)。已有真组件PNG的，拆槽=组织真组件清单+标"需新出"项，不必再AI切。

**实证产物**：养成手册双版本 `KB\产出-本地化与美术\X3\英雄养成手册\英雄养成手册_双版本_效果图_AI合成v1_reskin.png`(成品) + `_layout稿_图1_拼装布局.png`(图#1) + 拼装原型 `KB\产出-交互原型\X3_英雄养成手册\*_真素材拼装_v1.html`。

### 12b. 变体：竞品/异方向参考 → 横竖转换 → reskin 两段式（2026-06-18 养成手册二选一弹窗案）
当**图#1的来源是竞品截图且方向不对**（如竞品是横屏 BP 二选一，我们要竖屏弹窗）：别直接 reskin（方向不对硬塞会变形），**先转方向再 reskin**：
1. **第一段·横竖转换**：Morphix `buildFlipPrompt`(全文见 Morphix prompt库 §D；横→竖=把竖→横分支的方向词镜像替换)。reference=竞品截图，gemini，aspect 目标方向。**只翻布局不换风格**——关闭键/货币/标题按目标方向规范重排、网格列数 reflow(横4-6列→竖2-3列)、轴向翻转、保两选项并列可比、文字逐字留、禁"只拉宽不翻轴"。产出竖屏结构稿。
2. **第二段·X3 reskin**：把第一段竖屏结构稿当**图#1**，真实 X3 界面当**图#2**，走 §12 步骤4 的 buildReskinPrompt 职责切分；可在 prompt 里顺带**套内容映射**(竞品档位名→我方版本名)+**删/改元素**(如删价格数字)+**加新文案行**(如二选一描述)。
> 实证产物：竞品高级/豪华通行证(横)→`二选一弹窗_竖屏转换_stage1.png`(竖结构)→`二选一弹窗_X3风格设计稿_v1[_alt1].jpg`(X3风格成品，alt1更原生)。要点:竞品奖励图标会残留→第二段prompt强调用图#2格子风格,实际道具图配置时换;竞品业务文案(如"达到45级返还")要改成我方口径。
