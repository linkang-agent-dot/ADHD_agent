# X3 美术出图工作流总览

> 唯一入口文档：X3 现阶段所有 AI 出图的分类 / 主干流程 / 出图后沉淀。
> 冷启动一个新 agent，读这一篇就能知道「该走哪条线、用什么工具、出完往哪放」。
> 各条线的实战细节深挖见文末「关联深挖文档」。
> 更新：2026-06-24

---

## 0. 一句话地图

```
意图分类 → 选线（端到端2D / 重型工作流 / 上游概念稿）
        → 走 x3-media（GRFal 主后端，派 media-worker 异步出图）
        → 出图后核验（尺寸/透明/边距/占比 四项对参考）
        → 沉淀（归档KB + 脚本为真源 + S6拆槽 + 落地client+DK + 回写知识库）
```

核心信条（贯穿所有线）：
1. **方向先商量、不急着出图**——排版/配色/元素先用文字或 ASCII mockup 对齐，再烧 GRFal 时间。
2. **每轮只改一处**——锁住已认可部分当 reference，prompt 只动一个变量，否则整体漂移。
3. **都是给美术对齐方向的概念稿**——最终资产美术会重绘，micro 细节不死磕（纯 2D 端到端类除外）。
4. **真源=生成脚本**——改图改脚本重跑，不手改产物。

## 1. 出图分类（两个维度）

### 维度一：按「AI 落地度」分三档（决定 AI 能不能省美术）

x3-media/GRFal 只能产 **静态 2D 图 / 视频 / 3D mesh 毛坯 / 序列帧 / 音频**；产不了 Spine 2D 骨骼动画，也产不了可直接进游戏的特效 prefab。

| 档 | 模块 | AI 能做到哪 |
|---|---|---|
| ✅ **端到端**（纯 2D PNG） | 头像框 / 纪念卡 / 聊天表情 / 活动图标 / banner / 道具·资源·小图标 / 按钮 / UI框 | 一个 worker 出成品 PNG + DK 注册即进游戏 |
| 🟡 **只能做上游**（出概念/毛坯，美术重做） | 英雄皮肤原画 / 岛屿·主城皮肤·家具·装饰三件套（3D 模型） | `generate_image` 出原画锚 / `generate_3d` 出 FBX 毛坯，带绑点·LOD·规范的 prefab 仍靠美术 |
| ❌ **AI 做不了** | 英雄皮肤 Spine 骨骼动画 / 航迹等特效 prefab | 搭骨骼+蒙皮+K帧+导出 .skel/atlas / 引擎里搭粒子，绕不过美术 |

> ⚠️ 换皮场景优先「找现成」不是生成：换皮投放的「新东西」（皮肤/家具/船皮肤）绝大多数是游戏里已存在的资源，去配置表(HeroSkin/FurnitureSkin/Item)+客户端搜出来；搜不到才标「🔧需新出」。

### 维度二：按 x3-media「类型 type」分（决定走哪套出图配方）

统一入口 = **x3-media skill**（`~/.claude/skills/x3-media/SKILL.md`），按意图路由到对应 type 配方：

| type | 用途 | 关键约束 |
|---|---|---|
| skill_icon 技能图标 | 技能表查询 + 后处理贴边 | 256×256，必跑 postprocess |
| card_gallery 集卡册 | 确认人物 + 图鉴卡 | 640×900，带框非透明 |
| march_emoji / dynamic_march_emoji 行军表情 | 贴纸式表情包 | 透明底 256×256；动态=4×4 半身循环逐帧 |
| achievement_badge 成就徽章 | 1中央+5底板 | 128×128 |
| game_video 游戏视频 | 视频生成+处理+Unity保存+C#代码 | 见视频线 |
| ui_extract / ui_extract_fine UI素材提取 | 从参考图拆 UI 元素去文字 | 分档 resize+pngquant 量化到 1-80KB；精细拆走双底差分 |
| activity_icon 活动图标 | 活动入口/奖励图标 | 透明底 256×256 |
| effect_texture 特效贴图 | Unity 粒子/特效剪影贴图 | RGB全白+Alpha蒙版，2^N 正方形 |
| button 按钮 | 多色/多状态/多形状 UI 按钮 | 透明底，9-patch 兼容 |
| uiframe UI框/背景 | 头像框/弹窗背景/标题条/列表项底 | 9-patch 兼容 |
| general 通用 | 生图/修图/生视频 | 查 default-styles |

**★透明化硬规则**：行军表情 / 各种 ICON / 道具类 DK 资产 **默认走双底差分**（`transparify_asset.py`：生白底→生黑底→反推 alpha），**禁用** `generate transparent` 参数和 `remove_background`（会白边/穿洞/吃毛发）。例外：effect_texture 走 Alpha 蒙版；ui_extract 是拆现成图；card_gallery 带框非透明。

### 维度三：按「重型工作流」分（跨多步、多轮迭代的大活）

| 重型线 | 场景 | 核心打法 |
|---|---|---|
| **A. 角色皮肤换装** | 英雄皮肤换装概念稿 | 本人立绘当 ref → 逐轮只改一处 → 质量锚+逐条改设计出 FINAL；模型 **gpt**（保身份） |
| **B. 活动 UI 换皮效果图** | 新活动界面效果图 | 多图参考(原UI布局锚+皮肤主稿+道具) → gemini(保布局) → S1-S6 编排 |
| **C. 改造现有界面效果图** | 给已上线界面加/改元素 | **真组件拼装→AI reskin 五步法**（首选）：CSS拼装定布局=图#1 + 真实界面截图=图#2 → buildReskinPrompt 职责切分合成 |
| **D. 皮肤展示视频** | 替代 Spine 的皮肤展示视频 | seedance 首选(脸稳+动得足) → SBS 透明化 → 1080×1920 直出不拼接 |

### 维度四：UI 切图 / 拆素材（先分清「切现成的」还是「切生成的」）

「切图」有三条路线，**第一反应应该是去游戏里找现成切片，不是 AI 切**——能拿现成就别拆：

| 路线 | 什么时候用 | 怎么做 |
|---|---|---|
| ① **从 prefab 反查真实 sprite**（不切，直接拿） | 目标元素游戏里已有（改造现有界面、拼真组件原型） | 目标界面 prefab 抽 `m_Sprite` GUID → 扫 `.meta` 反查 PNG → 拷进 `_assets/`（详见《X3客户端GUI知识》控件模式库） |
| ② **ui_extract / ui_extract_fine**（AI 拆一张现成参考图） | 有一张参考图(竞品/旧界面)，要抠出 UI 元素 | x3-media type=ui_extract：去文字 + 分档 resize + pngquant 量化到 1-80KB；毛发/半透明走 ui_extract_fine（双底差分反推 alpha） |
| ③ **Morphix E 元素拆分四步**（把自己生成的效果图切成切片） | 效果图认可后拆成 F 固定层切片交美术/程序 | 见下方四步 |

**③ Morphix E 元素拆分四步流水**（全 AI+脚本，零手裁）：
1. **雪碧图**：`generate_image(gpt)` + 效果图当 reference + buildSplitPrompt（#B8B8B8 灰底防白边 / 原子化 / 严格去重 / 无文字 / 行排列 20px 间距 / 不重绘）。
2. **去底**：`remove_background`（result 是嵌套 list `[[url],null]` 要解开；验真透明 alpha 0-255）。
3. **切块**：PIL 投影递归分割（行带 gap18 → 列段 → 粘连块二次行分割）；脚本 `split_blocks.py`。
4. **归档**：按槽位命名（`F1_xx_九宫格_无字.png`）+ README 映射规格表。

> ★ 切图 vs 拆槽别混：本节「切图」=把一张图拆成独立**素材切片**；§4.3 的「S6 拆槽」=把效果图按**更新频率分四层**(F固定/T主题/B背景/TXT文本)的交付策略。先拆槽定层，再对 F 固定层用本节切图流水。

## 2. 工作流主干（所有线共用的底座）

### 2.1 工具链 & 认证

| 组件 | 路径 |
|---|---|
| 入口 skill | `~/.claude/skills/x3-media/SKILL.md`（type router / 默认风格 / 模型列表） |
| call_grfal.py | `C:\ADHD_agent\.cursor\skills\grfal-api\scripts\call_grfal.py` |
| base url | `https://grfal.tap4fun.com` |
| **★认证（2026-06-12 起）** | 走 `~/.config/grfal-api/token_store.json` 的 **Bearer access_token**（长效到 2036），call_grfal 无 GRFAL_COOKIE 时自动 fallback——**什么都不用设**。旧 cookie 注入法已废 |

认证铁律：脚本开头 `os.environ.pop('GRFAL_COOKIE', None)` 兜底（残留过期 cookie 优先级更高会顶掉好 token→401）；下载结果也带 `Authorization: Bearer`。验活 `GET /api/tasks?page=1&page_size=1` 带 Bearer 应 200。

### 2.2 执行模式：异步 media-worker（强制）

**主 agent 不得直调** call_grfal/生成脚本——一律派 `media-worker` 子 agent 后台异步跑：
- 一次原子 GRFal 调用 = 1 个独立 worker（3 张图 = 3 个并发 worker，不在一个 worker 内循环）。
- `run_in_background: true` + `model: "sonnet"`（必须显式指定，否则继承 fable 等不可用模型整批挂掉）。
- task_id = 单一事实来源，主 agent 只读 `state/tasks/<id>.json`，不接子 agent 长文本。
- worker prompt 只传 `TASK_ID=` + `SKILL_ROOT=` 两行。

### 2.3 GRFal 调用骨架（worker 内）

```python
os.environ.pop('GRFAL_COOKIE', None)
sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8', errors='replace')  # 防 GBK 控制台崩
# generate_image / generate_video 是 long_running → 必须 --submit-only 拿 task_id → --check-task 轮询
# 千万别加 --sync（300s 硬超时）；gpt 带参考图常 4-9min，video 5-12min
# --file reference_images=path 可重复写=多图参考(append)
# result 可能嵌套 list，递归 flat；URL 拼 base，下载带 Bearer
```

### 2.4 模型选型速查

| 模型 | 强项 | 用在 |
|---|---|---|
| **gpt** | 保角色身份/质量好，但慢(4-9min/2张) | 角色换装、风格保真要求高的 reskin |
| **gemini** | 快/稳/保 UI 布局好 | UI 换皮、结构确认稿、改造现有界面 reskin |
| **seedance** | 2D 立绘脸稳 + 运动足(3.2) | 皮肤展示视频首选（推翻早期 kling 首选） |
| **kling** | fflf 首尾帧锁得紧(0.6)循环无缝 | 视频需要严格无缝循环时；但 2D 脸必张嘴/warp 是硬伤 |

### 2.5 ★出图后默认核验（用户定，所有换皮出图后必做）

拿**对应参考图(复用源)**核 4 项，不只看尺寸（脚本：PIL `getbbox()` 算 bbox+四边 margin+占比逐项对参考）：
1. **尺寸** W×H = 参考（gpt 默认出 1024~2048 大图必须缩到目标）
2. **格式/透明** mode（图标/tile=RGBA 真透明；背景=RGB 不透明）。gpt「透明底」常是假透明→走 remove_background 或双底差分；验真 `Image.getchannel('A').getextrema()[0]<255`
3. **上下左右边距** = 参考（决定 sprite 在槽位对齐）
4. **主体占比** = 参考（不能简单居中；岛底座要落格子同位）

## 3. 三大重型工作流详解

### A. 角色皮肤换装线（gpt）

1. 选基础英雄 → 找客户端立绘 `Role/FullLength/Role_F_{id}.png`。
2. **以本人立绘当 reference**，prompt=「保身份(脸/发/身形/画风)，只换造型(服饰/道具/姿势)」→ 出 R1。
3. 后续每轮用上一版选中图当 ref，**逐轮只改一处**，版本名带语义后缀。
4. **质量好但带违规元素的图**（用户常给）：以它当质量/姿势锚 ref，prompt 逐条列要改的设计点（红→白金 / 国徽→奖杯 / 去脸颊国旗）——既得质量又合规。FINAL 常这么来。
- 坑：基底选错会白跑，动手前确认 ref 是哪一版；IP 合规用原创风格化不精确复刻商标。

### B. 活动 UI 换皮效果图线 + S1-S6 编排（gemini 为主）

策划案到「界面交互」段需出新界面效果图时的完整编排：
- **S1 结构设计**（不出图）：收参考图→拆「每张取什么基因」→ ASCII mockup + 要素清单对齐 → 拍板布局分区/交互状态机/红线(IP·国家元素)。
- **S2 结构确认稿**（gemini，快）：多参考合成出 2 候选，用户确认「结构对」。只验结构不追风格。
- **S3 风格化**（Morphix buildReskinPrompt 职责切分）：图#1=S2 结构稿（视觉全 ERASE 只留 WHAT/WHERE）+ 图#2=**本游戏真实 UI 截图**（design system 整套照搬），模型 gpt。
  - ★图#2 来源优先级：`C:\Users\linkang\Pictures\X3验收\` 验收截图库（X3 新版界面风格真源），挑与目标框架元素最接近的一张。
- **S4 细节迭代**：每轮只改一处；★变体覆盖检查——规则里写的 UI 元素(角标/锁定态/结算态)必须有图可对，只出默认态=漏交付。
- **S5 回填闭环**：效果图路径回填策划案「界面交互」段 +「美术需求」页签（双参考交付）。
- **S6 拆槽**（见 §4.3）。

### C. 改造现有界面效果图 = 真组件拼装→AI reskin 五步法（首选）

给「已上线界面加/改元素」（如养成手册加双档切换）出落地效果图，比纯 AI 整张生成（布局飘）和纯 CSS 拼装（接缝/光影不统一）都强：
1. **拆真组件**：目标 prefab 抽 `m_Sprite` GUID → 反查 PNG → 真 sprite 拷进原型 `_assets/`。
2. **CSS 拼装定精确布局**（=图#1 结构稿）：HTML 用真组件按真实坐标拼，新控件也摆到位；chrome headless 截设备区。
3. （选）出适配新控件的底图。
4. **★AI reskin 合成成品**（关键步）：`reference_images=[图#1拼装截图, 图#2真实界面]`，gemini，9:16；图#1 只取布局/文字/控件位置（视觉全 ERASE），图#2=design system 整套抄；新元素显式声明「按图#2 视觉语言原生渲染」。
5. **S6 拆槽交付**。

> ★血泪教训：拼装(图#1)看着糙完全没关系（它的视觉在第4步全被 ERASE，只取布局）——**别在第2步死磕拼装精度，赶紧进第4步**。把结构锚当成品交=必烂。
> 变体：图#1 来源是竞品横屏截图但要竖屏 → 先 `buildFlipPrompt` 横竖转换再 reskin（两段式）。

### D. 皮肤展示视频线（替代 Spine）

- **目标格式**：SBS 透明视频（左半彩色+右半 alpha 烧成白模 on 黑底），Unity shader 重建 alpha；格式锚 `AllianceCard_icon_1_a.mp4`。不透明 showcase 视频走普通满幅 mp4。
- **统一 1080×1920(9:16) 直出**，不做后期裁切（2026-06-15 客户端拍板）。
- **模型 seedance 首选**（脸稳+运动 3.2 真 Spine 量级；kling 给 2D 脸必张嘴/warp 压不动）；kling 仅用于需严格无缝循环。
- **无缝循环**：kling `ref_types="首帧图像,尾帧图像"` 同图传两次。
- **透明化链路**：`generate_video → video_remove_background(.webm 原生 alpha) → export_sbs_video(quality=CRF，q12 是甜点不是百分比！)`。
- **演出心法**：撩不撩的根在起点图（主稿端正则视频再调也端正）；眼神给镜头=死活线；写「一条贯穿全身的动作线」不写零件清单(否则糖人)；压制词(minimal/never)会叠加压死动作；prompt 长度=注意力预算，token 砸给最易被丢的那段。
- 视频处理一律用项目自带 `C:/x3-project/client/Tools/VideoTools/ffmpeg/ffmpeg.exe`（cv2 写不了可控码率，且中文路径静默失败）。

## 4. 出图后的沉淀（最易漏，按此收口）

### 4.1 归档路径规范

| 产物 | 根路径 |
|---|---|
| 出图/主稿/效果图归档 | `C:\ADHD_agent\KB\产出-本地化与美术\X3\{主题}\` |
| 节日活动主背景 | `C:\ADHD_agent\KB\产出-本地化与美术\X3\活动主背景\` |
| 美需本地存档 | `KB\产出-本地化与美术\{项目}_{年份节日}_{类型}_美需.md` |
| 视频/拆槽脚本 | 同主题目录下（gen_*.py / draw_slots_*.py / split_blocks.py） |

### 4.2 命名与版本

- 命名 `{活动名}_{模型}_{YYYYMMDD_HHMMSS}_v{N}.png`，版本带**语义后缀**（`_r5白金加徽章` / `_v7_礼盒下移`），定稿标 **`_FINAL`**。
- **真源=生成脚本**：改图改脚本重跑，不手改产物；下次换皮复制最接近的脚本改 REF/PROMPT/OUT。

### 4.3 ★S6 图槽模板化拆解（效果图认可后、交美术前必做）

效果图别直接交美术——先**拆槽**，否则文字/渐变烧死在图里=美术量×场次×语言爆炸：
- **四层拆解**：F 固定层(1套切图无文字永不换) / T 主题层(按复用单元出，对称设计一图翻转省一半) / B 背景层(整图按阶段几张) / TXT 文本层(全 TextKey 叠加，i18n 一次翻不烧图)。
- 成本判断：「出图便宜，改程序贵」——渐变/材质烧进图里批量出，不做程序染色。
- 维表驱动：主题层做成维表(TeamID|名TextKey|横幅DK|徽DK)，运营每场只填 ID。
- 槽位示意图**独立画**（PIL 线框 / HTML+chrome 截图），别叠在效果图上。配**槽位尺寸规格表**（px/比例/九宫格vs整图/数量/基准1080×1920）。

### 4.4 落地到 X3 客户端的链路

```
1. GRFal 生图 → 下载到 KB\产出-本地化与美术\X3\{主题}\
2. 用户选定 → 复制到 client\Assets\Res\UI\Spirits\{对应目录}\
3. Unity import 生成 .meta(GUID)
4. 编辑 DisplayKey asset 加 DK→path（或 Unity Ctrl+T 面板）
5. git commit + push（feature branch + MR，dev 受保护）
6. gdconfig 配置表字段 = 新 DK
7. push gdconfig + jolt 自动导表
```
各模块资源落地目录见 [reference_x3_cosmetic_resource_paths]；DK 注册见 [reference_x3_client_resources]。

### 4.5 ★交美术的双参考（换皮活动缺一不能出图）

每个美需项必须给两类参考：
1. **老图参考·格式结构** = 复用源旧界面图（告诉美术版式/尺寸怎么对齐）。
2. **新图参考·投放物元素** = 本期实际投放新东西的资源图（告诉美术新主题该画成啥样）。背景类无独立投放物→挂节日主视觉锚。
- 投放物绝大多数是游戏现成资源，去配置表+客户端搜出来，写完逐路径验存在。

### 4.6 知识库回写（收工自检）

- 新规律/踩坑/新链路 → 当场沉淀进 memory / 本文档。
- 三件套（大案子）：决策记录(按模块·为什么+被否决方案) / 换皮清单(骨vs皮) / 产物目录 FINAL 标注。

## 5. 踩坑速查（最值钱）

| 坑 | 解 |
|---|---|
| generate_image/video 是 long_running | 必须 --submit-only + --check-task 轮询，禁 --sync |
| Windows GBK 控制台 print 中文崩 | 脚本开头 `sys.stdout=io.TextIOWrapper(...,errors='replace')` |
| 残留过期 cookie 顶掉好 token→401 | 脚本开头 `os.environ.pop('GRFAL_COOKIE', None)` |
| gpt「透明底」是假透明 | 双底差分 / remove_background，验 alpha 真透明 |
| 角色被画太大顶到 UI 文字 | 用原 UI 当尺寸位置锚 + prompt「中等场景插画不 oversize」；ref 整体缩 90% 让安全边距 |
| 角色与道具都居中会叠 | 道具缩小下移到脚前 / 角色偏一侧二选一 |
| cv2 写不了中文路径(静默 False) | `cv2.imencode('.png',fr)[1].tofile(path)` |
| export_sbs_video quality 是 CRF 不是百分比 | q12=甜点(7MB 肉眼无损)，传 95 会糊成马赛克 |
| export_sbs_video 并发互相覆盖输出 | 永远串行跑 |
| 改造现有界面整张 AI 重画会「风格对但底飘」 | 真组件拼装当图#1，必走第4步 reskin，别拼装当成品交 |
| GIF 表情聊天里显灰底 | LSD 背景色索引必须==GCE 透明色索引；.bytes 改完需重建资源包 |
