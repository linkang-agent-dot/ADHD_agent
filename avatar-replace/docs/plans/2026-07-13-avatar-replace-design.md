# avatar-replace 设计文档

> 2026-07-13 · 个人侧项目，与公司 GRFal / 公司仓完全脱钩
> 用途：投放/宣传素材合规——把视频中的未成年人替换为数字人形象

## 背景与需求

- 投放素材（15-60s）中出现未成年人需要合规处理，替换为**明显成年**的数字人形象（保护方向：真实儿童从素材中移除）。
- 主要使用者为合作方；先本地跑通，部署形态（公网 / 合作方自建交付包）效果验证后再定。
- 后端**直连火山引擎**：Seedance（视频替换）+ 豆包 VLM（打轴识别）+ 即梦文生图（建形象库），同一账号。
- 关键技术约束：Seedance 参考视频单次上限 **15s**；策略 = 全片打轴 → 只切出含未成年人的片段替换 → 拼回。

## 架构

```
avatar-replace/
├── core/                        # 管线核心，零 Web 依赖，CLI 可独立跑
│   ├── providers/volc.py        # 火山引擎唯一封装点（Seedance + 豆包VLM + 即梦文生图）
│   ├── annotate.py              # 打轴：抽帧→VLM 识别未成年人时段+外观描述
│   ├── cut.py                   # ffmpeg 切段（镜头对齐 + ≤15s）
│   ├── replace.py               # 调 Seedance 替换（prompt 模板在此）
│   ├── stitch.py                # 拼回 + 原音轨铺回
│   ├── pipeline.py              # 编排 + 任务状态机（job.json 落盘，断点续跑）
│   └── cli.py
├── web/                         # 薄壳：FastAPI + 无框架单页
├── avatars/                     # 数字人形象库：<形象名>/{多角度参考图, meta.json}
├── jobs/                        # 任务工作目录（每任务一目录：源片/帧/轴/段/结果）
└── config.yaml + .env           # key 只在服务端环境变量
```

设计原则：所有 AI 能力收口在 `providers/volc.py` 一处；换模型/加备用通道只动这一个文件。

## 核心流程（五步）

1. **打轴**：ffmpeg 1fps 抽帧 → 豆包 VLM 两遍扫描（第一遍粗定位含未成年人时段，第二遍只对命中段细化描述：画面位置/着装/动作）→ 产出 timeline JSON `[{start, end, person_desc, sample_frame}]`。送图不送视频，打轴成本低。
2. **用户确认**：UI 展示时间轴+样帧+描述，用户勾选确认、可微调边界。此步骤兼作 VLM 年龄误判的人工兜底；确认页显示预估调用段数（=成本），确认前即知花费。
3. **切段**：确认时段外扩 ±0.5s，场景检测对齐最近镜头切点（避免镜头中间硬接）；单段 >15s 自动二分；未命中片段原样保留。
4. **替换**：每段调 Seedance——参考视频=该段，元素参考=数字人形象图 2-3 张（正/侧），prompt 模板：「将视频1中的{person_desc}替换为图片1中的数字人形象，其余人物、场景、运镜、动作节奏完全保持不变」。段级并发、段级重试。
5. **拼回**：ffmpeg concat 视频轨；**音轨用原片完整音轨铺回**（丢弃 Seedance 生成音频），音频零损失。已知风险：Seedance 输出段时长与原段偏差 → 超阈值对视频轨变速微调对齐。

## 数字人形象库

`avatars/<形象名>/`：正/侧/半身参考图 + `meta.json`（性别/风格/适用场景）。建库用即梦文生图批量生成**明显成年**形象，人工挑选入库；任务从库选形象，跨视频一致。

## Web 壳

FastAPI + 无框架单页，四页面：上传 → 打轴确认（时间轴交互）→ 进度（段级）→ 结果（对比预览+下载）。历史记录 = 扫 `jobs/` 目录，无数据库。本地 localhost。

## 安全（本地版预埋，公网版启用）

- **key 隔离**：API key 仅服务端环境变量；火山引擎子账号最小权限 + 控制台消费限额/预警（被打穿损失有硬顶）。
- **防刷中间件预埋**（本地关/公网开）：登录闸门（白名单账号）、每用户每日配额、单任务上限（≤60s/≤200MB）、IP 限流。
- **上传安全**：仅视频 MIME、文件名清洗、任务目录隔离、结果链接过期签名。
- 「合作方自建」路线下各家自用 key 自跑，公网项全部不需要。

## 阶段与验收

| 阶段 | 内容 |
|------|------|
| P1 | 管线核心，CLI 跑通全流程 |
| P2 | Web 薄壳 |
| P3 | 形象库建库 |
| P4 | 部署形态（公网 or 交付包，看效果定） |

- 前置条件：开通火山引擎账号 + 方舟 API key（豆包 VLM + Seedance/即梦视频）。
- 验收（key 配好后由用户触发）：一条真实素材全流程，验 ①打轴准确率 ②形象跨段一致性 ③非替换区保真 ④音画同步。
- **最大不确定性 = Seedance 人物替换实际效果**：P1 跑通即先验真素材，不行早止损。

## 被否决的方案

- **走公司 GRFal 网关**：用户明确要求自建、部署自家、与公司资源脱钩。
- **一步到位公网平台（方案B）**：核心效果未验证先背平台开发量；若终局是合作方自建，账号/配额系统白做。
- **纯 CLI 无 UI（方案C）**：打轴确认需看帧交互，命令行体验差；合作方非技术用户，堵死自建路线。
- **整片送 Seedance**：15s 上限 + 全片替换成本高；改为打轴后只替换命中段。
