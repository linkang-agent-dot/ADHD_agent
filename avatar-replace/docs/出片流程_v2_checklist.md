> 背景：首批 16 条实付 520 元、重掷浪费约 2/3。废片三来源：①打轴读错原片（整条报废）②Seedance 概念性画错（学费坑）③QC 粒度不够（细节漏给需求方多一轮）。

## 八维度判定标准（总表：事前进 prompt、事后 QC 同一张表）

| # | 维度 | 判定标准（验哪帧 → 挂线） | 层级 |
|---|---|---|---|
| 1 | 颜色 | 内衣面积最大帧（原分辨率）取色：主体=粉底肤色；反面对比色必须深/高饱和扎眼。挂：白/米白当主体、绿蓝调、反面不扎眼 | B 拿不准附图问 |
| 2 | 肩带 | 肩部特写帧：两条带子独立根部、平行、皮肤间隙≥一条带宽；肤色带更细+调节扣；宽景若隐若现。挂：共根/贴死/宽景太显眼 | A 恒定 |
| 3 | 露的度 | 三档（全藏/露一小截/直接展示）。**档位=打轴逐条读出（B）；「定档后全程不越档+外衣永不内衣化」=恒定规则（A）**。挂：越档、外衣出现罩杯/骨架/网纱 | A+B 拆分 |
| 4 | 谁说话 | 逐句标出镜/画外：出镜=嘴动+面向镜头；画外=无人对镜张嘴（可有画外手/话筒）。挂：任一句归属错 | B 歧义才问 |
| 5 | 关键动作 | 每条写死 1-2 个卖点动作（动词+对象+幅度）。挂：没发生或幅度越界（拉链变露背/勾带脱肩） | **C 每条必确认** |
| 6 | 道具 | 一行道具卡：名称+颜色+谁拿+用途。挂：认错、吐槽道具穿上身、凭空消失 | B 歧义才问 |
| 7 | 正反角色 | 一句锁死反面=谁+扎眼元素/正面=谁+隐形元素。挂：互换、同现一人 | A 恒定 |
| 8 | 服装一致 | 每个场景段的段首+段尾抽帧、逐人逐件对照。挂：中途变化、原片没有的附加物（安全带等） | A 恒定 |

- **A 恒定**：进 prompt 模板禁令+QC 清单，不占需求方注意力
- **B 条件确认**：打轴客观读出，辨别不清时附原片截图选择题式问
- **C 必确认**：确认卡=一句卖点+1-2 行关键动作+（如有）B 类带图问题，需求方 10 秒/张
- QC 分辨率分派：**维度 1/2/3/5/6 必须原分辨率关键帧**；4/7/8 构图级可用宫格图

## 第 0 步：卖点确认卡（硬门禁）

- 工具：`scripts/gen_sellpoint_confirm_html.py --ids .. --src-dir <batch>\开头 --out <batch>\卖点确认_<ids>.html`（原片帧图+确认点+可编辑反馈框+一键复制反馈）
- 卡内容只放 C 类（+待定的 B 类），存 prompts json 每条 `卖点` 字段；**卡带版本号 v1/v2**
- 需求方逐条回「确认/改X」才开跑
- **变更单半步**：确认后需求方改口 → 标记受影响 id → 重发确认卡 v2 → 已出旧片移 `<batch>\废弃\` 并在验收台账记一笔增量成本

## 第 1 步：打轴（产物=打轴卡）

- ffmpeg 3fps 宫格（`fps=3,scale=320:-1,tile=4x3`，勿加 drawtext 会段错误）+ faster-whisper small 转写（错字多时序准；准确台词以原片硬字幕为准）
- **产物落点：每条一份「打轴卡」写进 prompts json `打轴` 字段**：时间轴分镜 + 八维度 B 类读数（4 谁说话逐句 / 6 道具卡 / 3 露的档位 / 1 颜色读数），7 正反角色一句锁死
  - 历史脚注：10 号把吐槽道具当新品=整条报废；16 号正反角色写反
- **多人/多场景片扩展**：≥3 人或 ≥2 次场景切换时，逐人建「人物卡」（服装逐件+立场+扎眼/隐形件），服装一致性抽帧改为**每场景段首尾各一帧**

## 第 2 步：prompt 模板化拼装（顺序敏感）

```
[1] 外衣锁死句（最前：面料/廓形/无罩杯无骨架无网纱/完整不露背不开洞）
[2] 场景与人物（服装逐件+「从第一秒到最后一秒不换装」）
[3] 分镜与台词（画外音写法：「镜头外传来女声…提问的人不出现在画面里」）
[4] 场景禁令块（按片型选贴，见下）
[5] extra 个别覆盖（后置压冲突；少用"内衣"字样防文本风控，用"衣物层/色块"替代）
```

**场景禁令库**（实测踩坑得来）：
- 吊带片：两条独立带子（外衣带明显+肤色细带很不明显）、隔一指宽皮肤、不共根；只有本人的手
- 隔衣展示片：外衣完全不透光、任何位置无米色/肤色色块；**不画痕迹**，对比靠标注+台词
- 正反角色片：反面=X 穿扎眼件、正面=Y 穿隐形件，元素不互换、不同现一人（除非原片如此）
- 标注图形：白虚线圈/黄箭头=「叠加在画面上的贴纸特效，不改变衣服」
- 套头罩衫：套头穿好、前襟不敞开、只结尾掀下摆短暂露
- 字幕：prefix 强措辞禁字幕（仍有漏网，QC 必查）
- 双人片：两人服装逐件锁死+不换装

**prompts json 最小样例**（字段 schema）：
```json
{"_meta": {"batch_root": "E:\\222", "prefix": "…", "suffix": "…", "suffix_hidden": "…",
           "ref_image": ["refs/front.png","refs/back.png"], "generate_audio": true},
 "clips": [{"id": "08", "dur": 8, "garment_mode": "hidden",
   "卖点": {"卖点句": "…", "关键动作": "…", "需你确认": "…"},
   "打轴": {"时间轴": "0-2s …", "谁说话": "首句=画外", "道具": "无", "露的档位": "B露一小截", "正反": "单人正面"},
   "prompt": "[1]外衣锁死句…[2][3]…", "extra": "特别说明（以此为准）：…"}]}
```

## 第 3 步：审阅页过目（可选）

`python scripts/gen_prompt_review_html.py` → 需求方扫分镜台词。已过确认卡的批次可跳过。

## 第 4 步：出片 + 失败处理 + 熔断

- `python scripts/batch_t2v.py --json prompts/<batch>.json --out <batch>\成片_vN --workers 3 --only <ids>`
- ⚠️ batch **失败也 exit 0**：链式命令必须插成片文件存在性检查；看 `[batch] id: ok/fail` 行
- 文本风控 400：不计费，降敏改措辞重提（少用"内衣"+穿戴方位组合词）
- 网络超时：任务已建照常计费——**拿日志任务 id 手动轮询捞回**（GET /contents/generations/tasks/{id} → succeeded 后下载 video_url），别盲目重提
- **熔断规则：同一条同一维度挂 2 次 → 停掷**，升级处理：换表达方案（画不出的改标注/台词承担——01 的教训）或带图问需求方，绝不硬掷第 3 次

## 第 5 步：QC → 路由 → 烧字幕 → 交付

- 按该条「打轴卡+确认卡」逐维打勾；维度 1/2/3/5/6 抽原分辨率关键帧（`-ss <t> -frames:v 1`）；查 Seedance 漏网自带字幕
- **挂件路由表**：挂 1/2/3/5/6 → 回第 2 步改 prompt 重掷（计熔断次数）；挂 4/7 → 回第 1 步复核打轴；挂 8 → 补锁死句重掷
- 全过 → `python scripts/post_subtitle.py`（已修中文 13 字折行）→ 覆盖 `<batch>\最终交付\` 同名文件 + 更新 `<batch>\样片验收反馈.md`
