## 树 01 · 目的：把中大R的付费深度做起来
来源：X3 尼罗"breadth成功/depth没起来"复盘（`project_x3_nile_next_phase` / `reference_p2_depth_lessons_for_x3`）

**因果假设（先问）**：这些模块**凭什么**能让中大R掏更多/持续掏？最脆弱假设是啥？（常见跑偏：靠外显小额堆量 ≠ 深度；靠排行榜 ≠ 深度）

```
模块（深度引擎，如 限时GACHA / 储蓄罐 / 高阶养成）
├─ 表现【目的：抽取/投入过程要爽 + 稀有产出强感知 → 勾持续投入】
│   ├─ 投入前有没有期待铺垫？(蓄力/光效渐强)            → 查 prefab
│   ├─ 出稀有有没有强反馈？(全屏金光/音效/震动/慢镜)    → 查高品质分支表现
│   ├─ 数字/进度有没有即时跳动反馈？                    → 查
│   └─ 连续操作节奏拖不拖？(单次时长/可否跳过)          → 查
└─ 数值【目的：深度有底可挖 + 价值感 + 不伤性价比锚】
    ├─ 最高档价位多少？大R够不够得着稀缺？(vs 上期/竞品基线) → 查档位表
    ├─ 保底在第几抽？期望产出 vs 价位价值感对不对？        → 算期望值
    ├─ 有没有可持续投入结构？(多池/复刻/碎片养成/无底洞)    → 查模块
    ├─ 中→大R价格梯队连不连续？(相邻档价差有无断层)        → 查档位倍率
    └─ 改 floor/概率有没有误动锚点？                      → `reference_numerical_design_random_pkg`
```
**副作用**：深度投放有没有挤掉小R的低门槛参与？
