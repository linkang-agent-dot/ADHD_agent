# 数据分析报告验收清单（type=data-analysis）

> task-checker 跑 data-analysis 验收时对照的**单一来源**。
> 标准真源 = memory `workflow_data_analysis_must_check.md`（开工清单）+ `reference_data_analysis_module_registry.md`（模块规则卡）+ `reference_data_analysis_playbook.md`（方法论）+ 各案子「口径统一.md」（皮）；**本文件 = 可机检子集**——完整知识看 memory，这里只放能机检的客观项，别写全(防两份标准漂移)。
> 核心结构:回归=模块装配(框架→模块→规则)。机检重点=每个模块章的窗口/分母标注是否齐、是否按模块各自口径(而非全局一套)。
> **回流**：验收/评审发现清单没覆盖的新坑 → 当场补成规则并记『清单进化日志』。

## 类别
- **口径**：窗口/服段/分母/对照选错或没标注
- **指标**：三件套缺列、分母基底错、没归一化
- **呈现**：结论与图不成对、口头语、缺设计对账

## 怎么读产物
- **报告 HTML**：Read/Grep 直接读（注意生成器产物可能单行超长，用 Grep 带 -o / context 查）。
- **本案口径统一.md**（主 Claude 会随产物一起传路径；没传则该项 cannot_verify）：Read。

## 客观项（直接判 pass/fail）
| rule | 类别 | level | 怎么查 |
|---|---|---|---|
| `窗口口径标注` | 口径 | block | 报告每个模块章节应标注该模块**自身**统计窗口（日期区间）与服段；grep 各章节标题附近有无日期区间/服段字样。整报告只有一个全局窗口、模块章无各自窗口标注 → 报出哪些章缺 |
| `分母写进表注` | 口径 | block | 凡出现「付费率/ARPU」的表，附近应有表注写明分母（"分母=…总付费人数 N"式样）。grep `分母`，比对含付费率的表数量，缺注的列出 |
| `表带三件套` | 指标 | block | 凡含 ARPPU 或 收入 列的模块表，应同时含 **付费玩家付费率** 与 ARPU 列；逐表查表头，缺列的报表名/位置 |
| `口径文档一致` | 口径 | block | 若传入了本案口径统一.md：报告用的窗口/服段/分母数字应与该文档一致（抽查每模块窗口日期与分母 N）；不一致逐条列。未传文档 → cannot_verify 转人工 |
| `结论配图成对` | 呈现 | block | 每个图表容器（canvas/svg/chart div/.view）应配一条结论元素（.vc/.bigcon 或含"结论"class/标签），先结论后图；数图表数 vs 结论数，裸图无结论、或纯文字长段无锚点数据的"结论"，逐个报 |
| `大结论带跳转` | 呈现 | warn | 章级/总判词大结论（.bigcon 或等价元素）内应含 `<a href="#…">` 章节跳转；无跳转即报 |
| `设计对账章存在` | 呈现 | warn | 回归/复盘类报告应含「设计 vs 实际」对账内容（grep 设计回溯/设计对账/预期 vs 实际/mindmap 等价物）；纯现状描述无设计侧 → 报 |
| `禁口头语与抽象话` | 呈现 | warn | grep 可见文本中 `对打`、`本质上`、`定位是`（无数据支撑的抽象判词）；命中列位置。（"历史数据对比"是"对打"的正确写法） |
| `免费投放报人均` | 指标 | warn | 若报告有免费投放/补贴段：grep 该段应含「人均」；只有总名义（$X 万级总额无人均换算）即报 |
| `跨窗对比带警示` | 口径 | warn | 若报告含 vs 前代/跨窗对比：附近应有口径警示（窗口短/服少/分母窄/仅参考 等字样）；无警示即报 |
| `事件型归一化` | 指标 | warn | 若模块是轮次/场次驱动（grep 轮/场/淘汰赛）且有走势判断：应存在单场/单事件口径的归一化数据；只有按日/按轮总额走势即报 |

## 主观项（标 level=human，不计 pass/fail）
- 对照先例是否形式同构（机检只能确认"选了对照"，同构性需人判）
- 结论判词质量、是否点名主要增长因子（ARPPU×付费率×盘子哪个在动）
- 受众裁剪（给 +2 的版本是否只留四类核心数）
