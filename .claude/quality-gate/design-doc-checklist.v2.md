## 0 · 工作姿态（checker 先读，决定怎么跑下面每一条）
- **默认有罪**：假设这份策划案**已经有缺件/悬空引用/不一致**，你的任务是把它揪出来；找不到才给 pass。不要去找"它齐了"的证据。
- **只信产物，不信摘要**：判定输入是 GSheet/xlsx 的**实际页签和单元格**，不是 producer 说的"我写全了"。
- **每条 block 必须贴证据**：命中要**列出具体位置**（页签名 + 单元格/行 + 命中的原文）。只写 "fail" 不贴位置 = cannot_verify，不算结论；只写 "pass" 不说明查了哪些范围 = 同样不可信。
- **失败模式驱动**：下表每条都对应一次真实翻车（多为大富翁案）；逐条按"这次有没有再犯"去查。

## 1 · 类别（按问题性质打标，看哪类高发；**不拆文件**——单人维护下跨类归属难+破契约+拖累回流，故打标代替）
- **排版**：写入写错(插表头/列错位/页签名) / 配色区块 / 列阶梯·横铺schema·拆页签 / 不好看
- **逻辑**：缺件(本地化/配置/机制规则) / 改A没弄清连带B(删除链·依赖链) / 跨处一致(命名·三层·摘要↔详细)
- **交互**：缺状态(状态机) / 表现缺动态(客户端表现·动效·音效)
> 跨切面规则(如删改一致性)允许多标。task-checker 输出 JSON 按类别分组 checks（报告更清晰，读取契约不变）。

## 2 · 怎么读产物
- **GSheet**：`node C:\ADHD_agent\scripts\gws_stdin.js`（stdin 传 JSON）。列页签=spreadsheets.get 拿 sheets[].properties.title；读内容按页签/范围。中文用 UTF8、必要时按行号范围读（gws stdout 是 GBK，关键词搜易乱码）。

## 4 · 主观项（验收员判不了，标 level=human，不计 pass/fail）
- 颗粒度对不对（对齐范本深浅）／顶层组合选型（单玩法/节日包装/系统 rollout）／包装层文案质量

## 5 · 删改老模块原则（删除链 · 2026-06-03 大富翁案沉淀）
删除/取消一个老系统 = 清一条**依赖链**，不能只删主条：
`主条(升级) → 衍生表现(变色/升级特效/环形经验条) → 关联字段(Event.Level) → 关联配置表(ActvVoyageLevel) → 界面元素(等级UI/左右切级) → 其它处效果引用(神秘岛「经验+2」、岛屿表「可升级=是」)`。
**中途改过决策**(如「弱化→改直接删」)时，旧决策残留(如③等级变色)必须跟着删——最常见的悬空来源。落地后靠上表 `删改一致性` 兜底扫。
