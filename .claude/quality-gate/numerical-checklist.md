# 数值设计验收清单（type=numerical）

> task-checker 跑 numerical 验收时对照的**单一来源**。
> 标准真源 = skill `x3-numerical-design`（SKILL.md + references）。本文件 = 它的「可机检子集」——只放能客观判的项，完整方法看 skill。
> 改标准就改这里，验收员运行时读这里，不另存副本。
> **回流**：验收发现清单没覆盖的新坑 → 当场补成规则并记『清单进化日志』。

## 验收对象
一份 X3 活动**数值表**（道具价值 + 各礼包ROI + 投放内容），配套一份**策划案**（需求源）。
两份都要拿到——维度A 比数值表内部 + 往期，维度B 比数值表 vs 策划案。

## 输入（主 Claude 传）
- `type` = numerical
- 产物位置 = ① 数值表（GSheet ID 或本地 md/html）+ ② 策划案（GSheet ID 或本地文件）
- 任务名

## 维度 A — 数值自洽（数值表内部 + 跟往期比）

| rule | level | 怎么查 |
|---|---|---|
| `ROI未投爆·同模块对齐` | block | 对每个礼包算 ROI=总价值÷售价，跟 past-references 里**同模块**的基准区间比。漂出区间（尤其偏高）= block，列出哪个礼包多少 ROI vs 基准多少。**不跨类型比**（链式≈2.25x、周卡≈2.75x、皮肤档17~19x、抽奖整活动≈8x 各自不同）。 |
| `多档ROI线性一致` | block | 同一模块多个价位档（$9.99/$19.99/$29.99…），ROI 应各档一致、价值随价格线性。某档 ROI 明显高于其他档（玩家只会买那档）= block。 |
| `道具价值无瞎编` | block | 道具价值来源必须是以下之一：①已知锚（券$0.25/宝石$0.002/兑换币$0.01）②反查 tsv 实际钻石售价（钻石商店/直购/黑市/兑换）③纯数量缩放相对守恒（同款道具按价格倍数放大量、不给绝对值——**这是合法且常见的，别因"没给绝对美元数"判 fail**）。只有**查不到估值却凭空捏了一个绝对价**才=block（护栏①最常翻车处）。 |
| `玩法ROI模型用对` | block | 抽奖活动类（转盘/推币机/大富翁/开箱）是否用了**整活动ROI**而非单礼包ROI。若拿单礼包口径算抽奖活动=block。整活动还应有固定评估锚抽数 + 跨越/隐藏产出是否算入。 |
| `投放结构对齐往期` | warn | 内容构成/角色是否对得上同模块往期（如A线=券+宝石+VIP、周卡=10选4自选+券打底）。结构明显偏离往期定位=warn。 |
| `撞空白玩法未自编` | warn | 是否涉及空白清单玩法（累充/养成线深调/福袋/矿机/秒杀/联动皮肤/装饰头像框）却自己定了数。命中=warn，转人工：该找策划。 |

## 维度 B — 兑现策划案（数值表 vs 策划案逐条对）

| rule | level | 怎么查 |
|---|---|---|
| `模块全覆盖·无漏配` | block | 策划案列的每个礼包/活动模块，数值表里都有对应条目吗？逐个点名，缺哪个报哪个。 |
| `价位档对得上` | block | 策划案写的价格档（如三档 $9.99/$19.99/$29.99），数值表是否真按这些档配、不多不少。档数/档位对不上=block。 |
| `大奖/核心付费点兑现` | block | 策划案的"最终大奖/核心付费点"，奖池里真给了吗？且价值够不够撑得起"大奖"定位（大奖却塞低价值物=block）。 |
| `投放定位匹配设计意图` | warn | 策划案的投放定位（拉新广度 / 中高R深度），数值是否照做（主攻深度却把价值堆在低门槛档=warn）。 |

## 主观项（验收员判不了，标 level=human，不计 pass/fail）
- ROI 区间定得合不合理（这一期该比往期紧还是松）
- 大奖吸引力够不够、付费节奏好不好
- 投放结构创意层面的优劣

## 输出
按 task-checker 标准 JSON 输出，checks **按维度A/维度B 分组**，最后一行 `CHECK_DONE type=numerical passed=K/N blockers=数量`。

## fail-closed
拿不到数值表或策划案任一 / 内容为空 → 标 passed:false +「无法读取，需人工确认」，blocking_failures 带 `cannot_verify`，**不放行**。尤其**没拿到策划案就无法跑维度B**，必须明确报出来，不能只跑A就放行。
