Obsidian 知识库 `C:\ADHD_agent\KB`（vault 根，含 `.obsidian`，266 篇@2026-07-06）的标签体系。2026-06-04 把原来扁平、同义词各写各的标签统一重做成 **5 维嵌套命名空间**，并开启图谱 `showTags`。2026-07-06 第二轮全库梳理（补 103 篇标签 + 英文文件名统一中文）。归档新笔记或再整理时，frontmatter `tags:` 必须按这 5 维填，别再退回扁平词。

## 5 个维度（前缀固定）
- `kind/` 笔记性质：产出 · 方法论 · skill · 脑图 · 计划 · moc · **交接**（2026-07 收编：交接包/教学包内笔记用它）
- `domain/` 领域：**数据分析 · 数据复盘（两者不同！）** · 配置换皮 · 活动部署 · 美术媒体 · 本地化 · 竞品 · 通知协作 · 文档 · agent开发 · 前端 · 运维 · 媒体分析 · 个人助理（Token周报等个人基建归这）
  - `数据复盘` = 活动结束后的回顾：复盘/回归/评级/衰减类报告
  - `数据分析` = 探索与调研：白皮书/分布/漏斗/饱和度/查询工具/看板（2026-07-06 用户纠偏：6月曾把两者错误合并成数据复盘，已拆回）
- `proj/` 项目：P2 · X2 · X3 · 通用（巨猿=X2 别名）
- `fest/` 节日：春节·情人节·科技节·复活节·夏日节·圣诞节·占星节·推币机·挖矿·拓荒节·深海节·登月节·周年庆·音乐节·万圣节·感恩节·弹珠·**世界杯·尼罗河**（2026-07 新增）
- `year/` 时间：**月粒度 `year/YYYY-MM`**（如 year/2026-05；2026-07-06 用户定：裸年份太粗没信息量。日期来源优先级 = frontmatter created: → 文件名日期 → git 首次提交日期，**别用 mtime**——批量改标签会污染它）

## 文件命名规范（2026-07-06 起）
- **笔记文件名统一中文**（图谱节点名=文件名，英文名在星图里扎眼）。例外：① `notion_content.md`/`wiki_content.md` 是 generate_event_review 流水线固定产物名，别改（改要动 3 套 skill 脚本）；② 代码类名（如 `客户端补丁_UIPiggyBankContent.md`）保留英文类名部分。
- 目录说明文件统一 `_目录说明.md`（不用 README.md）。
- **机器流水统一归 `KB\_自动流水\`**（2026-07-06 用户定）：`Token周报\`（token_weekly_report.py/_html.py 每周五写）+ `工作line备份\`（daily_report_scan.ps1 每晚写）。图谱过滤器一条 `-path:_自动流水` 全屏蔽；gen_moc/check_orphans 也排除它。以后新增机器自动产的 md 流水都往这放。

示例：一篇 X2 夏日节复盘 → `tags: [kind/产出, domain/数据复盘, proj/X2, fest/夏日节, year/2026]`

## 打标签的可靠信号（按优先级）
1. **文件夹路径**最可靠（如 `产出-数据分析/`→数据复盘，`产出-数值设计/`→配置换皮，`运营skill/Skills-xxx/` 子目录名直接对应 domain）
2. 文件名里的无歧义名词（换皮/集卡/卡册/美术/本地化/竞品/前端）
3. 旧标签
4. ⚠️**别扫正文前若干字推断 proj/fest/year**——会把"顺带提到"的项目/节日/年份全抓进来造成大量误报（首页索引尤其严重）。proj/fest/year 只从 路径+文件名+旧标签 推。
5. ⚠️domain 关键词别用贪婪词："设计"会误命中前端、"工具/审核"会乱挂——只用无歧义名词。

## 图谱配置
`.obsidian/graph.json`：`showTags:true` + colorGroups 按 proj/P2·X2·X3 和 kind/skill·方法论·产出 上色；search 过滤 `-path:工作计划/_工作line备份 -path:产出-数据分析/Token周报`（流水文件不上图）+ `hideUnresolved:true`。
**🔒 改 graph.json 唯一正确姿势（2026-07-06 两次实测坐实）**：Obsidian 开着时写盘必被覆盖，且 **Ctrl+R 重载也不行**（重载先把内存旧态写回盘再读，照样覆盖）。必须：让用户**完全退出** Obsidian（含托盘）→ 写入 → 再打开。用户不便立即退出时，挂后台守护轮询进程消失后自动 Copy-Item 写入（本次即此法成功）。

## 索引层（MOC，2026-07-06 建）
- 结构 = 三层树：`_首页.md`（四层导航：目录索引→案子入口→方法论→Skill索引）→ 各目录 `_索引_<目录>.md`（8 份，≥4 篇的目录才建）→ 笔记；小目录/`_速记\`收件箱由首页直链。全库（排除 _自动流水）孤儿=0。
- **维护工具 `C:\ADHD_agent\skills\kb-maintenance\`** 三件：`gen_moc.py`（重建目录索引，幂等，`_索引_*` 别手改）· `tag_untagged.py`（新笔记补5维标签，先 dry-run 再 --apply）· `check_orphans.py`（孤儿校验，梳理后跑一次应为 0）。链接用路径式 `[[目录/子路径/名|名]]`，防重名歧义。
- 根目录只留 `_首页.md`；散笔记随手记进 `_速记\`，成案后挪进对应案子目录。
