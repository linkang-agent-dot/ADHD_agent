# 教学/交接包必须自包含：把"被调用的知识库"搬进包，别只留指针

做给别人的教学包/交接包（如 `KB\产出-数值设计\X3_英雄养成手册\_教学包_英雄养成手册出图流程\`）时，**链路上引用到的知识库要拷副本进包**，不能只在文末写"详见 KB\方法论\xxx"。

**Why**：同事冷启动看包，写"详见"=他还得回你的 KB 翻、翻不到就卡（本案同事连续两次反馈："morphix 那块只浅说""调用的知识库没搬进来"——根因都是只指针没搬实体）。判据同 [[workflow_handover_assetization]]：新人能否在包内拿到全部 keypoint，不能就没做到位。

**How to apply**：
1. **走一遍流程**，扫出链路上所有被引用的 KB/memory/skill 脚本路径（grep `KB\\` / `memory\\` / `[[...]]` / skill 路径）。
2. **搬副本进包**，按"对应流程哪一步"分目录（如 `知识库源\01_方法论_核心 / 02_memory工作流 / 03_skill底层CLI`），写一份 `_知识库索引_对应流程哪一步.md`。
3. **可复用脚本搬真身**（不是示意），标"改哪几个变量复跑"。
4. **闭包一层但不无脑全搬**：被引用文件内部再引用的，只补"和本主题直接相关"的（信息有效度）；属别的阶段/别的案子的明确不搬，并在索引里一句话说明为什么没搬。
5. 注明"包内是快照副本，改方法本身去权威源"，避免分叉。

范例落地：英雄养成手册出图教学包（00讲义+01方法+附录A实操调用+附录B Morphix全文+知识库源19文件+脚本可复用+参考图FINAL/废弃标注）。出图方法本身见 [[reference_morphix_reskin_prompts]] 及 `KB\方法论\X3_AI出图工作流…`。
