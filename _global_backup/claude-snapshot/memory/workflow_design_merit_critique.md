判策划案**「设计好不好」**的方法（区别于 design-doc 查"文档全不全/对不对"）。2026-06-05 共创落地，文件在 `C:\ADHD_agent\.claude\quality-gate\design-merit-checklist.md` + `design-merit-qbank.md`（type=design-merit，按需触发的"定稿前设计压测"）。

**核心认知（几轮纠偏沉淀）**：
1. **好不好是相对「目的」的，不是相对万能清单**：好 = 对它自己声明的目的实现到位、对自己要解决的问题解决彻底。**不能无脑套历史问题清单**（一份拉活跃的案子被"付费深度够不够"批 = 噪音/假阳性，违反信息有效度）。
2. **唯一硬框架只有一条：必须声明目的**（+要解决的问题+可衡量成功指标）。没目的就无从判好坏 → 这是唯一能机判 block 的客观项。
3. **"好不好"在概念/思维层判不动**（没有"好设计=X"规则）→ 不去判那个大问题，而是**把颗粒度拆细到可查叶子**（"有没有/够不够"+证据），主观就化成一串客观小问。
4. **拆解骨架：目的 → 模块 → 表现 & 数值**，且**每层自带局部目的**（表现的目的=让人感知到爽/期待；数值的目的=撑深度/控节奏/立锚）。好不好 = 每层局部目的实现度逐层 roll-up。
5. **触底规则**：拆到能指向"实际一段表现/一个实际数值"、可答"有没有·够不够"就停。**颗粒度预算**：细颗粒只花在主目的模块，辅目的浅拆，无关目的只做副作用检查。**落地基底可扩展**：默认表现&数值，但留存→节奏排期、情感→文案外显等触底基底不同，别套死。
6. **三个最易漏的横向维度**（必带）：①**因果假设(手段对不对路)**——执行满分≠手段对路(排行榜拉深度=跑偏)，点出最脆弱假设；②**基线参照**——"够不够"必须对历史同形式/竞品(调节日知识图谱+收益数据)，禁绝对判；③**副作用**——为主目的有没有伤别的盘子。
7. **判断没消灭、是被挪到最省力两件事**(标 level=human)：①每层局部目的写清没 ②拆得对不对。其余全是可查叶子。
8. **闭环**：设计期声明的目的+指标=上线后回归标尺(接 festival monitor/复盘)，回归结果回灌 qbank。子问题库**自下而上长**(不预设分类)。
