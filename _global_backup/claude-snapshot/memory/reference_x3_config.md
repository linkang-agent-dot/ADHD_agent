## 道具 Item ID 规则

| 品类 | Item ID 格式 | 示例 |
|------|-------------|------|
| 岛屿皮肤（永久/限时） | `Item_81xxx` | 81001=南瓜灯岬永久，81051=柔情海湾永久，81151=金字塔之城永久 |
| 英雄晋升皮肤 | `Item_530xxxx` | 5301501=甜心咖啡师·海泽尔(26情人节)，530202=红绸剑姬·阿米娜(26春节) |
| 家具（摆件/地毯） | `Item_151xxx` | 151059-151061=26情人节家具，151057-151058=26尼罗家具 |
| 主城装饰三件套（地板/墙纸/横梁） | `Item_152xxx` | 152027-152029=26春节新春典藏，152024-152026=25圣诞庆典 |
| 纪念卡 | `Item_180xxx` | 180077=我对你的誓言(26情人节)，180078=新春特辑(26春节)，180076=尼罗回响(26尼罗) |

## 数据库 asset_id 格式对照

| 前缀 | 对应品类 |
|------|---------|
| `Item_XXXXX` | 通用道具（家具/皮肤兑换券/节日道具等） |
| `FurnitureDecorateSkinID_XXXX` | 主城装饰三件套（ID 1001001-3001013）+ 雕像皮肤（ID 1000-1009） |
| `ShipSkin_XXXX` | 航迹皮肤 |
| `SoldierID_XXXX` | 兵种皮肤 |
| `HospitalSoldierID_XXXX` | 医院兵种皮肤 |
| `PlayerTech_XXXX` | 科技树 |

> 注意：岛屿皮肤在数据库中是 `Item_81xxx`，不是 `FurnitureDecorateSkinID_` 格式。

## 配置文件路径

**仓库根目录**：`C:\x3\gdconfig\`（remote: `https://git.tap4fun.com/x3/gdconfig`）
**配置真源**：`C:\x3\gdconfig\tsv\`（导入只认 tsv，命名 `{文件名}__{Sheet}.tsv`）。**改 tsv 不碰 xlsx**，详见 [[reference_x3_tsv_export_migration]]。

> ⚠️ `C:\x3\gdconfig\data\*.xlsx` 仅历史备份（下周删），`C:\X3\`、`C:\x3dev\` 是更早的弃用目录——**导入/改表都不找这些，全部走 `tsv\`**。

下表是「哪个表定义什么」（业务逻辑），改时对应 `tsv/{文件名}__{Sheet}.tsv`：

| 品类 | 表 | 说明 |
|------|------|------|
| 岛屿皮肤定义 | `Skin` | SkinType=1=岛屿皮肤，SkinType=2=航迹皮肤 |
| 主城装饰三件套 | `FurnitureSkin` | Type=1地板/2墙纸/3门；春节三件套ID：1001012/2001017/3001013 |
| 英雄晋升皮肤礼包 | `PackHeroPromotion` | 直接用Pack ID作为配置ID（如210716） |
| 纪念卡 | `MemorialCard` | PropertyGroup=1011情人节系，1010春节系 |
| 礼包 | `Pack` | 多 sheet：Pack/ChainPack/PackTypeInfo 等 |
| 排行榜奖励 | `Rank` | 各 Actv 排行榜挂这里 |
