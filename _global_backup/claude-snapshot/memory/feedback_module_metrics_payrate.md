报任何模块/档位的付费数据时，**付费玩家付费率（=模块买家÷当期该服段总付费人数）必须和 ARPU（同分母）/ARPPU（买家分母）一起出现**，缺一不可；档位拆分表也要逐档带付费率列。

**Why:** 2026-07-16 双节回归中用户连续两次补要付费率（对打表一次、竞猜档位表一次），并点名"我发现你特别不喜欢做付费率这个数据"——ARPPU 只讲深度，付费率讲渗透，用户的分析框架始终是渗透×深度两条腿（金标准结论 2.2×=渗透1.24××深度1.79×），漏付费率=只给半边。

**How to apply:**
- 模块表标准列序：收入 / 买家 / **付费玩家付费率** / ARPU / ARPPU / 复购 / max。
- 分母用当期该服段总付费人数（口径见 [[feedback_x3_festival_arpu_denominator]]），跨代对比时分母口径必须标注（部署面不同会失真，见 [[project_x3_dual_festival_regression]] 的夏日20.3%案例）。
- **回归/前后对比的结论要用三件套点名"主要增长点"**（2026-07-17 储蓄罐触达回归用户拍板范式）：增长 = ARPPU × 付费率 × 盘子，明说哪个因子在动、哪个不动——如"ARPPU 不变、ARPU +67%、主要增长的是付费率（5.7%→11.3%），符合预期（触达改造拉人头不动客单）"。别只报"没稀释"这种守势表述，要指出增长来源。
