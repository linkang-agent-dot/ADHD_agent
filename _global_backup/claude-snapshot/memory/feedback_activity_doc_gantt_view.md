写活动排期 / 活动记录 / 活动类交接文档时，**按「活动」分类，用甘特图视角**：一行一个活动，核心就是「**几号上什么活动**」（上线日期 + 活动名/玩法）。

**不要写配置落地的内容**——Pack ID、哪张表哪行哪列、GroupId、ContentID、TimeCycle id 这类实现细节不进文档（查表/查代码就有，占地方还过期）。

**Why**：受众（制作人/策划/协作者）看活动文档只关心「时间线上排了哪些活动」，配置细节是噪音。对齐 CLAUDE.md「信息有效度」原则。

**How to apply**：活动类输出先答「哪天 → 上什么活动」，按活动逐行列；配置落地细节最多给一个引用源指针（"配置见 X 表"），不展开。区别于 [[workflow_handover_assetization]]（那是接管化三件套，针对模块/换皮，不是活动排期）。

**交付格式（2026-06-22 补）**：HTML 类交付物默认做成**标准独立 HTML**（`<!DOCTYPE>` + `<meta charset="UTF-8">` + viewport + title，自包含无外链 CDN），用 `Start-Process <文件>` 本地浏览器打开即可。**不要默认走 Claude Artifact**（用户叫它"糖人风格"）——那产出的是无 head/charset 的片段，本地直接开会中文乱码；**也不默认上传 demo**（html-deployer skill）。要 artifact 或 demo 链接时用户会明说。

**视觉风格（家族范本·别自创难看的）**：排期/信息图 HTML 用本机既定「一图看懂」深色风——**范本 `KB\产出-数值设计\X3_世界杯\世界杯竞猜_进包一图看懂.html`**：深色底(#15171f)、窄栏、顶部 Tab 切页、内容用卡片行(日期色标 + pill 标签)。排期页签要**真甘特图**(横向日期轴+彩色条形+marker)时也套这套深色配色，别用浅色密集网格(被批"太丑")。关键时点(对阵确定/锁盘/开打)要**精确到小时且直接写进甘特条/时间条**，不要只埋在下方表格。范例成品=`KB\产出-数值设计\X3_7月排期_世界杯+深海节甘特.html`(两模块:甘特页签+详细信息页签)。

**HTML 编辑卫生坑**：用整文件 Write 重写时，之前靠 Edit 增量加的 CSS 类容易漏掉 → 引用该类的元素裸渲染(只剩文字、无底色框，看着像"格式错")。重写后或新加带 class 的元素后，`grep '\.类名{'` 确认样式定义还在；优先用 Edit 局部改、慎用整文件 Write。
