做数据回归时，不能纯看玩家购买数据来倒推活动的礼包结构和档位数量。正确流程是：先了解设计方案（设计了多少档礼包、定价梯度、预期目标），再用数据去验证设计是否达预期。

**Why:** 只看购买数据会遗漏"设计了但没人买"的礼包档位，导致分析结论偏差。例如 X2 大富翁可能设计了 8 档成就礼包但只有 5 档有购买记录，那"3档无人购买"本身就是重要发现。纯数据反推会丢失这个维度。

**How to apply:**
- 做数据回归前，先向用户反咨询：活动设计了哪些礼包/模块？各档位定价和预期目标是什么？
- 如果用户提供了 Google Sheet（案子/数值表），优先读取设计方案，再查数据
- 报告中应包含"设计 vs 实际"对比维度，而非仅呈现实际数据
- 适用场景：任何活动数据回归、礼包效果分析、节日复盘
