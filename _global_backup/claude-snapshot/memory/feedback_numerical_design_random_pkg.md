name: 随机礼包期望值调整原则
description: 调整随机礼包 drop 权重（期望值）需有数据支撑，不能仅因配套数值变动而联动改动
type: feedback
originSessionId: 86e04cb5-a405-49fa-94a1-2ed0f5d80a02
---
随机礼包的期望值（drop 权重）不随 floor/锚点消耗调整而自动跟着改。

**Why:** 期望值变动会直接影响礼包付费价值感知和实际收入，没有历史数据或 A/B 测试支撑时，调整属于无依据操作，违反数值设计原则。

**How to apply:** 遇到 floor 消耗、锚点礼包内容变化等配套调整时，不要顺手建议同步修改随机礼包权重。如果认为有必要调整，需先提出"需要哪些数据来支撑"，由设计师决策后再动。
