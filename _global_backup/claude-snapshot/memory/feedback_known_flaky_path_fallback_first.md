# 已知不稳路径别选；排查先回退稳定路径

**Why:** 2026-07-16 船只手册D7验证：部署330时为了不踢用户下线选了「热更workflow」，但 [[reference_x3_kadmin_deploy]] 07-15 就记过「测试配置生效别只热更，deploy_app重部署最稳」。热更两次假成功（workflow success但配置没重读）→ 用户新号D0复现异常 → 我花40+分钟排查Mongo/服务端代码/导表工具，最后绕回重部署才解决。用户直接问「你为啥搞个环境能搞这么久」。一开始重部署，整段排查都不存在。

**How to apply:**
1. 动手选路径前，先查对应链路 memory 有没有「这条路不稳/别用」的标记——有就直接走稳定路径，省事/少打扰用户不构成选不稳路径的理由（踢下线重登 30 秒 vs 排查 1 小时）。
2. 出现「明明操作成功了但行为不对」时，第一步=回退到已知稳定路径重做一遍再复测（如重部署/重启/干净clone），复测还坏才往深挖；别带着「工具应该没问题」的侥幸在不稳定路径上正向排查。
3. 深挖过程有产出（新探针/新判据）照常沉淀，但要意识到那是补偿收益，不能反过来正当化选错路径。
