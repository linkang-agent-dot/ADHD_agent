任务/项目收口时的**归纳标准**（已写进 CLAUDE.md 行为规范「操作后收口 ④」，每次任务完成都做一次）。

**Why:** 之前收口（道2 Stop hook + 收工自检①②③）只兜「有没有沉淀」，不兜「沉淀得能不能被接管」。结果是积累极厚（150+ memory）但都停在两个高度：太碎的事实（reference）+ 太具体的**时间线流水账**（project 日记 v0.x→v0.y）。流水账是编年史，给旁观者看，不给接手人看——理由埋在时间线里 = **做完就没了**。缺的是「中间高度」：按**接管入口**组织的可复用结构。

**How to apply:**
- **判据**（唯一标准）：一个新 agent 冷启动，能否在「修BUG / 接管模块 / 换皮」**任一入口**立刻拿到全部 keypoint？不能 → 当场归纳。
- **深度随任务大小伸缩**：小任务 = 回写一行 memory / 给改动补一句「为什么这么改、解决了什么」；一个案子/项目 = **三件套**：
  1. `_决策记录_按模块.md` — 每条四栏：决策 / 当时卡哪 / 试过什么失败了 / 解决了什么。**显式标 ⚠️ 被否决的方案**（防接手人重蹈）。按模块排，不按时间排。
  2. `_换皮清单.md` — 骨vs皮 + 美术/配置 copy-and-swap 表 + checklist。注意：换皮清单是**给下一轮的遗产/输出**，不是建造期工具 → 不会被任何 build-time skill 自动触发，只能靠收口仪式逼出来。
  3. 产物目录 `_目录说明.md` — 标 FINAL / 废弃 / 留档（试错版和上线版混在一起时，接手人会拿错废稿）。非破坏式（写索引，不删不改原文件）。
- **挂进"唯一入口"文档**（如交接总文档的文档指针段），否则归纳了也没人找得到。
- **禁止只留时间线流水账**。流水账可留作深挖版，但不能是唯一沉淀。

**范式样板**：`KB\产出-数值设计\X3_世界杯\`（_决策记录_按模块 / _换皮清单）+ `KB\产出-本地化与美术\X3\世界杯竞猜界面\_目录说明.md`。这套也是"项目剧本/playbook"的**案例级版本**——跨案子抽象一层即 playbook。
