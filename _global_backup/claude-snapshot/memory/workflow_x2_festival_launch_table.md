# X2 节日活动上线表工作流（2026-06-11 拓荒节首跑）

**表**：GSheet `1QfV-hLxER8ou0t-FBcc4SMnrt0esFjcMDCUKhi_pfHg`，每节日一个页签（占星范本 gid=15932925）。新节日 = 照占星页签结构生成新页签内容，用户再人工调。

## 结构（17 列 × 双批次）
- 列：`批次标识 | 部署日期 | configId | 活动名称 | 跨服(0/1) | 开始时间 | 结束时间 | 结束展示截止 | 服务器模式(schema/list/all) | 服务器值 | 排除服 | 同步审核服(默认NO) | 备注 | 状态🔒 | iGame ID🔒 | 部署时间🔒 | 错误信息🔒`（🔒=脚本回写）
- **灰度批次是可选的，每节日先问用户做不做灰度**（占星做了，2026拓荒明确不做）。做灰度时：灰度批次在前（X2 灰度服=`[1002102]`，仅 6 个先行活动：大富翁/掉落转付费×3/巨猿/周卡），**正式批次**在后（行内再放一次表头行）；灰度先发过的活动正式批次剔除 1002102，开始时间比灰度晚 1 天。
- **不做灰度的变换**：删灰度块、全行恢复含 1002102 的全量服、原灰度先行活动的开始时间提前回原灰度日（即正式-1天）。
- ⚠️ `backup_tab` 同一页签同一天第二次备份会返回 False（同名冲突），当天首备份仍在即可，别误判成写入失败。
- 时间 = 北京时间，仅写日期默认 08:00。

## 生成法
1. 读占星页签全量 → 逐行对照换 configId（新节日 id 用户会先贴在页签里）。
2. **日期整体平移**：占星主力开 05-11 → 新节日主力开 X，全表 + 同一偏移（拓荒 +32 天）；各活动相对节奏（掉落只开6天/7日开8天/抢购两期各2天/强消耗晚6天开/装饰礼包晚6天开）自动保留。
3. 服务器值：问用户口径（拓荒 = schema≥3 共81服，见 [[x2-server-schema-lookup]]），从 iGame 现拉（gameId 1089）别用缓存名单。
4. 占星没有对照的新活动（拓荒多了 BP礼包/gacha累计/挖矿累计/合成小游戏 4 个）：按常理填 + 备注「⚠️新增·无占星对照」让用户拍板；合成小游戏注意可能要按 schema 拆多活动。
5. 写前 duplicateSheet 备份；页签里用户已有内容（configId 对照）不删，挪到表格下方保留。

跨服口径：累充=1+跨服排行、**两个 BP（基金+BP本体）都=1+跨服排行**（2026拓荒被纠正：占星表上 BP 写"跨服无排行榜"别照抄，提交漏 rank=1 导致 13572 重发为 13589）、其余 0。

- 🔴 **时间语义（2026-06-11 被纠正两次后定稿）**：startTime=开始日 00:00:00 UTC；**endTime=最后活跃日的次日 00:00:00 UTC**（用户原话"到次日的0点"）。例：活动跑 06-12~06-18 共7天 → end=06-19 00:00 UTC；单日抢购 06-15 → end=06-16。⚠️ 我先后错过「前一天23:59:59」和「当日0点」两版（后者整批短一天、21条全部取消重发），别再用。表「结束时间」列写的就是这个次日日期。

## 配套产出
- **格式**：上线表页签写完后用 batchUpdate `copyPaste PASTE_FORMAT` 从占星页签逐块拷格式（表头/说明/灰度块/正式数据行铺满）+ updateDimensionProperties 拷 17 列列宽 + 冻结前 2 行。⚠️ PS 里含 `[模版]` 这类方括号路径要用 `-LiteralPath`（会被当通配符）。
- **甘特图页签**（用户会要求）：同表新建页签，列=活动名/configId/开始/结束+逐日 12 列（表头带周几、周末加深），灰度块橙条 f6b26b / 正式块蓝条 6d9eeb 分块排，⚠️新增活动名前带标记，冻结 2 行+首列；做法=addSheet → update_range 写值 → repeatCell 按各行开始~结束铺背景色（2026 拓荒范例脚本结构见本条，数据每节日不同现写即可）。
- 上线表/甘特图同源，**方向可反转**：用户常直接改甘特图再喊「以甘特图为主/重算时间」。⚠️ **用户改的是色条不是开始/结束列——列值会过期，真源=色条**（2026拓荒实测：列值原封不动、色条全重排）。读法：`spreadsheets get` + `fields=sheets/data/rowData/values/effectiveFormat/backgroundColor` 取日期列区背景色，非白=活动日。**色条语义（定稿）**：每格=一个活跃日；首格=开始日，**结束=末格的次日**（end 次日 UTC 0点，见上方🔴时间语义）——所有时长统一按此，不分单日多日；部署日期沿用各行原规则（D0活动=06-12、晚开=开始当日、抢购后期=开始前1天）。同步完提醒用户甘特图的开始/结束列已过期。
- ⚠️ gws wrapper 的 `ranges` 参数只吃**单字符串**不吃数组（数组会 400 Unable to parse range）；中文页签名要带单引号 `'页签'!A1:R28`。
- ⚠️ **run_gws 读含中文 formattedValue 的 `spreadsheets get` 会静默返 None**（gws stdout GBK 解码炸 json，同族坑见 [[gws-gbk-search]]）；改用 `node gws_stdin.js`（UTF-8 管道）调同一 API 即可。values get 走 gws_stdin 一直没问题。
- `backup_tab(sid, tab, '0611_v2')` 第三参传自定义后缀可避开同日同名备份冲突（默认日期后缀当天第二次会返 False）。
