出任何「现有界面改造/加入口/换皮」类效果图之前，**先查实装**（客户端代码+prefab 或实机截图），不要拿策划案/设计母表里的界面稿当底。

**Why:** 2026-07-14 海妖手册案实翻车：按海妖设计母表的「子活动入口卡片列表」设计稿出了猎场界面 mockup，用户随后发来实机截图——实装是自由布局立绘式（`UIMechaActiEntrance.cs` 硬编码入口），设计稿根本没实装，一次 gpt 生成白烧。用户指正原话「你应该先查代码，再去出图」。

**How to apply:** 界面类媒体任务派发前的前置检查：①有没有实机截图（找用户要/自己起 Editor 截）②界面类是哪个 .cs/prefab、入口机制是配置驱动还是硬编码（决定"对应配置"答案）③设计文档只当意图参考，视觉与结构以实装为准。顺带收益：查代码常能顺手解掉配置/路由疑问（本案 type27 跳转冒烟项就是查代码时解除的）。
