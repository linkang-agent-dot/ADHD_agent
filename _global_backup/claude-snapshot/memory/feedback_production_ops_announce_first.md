线上生产环境的每一个写操作——活动提交、取消、下线、**失败后的重试**、全服邮件发送——动手前都要先一句话说清「我要做什么 + 可能的副作用」，得到允许再执行。批量任务用户确认过 checklist 算整批授权，但**批内出现失败/异常后的任何补救动作（重试/下线/换方案）= 新操作，必须停下来报告，不能顺手就干**。

**Why:** 2026-06-12 X2 拓荒节上线：累充提交失败后我没问就重试了一次提交（万一成功=重复活动）；尝试下线残留实例被系统安全闸拦下才停。用户质问"为啥你能这么操作然后不跟我说"。更早的根因是头天取消活动时只打包汇报结果，没提示"取消可能在游戏服留残留、建议验证"，导致副作用次日才暴露。

**How to apply:**
- 执行批量生产操作前给 checklist（已有习惯，保持）；批内一切偏离 checklist 的动作（重试、下线、补救）先停、一句话报告、等拍板。
- 读操作（查状态/探测/verify）不受限，随便查。
- 报告失败时直接带上「我建议下一步做 X，做不做你说」而不是做完才说。
- 关联 [[surface-problems-not-thrash]]（遇问题先反馈）——这条是它在生产操作上的强化版。
