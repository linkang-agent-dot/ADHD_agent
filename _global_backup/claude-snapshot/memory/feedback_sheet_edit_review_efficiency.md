改 GSheet / 过审策划案这类活，反复在同样的地方慢 + 返工。2026-06-03 大富翁策划案过审复盘出的提速纪律：

**1. 工具复用，别现写**：GSheet 读写一律用 [[gsheet-toolkit]]（`C:\ADHD_agent\scripts\gsheet_utils.py`），别临时写脚本——否则重踩 batchGet 崩 / 控制台乱码 / gws.cmd 角括号，每次试错好几轮。

**2. 改单元格按内容定位，别手算行号**：用 `gs.find_row_by_value` 找目标行再改，别数行号——大富翁验收 B16/B17 就是手算错位，覆盖了珍珠贝 #14、漏改等级变色 #15，最后靠全量复查才抓回。

**3. 改前先全量扫，一次列清单批量改**：删/改老机制前，先全表 grep 删除链关键词（升级/等级/经验/神秘宝藏…），把**所有**残留点一次列全、批量改；别改一处发现一处分批返工（这次删除链分了 B16/B17 → 又 B9/美术/E20 两批才扫干净）。

**4. 删/改用户内容先问**：见 [[删改用户内容模块前先问]]——授权范围精确对齐，不连带删"我觉得也该删"的。

**5. 派 task-checker 提速**：① 把已 dump 好的全量内容文件路径喂给它（省它自己读 sheet 的往返）；② prompt 明确"**最后一条消息只输出 JSON、那条里不要再调任何工具**"——否则它会卡在中间工具调用、输出崩了得重跑（这次第一次就崩，白费 ~3min）。

**复盘**：这次"久"的大头是**可避免的浪费**（工具试错 + checker 重跑 + 分批返工 + gws 调用碎），不是流程本身必须。按上面纪律，同类活下次可快一半以上。关联 [[x3]] 策划案流程模板、[[gsheet-toolkit]] 工具。
