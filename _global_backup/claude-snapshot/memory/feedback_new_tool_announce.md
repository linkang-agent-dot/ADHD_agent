新造可复用工具（脚本/skill/探针）时，收工消息里必须明说一句「新增了工具 X，干什么用的、怎么调」，不能只沉淀进 memory/KB 就完事。

**Why:** 2026-07-07 用户发现 `unity_prefab_tree.py` 已存在近一个月（6-11 世界杯案的副产品）而他完全不知道——当时只在 workflow memory 里记了一行用法，从没报备。工具对我是标配、对用户是隐形，等于复用价值砍半。

**How to apply:**
- 收工自检第③条（复用固化）执行时加一问：这工具用户知道吗？不知道 → 收工消息里带一句。
- 周五每周归纳清单里"新增工具"单列一节兜底核对。
