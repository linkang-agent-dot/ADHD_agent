X3 活动数值设计的**专属 skill** = `x3-numerical-design`，位置 `C:\Users\linkang\.claude\skills\x3-numerical-design\`（Claude Code 已注册，触发词见其 description）。

**它干嘛**：把一份 X3 策划案变成可落地的数值表（道具价值 + 礼包 ROI + 投放内容），面向**对数值零基础的程序员**——从头讲清 ROI 和道具价值，按玩法路由到往期参考，最后独立复查防投爆。是 [[x3]]（策划案模板）的下游、[[reference_x3_config]]（落配置）的上游。

**5步主轴**（SKILL.md）：第0步读策划案拎5样(模块/玩法类型/价位档/投放定位/大奖) → 第1步定盘(道具价值+分模块ROI) → 第2步填数(按往期同模块投放定位) → 第3步落配置(交 `x3-config-export`) → 第4步复查(派 `task-checker` type=numerical 双维度)。

**核心数值逻辑**（用户拍板）：核心=ROI+单个道具价值；**北极星=不投爆ROI**；同模块ROI一致($9.99给X/$19.99给2X线性)；**ROI无全局标准、只跟同模块往期比**(链式≈2.25x/周卡≈2.75x/皮肤档17~19x/抽奖整活动≈8x)；抽奖类(转盘/推币机/大富翁)走单独"整活动ROI"逻辑不是单礼包。

**🪤 跨项目搬运"件数/分钟数 1:1 平移"陷阱（2026-07-09 马戏节扭蛋机实证）**：加速/通用填充道具在两个项目里**相对价不同**（X3 30m加速=0.4×传奇技能书 vs X2 同类=0.14~0.25×金书），按件数或分钟数 1:1 平移=按价值超发。马戏节普通池平移 X2 分钟数后，免付池比值 45.5% vs 基准带 P2≈11-18%/X2≈15-29%，超标 1.6 倍。**正解=平移"价值占比"，用「免费池EV/付费池EV 比值」做跨项目校验**（比值对口径系数不敏感，绝对值不可比但比值可比——项目内同口径算 EV 再取比）。对比底稿范例：`config-library\cases\2026_circus_strong_consume\x3_mapping_research\pool_ratio_compare.md`。

**🪤 券类价值双口径坑（2026-07-08 马戏节扭蛋机映射实测）**：X3 抽奖券类道具存在两套价值——**道具表口径 ≈500钻/张**（1042/1096/1128 实查均 500，前端"价值%"展示用它）vs **设计锚 $0.25=125钻/张**（foundation 口径，ROI 评估用它），差 4 倍。做 X3 数值时两处别混：算 ROI 用设计锚，配前端展示价值用道具表口径。同案还实锤：**X3 券礼包惯例=必搭钻石(面值≈售价1:1)+VIP点(2022)、券≈4张/$平铺**（深海锚点包 13021-024）；**低流通孤品禁用清单**（映射/换皮选道具先查 Reward 组数）：1004玩家经验=0组、4301自选袋=0组、3103高级资源箱=1组、11005通用3h加速=7组。
