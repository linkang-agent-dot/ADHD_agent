# Unity 图标提取全自动归档

## 规则
当用户提供 Unity Inspector 截图并提到"提取图标"、"复制图标"、"归档"、"保存这个"时，**必须全自动执行**，不询问任何中间步骤。

## Why
用户期望的是"看到截图→直接得到归档结果"的一键流程，不希望被问"保存到哪个项目"、"是否需要保存"等问题。项目信息从对话上下文中自动判断即可。

## How to apply

### 自动判断项目
- 从对话上下文中识别项目关键词（P2/机甲、X2/巨猿、X3/TimeCycle）
- 如果上下文没有明确项目，从截图路径或最近的工作目录推断
- 只有在完全无法判断时才询问项目

### 自动提取图标名称
- 优先从截图中的 Inspector 面板读取资源名称
- 如果截图不清晰，从用户描述中提取（"这个 icon_xxx"）
- 如果都没有，使用通用命名 `unity_icon_{timestamp}`

### 直接归档路径
```
C:\ADHD_agent\KB\产出-本地化与美术\{项目}\UI图标\{图标名称}.png
```

### 执行流程
1. 读取截图 → 裁剪图标区域
2. 确定项目 + 图标名称
3. 创建目录（如不存在）
4. 直接保存到 KB
5. 向用户报告："已保存到 KB/产出-本地化与美术/{项目}/UI图标/{名称}.png"

### 禁止的行为
- ❌ 询问"保存到哪个项目"
- ❌ 询问"是否需要保存"
- ❌ 先保存到桌面再询问是否移动
- ❌ 展示多个路径选项让用户选择

## 实战案例

**案例 1：X3 全球积分图标**（2026-05-25）
- 用户："我想单独把这个图片复制出来"
- 错误做法：询问"保存到哪个项目"、"用哪种方式提取"
- 正确做法：从上下文识别 X3 项目 → 从截图读取 `icon_global_integral` → 直接保存到 `KB/产出-本地化与美术/X3/UI图标/` → 报告完成

**案例 2：用户说"归档到 KB"**
- 这已经是明确指令，直接执行，不再确认任何细节
- 自动判断项目、自动提取名称、自动保存
