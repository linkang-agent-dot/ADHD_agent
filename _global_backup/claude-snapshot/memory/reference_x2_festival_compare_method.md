# X2 两节日开局窗口收入对比方法论（2026-06-22 占星vs拓荒沉淀）

做「为啥这期节日比上期少收入 / 差在哪几个模块 / 还能不能追平」这类对比时套这套。数据源 Datain `v1089.dl_user_order o` LEFT JOIN `v1089.dim_iap i`，用 `C:\ADHD_agent\skills\ai-to-sql\scripts\_datain_api.py::execute_sql(sql,'TRINO_HF')`。

## 口径对齐三铁律（不对齐=结论全错）
1. **服务器必须可比**：先 `count(distinct o.server_id)` 数两期活跃出单服。只有服数一致才公平（本次两期都=30服才敢直接比）。不一致要么限定同一批服、要么明说规模因素。
2. **窗口对齐用 D0–D9 完整10天**：最新节日的 D10=今天当天数据不全，剔除；两边都取 D0–D9。`Dn=(partition_date-D0).days`。
3. **模块归类靠名字关键词、不靠 id 段也不靠"谁家名字"**：⚠️X2 节日 BP/GACHA/七日/抽奖券/掉落转付费**复用旧 id**，dim_iap 名字常是旧节日的（拓荒 BP/GACHA 名字写"2025占星节"也有）→**按模块语义归类，只剔真·往期残留**（名字含`夏日节/万圣节/2024`的才是别的活动尾巴）。

## 节日收入判定口径（FEST，本次实测全覆盖）
```sql
FEST = (i.iap_type IN ('混合-节日活动','活动礼包')   -- 抓复用旧id的BP/GACHA/大富翁/七日
   OR substr(o.iap_id,1,7)='2013920'                 -- 当期节日新段
   OR substr(o.iap_id,1,8)='20135001'                 -- 强消耗抽奖券复用槽
   OR substr(o.iap_id,1,8)='20132302')                -- 挖矿关卡scene礼包
```

## 模块 classify（名字关键词，剔往期残留优先）
往期残留(夏日节/万圣节/2024)→剔。挖矿(名字含挖矿 or id 20132302)→有"翻倍/特权"=挖矿特权,否则挖矿关卡礼包。其余:GACHA / 大富翁(大富翁|骰子|存钱罐) / BP通行证(通行证|节日活动BP|bp礼包|BP集结) / 七日活动 / 自选周卡 / 每日礼包 / 掉落转付费 / 强消耗抽奖券 / 限时抢购 / 行军表情 / 巨猿砸蛋(砸蛋) / 装饰礼包(装饰) / 连锁&节日礼包(自选卡包|连锁|礼包) / 其他。

## 缺口拆解 = 广度 × 深度（核心框架，2026-06-22 命名校正）
**付费玩家ARPU = 付费玩家付费率(广度) × ARPPU(深度)**，分母都用付费盘(总付费人)，三者自洽：
- **付费玩家付费率(广度)** = 模块买家 / **全部付费玩家(总付费人)** —— 该模块在付费盘里的渗透。SQL 层 `count(distinct user_id)` 按模块去重（**不能按 iap buyers 相加**=重复计人）。
- **ARPPU(深度)** = 模块收入 / 模块去重买家 —— 只看买家深度，⚠️**会掩盖广度**。
- **付费玩家ARPU** = 模块收入 / 全部付费玩家 = 广度×深度，**最准**(见上节口径排序)。
**模块病灶三分类（每个缺口模块对号入座，决定怎么救）**：
- **广度型双降**（核心引擎，付费玩家付费率掉为主+ARPPU也降）：本案大富翁(−55%=广度−44%×深度−20%)/GACHA(−37%=−29%×−11%)。**靠排期/玩法把引擎做旺救回**。
- **纯广度崩·搭车模块**（付费玩家付费率大跌、ARPPU 持平甚至涨）：本案周卡(−38%=−38%×0%)/抽奖券(−42%=−51%×+18%)。**不是配价问题，是节日整体热度起不来没人进场**→靠把核心引擎做旺带动整体广度。
- **纯深度崩·配价问题**（ARPPU 腰斩、付费玩家付费率没怎么动）：本案连锁礼包(−63%=广度−21%×深度−53%)。**单独重配档位即可**。
- **健康**（广度降但深度涨自抵消）：本案 BP(−5%=−22%×+22%)。

## 三个口径选哪个（2026-06-22 校正，权威）
评"效果好不好"按可信度排序，**优先付费玩家ARPU**：
1. **付费玩家ARPU = 模块流水 / 全部付费玩家(总付费人,不是节日付费人)** —— ⭐最准。分母=付费盘(稳定,不受活跃口径污染),且**同时含广度+深度**:买的人变少会直接拉低它。`节日付费玩家ARPU = 节日流水/总付费人`,本次 $18.6→$12.9(−31%);全盘 $67→$54(−20%)。**符合用户既定规矩**(节日ARPU分母=所有付费玩家,见 [[feedback_x3_festival_arpu_denominator]])。
2. **ARPPU = 模块流水 / 买了该模块的玩家** —— ⚠️**会骗人**:只看买家深度,**掩盖"买的人变少"**。本次周卡 ARPPU $48→$48(持平)、抽奖券 $28→$33(+18%),但两者付费玩家ARPU 都 −40%(买家塌)。单看 ARPPU 会误判"没事"。
3. **付费率 = 付费人 / 活跃(DAU)** —— ❌两期不可比,别用作结论。`ods_user_daily` 累计活跃因**合服+老服衰减**大幅变化(占星日均DAU 15595→拓荒11048,−29%;累计108k→62k)。**累计口径 vs 日均口径会给相反方向**(本次累计付费率0.22%→0.31%"升",日均0.57%→0.51%"降")——这本身就证明它不可靠。只可看**同一期内每日付费率轨迹**(趋势,非跨期水平)。
关系式:付费玩家ARPU = ARPPU × (模块付费人/总付费人)。

## 追平预测算法
缺口÷剩余天数=**所需日均**;对比**实际近期日均**(取尾段D7-D9均值,已进衰减区,最代表"今天水平")。节日尾段只衰减不反弹→所需日均若>实际1.5x基本判"追不平"。**还要看逐日付费人数**:塌方=结构性枯竭,延长天数补不回。

## 本次结论（拓荒 vs 占星 D0–D9，~30服，2026-06-22；付费玩家ARPU 校正版）
占星 $25,733/233节日付费人 vs 拓荒 $17,288/193,缺口 **$8.4k**。**用付费玩家ARPU 重新诊断(比原"广度塌"版更准)**:
- **付费盘几乎没掉**:总付费人 1383→1338(−3%)。掉的是**每个付费玩家从节日榨出的价值**:节日付费玩家ARPU $18.6→$12.9(−31%)。
- **不是"没人了",是节日对每个付费玩家拉动变弱**。按模块付费玩家ARPU:GACHA $5.25→$3.31(−37%)、大富翁 $3.96→$1.77(−55%)、连锁礼包 $1.21→$0.45(−63%)、周卡 $1.47→$0.90(−38%)、抽奖券 $1.35→$0.78(−42%);仅 BP $3.30→$3.13(−5%)稳。
- ⚠️**修正原"ARPPU普遍持平"的错判**:周卡/抽奖券 ARPPU 持平甚至涨,但付费玩家ARPU 大跌=买家塌(广度),ARPPU 口径掩盖了。深度也确有降(节日ARPPU $110→$90,−19%,集中大富翁/GACHA)。
- DAU 15595→11048(−29%,合服+衰减)流失的主要是非付费/轻度玩家(付费盘只−3%),所以别把缺口归到"付费率/活跃"。
**对 X3 的含义(本案直接动因)**:占星(5月)大富翁+GACHA **D0 同台**→付费玩家跨模块买、人均更高;拓荒拆成 D0-D6 GACHA / D7 大富翁→大富翁被推到低流量尾段、付费玩家ARPU 被拖崩。**X3 回到 D0 同时上=对,能救付费玩家ARPU;但救不了 DAU 大盘(靠开局拉新/留存);世界杯补深度天花板不解大盘。**
