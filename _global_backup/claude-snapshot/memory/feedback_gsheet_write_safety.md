## 规则

### 写入授权（最高优先级）

0. **必须等用户明确说"可以写"、"确认"、"写入"才能执行写操作**，上下文推断（如"这两个不用改"）不算授权。即使在 Auto 模式下，GSheet 写入属于影响共享系统的不可逆操作，必须显式确认。

### 写入前（必做）

1. **备份必须在第一次写入之前** → `_bak_{日期}_{页签名}`（前缀 `_bak_` 避免被搜索工具优先选中）
2. **查目标 ID 是否已存在**，跳过已有行，防止重复写入
3. **校验列数 = 表头列数**，不足前补空列，防止列偏移
4. **禁用 `append INSERT_ROWS`** → 只用 `insertDimension + values.update`（append 会插到表头而非表尾）
5. **找最后数据行必须遍历 ID 列** → `max(i+1 for i, b in enumerate(all_b) if b and b[0].strip())`。**严禁用 `len(values)`**！API 返回的行数 ≠ 最后数据行（中间空行导致截断）。统一用 `gsheet_utils.find_last_data_row()`

### 写入中

5. **克隆行必须全列**（读源行用 A:AZ），不能截断，否则 INT 列为空 fwcli 报错
6. **新行必须有 p2_title 空列**，跟已有行格式一致
7. **超长 JSON（>7000字符）自动 fallback 逐 cell 写入**

### 删除行

8. **禁止按行号盲删**（每删一行后续行号全部偏移，连续删必错）
9. 正确做法：**先读目标行内容 → 确认是要删的行 → 从后往前删**

### 崩溃恢复

10. 脚本报错/崩溃后，**先验证 GSheet 实际状态**再决定是否重跑（API 调用可能在崩溃前已生效）

### 回滚

- 如果改错了，从 `_bak_` 页签复制数据回主页签
- 确认无误后删除 backup 页签

**Why:** 2026-05-18~19 拓荒节换皮时：
- append INSERT_ROWS 把数据插到表头，挤掉原始行（2121/2118/2115）
- 崩溃后重跑导致重复写入（2112）
- 按行号盲删误删 IPO 行
- 备份在污染后创建无法恢复（2121 的 21211001 丢失）
- 克隆行列数不全导致 fwcli 逐列报错（1365）

**How to apply:** write_activity.py 已内置以上规则（B1~B6）。手动写入时也必须遵守。
