## 策划案大纲逻辑原则

策划案结构必须遵循**项目内部逻辑**，不能按"清单填空"方式写。

**Why:** 用户明确反馈：好的策划案是根据玩家行为链、设计假设、机制逻辑来展开的，不是套模板逐项填写。按模板填出来的内容是表面的，缺乏策划案应有的逻辑结构感。

**How to apply:**
- 写大纲前先想清楚：这次优化的核心逻辑链是什么？（付费断层在哪 → 原因是什么 → 对应动作是什么）
- 章节顺序要能推导出下一章，而不是并列的"todo list"
- 每个设计动作要有数据/行为链支撑（即使是假设，也要写出依据）
- 内容结构参考现有策划案布局（如 `尼罗活动优化` 页签），但根据这次优化的具体逻辑重新组织，不要照搬



**How to apply:** 每次写策划案到 GSheet 后，跑格式脚本，再对照 checklist 与用户确认。

## 策划案内容结构模板

```
大标题：付费深度提升思路
  1、现有模块优化...
    1）具体动作
  2、无显投放新增...

大标题：付费深度增量预估
  表格：模块 | 现状ARPU | 预测ARPU | 增量 | 提升幅度 | 依据

大标题：{节日名}-活动排期总览
  1、活动开启list
    表格：活动 | 排期 | 功能开发量评估(颜色) | 活动ID
  2、活动开启范围
    表格：模块 | 解锁条件 | 配置表

大标题：{节日名}-各投放情况
  表格：类型 | 名称 | 定价 | 调整(颜色) | 说明

大标题：{模块名}（分模块详细设计）
  一、设计目标（bold11）
    1.1 核心目的
  二、活动情况（bold11）
    2.1 活动开启
      2.1.1/2/3
    2.2 [功能点]
      [配置说明] / [客户端处理] + 详细内容
    数值配置表（如有）
```

## ★结构规范（2026-06-02 大富翁返工沉淀 · 最易违反）

**根因教训**：把"分页签 + 分列分层"压成"单页签 + 单列流水账" → 逻辑/规则/配置/美术全糊在一起（被用户指"很变态"）。金标准范本 = 推币机单服版(`1MJbuYbnPS-M092mImveO0UYyi-RAg7t9_LXu0Aouuzg`)，9 个独立页签 + 规则说明列阶梯；航海之路原版同理(5 个分页签)。

### 规则1：按关注点拆独立页签（绝不混页）
单玩法案标配页签：`规则说明 / 数值设定 / 界面键值 / 美术需求 / 音效需求 / 配置表 / 打点 / 文案 / 验收` —— 每个关注点一个 tab，规则≠配置≠美术，物理隔开。**绝不把规则/配置/美术堆在同一页。**

### 规则2：规则说明用"列阶梯"表达层级（不要全塞一列）
每深一层右移一列：A=大节(一/二/三) → B=一级编号(3.1) → C=二级编号/功能点名(3.1.1 活动类型) → D=标注格[客户端处理]/[配置说明] → E=正文(WRAP) → F~=参数/事件矩阵(带表头行)。参数用**真表格横铺**，不用斜杠串。

### 规则3：配置表 4-5 行横铺 schema（不是句子）
每张表写成网格，行=类型(cs)/数据类型(int)/引用(Item·外表.字段)/中文字段/英文字段[/示例]，**字段横向铺列**，程序可直接照列建表。禁止 `ID/IslandGroup/.../...` 斜杠串成一格。

### 规则4：[客户端处理]/[配置说明] 用标注格分隔
表现/客户端逻辑/配置说明**不要混进规则正文句子**——单独放标注格列：[客户端处理]=粉 #FF99FF，[配置说明]=米黄 #FCEBCC。

## ★写完 Double-Check 流程（强制，claim 完成前不跳）
1. **自查对照 checklist**（上方格式 checklist + 本条结构规范 1-4）。
2. **读回验证**：拉关键页签实际单元格，确认列阶梯/横铺 schema 真落对列（不是全挤 A 或 C 列）。
3. **存疑则喊 sub-agent 独立审**：对照范本(推币机/航海之路原版)，要"哪儿挤/哪儿混 + 重构蓝图"。
4. 三步过了再告诉用户"完成"。
> Why：2026-06-02 大富翁第一版没自查直接交，被指"全挤一起、逻辑配置混、规则美术混"，整页返工重建。

## ★换行策略(2026-06-10用户反馈)
构建脚本**禁止给单元格设 WRAP 换行**——用户偏好全表 OVERFLOW 溢出显示,每次设回换行他都要手动改。新建任何 sheet 构建脚本时格式重置统一 OVERFLOW_CELL,不加 wrap 段。
