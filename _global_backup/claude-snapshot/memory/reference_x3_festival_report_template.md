## 报告区块（生成顺序）
1. **日期 tabs + KPI**：总流水/付费人数/节日流水/非节日/节日累计/累计占比
2. **ARPU 增量分析**：节日/非节日/综合 ARPU vs 14日基线
3. **节日模块构成（单日）**：选中日各模块条形
4. **节日模块构成 · 全程累计总量**：D0~当前各模块累计 + 日均
5. **每日流水趋势**：总流水 + 节日流水两张折线（独立坐标）
6. **模块变化趋势**：单模块逐日折线（模块 tab 切换）
7. **分时收入趋势**：①各天叠在 0-24 轴对比（日 chips 多选）②D0–D{n} 全程连续分时；**R级筛选 chips**（全部/超/大/中/小/非R）+ 每时/累计切换，驱动两张图
8. **节日收入 · R级分层（累计到当前）**：各档节日流水/占比/付费人数/层ARPU
9. **R级付费玩家付费率 & ARPU · vs 上期情人节**（累计同期，排除免费玩家）
10. **节日模块 · R级对比 · vs 情人节**：模块×R级，付费率/ARPU/ARPPU，**R级筛选 chips**（全部/超/大/中/小R）
11. **按日同期对比 · D对D**（本期 vs 上期，逐日同序对齐）：日期 tabs（D0/D1/.../累计），每天指标表（带「差异」列）+ 左右模块条 + **R级对打表** + 结论；末位「累计」= 累计到当前日 *(2026-06-02 大改版新增)*
12. **关注点**：自动告警

## ⚠️ 口径决策（硬骨头，别推翻重走弯路）
1. **双服段 + 「服务器生命周期一致」原则**（核心，别搞错）：
   - **生命周期一致 = 各期取「D0 时距开服天数(服龄)处于同一阶段」的服（按服龄阈值匹配），不是同一批物理服**。
   - 目的：排除"新服爆发期"对对比的干扰（新服开服初期收入/付费天然更高）。
   - **夏日(本期，主报告+对比夏日侧) = `server_id 1000~1880`（89服）**：基本都是夏日 D0(05-29) 已过 D35 的成熟服（仅末位 1880 服龄32天略低，可忽略）。
   - **情人节(对比上期侧) = `server_id 1000~1540`（55服）**：情人节 D0(02-06) 已过 D35；1550+ 当时未开服/未过D35=爆发期新服，排除。
   - 实现：`SERVER_FILTER`(89,夏日默认) + `SERVER_FILTER_VAL`(55,情人节)；查询函数带 `sf` 参数，情人节侧调用传 `sf=SERVER_FILTER_VAL`。
   - 服龄查 `dim_open_server.open_time`；判定 = 该服在各自 D0 的 `date_diff(D0, open_time) >= 35`。
   - ⚠️ 绝对值(流水/人数)跨不同服数(89 vs 55)→只可比率(付费率/ARPU/ARPPU)；绝对值差含"服数不同"成分。
2. **节日包口径**：夏日 = ActvOnline **100595 累充白名单(权威17包)**；情人节 = **硬编码 `iap_id LIKE '2107%' OR in('130020','130021','1002001')`**（情人节当期配置已被夏日复用覆盖，查不到，用前缀近似；已核对 2107% 把婚礼/情人节主题包都框到、未漏，未命中的都是日常养成包）。
3. **R级分类 = 快照优先 + 累充兜底**：`RCLASS=COALESCE(rl.rb, ltv兜底)`。兜底阈值（USD累充）≥1500超R/≥500大R/≥50中R/>0小R，由已分级用户累充分布反推（中位 小R$4.7/中R$344/大R$825/超R$2064）。**非R=零充免费玩家**。`rl_join(asof)`/`ltv_join(asof)` 都按 as-of 日期取快照/累充。
4. **付费率必须用「付费玩家付费率」= 节日付费人数 / 当期总付费人数**（分母仅付费玩家）。**绝不用 /活跃**——老服免费玩家流失会把活跃分母带歪、付费率虚高（实测全88服旧口径 17.8%→49.2% 假象，成熟服付费玩家口径才 29.3%→43.7%）。ARPU=节日流水/总付费；ARPPU=节日流水/节日付费。
5. **同期对齐**：累计 D0~当前；进行中当天两边都截**首 N 小时**（report_max_hr=当天最大有数据小时）。完成日整日对整日。
6. **🔴 待办（最终版）**：R级分类时点要**钉到「节日后」的固定快照**（节日结束那份），档位才稳定+反映经此节日的最终身份。进行中(as-of当前)会随累充兜底含本期消费而**漂**，只动小R/非R边界、超/大/中R稳定——节日进行中**不改**，节后做最终版只需把 `rl_join/ltv_join` 的 as-of 改成节后快照日。详见 [[x3]]。
