# 用户已备好的产物 = 既定方案，改路线先问

**事故（2026-07-17 马戏扭蛋机 UI）**：用户早已在 X2 把扭蛋机界面换皮完成并整包导出（UIactvfesstrong.prefab+64图+动画），既定方案=prefab 迁入 X3（协议规格文档就是为此写的）。我评估"X2 prefab 跨项目移植组件全断风险大"后**擅自改走克隆 X3 拓荒骨架**路线，做完后用户进游戏发现形态完全不对（格子翻奖≠扭蛋机器），一句"我不是给了你换皮出来的 prefab 了吗"打回，白做一轮。

**Why**：用户备好的资产承载了他已拍板的方案与已投入的工时；我看到的"技术风险"他可能早有解法（本例：工程内就有成熟的 x2-to-x3-migration skill+文档体系，跨项目迁移是被支持的标准路径——我没先查就下了"风险大"的结论）。

**How to apply**：
1. 接手带既有产物的任务，第一步=盘点用户已备好什么（prefab 包/文档/脚本），这些就是方案本身。
2. 想换技术路线：先查项目内有没有支持既定路线的基建（skill/工具/先例），确实走不通再**带着证据问用户**，不自行切换。
3. 判"风险大"必须实证（本例 manifest 导出包+迁移 skill 都在，5 分钟就能查到），不能凭直觉。
