**先追链路、再写配置，不能反过来。**

**Why:** X2 2026占星节掉落转付费模块配置时，先写了 2112，后面才发现礼包走的是 2121(discount)→2011→2013 直连而不是 2135→2011→2013 链路。先入为主以为是 2135 链路，导致配错返工。同一次还复用了旧 constant（event_labor_2025_2_drop/pkg/shop），与线上已有活动冲突。

**How to apply:**

## 动手前必须做的（每次配活动）

1. **找一个同类型的已上线活动**，完整读一遍它的 2112 → 子表 → 子子表引用，画出实际链路
2. **确认链路模式**：礼包是走 2135→2011→2013，还是 2121(discount)→2011 直连？任务奖励是直接给 item 还是引用 drop？
3. **确认 constant 唯一性**：在 2112 QA 表搜索拟用的 constant 关键词，确认不与任何已有行重复
4. **确认所有被引用的 ID 都已存在**：不只是第一层（2112→2115），也要追第二层（2115.reward 里的 item ID 在 1111 是否存在、2135.iap 指向的 2011 是否存在）
5. **2011 的绑定字段必须逐个确认**：time_info.actv_id 绑当前礼包活动、iap_status.recharge_actv 确认归属，不能沿用模板值

## 实战案例：X2 2026占星节 drop_topay

这条链路涉及 3 个 2112 活动 + 7 张子表，不是单个活动问题：

```
2111（3条日历）
  ├── 21127361 主活动 → 2121(actv_links/drop_topay/progress) + 2137(retake) + 2116(exchange)
  ├── 21127359 掉落活动 → 2124(drop)
  └── 21127360 礼包活动 → 2121(discount) → 2011:2011920111 → 2013(×4档)
                                               ├── time_info → 21127360
                                               └── iap_status.recharge_actv → 211200093
```

如果只看 2112.components 停在第一层，会漏掉 2121→2011→2013 这条隐藏链路和 2011 上的两个绑定字段。

## 实战案例2：X2 2026占星节 每日Gacha礼包

标准 2135 链路，但有累充归属和深层奖励追踪：

```
2112:21127355 → components:
  ├── package(×5) → 2135 → 2011 → 2013（标准链路）
  │    └── 2011.recharge_actv → 211200213（曾误挂夏日节 211200093）
  ├── weekly_pay_ratio → 2121 → 1111(select_box) → 真实奖励道具（3层）
  └── task → 2115
```

教训：
- **2135→2011→2013 必须同时新建**，不能只改 2135 指向新 2011 但 2013 还指旧的
- **recharge_actv 必须显式确认**，不能凭经验复用——本次就是误挂了另一个节日
- **写完后搜索旧 ID**：在 2011 全表搜旧 recharge_actv ID，确认无残留
- **weekly_pay_ratio 要追3层**：2121 外层 → 1111 item_select_box → 真实奖励道具

## 实战案例3：X2 2026占星节 限时抢购

标准 2135 链路，但 2121 里有 flash_sale_* 特殊组件也引用 2013/2135：

```
2112:21127357 → components.package → 2135(×9) → 2011 → 2013
                                                        ↑
2121 flash_sale_gacha (21217033) ─── 也引用 2013920115–122
2121 flash_sale_popup (21217034) ─── 也引用 2135:213521211/212
2121 flash_sale_raffle (21217035) ── 也引用新 2135
```

教训：
- **2121 特殊组件是隐藏引用层**：只看 2112.components 完全看不到 flash_sale_* 也引用了 2013/2135
- **节日道具要区分类型**：BP/gacha/节日专属必须替换当前节日道具，大富翁类通用道具可保留
- **2013 列数到 AE**：A_INT_country_use_type 在最后一列，写/查到 AD 会漏
- **2135 空行问题**：中间有空行时 fwcli 会解析失败，必须保证 B 列 ID 连续

## 识别跨表引用 vs 普通参数

- 字段语义是"引用"才算跨表引用：`A_INT_iap`→2011、`A_INT_rank_components`→2118
- 8位/10位数字出现在 `arg1`、`display_key`、`expr` 等字段里，**不一定是跨表引用**（如 1511018855 是 recharge_actv 的 arg1 参数，不是 1511 表 ID）
- 判断方法：看字段名前缀和该 type 的含义，不要单纯按数字位数推断
