## 节日开发月度周期

每个月一个节日活动，按以下固定节奏推进。

### 关键里程碑

| 时间节点 | 事项 | 产出 |
|----------|------|------|
| 每月第2周 周三 | **部署** | 活动配置部署到灰度环境 |
| 每月第2周 周四 | **灰度上线**（D0） | 灰度服玩家可见 |
| 每月第2周 周五 | **全服上线**（D1） | 全量玩家开放 |
| 活动结束后下一周 | **回归 + 出方案** | 回归报告 + 下一期方案，同一周内交付（X2 回归须含优化方案） |

### 三阶段 SOW

```
━━━ 阶段一：设计 & 开发（上期D1 → 本期部署日，约3~4周）━━━

  上期全服上线后 → 立刻启动下一期迭代
  │
  ├─ 出方案（上期D21产出的设计/数值方案）
  ├─ 美需包装 → 美术出图
  ├─ 配表（数值换皮 / 新模块配置）
  ├─ 翻译（18语言本地化）
  ├─ QA & 活动检查
  ├─ 提测 → 测试改BUG（3天）→ 验收（1天）
  └─ 部署前 review

━━━ 阶段二：上线（3天）━━━

  周三  部署到灰度
  周四  灰度上线（D0）
  周五  全服上线（D1）→ 同时启动下一期迭代（回到阶段一）

━━━ 阶段三：运营 & 复盘 ━━━

  活动运行中    监控数据
  活动结束      ┐
  下一周        ├─ 交付：回归报告 + 下期方案
                │  （X2 回归须含优化方案）
                └─ 同时下一期已在开发中
```

### 周期重叠

阶段二（上线）和阶段一（下一期开发）是同时启动的：
- 全服上线当天就开始迭代下一个节日
- 所以在 D1~D21 期间，实际上同时在做两件事：
  - 当期活动的运营监控 + 复盘
  - 下一期活动的设计 + 开发
