## 规则1：锚点礼包是价格锚，不是目标销售品

锚点礼包的设计目的是**对比参照**（价格锚定效应），让成就礼包等主力礼包显得更划算。锚点本身购买率低是正常的，不需要"优化锚点购买率"。

**Why:** 作者review时指出"锚点本身设计就是对比，不影响"。把锚点低购买率当问题来分析，暴露了对礼包设计意图的误判。

**How to apply:** 分析锚点数据时，重点看它是否有效抬高了成就礼包的购买率/ARPPU，而不是锚点本身的转化率。

## 规则2：探究关卡/进度节点的转化断崖，应该用全量+付费玩家的停留曲线

不能仅靠"免费资源投放量刚好覆盖到N圈"就断言"N圈断崖是因为免费资源用完"。因为付费玩家会在免费资源基础上额外购买骰子，他们的进度应该远超免费上限。如果付费玩家也卡在同一个节点，那问题就不在免费投放量，而在进度驱动力。

**Why:** 作者指出"如果玩家有付费，他会额外在免费投放后付费，所以170→30圈这点讲不通"。正确做法是拿全量玩家和付费玩家两组的停留关卡曲线对比，看各自在哪里掉。

**How to apply:** 
- 遇到"某个圈数/阶段之后全部0购买"的现象时，不要直接归因于免费资源上限
- 应该查 **全量玩家进度分布** 和 **付费玩家进度分布** 两条曲线
- 如果付费玩家也大量停留在该节点 → 问题在深度驱动力/奖励吸引力
- 如果付费玩家突破但非付费没突破 → 问题才是免费资源不够
- 需要活动参与/进度埋点数据（如 ods_user_activity 或 counter 数据）
