删除或大改**用户做的内容模块**（策划案段落、给特定受众的版本、概述/总览区等）前，**先单独问用户确认**，不要自行判断"这是草稿残留/冗余/重复"就删。

**Why:** 2026-06-03 大富翁策划案过审：把规则说明页签 A3–A85「本期改动一句话总览」（用户专门给**制作人**看的"这期改了啥"版本）误判成"草稿残留"删掉了。而且用户 3A 授权的只是删"重复的**美术&本地化块**"，我删行时连那个总览一起删了（删 rows 2–139），既**超出授权范围**、又**没在动它前单独问**。用户明确反馈："你把我给制作人看的一句话版本全删了，能不能做改动模块的事情之前先问问我。"

**How to apply:**
- **授权范围精确对齐**：用户说删 X，就只删 X，不连带删"我觉得也该删"的 Y；删行/批量操作前，把"将删除的具体内容"逐条列出来，对照授权范围核一遍。
- **拿不准用途就问**：某块内容是"草稿"还是"给某受众的正式版本"分不清时 → 问，别猜。给制作人/策划/程序看的精简版常常**故意**和详细规则重复，重复≠冗余。
- 呼应 [[feedback_cleanup_checklist_first]]（删文件前出清单确认）——内容块删除同理，哪怕已有总体授权也要单独确认。
