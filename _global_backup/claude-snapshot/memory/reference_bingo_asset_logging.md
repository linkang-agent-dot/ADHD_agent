## 关键结论

BINGO 连线任务奖励的卡包在 `ods_user_asset` 里以**具体任务ID**记录，而非活动ID。
用活动ID（如 `21127035`）做 `reason_sub_id` 过滤会漏掉白包/绿包，只能查到蓝包等其他奖励。

## 正确查询方式

```sql
-- 查BINGO连线任务发出的卡包（白/绿包）
SELECT reason_sub_id, asset_id, COUNT(DISTINCT user_id) AS users, SUM(change_count) AS total
FROM v1041.ods_user_asset
WHERE reason_id = 'event_task'
AND reason_sub_id IN (
    '211550665','211550666','211550667','211550668',  -- 右1-右4
    '211550669','211550670','211550671','211550672',  -- 下1-下4
    '211550673'                                       -- 斜线
)
AND asset_id LIKE '111110%'
AND change_type = '1'
GROUP BY reason_sub_id, asset_id
```

## 日志规则（运维注意）

- `reason_id = 'event_task'`
- `reason_sub_id` = **具体任务ID**（非活动ID）
- 用活动ID（21127035）查 event_task 只能找到蓝包（111110004），白包会漏
- 蓝包（111110004）单独通过另一机制发放，reason_sub_id 为活动ID
