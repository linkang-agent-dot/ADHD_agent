## 单条礼包模版

每个礼包单独一个 `.md` 文件，存放在对应游戏文件夹下的 `礼包/` 目录。

文件命名：`礼包名称.md`（用游戏内原名，如 `精致月卡.md`）

```markdown
---
tags:
  - 竞品调研/礼包记录
  - game/{游戏缩写}
  - 商业化/{定位}         # 可多个：破冰/付费粘性/付费深度/活动付费/常驻付费
  - 付费内容/{类型}       # 可多个：英雄/加速/资源/装备/VIP/货币/皮肤外观
aliases:
  - {礼包简称}           # 方便双向链接引用
  - {礼包别名}           # 可选
game: {游戏全名}
记录日期: YYYY-MM-DD
礼包类型: 限时 / 常驻 / 首充 / 周卡 / 月卡 / 赛季通行证 / 节日 / 回流 / 新手 / 成长基金 / 累充 / 特价
价格: $X.XX
---
# 📦 礼包名称

- **记录日期：** YYYY-MM-DD
- **价格：** $X.XX / ¥X / 免费 / 礼包码
- **礼包类型：** 限时 / 常驻 / 首充 / 周卡 / 月卡 / 赛季通行证 / 节日 / 回流 / 新手 / 成长基金 / 累充 / 特价
- **商业化定位：** `商业化/XXX`（选一个主要定位）
- **核心付费内容：** `付费内容/XXX`（可多选）
- **目标人群：** 免费转付费 / 小R / 中R / 大R / 超R / 全体
- **限购：** 1次 / 每日1次 / 不限 / 活动期间X次
- **有效期：** 常驻 / YYYY-MM-DD ~ YYYY-MM-DD
- **展示位置：** 主城弹窗 / 商店页 / 活动页 / 邮件推送 / 登录弹窗
- **触发条件：** 无 / 等级达到X / 首次通关X / 活动期间 / 回流触发
- **标注增值：** X%（如礼包上有标注）
- **关联活动：** （用 `[[活动名]]` 双向链接，链接到独立笔记）
```
