## 单条活动模版

每个活动单独一个 `.md` 文件，存放在对应游戏文件夹下的 `活动/` 目录。

文件命名：`活动名称.md`（用游戏内原名，如 `寻医之道.md`）

```markdown
---
tags:
  - 竞品调研/活动记录
  - game/{游戏缩写}
  - 活动类型/{类型}       # 可多个：任务积分/通行证/签到/竞赛/累充/限时充值/抽卡/集换/联盟/跨服/导航
  - 商业化/{定位}         # 可多个：破冰/付费粘性/付费深度/活动付费/常驻付费
aliases:
  - {活动简称}           # 方便双向链接引用，如"寻医之道"
  - {活动别名}           # 可选，多个别名方便搜索
game: {游戏全名}
记录日期: YYYY-MM-DD
生命周期: 预热 / 进行中 / 结算 / 已结束
---
# 🎪 活动名称

- **记录日期：** YYYY-MM-DD
- **活动类型标签：** `活动类型/XXX`（可多选）
- **商业化定位：** `商业化/XXX`（可多选）
- **起止时间：** YYYY-MM-DD HH:MM ~ YYYY-MM-DD HH:MM（共X天）
- **生命周期阶段：** 预热 / 进行中 / 结算 / 已结束
- **入口位置：** 主城图标 / 活动页Tab / 弹窗 / 邮件 / 横幅
- **是否常规/首次：** 常规复刻 / 首次上线 / 改版复刻
- **关联活动/礼包：** （用 `[[活动名]]` 双向链接，链接到独立笔记）
- **设计目的：** （这个活动解决什么商业化问题？拉新付费/提升ARPPU/促活/回流/引导成长/付费转化）

## 活动规则
- **参与条件：** （等级/解锁要求等）
- **核心机制：** （积分制/排名制/进度条/抽奖/兑换/任务链...）
- **参与方式：** （简述玩家需要做什么）
- **活动货币/道具：** （活动专属代币名称、获取途径、兑换比例）
- **每日/每周上限：** （如有产出上限）
- **其他规则：** （特殊机制、组队要求等）

|---------|---------|---------|------|----------|
|         |         |         |      |          |

## 付费相关
- **活动内付费入口：** （直购/礼包/通行证...）
- **免费可达程度：** 全部免费可达 / 免费可达约X% / 必须付费参与
- **付费天花板：** 约 $X（达到满奖励的预估花费）

## 受众人群分析
- **免费玩家：** （能玩到什么程度？体验如何？）
- **小R：** （付费后体验提升多大？）
- **中R/大R：** （是否有追求空间？）
- **核心受众：** （这个活动主要服务哪个群体？）

## 复用性判断
- **是否可复用模板：** 是（换皮复用）/ 否（一次性）/ 待观察
- **预估开发成本：** 高 / 中 / 低
- **复用周期：** （如果是复用模板，多久出现一次？）
```
