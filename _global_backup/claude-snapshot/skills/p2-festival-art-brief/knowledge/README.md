## 每份 MD 的 9 章结构

1. 档期演变（历年主题方向对照表）
2. 节日定位（档期、主题、循环身份、情绪词）
3. 视觉调性（色系、风格、手法、避坑）
4. 模块拓扑（当前标准模块清单 & 产出要点）
5. 命名风格（样本 + 跨年对照 + 公式）
6. 核心元素库（按年分主元素 / 辅助 / 规避）
7. 美需描述范式（从真实行抽提的结构化模板）
8. 新美需生成建议（已用方向 + 可探索方向 + 写作 Checklist）
9. 相关链路（数据源、相关 skill、配置表）

## 更新规则

年末 / 新版本上线后，从节日本地化主表 `1pPlo0ccRBmlQj9OFWnGXhQg1wLiihfQfSxEsoKA0hLA` dump 最新档期，然后人工增量更新对应节日 MD 的"档期演变"和"元素库"章节。回源脚本保留在 skill 作者（liusiyi@happyfactory.com）本地仓库。

## 产出落地（2026-04-21 升级）

Skill 生成美需后，会**自动在主表建新 tab + 贴参考图**，美术直接看 Sheet：

- 脚本：`.claude/skills/p2-festival-art-brief/scripts/create_art_brief_tab.py`
- 新 tab 紧贴历年对应月份档期（如 2027 科技节 tab 插在 2026 科技节后）
- 列布局：A 模块名 / B 功能说明 / C 功能描述 / D 文案需求 / E 往年图片（留空手动拖）/ F 文案(本地化) / G 美需描述+参考链接 / H =IMAGE() 主图缩略
- 每模块配 2-5 张参考图（WebSearch 拿 URL，H 列 `=IMAGE()` 显示主图，G 列追加所有链接）
