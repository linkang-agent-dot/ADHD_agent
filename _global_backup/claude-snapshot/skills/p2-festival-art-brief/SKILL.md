# P2 节日美需包装手册

## 这个 skill 解决什么问题

每个节日运营策划要产出"主题方向 + 模块美需描述 + 文案命名"，过去痛点：
- 方向重复（年年都想蒸汽朋克 / 星际）
- 模块遗漏（套装循环节日忘配 LV1-6、手札节日漏掉子模块 1-5）
- 命名风格漂移（2024 洋风 / 2025 国风 / 2026 又翻车）
- 美需描述不规范（缺底座 / 缺动态 / 缺方向备选 / 缺避坑）

本 skill 把 **2021-2026 年 60 份节日美需**压成 13 份知识库 MD，每次出新美需**先查档 → 结合新创意 → 按范式生成**。

## 标准执行流程

### Step 0 自主学习（必做，不跳过）

1. **识别目标节日**：从用户请求里提取节日名（"科技节""情人节"……）
2. **读对应 MD**：`knowledge/<NN>_<节日>.md`（相对 skill 根目录）
3. 确认：循环身份、过去 2 年主题方向、已用过的创意方向、历年命名样本
4. **向用户汇报**你读到的现状（1-2 段），等确认再产出

### Step 1 理解用户输入

用户输入通常有 3 种形态：

**A. 题材指定型**："今年科技节做蒸汽朋克方向"
→ 查 MD 核对蒸汽朋克是否与历年重复 → 若 OK 直接按题材展开

**B. 方向放空型**："下一期音乐节帮我想新方向"
→ 从 MD 的"可探索方向"池 + 互联网创意，提 3-5 个候选 → 让用户选

**C. 补写美需型**："给这个主城皮肤 LV3 补美需描述"
→ 查该节日 LV3 历年美需样本 → 按范式补齐底座/动态/方向备选/避坑

### Step 2 互联网创意融合（如果用户要脑洞）

被要求"帮我发散题材"时，结合 3 类创意池：
- **流行文化**：近 1 年热门影视/游戏/综艺/新闻题材
- **亚文化**：小众但有识别度的美学（如大地美学 / 液态金属 / 生机符号学）
- **地域文化**：国内外具象传统元素（敦煌 / 和风 / 北欧 / 南美…）

对每个候选题材，给：
- 一句定位：**[主题][差异化切入点]**
- 与本节日历年方向的差异点（重点避坑）
- 3-5 个视觉元素关键词
- 可能命名示例（2-3 组）

### Step 3 按节日范式产出

按目标节日 MD 的"**美需描述范式**"章节，每个模块必须包含：
1. 核心主体（≥2 特征）
2. 底座/支撑处理（装饰物类）
3. 动态效果（LV4-6 / 高级款）
4. 配色方案
5. ≥2 方向备选（复杂模块）
6. 避坑项（"避免像xxx""避免与去年xxx撞型"）

### Step 4 命名产出

参考 `p2-translation-style` skill 的风格指南，结合本节日 MD 的"命名风格"章节：
- 主城皮肤: 4 字主名 + 10-14 字描述
- 机甲染色: `机甲名（四字意象）` 双层
- 行军特效: 4 字主名 + 10-14 字描述（含动作词）
- 行军表情: 2-4 字主名 + ≤10 字描述
- 头像框/铭牌: 4 字主名 + 10-14 字描述
- 装饰物: 可共用同名 + 描述增量（春节/拓荒常用）
- 活动名: 4 字主名，动词+名词结构

### Step 5 参考链接搜集（每个模块必做）

美需不是光写字——美术要"一眼定位方向"。每个模块配 **2-5 条参考链接**，H 列会渲染成可点击的蓝色链接列表（①主图 / ②底座 / ③动态 / ④配色 …），美术点进去直接看 Google 图片瀑布流挑图。

**链接数建议**：

| 模块类型 | 建议条数 | 关键词策略 |
|---|---|---|
| 主城皮肤 / 机甲皮肤 | **4-5 条** | 主体 + 底座 + 动态参考 + 配色参考 + 避坑对照 |
| 机甲染色 / 行军特效 | **3 条** | 主意象 + 特效对照 + 配色氛围 |
| 行军表情 / 头像框 / 铭牌 / 装饰物 | **2 条** | 主意象 + 风格氛围 |
| 手札子模块 | **3 条** | 主物件 + 使用场景 + 升级对照 |

**两种写法（按需选）**：

1. **searchQuery**（默认）—— 脚本自动转成 Google 图片搜索 URL
   ```json
   {"label": "主图", "searchQuery": "steampunk gear castle concept art"}
   ```

2. **url**（已有具体直链）—— 点击直接跳该 URL
   ```json
   {"label": "主图", "url": "https://cdna.artstation.com/xxx.jpg"}
   ```

**搜图关键词原则**：
- **英文 + 风格词**（`steampunk castle concept art` 比"蒸汽朋克城堡"效果好得多）
- 关键词里带 **artstation / pinterest / concept art** 等词，能把结果导向游戏美术站
- 每条链接打一个用途标签（主图 / 底座 / 动态 / 配色 / 避坑对照）

> **注**：脚本不联网，不贴缩略图，只生成 Google 搜索 URL。不需要配任何 API key。

### Step 6 落 Google Sheet（主产物）

**硬规则 · `insertAfterTabTitle` 必填**：主表是按"年倒序 + 月正序"组织的（2026 各月 → 2025 各月 → ...）。新美需 tab 必须按日历顺序插入，紧贴**同年 (N-1) 月节日 tab** 之后。禁止追加到末尾、禁止放到最前。

| 目标节日 | `insertAfterTabTitle` 锚点 |
|---|---|
| 10 月万圣节 | `2026 9月上线-音乐节-主城+行军特效补包`（最后一个 9 月 tab） |
| 2 月情人节 | `2026 1月上线-春节` |
| 1 月春节 | `2025 12月上线-圣诞节`（跨年 → 接上一年 12 月） |
| 4 月复活节 | `2026 3月上线-科技节套装`（最后一个 3 月 tab） |

同月有多个 tab 时（主包 + 补包等），取 index 最大的那个。遇到陌生月份或跨年场景，**先用 gws 读一遍 tab 列表确认锚点**，别猜。

**脚本层强制**：`insertAfterTabTitle` 为空时脚本直接报错退出（除非显式加 `--allow-append` —— 只在一次性烟雾测试时用）。

调用 `scripts/create_art_brief_tab.py` 建正式档期页签：


**脚本行为**：

## 常见场景示例

### 场景 1: "帮我想 2027 科技节的新方向"

1. 读 `03_科技节.md`
2. 现状汇报：过去 5 年方向 工具铲/废土工业/摩登都市/星际飞船/军事坦克 已用；未用过的 MD 候选池：蒸汽朋克/赛博朋克/量子计算/AI 觉醒/生化实验室/深空探测/复古 Synthwave/纳米机械/时空机器/仿生进化
3. 从互联网补充 3 条新热点（近年流行的：脑机接口 / 生成式 AI 可视化 / 元宇宙虚拟景观 等）
4. 推荐 3-5 个候选并列出差异化、视觉词、命名示例
5. **等用户选定**后，按 Step 3-5 产出 LV1-6 + 手札 + BP + 机甲染色全套美需，每模块附 2-5 条搜索关键词
6. **Step 6 落 Sheet**：调 `scripts/create_art_brief_tab.py`，新 tab 紧贴 "2026 3月上线-科技节套装"
7. 把 tab URL + MD 速览一起发给用户

### 场景 2: "给行军表情'火箭发射'包装一下美需"

1. 从用户上下文识别节日（如登月节）
2. 读 `07_登月节.md` 的"行军表情"章节（看历年表情包美需样例）
3. 查 `p2-unite-gift-pack` skill 的行军表情规范（动效帧数/底座规则）
4. 产出结构化美需（主体动作 / 帧序列 / 尾部流光 / 避坑）

### 场景 3: "这个主题（如蒸汽朋克）能套在哪几个节日上"

1. 遍历 12 份节日 MD 的"已用方向"
2. 排除已用 + 定位冲突的节日
3. 推荐 2-3 个最适配的档期，并说明为什么（情绪匹配 / 色调匹配 / 模块适配）

## 风格约束 & 避坑

1. **禁止**直接照搬互联网素材命名——必须本土化+四字化
2. **禁止**与 P2 配置表里 QA 的现有道具名重复（必要时查 `p2-translation-style`）
3. **禁止**做宗教具象符号（十字/佛像/教堂），复活节/圣诞节只做文化外衣
4. **禁止**猿族元素乱用——科技节/情人节起已去猿族化；拓荒节/周年庆反而强化猿族梗
5. **必须**给出"与去年差异"的明确陈述，否则就是变相复刻
6. **必须**给装饰物 1-3 级的进化逻辑（加特效 / 加材质 / 加部件，不要换主体）
7. **必须**给 LV4-6 动态描述（单静态美需会被驳回）
