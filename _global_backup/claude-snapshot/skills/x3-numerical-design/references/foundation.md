# 数值前置课(0 基础必读,约 10 分钟)

> 你可能写了很多年代码,但没碰过游戏数值。这页只讲两个地基概念。不懂它们,后面填的每个数都是瞎填。
> 看完你应该能回答:①一个礼包里装的东西,我怎么知道它值多少钱? ②为什么不能"多给点让玩家开心"?

---

## 概念一:每样东西都能换算成钱(价值单位)

游戏里发给玩家的道具(券、宝石、加速、皮肤…),在数值视角下**每个都有一个"值多少钱"**。
有了这个,你才能把"这个礼包装了 20 张券 + 2500 钻"翻译成一个数字:它值多少美金。

### X3 的换算锚(背下来)
```
1 美元(USD) = 500 钻石          ← 官方充值兑换比例,一切的尺子
1 钻石 = $0.002
```
| 常见道具 | 单价 | 怎么记 |
|---|---|---|
| 抽奖券 / 转盘券 | **$0.25 / 张**(=125钻) | 券类统一这个价,夏日/尼罗/世界杯三案一致 |
| 宝石 / 钻石 | **$0.002 / 个** | =1USD/500,基准本身 |
| 兑换币(活动积分) | **$0.01 / 个** | 兑换商店那种币 |
| VIP点数 | $0.004 / 点 | |
| 金币 / 资源 / 阅历 | 极低(<$0.0001) | 当填充物,不影响大局 |

> 完整道具价值表:「X3礼包投放整理」GSheet `1gOCYBTtnxUiviDNiGwIX1vMAGRgQIyv76Nmd7ngBDF0` 的 `道具表` 页签(1728个道具)。
> ⚠️ 这表**填充不全**——养成/加速/碎片类很多道具**没有估值**。查不到时**别自己猜**(见 SKILL.md 护栏①)。

### 算一个礼包值多少钱
```
礼包总价值(美元) = Σ(每个道具单价 × 数量)
例:20张券 + 2500钻 = 20×$0.25 + 2500×$0.002 = $5 + $5 = $10
```

---

## 概念二:ROI = "玩家花的钱"和"拿到的价值"之比

```
ROI = 礼包总价值 ÷ 售价
例:上面那个礼包值 $10,卖 $4.99 → ROI = 10 ÷ 4.99 ≈ 2.0x
```
ROI = 2x 意思是:玩家花 1 块,拿到 2 块的东西。

### 为什么"不能投爆 ROI"——这是数值最核心的纪律
直觉会说"多给点玩家更开心,卖更多"。**错。** 原因:
1. **直接亏钱**:ROI 越高,你卖的每一份都"送"得越多。
2. **破坏付费结构**:玩家不傻。如果 $9.99 档 ROI 特别高、$99.99 档反而低,所有人只买便宜档,高档没人要,总收入反而塌。
3. **养坏预期**:这期投爆了,下期回到正常 ROI,玩家觉得"变抠了",付费意愿掉。

所以数值的纪律是**克制**,不是慷慨。

### ROI 没有"标准值",只有"同模块基准"
这是最容易错的地方。不同类型的礼包,正常 ROI 差很多:

| 模块类型 | 正常 ROI 区间 |
|---|---|
| A线链式礼包(节日主力) | ≈ 2.25x |
| 自选周卡 | ≈ 2.75x |
| 晋升皮肤 / 高价值锚点档 | 17 ~ 19x(皮肤本身"标价"很高) |
| 抽奖活动(转盘/大富翁,整活动口径) | ≈ 8x |

**所以判断"投没投爆"的唯一靠谱办法 = 跟"同类型的上一期活动"比,不是看绝对数字。**
这就是为什么 SKILL.md 第二步要你去查 `past-references.md` 找"同模块往期"——你抄的是它的 ROI 和投放结构,守住就不会爆。

## 实战:大半道具查不到价值怎么办?(相对守恒法 = 默认路径)

上面"算绝对美元价值"的前提是每个道具都有单价。但真实场景里(尤其养成/加速/碎片类)价值表大片是空的——很多道具压根没在任何地方标过钻石价,你算不出绝对 ROI。

**这是常态,不是异常。大部分情况你根本不需要绝对价值——用相对守恒法(纯数量缩放)即可:**
> 你要做的礼包,在"参照对象"(同模块往期 / 同活动低一档 / BP 的进阶轨)基础上**同款道具按比例放大量**:
> **量 ×k、价格 ×k → ROI 完全不变**。不用知道每个道具值多少钱,就保证"没投爆、跟参照档对齐"。

例:进阶档 $9.99 发「信物×100 + 技能书×5」(没估值),终极档定 $19.99(价格 2×)→ 同款道具发 2× 量(信物×200 + 技能书×10)→ ROI 与进阶档严格一致,守住。
- 产出是**"相对 ROI 守恒"的结论**(对齐参照档),给不出绝对美元数字——**这是诚实且正确的**,不是缺陷。
- **X3 大部分场景(加档/换皮/superset 轨)就是同款道具数量翻倍,到此为止,不必凑绝对美元数。**

### 少数情况:确实要绝对价值(改了道具种类,不是纯缩放)
比如终极档想塞一个进阶档没有的新道具,纯缩放算不了,得知道这个新道具值多少钱。取值顺序:
1. **先反查 tsv 的实际钻石售价**——道具在游戏里以钻石卖过的地方就是它的"实际钻石价":钻石商店 `Shop__ShopItemCfg.tsv`、直购、黑市 `BlackMarket__*`、兑换 `ActvExchange__*` 里**以钻石计价**的售价。`grep` 该 ItemID,找到钻石售价 ÷ 500 = 美元价值。(注意确认那一行的计价货币真是钻石,不是别的代币。)
2. **反查不到 = 纯养成产出物**(从没在任何地方标过钻石价,养成线价值地图 X3 仍缺)→ 别瞎编:要么退回纯数量缩放守恒,要么找策划/数值给价(护栏①)。

---

## 一句话总结
> 把道具换算成钱(概念一)→ 算出礼包 ROI(概念二)→ 跟同模块往期比,守住别爆(纪律)。
> 这三句就是整个数值设计的内核,剩下的都是查表和落地。
