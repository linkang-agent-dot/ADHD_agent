# 按玩法分叉:你这个模块用哪个 ROI 公式

> 第1步定 ROI 时,先判这个模块属于哪一类——**两类的算法完全不同**,套错=错得离谱。

## 先做这道分流题

> **问自己(或问策划):玩家是"花钱直接买一个固定礼包",还是"花钱反复抽/投,产出散落随机"?**

- **买固定礼包** → 礼包类 → 用【单礼包 ROI】
- **反复抽/投** → 抽奖活动类 → 用【整活动 ROI】

| 类型 | 典型模块 | 用哪套 |
|---|---|---|
| 礼包类 | 链式礼包、累充礼包、周卡、月卡、储蓄罐、晋升皮肤包、装饰阶梯 | 【单礼包 ROI】 |
| 抽奖活动类 | 转盘、推币机、大富翁、扭蛋、开箱(消耗券抽) | 【整活动 ROI】 |

---

## 【单礼包 ROI】(礼包类)
```
ROI = 礼包总价值 ÷ 售价
礼包总价值 = Σ(道具单价 × 数量)        ← 道具单价见 foundation.md
```
- 跟**同模块往期**对齐(past-references.md),守在它的 ROI 区间。
- 多档时**线性**:$9.99 给 X,$19.99 给 ~2X,ROI 各档一致。
- 这是默认、最常见的情况。

---

## 【整活动 ROI】(抽奖活动类)
抽奖活动玩家是"反复投入、产出随机散落",不能看单次,要看**整个活动玩下来**:
```
整活动 ROI = 全程产出钻价值 ÷ 全程实付钻
全程实付钻 = 抽数 × 单抽消耗道具的钻价值
全程产出钻价值 = Σ(各产出来源折算的钻价值)
```

### ★投放结构:抽奖货币要"打包进礼包"卖,不是纯卖货币
玩家抽奖要消耗一种货币(大富翁=**罗盘**、转盘/开箱=**券**)。这种货币的**主投放方式 = 打包进礼包**(罗盘/券 + 其他奖励一起),**不是光秃秃直购罗盘/券**:
- **主力 = 罗盘+礼包(A线链式礼包)**:礼包里装罗盘/券 + 宝石 + VIP经验等,ROI 高、是变现主轴,也是"买了顺便多抽几把"的飞轮入口。
- **纯卖货币(B线)= 低 ROI 的直购底**:只给罗盘/券、不搭别的,ROI 偏低,只作补量兜底,**不能当主投放**。
- 所以配抽奖活动时:**先有"罗盘+礼包"的投放盘**(按礼包类规则定档位/ROI),再配抽奖玩法本身的产出 ROI(下面整活动口径)。两者都要,缺了"罗盘+礼包"=没有变现主轴。
> 范式见 numerical-manual:A线=券+宝石+VIP经验(ChainPack,ROI最高);纯券=B线(pack_type=15,ROI偏低=直购券底)。

### 关键做法(否则一定算错)
1. **固定一个评估锚抽数**:用一个固定抽数算整活动(深海大富翁案用 **700 抽**,约等于一个深R玩家的完整投入)。换皮前后用同一抽数对比。
2. **目标 ROI 锚同类玩法**:转盘/抽奖类锚尼罗/春节转盘,深海大富翁定 **8x**。
3. **把"随机落点"折成确定值**(命中率均匀模型):
   ```
   每格落点次数 ≈ 总抽数 ÷ 格数
   某格产出 = 落点次数 × 该格单次期望产出 E
   ```
4. **隐藏大头别漏算**:大富翁"跨越起点"额外给奖(代码 landIndex>24 触发,一次给2个),700抽≈跑102圈≈204次额外抽取——漏算严重低估产出。

### ⚠️ 抽奖活动的通病:"越抽越赚"(必须压平)
投入是线性的(每抽成本固定),但产出常有放大机制,导致深R**越抽 ROI 越高**——跟"越往后越不划算"的正常预期相反。两个加速器:
| 加速器 | 机制 | 怎么压 |
|---|---|---|
| 格子/岛屿升级 | 停留升级,产出暴涨3-6× | 换皮可整体砍掉(不升级、塌成单档E) |
| 阶段奖励后置 | 后段价值是前段10× | 价值前移 / 拉平梯度 |
最小改动 = **只调权重/E值让期望更平**,不改玩法内容。
