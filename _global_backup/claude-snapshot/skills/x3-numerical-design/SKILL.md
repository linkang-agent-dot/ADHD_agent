# X3 数值设计

## 这个 skill 是干嘛的 / 你要产出什么

把一份**策划案**(说清要做哪些模块、什么定位、什么档位、大奖是什么)变成一张**数值表**:
每个礼包/奖池"售价多少、总价值多少(ROI)、里面具体投放什么道具×多少"。
**产出物不是"填好的配置格子",是"先想清每样东西值多少钱、为什么,再倒推出格子"。** 最后才落配置。

> ⚠️ **如果你没接触过游戏数值,先读 `references/foundation.md`(10分钟)。** 那里讲清两个地基概念:
> ①每个道具值多少钱(价值单位) ②ROI 是什么、为什么"不能投爆"。不懂这两个,后面每一步都是瞎填。

## 北极星:不投爆 ROI

数值设计最容易闯的祸是**给太多**——玩家花 $9.99 拿到值 $50 的东西,你以为"慷慨",其实是亏钱 + 破坏整条付费结构。防它的铁律就一句:

> **同一个模块,ROI 要一致、跟往期对齐。$9.99 给 X 块道具,$19.99 就给 2X,线性不破。**

注意:**ROI 没有"全局正常值"**。A线链式礼包基准 ≈2.25x、周卡 ≈2.75x、晋升皮肤档可以 17~19x、抽奖活动按整活动算 ≈8x——差很多。所以**永远跟"同模块往期"比,不跨类型比**。这就是第二步要查往期参考的原因。

## 主轴:5 步(第0步起,别跳)

### 第 0 步 — 读策划案,拎出 5 样东西
策划案是这次数值的"需求源"。先把它读透,列出来:
1. **模块清单**:这次要配哪些礼包/活动模块?
2. **玩法类型**:每个模块是"礼包类"还是"抽奖活动类"?(决定用哪个 ROI 公式,见第1步)
3. **价位档**:每个模块有哪些价格档?($9.99 / $19.99 / $29.99 …)
4. **投放定位**:策划想主攻什么?(拉新广度 / 中高R深度 / 大奖钩子)
5. **大奖 / 核心付费点**:玩家冲着什么花钱?

> 拎不出来=策划案没写清,**先回去找策划补,别自己脑补**。第4步会拿这5样回头验。

### 第 1 步 — 定盘(算每个礼包"该值多少")
对每个模块,两件事:

**① 确定道具价值** — 把"礼包里装的道具"翻译成钱。
- 价值锚在 `references/foundation.md` 和「X3礼包投放整理」表(SheetID `1gOCYBTtnxUiviDNiGwIX1vMAGRgQIyv76Nmd7ngBDF0`,`道具表`页签)。
- 常用单价:券 $0.25/张、宝石/钻石 $0.002/个、兑换币 $0.01/个、1 USD = 500 钻。
- ⚠️ **查不到价值的道具(养成/加速类大量没估值)→ 见下方「护栏」,禁止自己猜一个填进去。**

**② 分模块定 ROI** — 跟**同模块往期**对齐。
- 去 `references/past-references.md` 找同类型上一期活动,抄它的 ROI 档。
- 礼包总价值 = 售价(USD) × 500 × ROI(钻石口径) → 这就是这个礼包"该装多少钻的东西"。
- **礼包类**用单礼包 ROI;**抽奖活动类**(转盘/推币机/大富翁)用"整活动 ROI"(全程产出÷全程实付),完全不同——见 `references/playbook-by-form.md` 选对公式。

### 第 2 步 — 填数(总价值的盘子里投放什么)
总价值锁定后,决定盘子里**具体投什么道具、各占什么角色**。这一步靠"抄往期同模块的投放定位":
- 去 `references/past-references.md` 看同模块往期是**怎么搭内容**的——不只看 ROI 数字,看**投放结构**。
  例:A线链式 = 券(打底) + 宝石(冲ROI) + VIP经验(钩子);周卡 = 10选4自选 + 券打底。
- 照"同定位"投:策划说主攻深度 → 堆中高R档;说拉新 → 低门槛广覆盖。
- 算出来的内容总价值要落回第1步定的盘子(±小幅),漂了就调道具数量。

### 第 3 步 — 落配置表
数值结论 → 具体 tsv 字段。**这一步交给 `x3-config-export` skill 做**(它管安全改 tsv + 导表 + 验证)。
关键映射:售价档→`Pack.Price`(填档位id不是美元)、奖池内容→`Reward` 表(Pack.Content=Reward.RewardID 连接)。
字段细节和必检见 `C:\ADHD_agent\.cursor\x3-config-library\table-reference.md`。

### 第 4 步 — 复查 ★(扫知识库 + 派独立 sub-agent,过了才收工)
自己算的自己不靠谱。派 `task-checker` 子 agent **独立复查**,跑两个维度:
- **维度A 数值自洽**:ROI 投没投爆(跟同模块往期比)、价值有没有瞎编、玩法模型用对没、投放定位对齐没。
- **维度B 兑现策划案**:第0步拎的"模块清单/价位档/大奖/定位",数值表里**逐条**做到了吗?有没有漏配、档位对不上、大奖没给够。

派法(就是你写策划案时那套验收员):
```
派 task-checker 子 agent,传:
  type = numerical
  产物位置 = <你的数值表文件/GSheet> + <策划案文件/GSheet>
  任务名 = <活动名>数值
```
它会读 `C:\ADHD_agent\.claude\quality-gate\numerical-checklist.md` 逐条对,输出 `CHECK_DONE`。
- 有 blocker → 别 claim 完成,列给用户问『修/跳过』,修完重派。
- 全过 → 收工。

## 护栏(第1-2步最容易翻车的三处,撞到就停)

1. **道具价值查不到,不许猜。** 价值表填充不全(养成/加速/碎片类大量空)。处理顺序:
   ① **默认:沿用参照对象的同款道具、纯数量缩放守 ROI**——换皮=被抄模板原道具;新增档/轨(如 BP 终极轨)=参照轨同款道具按价格倍数同比放大量。**X3 大部分情况就是数量翻倍,到此为止,不用估值**(foundation「相对守恒法」)。
   ② 确实换了道具种类、必须知道绝对价值时:**先反查 tsv 实际钻石售价**(钻石商店 `Shop__ShopItemCfg`/直购/黑市/兑换里以钻石计价的售价 ÷500=美元价值);
   ③ 反查不到(纯养成产出物,从没标过钻石价)→ **找策划/数值要价**,补估后回填价值表;
   ④ **绝不对空值道具假设价值**——这是 90% 准确率掉到 50% 的头号原因。
   ⑤ **换皮抄骨架的道具清单也要逐个过「流通度审计」**——每个道具查全 Reward 表被多少组投放/历史总量+有无功能出口。历史只投放过一两次的道具=玩家不认识、可能是废坑或稀缺性事故(2026-07-02 深海周卡实证:夏日骨架照抄的红宝石之血40032全游戏只投过19瓶且无功能出口,周卡档3一周就发63瓶)。审计五件套=存在性+i18n+图标+全表流通度+功能出口。

2. **别把抽奖玩法当礼包抄。** 转盘/推币机/大富翁的 ROI 是"整活动产出÷整活动实付",还有"越抽越赚"的加速器要压平。拿单礼包 ROI 套它=错得离谱。不确定属哪类→读 `references/playbook-by-form.md`。

3. **撞到空白玩法,交给策划。** 累充、养成线深调、福袋盲盒、矿机挂机、限时秒杀、联动皮肤——X3 **没有往期数值参考**(见 past-references 末尾"空白清单")。这些不要自己从零定数,标出来找策划,否则保不住 90%。
