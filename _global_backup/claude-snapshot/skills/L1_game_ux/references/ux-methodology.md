# 复杂博弈系统 UX 优化方法论

> 来源：贸易港口案例复盘
> 核心理念：将策划的"理性数值逻辑"转化为玩家的"直觉感性体验"
> 关键转变：从"比价计算器"进化为"风险博弈大厅"
> 最后更新：2026-01-15

---

## 四阶段流程总览

```
1. 核心定义 → 找准"魂"（挖掘冲突，确立层级）
2. 信息架构 → 做减法（去噪，定量转定性）
3. 交互流程 → 切场景（任务分流，渐进式披露）
4. 视觉表现 → 强感知（数据可视化，情绪化反馈）
```

---

## 阶段 1：核心定义 — 找准"魂"

**挖掘冲突**：
- ❌ 错误做法：只看表面的增删改查，把复杂系统当简单数据展示
- ✅ 正确做法：通过提问明确核心玩法，找到真正的"冲突点"
- **案例**：贸易港口不是"算减法"（利润=卖-买），而是"赌人性"（人多必跌）

**确立层级**：
- 一旦确立核心玩法（如"博弈"），关键信息成为 UI 的绝对主角
- 其他信息（固定的、无差异化的数据）必须让路
- **案例**：拥挤度（风险）和预期利润（收益）是主角，固定的航行时间（4小时）是配角

```
核心玩法 → 关键信息 → UI主角
其他信息 → 降级/隐藏/删除
```

---

## 阶段 2：信息架构 — 做减法

**奥卡姆剃刀去噪**：
- 原则：如果一个信息对决策没有差异化影响，它不该出现在列表里
- 判断标准：这个信息能帮玩家做决策吗？在不同选项间有差异吗？都是"否"→删除
- **案例**：全服航线时间统一为4小时 → 直接删除

**定量转定性**：
- 原则：快速决策时，模糊的红绿灯比精确数字更高效
- **案例**：将 `92%` 及硕大进度条简化为 🔴 **拥挤** 小圆点信号灯
- 适用场景：
  - 列表快速浏览：用定性标识（🔴/🟡/🟢）
  - 详情深入查看：显示精确数值
- 转换规则：高风险/高收益 → 🔴，中等 → 🟡，低风险/稳定 → 🟢

**关键词**：**决策降噪**、**定性优于定量**

---

## 阶段 3：交互流程 — 切场景

**任务模型分流**：
- 原则：降低认知负荷的最好办法，是一次只让用户思考一件事
- **案例**：将混杂的贸易界面拆分为 📤 **我要卖** 和 📥 **我要买** 两个页签
- 要点：卖时只看赚多少；买时只看成本多少；避免同界面同时展示买卖信息

**渐进式披露 (Progressive Disclosure)**：
- 原则：手机屏幕寸土寸金，把复杂性藏在点击之后，让首屏保持"性感"
- **案例**：
  - 列表：只看概览（港口名、拥挤度、预期利润）
  - 点击卡片：唤起 Bottom Sheet 显示 K 线图和明细
- 要点：首屏只展示决策必需信息；详情点击后展示；使用平滑过渡动画

**关键词**：**任务流拆分**、**移动端渐进式交互**

---

## 阶段 4：视觉表现 — 强感知

**数据可视化**：
- 原则：能用图绝不用字，能用颜色绝不用数字
- **案例**：用 🌶️/🧵 代替文字（辣椒=高风险，丝绸=低风险）；用 💰/📉 区分买卖按钮
- 要点：选择有语义的图标；保持图标风格一致；3-5个核心图标即可

**情绪化反馈**：
- 原则：UI 不仅传递信息，还要传递情绪（焦虑感或成就感）
- **案例**：
  - 高风险时：红色预警动画（闪烁、脉冲效果）
  - 推荐位：金边高亮（吸引注意力）
  - 成功交易：绿色闪光 + 数字飞入动画
- 要点：风险/警告→红色+动画；机会/推荐→金色+高亮；成功/完成→绿色+特效

**关键词**：**风险可视化**、**情绪化反馈**

---

## 适用场景

- 复杂博弈系统（贸易、拍卖、市场）
- 多变量决策系统（装备选择、策略规划）
- 实时竞争系统（排行榜、PVP匹配）
- 风险收益系统（投资、交易、竞拍）
