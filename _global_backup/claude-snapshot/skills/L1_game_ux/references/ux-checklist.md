# 交互自查验收标准

> 来源：Google Drive - 交互自查验收标准（ID: `17942KlGH0upfGz6iDNS5smU6lrR09pRG2HU4jppug5I`）
> 设计完成后对照此表检查，基本可避免大部分遗漏问题。

---

## 信息结构与流程

### 信息结构
- [ ] 是否已对现有信息流进行过简化
- [ ] 信息结构是否容易理解，信息主次分明
- [ ] 信息层级是否清晰，**不超过 3 层**
- [ ] 信息分类是否合理，相关内容靠近
- [ ] 信息视觉流是否流畅
- [ ] 频繁操作区、按钮位置是否好点

### 流程设计
- [ ] 产出与消耗是否容易对应进入
- [ ] 返回和出口是否符合用户预期
- [ ] 是否充分考虑操作容错，如异常退出

---

## 界面呈现

### 控件呈现
- [ ] 控件是否符合认知，同一个控件只代表一个含义
- [ ] 控件样式是否符合规范、一致性（包括文字）
- [ ] 控件交互行为是否符合一致性
- [ ] 控件的多态如何呈现

### 数据呈现
- [ ] 空态如何呈现
- [ ] 字数有限制时超限处理
- [ ] 无法完整显示的数据如何处理
- [ ] 数据过期如何提示
- [ ] 数据如何排序
- [ ] 数据显示是否有特定的格式要求
- [ ] 数据是否存在极值，保证常态下的界面美观性

### 入口与提醒

**系统级**：邮件预告、功能解锁弹窗、规则说明弹窗、HUD 入口、聊天、飘字提示、红点

**玩法级**：邮件预告、拍脸弹窗、副本目录、联盟主页、HUD 入口、HUD 气泡提醒、联盟聊天、活动日历、飘字提示、红点

**活动级**：邮件预告、拍脸弹窗、HUD 入口、顶部任务活动计数、对应入口的活动标记（雷达）、城市入口、联盟主页、聊天、飘字提示、红点

### 文案呈现
- [ ] 句式是否一致
- [ ] 用词是否一致及准确
- [ ] 文案是否有温度，还原

### 输入与选择
- [ ] 是否为用户提供了默认选项
- [ ] 输入过程是否提供提示和判断
- [ ] 是否可以使用互动条或者其他形式
- [ ] 是否需要定制键盘形式

---

## 交互过程与反馈

### 引导与正反馈
- [ ] 是否设计了界面入场，通过入场动画进行信息整合
- [ ] 是否周全地考虑了所有**操作成功**的反馈（点击瞬间、载入中、进行中、当前效果状态、状态变化、过程中取消、界面切换）
- [ ] 是否周全地考虑了所有**操作失败**的反馈
- [ ] 操作中是否允许中断、取消
- [ ] 过程中退出是否需要进行存储，保留操作
- [ ] 是否对强调内容进行突出显示
- [ ] 是否设计了必要且合理的动效
- [ ] 不可操作时是否有动效提示玩家游戏还在运行

---

## 特殊情况
- [ ] 是否存在 VIP 特权/角色权限不同的操作
- [ ] 不同特权的显示是否一致
- [ ] 是否存在特殊模式
- [ ] 是否有超出屏幕的情况，需不需要提示滑动
- [ ] 是否需要根据不同情况跳转到不同的界面（如活动界面跳转到可领奖）

---

## 颜色标签规范（交互文档标注）

| 颜色 | 用途 | 示例 |
|:---|:---|:---|
| 🟦 蓝色 | 主流程/正向操作 | 购买、升级、确认 |
| 🟩 绿色 | 成功状态/可领取 | 任务完成、奖励可领 |
| 🟨 黄色 | 警告/需注意 | 资源不足、即将过期 |
| 🟥 红色 | 错误/负向操作 | 取消、删除、失败 |
| 🟪 紫色 | 特殊流程/分支 | VIP特权、付费路径 |
