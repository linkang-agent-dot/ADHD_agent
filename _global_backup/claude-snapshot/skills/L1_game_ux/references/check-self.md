# 交互自查验收标准（CheckSelf）

设计完成后逐项过筛，覆盖 WCAG 2.2 + 启发式评估。

## 一、信息结构与流程

### 信息结构
- [ ] 现有信息流是否已简化
- [ ] 信息主次分明，层级不超过 3 层
- [ ] 相关内容靠近，视觉流流畅
- [ ] 频繁操作区/按钮位置好点到

### 流程设计
- [ ] 产出与消耗入口容易对应
- [ ] 返回/出口符合用户预期
- [ ] 异常退出有容错处理

## 二、界面呈现

### 控件
- [ ] 同一控件只代表一个含义，样式/行为一致
- [ ] 多态（默认/激活/禁用/加载）均有定义

### 数据
- [ ] **空态**：插图/图标 + 引导性文字（Empty state pattern，必须）
- [ ] **加载态**：数据密集型用骨架屏，非 Spinner
- [ ] 字数超限处理、截断处理、数据排序、极值处理

### 入口与提醒（按类型检查）
- **系统功能**：邮件预告 / 功能解锁弹窗 / 规则说明弹窗 / HUD 入口 / 红点
- **玩法**：邮件预告 / 拍脸弹窗 / 副本目录 / HUD 气泡提醒 / 活动日历 / 红点
- **活动**：邮件预告 / 拍脸弹窗 / HUD 入口 / 雷达标记 / 城市入口 / 红点

### 文案
- [ ] 句式/用词一致且准确
- [ ] 文案有温度，贴近玩家语境

### 输入与选择
- [ ] 提供默认选项
- [ ] 输入过程有提示和实时验证
- [ ] 按需考虑互动条 / 定制键盘

## 三、交互过程与反馈

### 引导与正反馈
- [ ] 入场动画整合信息（减少认知负荷）
- [ ] **所有操作成功反馈**：点击瞬间 / 载入中 / 进行中 / 状态变化 / 界面切换
- [ ] **所有操作失败反馈**均有覆盖
- [ ] 允许中断/取消，退出后保留操作
- [ ] 动效必要且合理；不可操作时有动效提示游戏仍在运行

### 特殊情况
- [ ] VIP/角色权限不同时的差异呈现
- [ ] 内容超出屏幕是否需要滑动提示
- [ ] 不同状态跳转路径是否覆盖（如活动界面 → 可领奖界面）

## 自测报告格式

```
## 自测报告

### ✅ 达标项
（列举）

### ❌ 必须解决
（问题 + 原因）

### ⚠️ 建议解决
（问题 + 建议方案）

### 💡 更佳方案
（可选优化项）

### 玩家视角犀利意见
- [大R - Alex]：...
- [大R - 军师老王]：...
- [中R - 琳琳]：...
- [小R - ...]：...
- [零氪 - ...]：...
```
