# 测试 Brief 模板库（分类）

开测时（流程第 0 步）挑对应**功能类别**的模板起草 brief，已预填该类的**典型触发条件 / 验证点 / 常见坑**，省得从零想。

- **起草时机**：第 0 步先填「功能/改了什么/成功断言」；触发条件&复现步骤等**第 4 步摸清链路后补全**。
- **完成去向**：测通后的完整 brief = 一条配方，挪进 `recipes.md`（按类别分区）。
- **模板 ≠ 成品**：这里是空白起点；`recipes.md` 是已验证实例。

## 通用骨架（任何功能都从这个长出来）
```
# 测试 Brief · <功能名>（关键 ID / ActvType）
- 来源：入参 brief / Jira 单 / 口头（注明）
- 改了什么：<逐处 文件:行 + 改动>
- 成功断言（先定，否则判断不了测没测通）：
    □ 运行时值 <X> 变成 <Y>
    □ 日志出现/消失 <关键字>
    □ UI <某窗/某元素> 显示
    □ flag/dedup <key> 置位
- 触发条件（前置状态清单）[第4步后补]：<等级/资源/时间窗/功能解锁/已有或未有某状态>
- 复现步骤（GM/eval）[第4-5步后补]：
- 验证点：① 代码路径 <flag/值/日志> ② UI <截图>
- 待确认 / 风险 / 回归面：<触及 CSShared/协议/配置结构 必填回归面>
```

## 按类别预填（挑一个 copy，改具体值）

### 1. 活动开启
- **触发条件常含**：`CheckActivityIsUnlock` 各关 —— PlayerLv 区间、TimeCycle 时间窗（注册相对要新号 / 开服相对 / 绝对）、RequireFunction 功能解锁、（充值类）TotalPayMoney 门槛。
- **验证点**：`GetActivityIdsByCfgID(cfgId)` 非空；服务端日志开启记录。
- **坑**：玩家触发型 ≠ 服务端部署型（部署 GM 对玩家活动无效）；跨门槛事件 vs 登录状态式重评估两条路覆盖与否。

### 2. 触发型弹窗
- **触发条件常含**：上游事件（充值/升级/解锁）+ 活动/功能已开 + **去重 key 未置位**（每角色/每日一次）。
- **验证点**：① dedup key（如 `ActvPopupCount_{id}`）0→1 = handler 推窗了 ② 截图 = 真显示。
- **坑**：`WndMgr.Push` 是队列式（按优先级、进主场景才弹）——推窗 ≠ 立即显示，可能排在引导奖励/VIP 等窗后；dedup 在"推"时置位（非"显示"时）。

### 3. 礼包 / 充值
- **触发条件常含**：礼包在售（未屏蔽/未下架）、购买模式（金砖模式要余额够）、限购次数未满。
- **验证点**：`BuyGiftReq`→发货；`TotalPayMoney`/充值积分增量；限购计数。
- **坑**：Editor 下购买走 `#else` 直发 BuyGiftReq（非真 SDK）；gem 档不涨 TotalPayMoney、现金档才涨；**无模拟真实充值 GM**，涨 pay 只能走现金档购买；pay 只增不减。

### 4. 奖励发放
- **触发条件常含**：领取入口可用、未领过（防重领 flag）、前置达成（任务完成/集齐/阶梯到点）。
- **验证点**：到账内容&数量（背包/资源 diff）、领取状态置位、邮件附件。
- **坑**：奖励组 RewardID 可能跨活动**共享**——改奖励先确认是否被复用源共享，否则连带改别处。

### 5. 任务 / 进度
- **触发条件常含**：任务已接、进度计数源事件、章节前置解锁。
- **验证点**：进度计数推进、完成回调触发后续、`HandleNewPlayerActivityInfos`。
- **坑**：进度来源事件可能多路；`TaskProgressInitType.New/None` 影响初始化。

### 6. 红点
- **触发条件常含**：红点对应业务状态变化（有可领/有新内容）。
- **验证点**：红点出现/消失时机、数字（`RedPointMode.Point/Num`）、已读策略生效。
- **坑**：红点不消失常是已读策略/刷新时机问题；改红点看 `client-ui-common-rules` skill。

### 7. 数值 / 属性
- **触发条件常含**：buff/加成来源生效、相关配置加载。
- **验证点**：算出来的值 = 预期（属性/容量/加成叠加）。
- **坑**：⚠️ **数值口径属策划需求**（容量公式/玩法规则等），核对可以、**不擅自改**；要改另开需求。

### 8. UI / 交互
- **触发条件常含**：界面入口可达、所需数据已加载。
- **验证点**：界面打开走完整链路（非直 Show）、按钮 `interactable`、本地化文本（`TFWText`）显示、数据变了刷没刷（顶部资源条/列表）。
- **坑**：场景切换型 UI 直 `WndMgr.Show` 会卡死（见 DebugUtils）；"数据变了但没人通知刷新"是高发隐藏路径。
