# X3 iGame 活动批量部署 skill

## 用户怎么用（同事看这一节）

跟 Claude 对话，把下面这些信息**给清楚**就能跑。Claude 缺啥会主动追问，但**一次性给齐最快**。

### 你要给 Claude 提供的信息（按重要性）

| # | 必填？ | 字段 | 取值 | 举例 |
|---|---|---|---|---|
| 1 | ✅ 必填 | **环境** | `prod`(生产) / `dev`(测试) | "生产服"、"线上"、"dev"、"测试服" |
| 2 | ✅ 必填 | **活动配置 ID** (configId) | iGame 上活动配置的 ID | `102001`(风暴)、`101901`(入侵)、`100702`(酒馆争霸)、`103001`(S1)、`300601`(S6) |
| 3 | ✅ 必填 | **是否跨服** | 跨服=acrossServer:1 / 单服=0 | "跨服活动"、"单服活动"、"按 sheet 走（跨服）" |
| 4 | ✅ 必填 | **服务器范围 + 分组** | 服 ID 列表，跨服多组用换行/分号分隔 | `1870/1880/1890 一组，1900/1910 一组`<br>或 `60 个全服：1170, 1270, …, 1970` |
| 5 | ✅ 必填 | **活动开始时间** | UTC 或北京时间都行（说明哪个），精度到分 | "6/29 00:00 UTC"、"北京时间 6/29 早上 8 点"、"sheet 上写的那个时间" |
| 6 | ✅ 必填 | **活动结束时间** | 同上 | "7/5 23:59:59 UTC"、"7 天" |
| 7 | 可选 | **活动名称** (name) | iGame 后台显示用 | 不给的话 Claude 按模板自动生成 `"风暴逐鹿OpenServer_1870,1880,1890"` |
| 8 | 可选 | **是否跨服排名** (acrossServerRank) | 一般跟 acrossServer 一样 | 不给的话默认跟 acrossServer 同值 |

### 触发样例

#### 样例 A：批量上跨服活动

> 帮我在 **X3 生产服**上 **6/29-7/5** 的**风暴逐鹿**（cfg=102001），分两组：
> **1870/1880/1890** 一组、**1900/1910** 一组

Claude 拿到：环境 prod ✓ / cfg 102001 ✓ / 跨服 ✓（风暴逐鹿默认跨服）/ 服分组 ✓ / 时间窗 6/29-7/5 ✓

→ 组装 payload → dry-run 预览 → 你说"发" → submit → 汇总返回的活动 id

#### 样例 B：按 sheet 一次性上多个活动

> 帮我把跨服匹配 sheet **6/29 那周**三线全上 X3 生产服：**风暴+S1+S6+酒馆**共 17 组

→ Claude 读 sheet 算 17 条 → **自动聚合** 17→8 → dry-run → 你确认 → 批量发

#### 样例 C：批量下架

> 把刚才那批活动下架，**id 13713-13729**

→ 17 个 id 逐个调 `batch_offline`

#### 样例 D：先在测试服跑通

> **先在测试服 (dev)** 上跑一遍验证：风暴 cfg=102001 配两组 **100,220** 和 **400,440**，时间 6/29-7/5

→ 用测试服 host + 测试服 auth，不动线上

### 模糊语言 Claude 也能识别

不用说得很死板，下列说法 Claude 都能转成正确 payload：

- "风暴逐鹿" / "风暴" / "storm" → cfg=102001（跨服 7 天）
- "世界入侵" / "入侵" → cfg=101901（跨服 8 天）
- "酒馆争霸" / "跨服酒馆" → cfg=100702（跨服 6 天）
- "魔海回声 S6" / "S6" / "新赛季" → cfg=300601（跨服 28 天 KVK）
- "魔海回声 S1" / "S1 重开" → cfg=103001
- "60 个全服" / "全服活动" → 单服 acrossServer=0，所有服合 1 组
- "按 sheet 走" / "跨服匹配 sheet 那一套" → Claude 读 sheet 自己拼 payload
- "7 天后下架" / "明天下架" → 自动算 endTime
- "跟上周一样的服" / "上次那 5 组" → Claude 可以查 sheet 历史

### Claude 缺信息会主动问

少给了哪个 Claude 会主动问，不会瞎猜。比如：

> 用户：上一个风暴
> Claude：好的，我帮你上**风暴逐鹿**（cfg=102001）。请补充：
>   1. 哪个环境？(生产/测试)
>   2. 哪几个服？跨服分组怎么分？
>   3. 时间窗（开始/结束）



### 触发样例

#### 样例 A：批量上跨服活动

> 帮我在 X3 生产服上 6/29-7/5 的风暴逐鹿，分两组：1870/1880/1890 一组，1900/1910 一组

Claude 会：
1. 识别活动=`风暴逐鹿`（cfg=102001）、跨服 2 组、UTC 时间窗
2. 显示 payload 让你看
3. 你说"发"，它走 submit、汇总返回的活动 id
4. 失败/有问题随时让它下架

#### 样例 B：按 sheet 一次性上多个活动

> 帮我把跨服匹配 sheet 6/29 那周三线全上 X3 生产服：风暴+S1+S6+酒馆共 17 组

Claude 会：
1. 读 sheet 算出 17 条 payload
2. **自动聚合**：同活动同时间的多组合并 → 17 条变 8 条 submit（节省 9 个 ID）
3. dry-run 给你看预览
4. 你确认后批量发，每条间隔 200ms 防压库

#### 样例 C：批量下架

> 把刚才那批活动下架，id 是 13713-13729

Claude 会：
1. 拆分逗号/连续段 → 17 个 id
2. 逐个调 `batch_offline`
3. 汇总成功/失败

#### 样例 D：先在测试服跑通

> 先在测试服 (dev) 上跑一遍验证流程：风暴 cfg=102001 配两组 (100,220) 和 (400,440)

切 `--env dev`、用测试服 auth，**不动线上**。
