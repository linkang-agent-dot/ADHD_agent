# 本机环境与已踩坑（改脚本/换机器前必读）

## 转写（faster-whisper）

- **模型用 small**：本机 commit limit 常年 86/91GB，medium 加载/推理常报 `mkl_malloc: failed to allocate memory`。
  脚本已带 medium→small 自动降级；批量前想用 medium 先关大内存程序（模拟器等）。
- **必须 `HF_HUB_OFFLINE=1`**：模型已本地缓存（~/.cache/huggingface），不加会联网检查 HF 且公司网常断。
- **vad_filter=False**：默认长音频解码也会整段跳过人声（实证漏 19s，medium/small 都漏）——
  所以精确定位一律走"切点 ±6s 窗口重转写"。
- small 会输出**繁体**和标点——文本匹配前先繁→简+去标点归一化（scripts 里有 norm()）。
- 排比/重复段+BGM 区域 small 会幻觉（"吃它喷"×N、片尾"点赞订阅"）——只影响读稿，不影响切割
  （切割靠窗口重转写+声学）。

## 控制台/路径

- **PYTHONIOENCODING=utf-8** 前缀跑 python；中文输出宁可写 UTF-8 文件再 Read（GBK 控制台乱码会破坏 JSON）。
- Git Bash 的 /c/ 路径传给 python 的 av/whisper 库会报 FileNotFound——python 内一律用 Windows 原生路径。
- 源片文件夹里 `.###` 后缀 = 网盘拷贝中，跳过。

## ffmpeg

- 产品目录自带 ffmpeg（`cr多功能剪辑软件/ffmpeg/`，4.x）；脚本自动探测，否则用 PATH。
- 帧精确切割：`-ss` 前置 + 重编码即帧精确；用 `-t` + `-frames:v` 而非 `-to`（-to 取整会多含 1 帧造成相邻段重叠）。
- 无损拼接：切片编码参数完全一致时 concat demuxer `-c copy` 即可。

## 台账/断点

- analyze.py / cut.py / remix.py 全部断点续跑：转写跳过已有 json；cut 全量重切时 remix 写回的
  used_in 需保留（cut 脚本会合并旧台账）；remix 出片即写回 used_in。
- 中途新增源片：正在跑的 analyze 不会捡（启动时 glob），跑完再跑一遍即可（自动跳过已完成的）。
