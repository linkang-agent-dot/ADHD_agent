# 边界质检协议（派 sub-agent 执行，人只看 FAIL/WARN）

切完一批后，用 Agent 工具派 general-purpose sub-agent 按本协议跑客观检查。
sub-agent 只读被检文件 + 写自己的 scratchpad，不得修改切片/台账。

## 给 sub-agent 的输入

- 切片库路径（A/B/C mp4 + ledger.json）、切点计划（pipeline/analysis/cut_plan.json，note 字段=预期"前句|后句"）、源片目录、ffmpeg/ffprobe 路径
- 环境须知：`HF_HUB_OFFLINE=1 PYTHONIOENCODING=utf-8` 前缀；whisper 用 small（cpu, int8, cpu_threads=4）；中文输出写 UTF-8 文件再读（控制台 GBK 乱码）

## 检查项（每项 PASS/WARN/FAIL + 证据数字）

**检查1·切点声学**：源片切点 t 处 [t-0.05,t+0.05] 的 RMS vs 前句人声区 [t-1.0,t-0.3] 的 RMS。
低 ≥6dB=PASS / 3~6dB=WARN / <3dB=FAIL（疑似切在人声上）。
⚠️ 已知假阳性：gap=0 贴字切时后句起音落进测量窗，会 FAIL 但转写完整——以检查2为准，声学单独 FAIL 降级为"人耳抽听"。

**检查2·边缘转写（金标准）**：每条切片提取开头/结尾 2.5s 音频，whisper small 转写（vad_filter=False），
对照 cut_plan 的 note：段开头应以"后句"开头词起且首字不缺（"千万别多吃"变"万别多吃"=FAIL 截字）；
段结尾应以"前句"收尾且末字完整。拿不准标 WARN 附转写原文。
增强手段：对疑似错位的切点，用词级时间戳（word_timestamps=True）复核前后词的精确起止，
直接给出建议切点（实战曾抓出整片转写时间戳偏 0.5s 导致"那"字被切进上一段）。

**检查3·物理**：每条切片 30fps、1080×1920、时长≈帧区间差/30（误差<0.1s）；同源三段帧区间
首尾相接零重叠（A止=B起、B止=C起、C止≈源总帧）。60fps 源折算允许 ±1 帧。

**检查4·音频**：有音轨、aac 48000Hz。

## 输出

报告写 scratchpad（8切点×检查1/2 表 + 每切片×检查3/4 表），回复总判（N PASS/N WARN/N FAIL）+
FAIL 明细（附建议切点）。

## 处置

- FAIL（截字/错位）：修 boundaries.json 的 hint（用质检给的词级时间戳），重跑 plan+cut。
- 声学 FAIL 但转写完整：留用，边界预览给用户人耳抽听。
- WARN：台账标注，随批次抽检。
