# 口播视频三段式混剪管线

## 产品目录约定（一个产品一条生产线）

```
<产品根目录>/                    # 例：C:\新建文件夹\000 源片
├── 000 源片/                    # 输入池：完整源片 mp4（.###后缀=拷贝中，跳过）
├── pipeline/
│   ├── analysis/                # 转写产物（<源片名>.json/.txt）
│   ├── boundaries.json          # ★人工确认的切分边界（数据，见下）
│   └── reference_finals.json    # ★已投放成品的三段文本签名（组合器排重用）
├── 切片库/                      # A/ B/ C/ 边界预览/ ledger.json
└── 输出池/                      # <批次>/成品_NNN__*.mp4 + 记录.json + 摘要.md
```

ffmpeg：优先用产品目录下自带的（`**/ffmpeg/ffmpeg.exe`），否则用 PATH。

## 全流程五阶段

| 阶段 | 动作 | 用户参与点 |
|---|---|---|
| 0 定标准 | 新产品先读 `references/切分标准.md`，用该产品的人工正确案例校准（A/B 分界判据可能因品类微调） | 拍板标准 |
| 1 实切验证 | 先切 3~5 条 → 边界预览 + 质检 agent → 用户验收 | 听 LOW 切点 |
| 2 批量切片 | 全量源片过管线入台账 | 只看复核队列 |
| 3 组合出片 | `remix.py` 指定数量出片 | 定产量、抽检 |
| 4 例行运转 | 新片进池重跑 1→3（脚本全部断点续跑，已处理的自动跳过） | 扔片拿片 |

**节奏铁律：每阶段先小批量给用户验收再放量，不要抢跑全切。**

## 单条源片六步流水

1. **整片转写**：`python scripts/analyze.py --base <产品根目录>`（断点续跑，跳过已有 json）
2. **语义定界（人/主模型做）**：读 `analysis/*.txt`，按 `references/切分标准.md` 找 A→B、B→C 转折句，
   写进 `boundaries.json`。核心判据速记：**A 只装钩子表演形式（排比/悬念/剧场/演示），一转入
   "对你说话的连续说服"（你是…/如果你…/…不可能自己好的）就是 B；C 首句必须进入成交语境**。
3. **双保险核验**：整片转写显示 ≥3s 无人声的区间必须抽帧看画面（可能是字幕演绎/产品特写/被漏听的人声——
   三种都实际发生过）；切点附近脚本会自动 ±6s 窗口重转写，不信任整片时间戳。
4. **锚定落点**：`python scripts/cut.py plan --base <目录>`——锁定下段开头词只在该句边界测真实停顿
   （梯级静音检测），输出置信度（HIGH≥0.3s / MID≥0.12s / LOW 需人耳）。检查 plan 输出的
   "前词┃后词"是否与预期句边界一致后再执行 cut。
5. **切割**：`python scripts/cut.py cut --base <目录>`——重编码、统一 30fps、精确帧数（零重叠）、
   30ms 音频淡化；产出切片+边界预览（命名 `切点1_A尾接B头_<t>s`）+ ledger。
6. **质检**：切完派 sub-agent 按 `references/边界质检协议.md` 跑三层客观检查（声学电平差/
   边缘转写截字比对/物理参数），只把 FAIL/WARN 报给用户。质检发现切点错位时，修 boundaries.json
   重新 plan+cut（脚本可全量重跑，台账 used_in 会保留）。

## boundaries.json 格式

```json
[{"file": "xxx.mp4", "hooks": 1,
  "cut1": {"hint": 9.84, "after": "颈椎越疼", "note": "…吃它(排比结束) | 颈椎越疼越要用"},
  "cut2": {"hint": 64.80, "after": "那今", "note": "直达疼痛处 | 那今天厂家首次开启"}}]
```
- `hint`=预估切点秒（从整片转写读），`after`=下一段开头 2~4 个字（锚定词，避开在窗口内重复出现的词），
  `note`=预期"前句|后句"（质检对照用）。
- 边界经人工/质检确认后不要随意改动 hint——脚本按锚定词自动精调，hint 只要落在 ±2s 内即可。

## 组合出片

```
python scripts/remix.py <批次名> <数量|max> --base <产品根目录>
```
内置约束：A/B 异脚本（转写文本相似度≥0.6 判同脚本，防"看似异源实则同话"）、历史组合防重（写回台账）、
排除与 reference_finals.json 中已投放成品三段全同款、优先三段异主播、均匀消耗素材。

## 关键经验（为什么这么设计，改动前必读）

- **切点只落完整句边界**：41 条人工旧切片 6 成毁于句内切——同源拼回听不出，跨源混拼立刻穿帮。
- **不信任整片转写**：whisper 整片跑会整段漏人声（实证漏 19s）；词级时间戳是首尾相连的，
  真实停顿必须声学实测。
- **BGM 会淹掉静音检测**：切点定位靠 ASR 词间隙+梯级静音，不靠单一静音阈值。
- **防重靠台账不靠文件名**：源片删除不伤切片和历史记录；"A 段很好 BC 普通"可单独保留。
- 环境坑（内存 commit limit/GBK 控制台/HF 离线/small 模型）见 `references/环境与坑.md`。

## 本 skill 不做的事

- 不调用 GRFal / 云端生成服务（全本地）；不做视频内容创作，只做切割重组。
- 语义定界不外包给脚本——转折句判断由主模型读转写完成，脚本只负责"定了界之后切得准"。
