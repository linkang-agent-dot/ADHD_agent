# 道具图标（item_icon）完整流程

游戏内道具/消耗品/宝箱/卡包等物件图标，用于背包、商店、奖励弹窗、活动界面等位置。3D 卡通渲染风格，强调物件的可交互感和价值感。

---

## 何时匹配

用户说：道具图标、宝箱图标、卡包图标、礼包图标、item icon、做个道具图、消耗品图标 等。

---

## 视觉规范

| 属性 | 要求 |
|------|------|
| **尺寸** | 256×256 像素 |
| **背景** | 透明底（alpha = 0），无边框 |
| **风格** | 2D 美术，卡通半厚涂渲染，奇幻轻蒸汽风，与 X3 项目一致 |
| **主体** | 单一道具居中，有体积感和质感，视觉紧凑 |
| **质感** | 金属铆钉、皮革扣带、宝石镶嵌、木箱铁锁等细节 |
| **价值层次** | 不同品质的道具通过材质和光效区分（铜→银→金→紫光） |
| **禁止** | 多个物品拼接、九宫格、文字标签 |

---

## 流程（必须按序执行）

1. **确认道具类型**：与用户确认要生成什么道具（如：宝箱、卡包、红包、钥匙、礼包等）
2. **确认品质/稀有度**（可选）：若道具有品质区分（普通/稀有/史诗/传说），确认对应的视觉档次
3. **模型**：未指定模型且 config 无偏好 → **必须先询问**
4. **传入参考图**：从 `references/icon-ref-item/` 中选 2-3 张作为 `reference_images`（用于风格锚定，非内容参考）
5. **构造 prompt**：default_prompt（从 `default-styles.md` 的 `item_icon` 行逐字复制）+ 英文逗号 + 道具描述（英文）
6. **调用 API**：见 `api-calling.md`，`aspect_ratio: "1:1"`
7. **后处理**：
   - 调用 `remove_background` 抠背景 → 得到 RGBA 透明底
   - 缩放至 256×256（Lanczos）
8. **保存**：按命名规则保存到下载目录

---

## 命名规则

参考项目 UI 素材命名规范（`L2_asset_export/references/naming-conventions.md`，X3 项目 asset_export skill 路径待确认）：

- 宝箱类：`Loot_Icon_{名称}.png`（如 `Loot_Icon_lootbox1.png`）
- 卡包类：`MiniGame_icon_{名称}.png`（如 `MiniGame_icon_CardPack1.png`）
- 通用道具：`Icon_Daoju_{PascalCase名}.png`（如 `Icon_Daoju_SummerBP.png`）
- 联盟道具：`Alliance{功能}_icon_{名称}.png`

> 优先匹配项目中已有的同类道具命名前缀，保持一致。
