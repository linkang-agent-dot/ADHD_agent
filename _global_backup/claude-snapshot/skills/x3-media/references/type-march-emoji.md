# 行军表情（march_emoji）完整流程

## 何时匹配

用户说：行军表情、制作行军表情、做行军表情、行军表情包 等。

## 流程

1. **索要人物参考图**：必须先向用户索要**人物参考图**；未提供则提示“请提供一张人物参考图”
2. **模型**：未指定则先询问；已指定则直接使用
3. **判断参考图类型**：
   - 用户给的是**立绘 / 全身像 / 常规角色图** → 先把角色转换成**游戏内贴纸式表情包画风**
   - 用户给的是**已有表情包样例** → 优先贴近现有表情包的线条、头身比、情绪强度和附属元素风格
4. **构造 prompt**：使用 `default-styles.md` 中 `march_emoji` 的 default_prompt 全文 + 用户补充描述（英文）
5. **调用 API**：`generate_image`，reference_images 传入用户提供的人物图
6. **检查构图**：
   - 必须为**半身像 / 大头贴纸式构图**
   - 人物必须居中，不能明显左右偏
   - 头顶到画布边缘要有稳定留白，不能贴边或被裁
   - 半身露出范围要稳定，不能忽然只剩头部或露出过多下半身
7. **检查风格**：
   - 必须像**游戏内表情贴纸**，不是普通 UI 图标，也不是常规立绘裁切
   - 画风优先贴近**圆润卡通 / 粗描边 / 大头短身 / 两段式赛璐璐阴影**
   - 表情读性优先于写实度，允许夸张五官、手势和情绪表达
   - 手势需要可读时，允许手部适度放大；不要做写实手比例
   - 禁止做成写实、厚涂、二次元立绘渲染、复杂光影插画
   - 若用户明确要求，可加入文字、气泡、情绪符号；未要求时默认不加
8. **下载 + 后处理**：产出须为 **256×256** 像素（下载后缩放）
9. **保存到下载目录**

## 关键规则

- `reference_images` 参数名必须是 `reference_images`，不是 `image_paths`
- 必须用 `--file reference_images=<路径>` 传人物参考图
- 默认风格：游戏内贴纸式表情包、透明底、大头半身、粗描边、强情绪读性
- 参考样式重点：**圆润轮廓、粗黑描边、五官放大、手势清晰、两段式阴影、暖色高饱和**
- 用户给立绘时，重点是**转成表情包画风**，不是直接裁一张半身图
- 文字 / 气泡 / 情绪符号属于可选附属元素，只有用户明确要时才添加
