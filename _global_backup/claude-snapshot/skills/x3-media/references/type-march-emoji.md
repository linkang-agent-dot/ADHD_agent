# 行军表情（march_emoji）完整流程

## 何时匹配

用户说：行军表情、制作行军表情、做行军表情、行军表情包 等。

## 流程

1. **索要人物参考图**：必须先向用户索要**人物参考图**；未提供则提示“请提供一张人物参考图”
2. **模型**：未指定则先询问；已指定则直接使用
3. **判断参考图类型**：
   - 用户给的是**立绘 / 全身像 / 常规角色图** → 先把角色转换成**游戏内贴纸式表情包画风**
   - 用户给的是**已有表情包样例** → 优先贴近现有表情包的线条、头身比、情绪强度和附属元素风格
4. **构造 prompt**：使用 `default-styles.md` 中 `march_emoji` 的 default_prompt 全文 + 用户补充描述（英文）
5. **调用 API**：`generate_image`，reference_images 传入用户提供的人物图
6. **检查构图**：
   - 必须为**半身像 / 大头贴纸式构图**
   - 人物必须居中，不能明显左右偏
   - 头顶到画布边缘要有稳定留白，不能贴边或被裁
   - 半身露出范围要稳定，不能忽然只剩头部或露出过多下半身
7. **检查风格**：
   - 必须像**游戏内表情贴纸**，不是普通 UI 图标，也不是常规立绘裁切
   - 画风优先贴近**圆润卡通 / 粗描边 / 大头短身 / 两段式赛璐璐阴影**
   - 表情读性优先于写实度，允许夸张五官、手势和情绪表达
   - 手势需要可读时，允许手部适度放大；不要做写实手比例
   - 禁止做成写实、厚涂、二次元立绘渲染、复杂光影插画
   - 若用户明确要求，可加入文字、气泡、情绪符号；未要求时默认不加
8. **下载 + 后处理**：产出须为 **256×256** 像素（下载后缩放）
9. **保存到下载目录**

## ⚠️ 抠图：AI 图生图的 Q版角色禁用双底差分（2026-06-15 实测踩坑）
- **现象**：march_emoji 默认走双底差分(transparify 白底+黑底各生一次再反推alpha)，但用 gpt 图生图/参考图生成角色时，两次生图会**漂移对不齐 → 严重重影**（实测半透明像素占比 **79.6%**，正常应 <15%）。
- **根因**：双底差分硬要求两次生图除背景外像素级对齐；AI img2img 做不到稳定对齐。
- **修法**：Q版角色是**干净粗描边、非真人毛发**，根本不需要双底差分 → **单次生成**(可生在 solid flat green 绿幕便于干净抠，或模型直接透明) + 普通 remove_bg 即可（实测降到 **4.3%**，高清 1.6%）。
- **适用**：所有"AI 生成/图生图的卡通角色贴纸"（应援表情/换皮角色表情等）。真人毛发/半透明边缘仍可保留双底差分。
- 派 worker 时在 task params 写 `postprocess: single_pass_remove_bg_NO_dual_diff` + override_note 明示禁双底差分。

## ★真实宠物/真人照片 → 走「图转表情 likeness-first」（2026-07-08 实证）

用户拿**自家宠物/真人照片**要Q版表情时，**不要**用"描述式生成"（把照片当特征参考、prompt 里描述 a cute chibi Shiba Inu…）——产出是"通用同品种"，用户一眼觉得**不像自己的**。正确姿势：

- prompt 主轴 = **"Convert this exact photo into …"** + **"CRITICAL: must look like the SAME individual dog/person as in the reference photo"**
- **姿势/表情/头部角度/特征标记（眉纹、口鼻白斑、毛色分布）全部从照片来**，明确逐条写进 prompt；风格段只负责卡通化（chibi、cel shading 等），加 "Stylization only … simplified but recognizable"
- 照片里成就神态的**关键道具保留**（如下巴搭的被子），写 "keep X as the only prop"
- 参考图只传用户照片，**不传风格模板**（模板会把长相拉向通用模板脸）
- 其余流程不变：绿幕实底 → remove_background → verify_transparency

实证：2026-07-08 柴犬案，描述式两连败（用户："这个不像我给你的参考图"），改 likeness-first 后一次过，姿势/卷尾/眉纹全对上。

## 关键规则

- `reference_images` 参数名必须是 `reference_images`，不是 `image_paths`
- 必须用 `--file reference_images=<路径>` 传人物参考图
- 默认风格：游戏内贴纸式表情包、透明底、大头半身、粗描边、强情绪读性
- 参考样式重点：**圆润轮廓、粗黑描边、五官放大、手势清晰、两段式阴影、暖色高饱和**
- 用户给立绘时，重点是**转成表情包画风**，不是直接裁一张半身图
- 文字 / 气泡 / 情绪符号属于可选附属元素，只有用户明确要时才添加
